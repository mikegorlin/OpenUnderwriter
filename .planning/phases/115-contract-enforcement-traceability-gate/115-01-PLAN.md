**Step 3: Create `src/do_uw/stages/analyze/do_context_engine.py`** with:

1. `SafeFormatDict(dict)` class -- subclass dict, override `__missing__` to return `""` and log a warning. This prevents `KeyError` on missing template variables.

2. `_select_template(do_context_templates: dict[str, str], status: str, threshold_level: str) -> str` function:
   - If status == "TRIGGERED": try `TRIGGERED_{threshold_level.upper()}`, then "TRIGGERED", then "DEFAULT", then ""
   - If status == "CLEAR": try "CLEAR", then "DEFAULT", then ""
   - If status == "INFO": try "INFO", then "DEFAULT", then ""
   - If status == "SKIPPED": return "" (no do_context for skipped signals)
   - Final fallback: `do_context_templates.get("DEFAULT", "")`

3. `render_do_context(template: str, signal_result: SignalResult, company_name: str = "", ticker: str = "") -> str` function:
   - Build `SafeFormatDict` with keys: `value` (str of signal_result.value or ""), `score` (alias for value), `zone` (signal_result.threshold_level), `threshold` (signal_result.threshold_context), `threshold_level` (signal_result.threshold_level), `evidence` (signal_result.evidence), `source` (signal_result.source), `confidence` (signal_result.confidence), `company` (company_name), `ticker` (ticker)
   - Flatten `signal_result.details` into dict with underscore-joined keys: for each `(k, v)` in details, add `details_{k}` -> str(v). For nested dicts, recursively flatten with `details_{k}_{subk}`.
   - Call `template.format_map(variables)` and return result
   - Wrap in try/except for any formatting error, return "" and log warning

4. `apply_do_context(result: SignalResult, sig: dict[str, Any], company_name: str = "", ticker: str = "") -> SignalResult` function:
   - Extract `presentation = sig.get("presentation", {})` -- handle dict or None
   - Extract `do_context_templates = presentation.get("do_context", {})` if presentation is a dict
   - If empty dict, return result unchanged (do_context stays "")
   - Call `_select_template(do_context_templates, result.status.value, result.threshold_level)`
   - If template is empty string, return result unchanged
   - Call `render_do_context(template, result, company_name, ticker)`
   - Set `result.do_context = rendered_string`
   - Return result

5. `validate_do_context_template(template: str) -> list[str]` function (used by brain health/audit in Task 3):
   - Parse template string for `{variable}` patterns
   - Check for unbalanced braces (odd number of `{` or `}`)
   - Return list of error strings (empty list = valid)
   - Known valid variables: value, score, zone, threshold, threshold_level, evidence, source, confidence, company, ticker, plus any `details_*` pattern

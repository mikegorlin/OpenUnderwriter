    Create `src/do_uw/stages/render/context_builders/probability_decomposition.py`:

    The core challenge: EnhancedFrequency uses a multiplicative formula (base_rate * hazard_mult * signal_mult) with only 3 components, but D-04 requires 7+ additive display components. Solution per research: transform multiplicative to additive by computing each component's marginal impact on the running total.

    Function `build_probability_decomposition(state: AnalysisState) -> list[dict[str, Any]]`:

    1. Read `state.scoring.enhanced_frequency` -- if None, return [].
    2. Start with sector base rate (base_rate_pct from EnhancedFrequency). Mark CALIBRATED, source="NERA 2024 Recent Trends / Cornerstone Research".
    3. Decompose hazard_multiplier into sub-components by reading factor scores:
       - **IPO Uplift**: If F4 (IPO/SPAC/M&A) has points > 0, compute marginal impact = running_total * (ipo_mult - 1.0) where ipo_mult = 1.0 + (F4.points_deducted / F4.max_points) * 0.5. Mark UNCALIBRATED.
       - **Market Cap Tier**: Derive from company market cap -- small caps have higher base rates. Use a simple lookup: micro (<$300M) = +3.0%, small ($300M-$2B) = +1.5%, mid ($2B-$10B) = +0.5%, large (>$10B) = -0.5%. Mark CALIBRATED, source="SCAC 2023 Annual Report (market cap filing rates)".
       - **Volatility Adjustment**: From F7 score, compute marginal = running_total * (vol_mult - 1.0) where vol_mult = 1.0 + (F7.points_deducted / F7.max_points) * 0.3. Mark UNCALIBRATED.
    4. Decompose signal_multiplier into sub-components:
       - **Insider Selling Signal**: From factor_signal component, if F6 (Short Interest) or insider data shows elevated activity. Derive from crf_signal/pattern_signal sub-components. Mark UNCALIBRATED.
       - **Litigation History**: From F1 score, if active SCA = large uplift. Mark CALIBRATED, source="Stanford SCA Database filing rates by prior litigation".
       - **Governance Quality**: From F9 + F10 scores, governance weakness increases probability. Mark UNCALIBRATED.
    5. Each component: compute value_pct as additive adjustment (+X% or -X%), direction ("base"/"increase"/"decrease"), running_total_pct after this component.
    6. Ensure final running_total_pct approximates adjusted_probability_pct. If there's a residual (due to interaction effects in the multiplicative model), add a final "Model Interaction" component with the difference, marked UNCALIBRATED.

    Use safe_float() for ALL values read from state. Guard against division by zero on max_points.

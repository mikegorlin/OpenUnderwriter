<objective>
Build `_signal_consumer.py` -- the typed extraction layer that context builders will use to consume signal results instead of doing raw dict access.

Purpose: Currently context builders access `state.analysis.signal_results` as `dict[str, Any]` and do ad-hoc `isinstance(result, dict)` checks with `.get("status")` calls. This is fragile, untyped, and makes builder rewrites (Phase 111) impossible without a clean contract. This plan creates the typed contract.

Output: `_signal_consumer.py` with `SignalResultView` frozen dataclass wrapping a signal result plus its brain definition, and 6 extraction functions that replace all raw dict access patterns.
</objective>

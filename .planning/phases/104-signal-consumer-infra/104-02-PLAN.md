    - safe_get_status(signal_results, signal_id, default="SKIPPED") -> returns status string or default
    - safe_get_level(signal_results, signal_id, default="") -> returns threshold_level or default
    - safe_get_result(signal_results, signal_id) -> SignalResultView | SignalUnavailable: returns view or unavailable marker
    - safe_get_signals_by_prefix(signal_results, prefix) -> list[SignalResultView]: returns empty list when no matches (never crashes)

  <action>
    Create `src/do_uw/stages/render/context_builders/_signal_fallback.py`:

    1. Import from _signal_consumer: get_signal_result, get_signal_value, get_signal_status, get_signal_level, get_signals_by_prefix, SignalResultView

    2. Define `SignalUnavailable` as `@dataclass(frozen=True)`:
       - signal_id: str
       - reason: str  (e.g., "not_found", "skipped", "data_unavailable")
       - `__str__` returns formatted string
       - `__bool__` returns False (so `if result:` works as expected -- SignalUnavailable is falsy)

    3. Implement `safe_get_result(signal_results: dict[str, Any] | None, signal_id: str) -> SignalResultView | SignalUnavailable`:
       - If signal_results is None -> SignalUnavailable(signal_id, "no_results")
       - Call get_signal_result; if None -> SignalUnavailable(signal_id, "not_found")
       - If result.status == "SKIPPED" -> return the SignalResultView (SKIPPED is a valid result, not an error -- context builders check status)
       - Otherwise return the SignalResultView

    4. Implement safe_get_value, safe_get_status, safe_get_level with null-safe wrappers:
       - If signal_results is None, return default immediately
       - Otherwise delegate to consumer function, return default on None

    5. Implement safe_get_signals_by_prefix:
       - If signal_results is None, return []
       - Otherwise delegate to get_signals_by_prefix

    6. Export via `__all__`.

    Keep under 120 lines. Pure wrappers -- no business logic.
  </action>

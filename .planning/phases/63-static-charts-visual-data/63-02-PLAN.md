    2. Create `src/do_uw/stages/render/charts/sparklines.py`:
       - Pure SVG generation (NO matplotlib dependency -- sparklines are too simple for matplotlib overhead)
       - Function signature: `render_sparkline(values: list[float], width: int = 60, height: int = 16, direction: str = "auto", color: str | None = None) -> str`
       - Implementation approach:
         a. If empty values, return ""
         b. If single value, render horizontal line
         c. Normalize values to fit within (0, height) with 2px padding top/bottom
         d. Compute x-coordinates evenly spaced across width
         e. Build SVG path: `M x0,y0 L x1,y1 L x2,y2...`
         f. Add filled area below the line (same color with 12% opacity)
         g. Auto-detect direction: compare last value to first value
         h. Use CHART_COLORS for color resolution: "up"->sparkline_up, "down"->sparkline_down, "flat"->sparkline_flat
         i. Override with explicit color param if provided
         j. SVG attributes: `viewBox="0 0 {width} {height}"`, `preserveAspectRatio="none"`, inline `style="width:{width}px;height:{height}px;vertical-align:middle"`
       - IMPORTANT: No matplotlib import. Pure string building. This keeps the module fast and lightweight.
       - Add `__all__ = ["render_sparkline"]`

  <action>
Create `src/do_uw/stages/render/manifest_audit.py` with:

1. A `ManifestClassification` StrEnum with values: `RENDERS = "renders"`, `WIRED = "wired"`, `SUPPRESSED = "suppressed"` (per D-01).

2. A `classify_manifest_groups(state, context)` function that:
   - Loads manifest via `load_manifest()` from `do_uw.brain.manifest_schema`
   - For each section's groups, classifies based on:
     - **SUPPRESSED**: Template file is empty/comment-only OR group has `display_only=True` and zero signals fired for that group (per D-06) OR group has `requires` clause and required data is absent in state
     - **RENDERS**: Template renders non-empty stripped HTML when given the context dict
     - **WIRED**: Template has a data path (context key or requires clause) but renders empty for THIS ticker -- data exists for other tickers
   - Returns `dict[str, ManifestClassification]` mapping group_id to classification

3. A `build_manifest_audit_context(state, context)` function that:
   - Calls `classify_manifest_groups(state, context)`
   - Returns a dict suitable for template context: `{"manifest_audit": {"total": N, "renders": N, "wired": N, "suppressed": N, "groups": {group_id: classification_str, ...}}}`
   - Per D-08, this dict goes into the render context for the audit trail section

4. For signal-based suppression (D-06), use `collect_signals_by_group()` from `manifest_schema.py` to check if any signals map to a group. If a group has `display_only=True` and no signals reference it AND the template renders empty, classify as SUPPRESSED.

5. For template rendering check, use Jinja2 `Environment` with the same `FileSystemLoader` path as production (`src/do_uw/templates/html/`). Render each group's template with the provided context dict. If `rendered.strip()` is empty string, it's either WIRED (has data path) or SUPPRESSED (no data path).

6. To determine "has data path": check if the group's `requires` list is non-empty OR if the group's template filename (minus .html.j2) appears as a context key pattern.

Keep this file under 150 lines. Use type hints throughout. Import `load_manifest`, `collect_signals_by_group` from `do_uw.brain.manifest_schema`.
  </action>

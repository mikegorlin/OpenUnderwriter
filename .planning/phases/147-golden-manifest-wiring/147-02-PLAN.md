**Step 1: Convert 3 stub templates** (subsidiary_structure, workforce_distribution, operational_resilience) per D-02:

```jinja2
{# Suppressed: no data path. Phase 147 audit classification: suppressed. #}
```

**Step 2: Add top-level guards to ~70 unguarded display_only templates** per D-02 and D-03:
For each template identified by the Plan 01 audit as needing a guard:

a. Read the template to identify what context variable it reads (e.g., `mkt.get('earnings_guidance')`, `fin.get('cash_flow')`)
b. Wrap the ENTIRE template content (including any `<h3>` or `<div>` headers) in a top-level `{% if %}` guard that checks the relevant context key
c. The guard MUST be the FIRST non-comment line of the template
d. Use the established pattern from research:
   ```jinja2
   {% set data = context_key | default({}) %}
   {% if data and data.keys() | list | length > 0 %}
   ... entire template content ...
   {% endif %}
   ```
   Or for simpler cases where a specific boolean flag exists:
   ```jinja2
   {% if has_esg_data %}
   ... content ...
   {% endif %}
   ```

**Step 3: For display_only templates governed by signals** (per D-06):
If a display_only group has signal references (via `collect_signals_by_group`), add a guard that checks if any signals fired for that group:
```jinja2
{% set group_signals = signal_results.get('group_id', []) %}
{% if group_signals %}
... content showing signal results ...
{% endif %}
```
If zero signals fire, template produces ZERO output.

CRITICAL per D-02: Guards produce ZERO DOM elements when data is absent. No empty divs, no hidden cards, no whitespace outside Jinja2 blocks. Use `{%- if ... -%}` with whitespace control if needed.

CRITICAL: Do NOT modify templates that already have proper top-level guards (the ~33 identified in research). Only touch the ~70 unguarded ones + 3 stubs.

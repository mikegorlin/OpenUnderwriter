  <action>
    Create `src/do_uw/stages/render/canonical_metrics.py`:

    1. **MetricValue model** (frozen Pydantic BaseModel):
       - `raw: float | int | str | None = None`
       - `formatted: str = "N/A"`
       - `source: str = "none"` (format: "xbrl:10-K:FY2025" | "yfinance:info" | "llm:10-K" | "none")
       - `confidence: str = "LOW"` (values: "HIGH", "MEDIUM", "LOW")
       - `as_of: str = ""` (period: "FY2025" | "TTM" | "2026-03-27")
       Use `model_config = ConfigDict(frozen=True)`.

    2. **CanonicalMetrics model** (frozen Pydantic BaseModel):
       All fields defaulting to `MetricValue()` (so missing data never crashes):
       - Identity: company_name, ticker, exchange, sic_code, sic_description, ceo_name
       - Financial: revenue, revenue_growth_yoy, net_income, total_assets, total_liabilities, total_debt, cash_and_equivalents, market_cap, shares_outstanding, employees
       - Market: stock_price, high_52w, low_52w, beta
       - Scoring: overall_score, tier

    3. **Helper `_sv(obj)` function**: Unwrap SourcedValue to primitive. If `isinstance(obj, SourcedValue)`, return `obj.value`. Import the existing `sv` from `_key_stats_helpers.py` or re-implement locally (3 lines). Must handle None input.

    4. **Helper `_xbrl_line_item(state, statement_type, field_names)` function**: Extract a value from XBRL financial statements. Navigate `state.extracted.financials.statements` (list of StatementData), find the latest period for `statement_type`, search `line_items` for any matching field name from `field_names` tuple. Return `(value, period_label)` or `(None, "")`. Read the existing `_extract_revenue` in `key_stats_context.py` to understand the XBRL navigation pattern -- replicate it as a generic helper.

    5. **Helper `_yfinance_info(state)` function**: Extract yfinance info dict from `state.acquired_data.market_data.get("info", {})`. Return empty dict if any path is None.

    6. **Individual resolver functions** for each metric. Each returns a `MetricValue`. Follow XBRL-first source priority from RESEARCH.md Section "Source Priority Chain":
       - `_resolve_revenue(state)`: XBRL income_statement (total_revenue/revenue/net_sales) > yfinance totalRevenue
       - `_resolve_net_income(state)`: XBRL income_statement (net_income/net_income_loss) > yfinance netIncomeToCommon
       - `_resolve_market_cap(state)`: state.company.market_cap (SourcedValue) > yfinance marketCap
       - `_resolve_stock_price(state)`: state.extracted.market.stock.current_price > yfinance currentPrice
       - `_resolve_employees(state)`: state.company.employee_count (SourcedValue) > yfinance fullTimeEmployees
       - `_resolve_exchange(state)`: state.company.identity.exchange (SourcedValue) > yfinance exchange
       - `_resolve_ceo_name(state)`: governance extraction ecd.ceo_name > scan officers list for CEO title
       - `_resolve_revenue_growth(state)`: derive from multi-period XBRL revenue > yfinance revenueGrowth
       - `_resolve_total_assets(state)`: XBRL balance_sheet > yfinance totalAssets
       - `_resolve_total_debt(state)`: XBRL balance_sheet > yfinance totalDebt
       - `_resolve_overall_score(state)`: state.scoring.overall_score
       - `_resolve_tier(state)`: state.scoring.tier
       - Identity resolvers (company_name, ticker, sic_code, sic_description): straightforward from state.company

    7. **`build_canonical_metrics(state: AnalysisState) -> CanonicalMetrics`**: Call all resolvers, return CanonicalMetrics. Wrap each resolver in try/except so one failure doesn't crash the whole registry. Log warnings for individual resolver failures.

    8. **Use `fmt_large_number` from `_key_stats_helpers.py`** (the em-dash variant) for formatting currency values. Import as `from do_uw.stages.render.context_builders._key_stats_helpers import fmt_large_number`. For employees, use comma-separated format (e.g., "142,000"). For percentages, use f"{val:.1f}%" format. For non-numeric metrics (company_name, ticker, exchange, ceo_name), `formatted` equals the string value.

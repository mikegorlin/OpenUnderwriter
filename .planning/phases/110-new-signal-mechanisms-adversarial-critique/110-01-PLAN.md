    **Step 3: Create mechanism_evaluators.py in stages/analyze/.**
    Three evaluator functions, each returns SignalResult:

    `evaluate_conjunction(sig, data, signal_results)`:
    - Read conjunction_rules from sig["evaluation"]["conjunction_rules"]
    - For each required_signal, look up status in signal_results dict
    - Match against signal_conditions (default: "TRIGGERED" means RED/YELLOW/TRIGGERED)
    - If matched >= minimum_matches, return TRIGGERED with level="red" and evidence listing matched signals
    - Otherwise return CLEAR
    - If >50% of component signals are SKIPPED, return SKIPPED (can't evaluate)

    `evaluate_absence(sig, data, signal_results, company)`:
    - Read absence_rules from sig["evaluation"]["absence_rules"]
    - For company_profile type: check company attributes to determine if disclosure expected (use condition string as documentation, implement specific attribute checks per signal)
    - Look up expected_signals in signal_results
    - If expected signal has data_status=DATA_UNAVAILABLE, return SKIPPED (didn't look)
    - If expected signal is SKIPPED with data_status=EVALUATED (looked but no data), return TRIGGERED
    - If expected signal is TRIGGERED or CLEAR, return CLEAR (disclosure present)
    - For always_expected type: fire if signal SKIPPED regardless of company profile

    `evaluate_contextual(sig, data, signal_results, company, pipeline_config)`:
    - Read contextual_rules from sig["evaluation"]["contextual_rules"]
    - Look up source_signal result in signal_results
    - Determine company context (lifecycle_stage, sector, size) from CompanyProfile using context_matrix.yaml classification rules
    - Apply threshold_adjustment: multiply the source signal's threshold by the adjustment factor
    - Re-evaluate the source signal's raw value against the adjusted threshold
    - Return TRIGGERED/CLEAR/YELLOW based on adjusted thresholds
    - If company context unavailable for required dimensions, return SKIPPED

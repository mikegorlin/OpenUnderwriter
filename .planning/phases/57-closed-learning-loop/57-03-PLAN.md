CONTEXT.md locked decisions for lifecycle transitions:
```
INCUBATING -> ACTIVE: 5+ runs, fire rate 5-80%, no feedback issues
ACTIVE -> MONITORING: fire rate >80% or <2% for 3+ consecutive runs, OR 3+ DISAGREE reactions
MONITORING -> ACTIVE: re-calibrated (fire rate back to 5-80%)
MONITORING -> DEPRECATED: fire rate still anomalous after 10 runs, OR confirmed by feedback
ACTIVE -> DEPRECATED: clearly broken
DEPRECATED -> ARCHIVED: 90+ days in DEPRECATED, no objections
DEPRECATED -> ACTIVE: un-deprecated
INCUBATING -> ARCHIVED: never worked
ARCHIVED -> nothing: terminal
```

**Enums/Constants:**
```python
from enum import StrEnum

class LifecycleState(StrEnum):
    INCUBATING = "INCUBATING"
    ACTIVE = "ACTIVE"
    MONITORING = "MONITORING"
    DEPRECATED = "DEPRECATED"
    ARCHIVED = "ARCHIVED"

VALID_TRANSITIONS: dict[LifecycleState, set[LifecycleState]] = {
    LifecycleState.INCUBATING: {LifecycleState.ACTIVE, LifecycleState.ARCHIVED},
    LifecycleState.ACTIVE: {LifecycleState.MONITORING, LifecycleState.DEPRECATED},
    LifecycleState.MONITORING: {LifecycleState.ACTIVE, LifecycleState.DEPRECATED},
    LifecycleState.DEPRECATED: {LifecycleState.ARCHIVED, LifecycleState.ACTIVE},
    LifecycleState.ARCHIVED: set(),  # Terminal
}

# Map legacy states to new states
LEGACY_STATE_MAP: dict[str, LifecycleState] = {
    "INACTIVE": LifecycleState.DEPRECATED,
    "RETIRED": LifecycleState.ARCHIVED,
    "ACTIVE": LifecycleState.ACTIVE,
}
```

**Pydantic models:**
- `TransitionProposal`: signal_id, current_state (LifecycleState), proposed_state (LifecycleState), reason (str), evidence (dict: fire_rate, total_runs, disagree_count, days_in_state, etc.), confidence (LOW|MEDIUM|HIGH)
- `LifecycleReport`: proposals (list[TransitionProposal]), total_signals_analyzed (int), by_state (dict[str, int] -- count per state), summary (str)

**Core functions:**
- `is_valid_transition(from_state: LifecycleState, to_state: LifecycleState) -> bool`: Check VALID_TRANSITIONS dict.
- `normalize_lifecycle_state(raw: str | None) -> LifecycleState`: Map raw YAML value to LifecycleState. None -> ACTIVE, "INACTIVE" -> DEPRECATED, "RETIRED" -> ARCHIVED. Unknown -> ACTIVE (safe default).
- `_get_signal_run_stats(conn, signal_id) -> dict`: Query brain_signal_runs for total_runs, fire_rate, last 3 run fire rates (for consecutive check), first_run_date (for age).
- `_get_disagree_count(conn, signal_id) -> int`: Query brain_feedback WHERE reaction='DISAGREE' AND signal_id=? AND status='PENDING'.
- `_get_days_in_state(conn, signal_id, current_state: str) -> int`: Query brain_changelog for last lifecycle_state change. If no record, use signal's first run date. Return days since that date.
- `evaluate_transition(conn, signal_id, current_state: LifecycleState, run_stats: dict, disagree_count: int, days_in_state: int) -> TransitionProposal | None`: Apply the locked transition criteria:
  - INCUBATING: if total_runs >= 5 AND 0.05 <= fire_rate <= 0.80 AND disagree_count == 0 -> propose ACTIVE
  - ACTIVE: if fire_rate > 0.80 or fire_rate < 0.02 (for 3+ consecutive runs) OR disagree_count >= 3 -> propose MONITORING
  - MONITORING: if fire_rate back in 0.05-0.80 range -> propose ACTIVE. Else if total_runs_since_monitoring >= 10 and still anomalous -> propose DEPRECATED.
  - DEPRECATED: if days_in_state >= 90 and no new feedback -> propose ARCHIVED
  - ARCHIVED: no transitions (terminal)
  Return None if no transition warranted.
- `generate_lifecycle_proposals(conn, proposals: list[TransitionProposal]) -> int`: For each proposal, INSERT into brain_proposals with source_type='CALIBRATION', proposal_type='LIFECYCLE_TRANSITION', proposed_changes={"from_state": X, "to_state": Y, "lifecycle_state": Y}, backtest_results containing evidence dict. Return count.
- `compute_lifecycle_proposals(conn) -> LifecycleReport`: Orchestrator. Load all signals via BrainLoader. For each, normalize lifecycle state, get run stats, get disagree count, get days in state, evaluate transition. Collect proposals. Generate proposals in DuckDB. Return report.

**Tests:** Create `tests/brain/test_brain_lifecycle_v2.py` using in-memory DuckDB fixture. Seed brain_signal_runs, brain_feedback, brain_changelog, brain_proposals with synthetic data for each scenario.

Co-occurrence mining SQL from research:
```sql
WITH fired AS (
    SELECT DISTINCT run_id, signal_id
    FROM brain_signal_runs
    WHERE status = 'TRIGGERED' AND is_backtest = FALSE
),
pairs AS (
    SELECT a.signal_id AS sig_a, b.signal_id AS sig_b,
           COUNT(DISTINCT a.run_id) AS co_fire_count,
           (SELECT COUNT(DISTINCT run_id) FROM fired WHERE signal_id = a.signal_id) AS a_total,
           (SELECT COUNT(DISTINCT run_id) FROM fired WHERE signal_id = b.signal_id) AS b_total
    FROM fired a
    JOIN fired b ON a.run_id = b.run_id AND a.signal_id < b.signal_id
    GROUP BY a.signal_id, b.signal_id
)
SELECT sig_a, sig_b, co_fire_count,
       ROUND(co_fire_count * 1.0 / LEAST(a_total, b_total), 3) AS co_fire_rate
FROM pairs
WHERE co_fire_count * 1.0 / LEAST(a_total, b_total) >= 0.70
ORDER BY co_fire_rate DESC;
```

**3. Create `src/do_uw/brain/brain_correlation.py` (~250-300 lines) with:**

**Pydantic models:**
- `CorrelatedPair`: signal_a, signal_b, co_fire_count, co_fire_rate, a_fire_count, b_fire_count, correlation_type ("potential_redundancy" | "risk_correlation")
- `RedundancyCluster`: prefix, signal_ids (list[str]), co_fire_rates (list[float]), recommendation (str)
- `CorrelationReport`: correlated_pairs (list[CorrelatedPair]), redundancy_clusters (list[RedundancyCluster]), total_pairs_analyzed (int), above_threshold_count (int), below_threshold_count (int), excluded_high_fire_rate (int), proposals_generated (int)

**Core functions:**
- `get_co_fire_threshold(brain_config_dir: Path | None = None) -> float`: Load threshold from brain/config/learning_config.json. Fall back to 0.70 if file missing.
- `_extract_prefix(signal_id: str) -> str`: Extract prefix from signal ID (e.g., "FIN.LIQ.current" -> "FIN.LIQ"). Split on "." and take first 2 segments.
- `_classify_correlation(sig_a: str, sig_b: str) -> str`: Return "potential_redundancy" if same prefix, "risk_correlation" if different prefix.
- `mine_cooccurrences(conn, threshold: float = 0.70, excluded_signal_ids: set[str] | None = None) -> list[CorrelatedPair]`: Execute the DuckDB cross-join SQL query from research. Pre-filter excluded_signal_ids by adding WHERE clause to exclude high fire rate signals. Return all pairs above threshold with classification. Also return below-threshold pairs separately for storage.
- `detect_redundancy_clusters(pairs: list[CorrelatedPair]) -> list[RedundancyCluster]`: Group same-prefix pairs ("potential_redundancy" type). If 3+ distinct signals in a prefix all appear in above-threshold pairs with each other, flag as redundancy cluster with recommendation "Consider consolidating N signals in {prefix} -- all co-fire >70%".
- `store_correlations(conn, pairs: list[CorrelatedPair]) -> None`: DELETE existing rows then INSERT discovered correlations to brain_correlations table. Store above-threshold pairs with above_threshold=TRUE.
- `generate_correlation_proposals(conn, pairs: list[CorrelatedPair]) -> int`: For each above-threshold pair, generate CORRELATION_ANNOTATION proposals in brain_proposals. proposed_changes = {"correlated_signals": [other_signal_id]}. One proposal per signal (both sides of the pair get a proposal listing the other). Return count generated.
- `compute_correlation_report(conn, threshold: float | None = None) -> CorrelationReport`: Orchestrator. Get threshold from config (or use parameter). Get effectiveness data via compute_effectiveness() to identify high-fire-rate signals (fire_rate > 0.80). Run mine_cooccurrences excluding high-fire signals. Detect redundancy clusters. Store correlations. Generate proposals. Return complete report.

    - Test: peer signal with both sector and overall percentiles -> evidence includes both

  <action>
    Add two new functions to mechanism_evaluators.py:

    evaluate_trend(sig, data, signal_results):
    - Extract field_key from sig["data_strategy"]["field_key"]
    - Get current_value from data[field_key]
    - Get prior_value: look for "{field_key}_prior" in data, or check for YoY comparison data pattern
    - For DISC.YOY signals: data comes from extracted.ten_k_yoy which has current/prior pairs
    - For FIN.TEMPORAL signals: data comes from XBRL quarterly metrics via _map_temporal_check
    - For FWRD.WARN signals: many will lack data (SKIPPED/DEFERRED candidates)
    - Calculate delta and pct_change
    - Compare against threshold from sig["evaluation"]["threshold"] (if present)
    - Return TRIGGERED if change exceeds threshold direction, CLEAR otherwise
    - Evidence dict includes: current_value, prior_value, delta, pct_change, direction ("improving"/"deteriorating"), period comparison
    - If data missing, return SKIPPED via _make_mechanism_skipped()

    evaluate_peer_comparison(sig, data, signal_results, *, benchmarks=None):
    - Extract field_key from sig["data_strategy"]["field_key"]
    - Get company_value from data[field_key]
    - Look up percentile from benchmarks (FramesPercentileResult dict)
    - The benchmarks parameter is passed from signal_engine when mechanism=peer_comparison
    - Compare percentile against threshold (default: 75th or 90th depending on signal direction)
    - Return TRIGGERED if company is an outlier vs peers, CLEAR otherwise
    - Evidence dict includes: company_value, overall_percentile, sector_percentile, metric_name
    - If benchmarks missing or metric not found, return SKIPPED

    Both functions follow the exact same signature/return pattern as evaluate_conjunction/evaluate_absence/evaluate_contextual.

    CRITICAL: These evaluators ENHANCE evaluation. Trend signals that previously evaluated via threshold fallthrough must NOT regress to SKIPPED. The trend evaluator should produce a result when data exists AND fall through to threshold evaluation if the trend-specific comparison can't be made but the base value exists.

    Extend tests/stages/analyze/test_mechanism_evaluators.py with the test cases listed in behavior block.

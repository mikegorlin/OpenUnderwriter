  <action>
Create `src/do_uw/cli_brain_trace.py` (~200-280 lines):

1. Import `brain_app` and `console` from `cli_brain`, plus chain_validator types.

2. Register `@brain_app.command("trace-chain")` with parameters:
   - `signal_id: Optional[str] = typer.Argument(None, help="Signal ID for detailed view")`
   - `json_out: Optional[Path] = typer.Option(None, "--json", help="Write JSON report to file")`

3. **Full table mode (no signal_id arg)**:
   - Call `validate_all_chains()`
   - Print summary Panel at top:
     ```
     Total: 476 | Complete: 135 | Broken: 341 | Inactive: 0
     Foundational: 20 complete, 0 broken
     ```
   - Print gap breakdown table:
     ```
     Gap Type               | Signals
     NO_ACQUISITION         | 42
     MISSING_FIELD_KEY      | 89
     NO_FIELD_ROUTING       | 15
     NO_EVALUATION          | 23
     NO_FACET               | 341
     FACET_NOT_IN_MANIFEST  | 12
     ```
   - Print full signal table with columns: Signal ID, Name, Status, Gaps
     - Status column: green checkmark for complete, red X for broken, dim "inactive" for inactive
     - Gaps column: comma-separated gap types (abbreviated: NO_ACQ, NO_FK, NO_FR, NO_EVAL, NO_FAC, NO_MAN)
     - Sort: broken first, then complete, then inactive
   - If INACTIVE signals exist, print separate dim table at bottom

4. **Single signal mode (signal_id arg)**:
   - Load signals, find the matching signal
   - If not found, print error and `raise typer.Exit(code=1)`
   - Call `validate_single_chain()` for that signal
   - Print vertical Panel showing each chain link:
     ```
     Signal: FIN.PROFIT.revenue
     Name: Revenue Trend Analysis
     Type: evaluate
     Status: BROKEN

     ACQUIRE  [green]OK[/green]  acquisition.sources[0].type = "xbrl"
     EXTRACT  [green]OK[/green]  data_strategy.field_key = "revenue"
     ANALYZE  [green]OK[/green]  evaluation.thresholds: 3 rules
     RENDER   [red]BROKEN[/red]  NO_FACET — not assigned to any facet
     ```

5. **JSON export (--json flag)**:
   - Run full validation (same as no-arg mode)
   - Serialize ChainReport to dict via `.model_dump()`
   - Convert ChainGapType enum values to strings
   - Write pretty-printed JSON to the specified path
   - Print confirmation: "Report written to {path}"

6. Register in `cli_brain.py`: Add `import do_uw.cli_brain_trace as _cli_brain_trace  # noqa: F401, E402` at the bottom of cli_brain.py, following the same pattern as cli_brain_apply.

Follow Rich formatting patterns from `cli_brain_health.py`: use `Panel`, `Table`, styled text with `[green]`, `[red]`, `[dim]`, `[bold]` markers. Use `console` from cli_brain (not creating a new one).

  <action>
Create `src/do_uw/brain/chain_validator.py` (~250-350 lines):

1. Define `ChainGapType` enum with 6 values: NO_ACQUISITION, MISSING_FIELD_KEY, NO_FIELD_ROUTING, NO_EVALUATION, NO_FACET, FACET_NOT_IN_MANIFEST

2. Define Pydantic models:
   - `ChainLink`: status (complete/broken/na), link_type (acquire/extract/analyze/render), detail (str describing what was found or missing)
   - `SignalChainResult`: signal_id, signal_name, signal_type ("evaluate"/"foundational"), chain_status ("complete"/"broken"/"inactive"), gaps (list[ChainGapType]), links (list[ChainLink])
   - `GapSummary`: gap_type (ChainGapType), signal_ids (list[str]), count (int)
   - `ChainReport`: total_signals, chain_complete, chain_broken, inactive_count, results (list[SignalChainResult]), gap_summary (list[GapSummary]), foundational_complete (int), foundational_broken (int)

3. Implement `validate_single_chain(signal: BrainSignalEntry, sections: dict[str, SectionSpec], manifest: OutputManifest, field_routing_keys: set[str]) -> SignalChainResult`:
   - **Acquire link**: Check signal.acquisition is not None (has sources), OR signal.acquisition_tier == "tier1", OR signal has a `data_strategy` with a recognized Tier 1 field key. If none, gap = NO_ACQUISITION.
   - **Extract link**: Check signal.data_strategy is not None and has a "field_key", OR signal_id is in field_routing_keys (FIELD_FOR_CHECK dict). If neither, gap = MISSING_FIELD_KEY.
   - **Analyze link** (evaluate signals only): Check signal.evaluation is not None (V2), OR signal.threshold has meaningful content (not just defaults). If neither, gap = NO_EVALUATION. For foundational signals, mark as N/A.
   - **Render link** (evaluate signals only): First check signal is assigned to a facet in any section YAML (search all sections' facets' signals lists). If not, gap = NO_FACET. If yes, check that facet ID exists in the output manifest. If not, gap = FACET_NOT_IN_MANIFEST. For foundational signals, mark as N/A.

4. Implement `validate_all_chains(signals_dir: Path | None = None, sections_dir: Path | None = None, manifest_path: Path | None = None) -> ChainReport`:
   - Load signals via `load_signals()`
   - Load sections via `load_all_sections()`
   - Load manifest via `load_manifest()`
   - Load FIELD_FOR_CHECK keys from `signal_field_routing.FIELD_FOR_CHECK`
   - Iterate all signals, call validate_single_chain for each
   - Separate INACTIVE signals (detect via provenance.status == "INACTIVE" or similar -- check actual signal YAML for status field)
   - Build gap_summary by grouping broken signals by their gap types
   - Return ChainReport with aggregate counts

5. Export: `validate_all_chains`, `validate_single_chain`, `ChainReport`, `SignalChainResult`, `ChainGapType`, `ChainLink`, `GapSummary`

For detecting INACTIVE: Check `signal.provenance` for a status or lifecycle field. If no explicit INACTIVE marker exists in signal schema, check if `work_type` or any other field indicates inactive status. Document the detection method in a comment.

For Tier 1 acquisition inference: The `acquisition_tier` field on BrainSignalEntry is the primary check. Also consider signals with BASE.* prefix as implicitly tier 1.

Create tests in `tests/brain/test_chain_validator.py` using pytest fixtures that create mock BrainSignalEntry, SectionSpec, and OutputManifest objects. Use `@pytest.fixture` for reusable test data. Tests should NOT require actual YAML files or DuckDB -- use constructed Pydantic models.

       Read officer forensic profiles from `state.extracted.governance.leadership.executives` (LeadershipForensicProfile objects). For each officer:

       - Collect personal_litigation from the profile's `prior_litigation` list
       - Build OfficerBackground objects
       Then call `query_officer_prior_sca()` with all prior company names in a single batch, `detect_serial_defendants()` to flag matches, and `assess_suitability()` for each.

    2. **build_shareholder_rights(state: AnalysisState) -> dict[str, Any]**
       Read from `state.extracted.governance.board` (BoardProfile). Build 8 provisions per D-07 using the RIGHTS_PROVISIONS mapping from research (board classification, poison pill, supermajority, proxy access, cumulative voting, written consent, special meeting, forum selection). For each:
       - Read SourcedValue field from BoardProfile, convert to Yes/No/N/A status
       - Look up defense_strength from provision definition (Protective vs Shareholder-Friendly)
       - Look up D&O implication from provision definition (yes_implication vs no_implication)
       Compute overall_defense_posture per D-09: count provisions with status="Yes" AND defense_strength="Protective". Strong >= 5, Moderate 3-4, Weak <= 2.
       Return dict: `shareholder_rights` (ShareholderRightsInventory as dict), `has_shareholder_rights` (bool).

       Read transactions from `state.extracted.market.insider_trading.transactions`. Get shares_outstanding from `state.extracted.financials.statements` XBRL data or `state.company.shares_outstanding` (try both, fallback to None per research open question 3). Call `aggregate_per_insider()` from officer_background.py. Format results for template: total_sold_usd as formatted currency string, pct_os as formatted percentage or "N/A", 10b5_1_badge as "10b5-1" or "Discretionary" per D-12.
       Return dict: `per_insider_activity` (list of formatted dicts), `has_per_insider_activity` (bool), `insider_count` (int).

  <action>
    Create `src/do_uw/stages/extract/officer_background.py` with these functions:

    1. **extract_prior_companies_from_bio(bio_text: str) -> list[PriorCompany]**
       Uses a simple regex-first approach for common patterns ("served as [ROLE] of [COMPANY] from [YEAR] to [YEAR]", "was [ROLE] at [COMPANY] ([YEARS])"). For complex bios, this will be called with LLM in future, but for now regex + pattern matching is the initial pass. Return empty list if no matches.

    2. **date_ranges_overlap(officer_start_year: int, officer_end_year: int, class_period_start: str, class_period_end: str) -> bool**
       Convert officer years to full ranges (start_year-01-01 to end_year-12-31 per research recommendation). Parse class period dates as YYYY-MM-DD. Return True if ANY overlap exists. Handle None/empty dates gracefully.

    3. **query_officer_prior_sca(prior_company_names: list[str]) -> list[dict[str, Any]]**
       Batch query Supabase for SCA filings at officer prior companies. Use `or=()` filter with `company_name.ilike` matching (per D-05, same pattern as query_peer_sca_filings). Split into batches of 20 if more than 20 companies. Return raw Supabase results with class_period_start/end.

    4. **detect_serial_defendants(officers: list[OfficerBackground], sca_results: list[dict]) -> list[OfficerBackground]**
       For each officer's prior_companies, check if any SCA result matches the company name AND the date range overlaps with officer tenure. If match, set is_serial_defendant=True, populate sca_exposures with OfficerSCAExposure records (per D-04, D-06).

    5. **assess_suitability(officer: OfficerBackground, has_full_bio: bool, has_litigation_search: bool) -> tuple[str, str]**
       Returns (suitability_level, reason). Per D-03: HIGH = full bio + litigation search done + cross-validated, MEDIUM = partial data, LOW = minimal bio. This is a data completeness indicator.

    6. **aggregate_per_insider(transactions: list[InsiderTransaction], shares_outstanding: float | None) -> list[PerInsiderActivity]**
       Group transactions by insider_name. For each insider: sum sell total_value, count sell transactions, detect if ANY sell has is_10b5_1=True, find earliest/latest transaction_date. Calculate pct_os if shares_outstanding available. Sort by total_sold_usd descending (per D-11). Skip compensation-type codes (A=award, G=gift, M=exercise per existing COMPENSATION_CODES pattern).

    Import PerInsiderActivity, OfficerBackground, PriorCompany, OfficerSCAExposure from governance_intelligence models. Import InsiderTransaction from market_events. Use safe_float() for all numeric values from SourcedValue fields. Use httpx for Supabase queries (follow supabase_litigation.py pattern).

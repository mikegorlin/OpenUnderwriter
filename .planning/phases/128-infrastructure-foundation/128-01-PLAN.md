The audit appendix currently has THREE overlapping audit views:
1. **Signal disposition audit** (audit.py): Shows signal status (TRIGGERED/CLEAN/SKIPPED/INACTIVE) by section
2. **Render audit** (render_audit.py): Shows data extraction coverage (excluded fields, unrendered fields)
3. **Health issues** (render_audit.py): Shows data quality warnings

These overlap significantly -- a signal that's TRIGGERED appears in disposition audit AND may appear in render audit's "rendered fields" list. Fields appear in both "extracted" and "signal" categories.

**Deduplication approach -- merge into a unified audit context:**

1. Update `build_audit_context()` in audit.py to accept BOTH disposition_summary AND render_audit as parameters:
   ```python
   def build_audit_context(
       disposition_summary: dict[str, Any] | None,
       render_audit: RenderAuditReport | None = None,
   ) -> dict[str, Any]:
   ```

2. Add a `audit_unified_summary` key that provides:
   - Total signals checked (from disposition)
   - Total data fields extracted (from render audit)
   - Coverage percentage (from render audit)
   - Single combined "issues" list that deduplicates:
     - SKIPPED signals AND unrendered fields referring to same data path get merged into one entry
     - TRIGGERED signals that also appear as health_issues get merged
   - Dedup logic: match on signal_id prefix against field path prefix (e.g., signal `FIN.revenue_growth` maps to field path `extracted.financials.revenue`)

3. Keep the individual audit_* keys for backward compat but add `audit_dedup_savings` showing how many entries were consolidated.

4. Update the assembly code (in assembly_dossier.py from Task 1) to pass both disposition_summary and render_audit to the unified build_audit_context. Currently they are called separately at lines 254-259 and 624-638.

5. Write test_audit_dedup.py verifying:
   - Passing both inputs produces `audit_unified_summary`
   - Duplicate entries are consolidated (count < sum of individual counts)
   - Backward-compat keys still present

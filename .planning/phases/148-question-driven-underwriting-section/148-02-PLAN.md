    Create `answerers/sca_questions.py` with `generate_sca_questions(state, ctx)` function that returns a list of question dicts ready for slotting into domains.

    **Data extraction:** Access risk card from `ctx.get("litigation", {})`:
    - `risk_card_filing_history` -> filing_history
    - `risk_card_repeat_filer` -> repeat_filer_detail
    - `risk_card_scenario_benchmarks` -> scenario_benchmarks (if available in ctx)

    Fallback: access via `state.acquired_data.litigation_data.risk_card` if ctx keys not available.

    **Four SCA question types per D-05:**

    **Type 1: Filing frequency & recidivism -> Litigation domain (SCA-LIT-01)**
    Always generated (even for no-history companies per Pitfall 5).
    - With history: "REPEAT filer -- {n} SCA filings. Settlement rate: {rate}%. Total exposure: ${exposure}M."
    - No history: "FIRST_TIME filer -- no SCA history. Clean litigation record." -> UPGRADE verdict.
    - Question text: "What is the company's SCA filing frequency and recidivism category?"

    **Type 2: Settlement ranges & severity -> Litigation domain (SCA-LIT-{SCENARIO})**
    One per scenario in `scenario_benchmarks`:
    - "For {scenario} cases, median settlement is ${median}M (P25: ${p25}M, P75: ${p75}M, P90: ${p90}M). {n_settlements} comparable settlements from {total_filings} filings."
    - DOWNGRADE if company has this scenario in history; NEUTRAL if informational only.
    - Question ID: `SCA-LIT-{scenario[:8].upper()}` (e.g., SCA-LIT-ACCOUNTI)

    **Type 3: Peer SCA comparison -> Market domain (SCA-MKT-01)**
    Generated if scenario_benchmarks exist:
    - "Sector has {total_filings} SCA filings for {scenario}. Dismissal rate: {dismissal}%."
    - Include avg_stock_drop_pct: "Average stock drop in matching cases: {drop}%"
    - DOWNGRADE if company's scenario count exceeds sector average.

    **Type 4: Trigger pattern matching -> Operational domain (SCA-OPS-01)**
    Generated if scenario_benchmarks have multiplier data:
    - "Company matches {scenario} pattern. SEC investigation multiplier: {sec_inv}x. Restatement multiplier: {restatement}x."
    - DOWNGRADE if any multiplier > 5x (indicates elevated severity).

    **All generated questions must include:**
    - `source: "SCA Data"` — for template badge
    - `domain: "litigation_claims"` or `"stock_market"` or `"operational_emerging"` — for slotting
    - `data_sources: ["supabase_sca"]`
    - Full answer fields (answer, evidence, verdict, confidence, data_found)
    - Evidence list with specific numbers

    **Also add `answer_sca_question(q, state, ctx)` function** that re-answers any SCA question from current data (for the case where questions are pre-generated but need refreshing). This function reads `_risk_card` or `_benchmark` from the question dict.

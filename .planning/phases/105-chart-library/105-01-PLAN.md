<objective>
Create a chart style registry YAML and reusable chart component generators that enforce consistent styling across all 15 chart types, replacing ad-hoc matplotlib formatting scattered across 9 chart files.

Purpose: Currently every chart file hardcodes its own colors (e.g., "#1A1446", "#FFD000"), font sizes, grid styles, header layouts, and spine formatting -- the same patterns repeated 9+ times. This makes visual consistency fragile and style changes require editing every file. A centralized style registry + component library solves both problems.

Output: chart_styles.yaml defining canonical styles for every chart type, a Pydantic-validated loader, reusable component generators, and all existing chart renderers refactored to use them.
</objective>

    ```python
    def create_styled_figure(chart_type: str, format: str = "png") -> tuple[Figure, dict[str, str]]:
        """Create a matplotlib figure with style registry dimensions and colors.
        Returns (fig, colors_dict) so the caller has resolved colors.
        SVG format always gets light theme (existing behavior preserved)."""

    def create_styled_header(ax, title: str, stats_items: list[tuple[str, str, str]], chart_type: str) -> None:
        """Render a stats header bar above a chart. Replaces 4 separate _render_*_header functions
        across stock_charts, drawdown_chart, volatility_chart."""

    def apply_styled_axis(ax, chart_type: str, period: str | None = None, show_xlabel: bool = True) -> None:
        """Apply axis styling from registry: spines, grid, tick params, date formatting.
        Replaces _apply_bloomberg_style, _apply_dd_style, _apply_axis_style."""

    def create_styled_legend(ax, chart_type: str, loc: str = "upper left", additional_ax: Any = None) -> None:
        """Create legend with registry-specified colors and positioning.
        Replaces 4+ inline legend calls."""
    ```

    2. Refactor ALL chart renderers to use chart_components instead of inline formatting. The refactoring pattern for each file:

    **CRITICAL: Do NOT change chart output appearance.** This is a refactoring. The same pixels must come out. Only the source of formatting values changes (from hardcoded to registry).

    For each chart file:
    - Replace hardcoded color dicts with `resolve_colors(chart_type, format)`
    - Replace `plt.figure(figsize=..., dpi=...)` with `create_styled_figure(chart_type, format)`
    - Replace inline header rendering with `create_styled_header(...)` where the header pattern matches (stock, drawdown, volatility all share the same header pattern)
    - Replace inline axis styling with `apply_styled_axis(...)`
    - Replace inline legend calls with `create_styled_legend(...)`
    - Keep chart-specific data logic (computations, annotations) untouched

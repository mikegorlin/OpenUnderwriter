**Evaluation mechanism classification guide:**

| Mechanism | When to use | Signal examples |
|-----------|------------|-----------------|
| `threshold` | Signal compared against fixed numeric/boolean cutoff | FIN.ACCT.auditor (opinion type), FIN.LIQ.current_ratio (>1.0), LIT.SCA.active (boolean) |
| `peer_comparison` | Signal compared against peer group metrics | FIN.PEER.* (XBRL peer benchmarks), STOCK.PRICE.returns_multi_horizon (vs sector) |
| `trend` | Signal tracks directional change over time | FIN.TEMPORAL.* (quarterly trends), DISC.YOY.* (year-over-year changes), FWRD.WARN.* (deterioration) |
| `conjunction` | Signal only meaningful when combined with others | Future use -- placeholder for Phase 110. Most signals are NOT conjunction yet. |
| `absence` | Risk comes from MISSING data/disclosure | Future use -- placeholder for Phase 110. Signals that fire when expected data is absent. |
| `contextual` | Interpretation depends on company context | Future use -- placeholder for Phase 110. Same data = different risk by company type. |

**IMPORTANT:** Most existing signals are `threshold` (simple cutoff). Use `peer_comparison` for signals with peer benchmark data. Use `trend` for temporal/YOY signals. The `conjunction`, `absence`, and `contextual` mechanisms are Phase 110 features -- do NOT assign them to existing signals unless the signal's current evaluation logic genuinely uses that pattern.

**Classification rules for mechanism:**

- `threshold`: Default for most signals. Any signal that compares a value against a fixed cutoff (numeric threshold, boolean check, tiered red/yellow/clear). This covers ~80% of signals.
- `peer_comparison`: Signals in `fin/peer_xbrl.yaml`, any signal with "peer" in its evaluation logic, STOCK.PRICE signals that compare against sector/market benchmarks, BIZ.COMP.* competitive position signals.
- `trend`: Signals in `fin/temporal.yaml`, `disc/ten_k_yoy.yaml`, FWRD.WARN.* (deterioration warnings), any signal tracking directional change over time periods. Key indicator: signal has `window_years` field or temporal keywords in formula.
- Do NOT use `conjunction`, `absence`, or `contextual` for any existing signal -- those are Phase 110 mechanisms.

# Phase 127 Plan 01: Self-Review Loop

## Objective
Build an automated post-pipeline HTML quality auditor that produces structured JSON reports and integrates with the CLI via --review flag.

## Context
- Existing QA infrastructure: `src/do_uw/validation/qa_report.py` and `qa_content.py`
- Boilerplate patterns: `src/do_uw/stages/render/formatters.py` (_BOILERPLATE_RX)
- CLI: `src/do_uw/cli.py` with typer

## Tasks

### Task 1: Create self_review.py module
type="auto"
Create `src/do_uw/stages/render/self_review.py` implementing:
- REVIEW-01: HTML audit (section count, N/A count, empty sections, boilerplate, data consistency)
- REVIEW-02: Structured JSON report with per-section scores
- REVIEW-04: Specific checks (LLM refusals, double-encoding, empty red flags, DDL discrepancy)

### Task 2: Add --review CLI flag
type="auto"
Wire --review into cli.py analyze command to run self-review after pipeline.

### Task 3: Write tests
type="auto"
Comprehensive tests for all review checks.

## Verification
- `uv run pytest tests/test_self_review.py -v` passes
- Module imports cleanly
- --review flag registered in CLI

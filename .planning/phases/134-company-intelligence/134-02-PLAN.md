  <action>
    Create `src/do_uw/stages/render/context_builders/_company_intelligence.py` following the `_market_volume.py` pattern. Pure data formatters -- no evaluative logic, no D&O commentary generation. Each function reads from state and returns a dict.

    **6 builder functions:**

    1. `build_risk_factor_review(state: AnalysisState) -> dict[str, Any]` (per D-01, D-02, D-03, D-04):
       - Reads `state.company.risk_factor_profiles` (list of RiskFactorProfile)
       - Reads `state.extracted.ten_k_comparison.risk_factor_changes` for YoY deltas
       - Calls `classify_risk_factors()` from Plan 01 to populate classification fields
       - Returns `{"risk_factor_review": [...rows...], "has_risk_factor_review": bool, "risk_factor_summary": {"total": N, "novel": N, "elevated": N, "high_severity": N}}`
       - Each row: `{"title": str, "classification": str, "severity": str, "yoy_delta": str, "do_implication": str, "source_passage": str, "category": str}`
       - Sorted by: ELEVATED first, then NOVEL, then STANDARD. Within each: HIGH severity first.

    2. `build_peer_sca_contagion(state: AnalysisState) -> dict[str, Any]` (per D-06, D-08):
       - Reads peer group from `state.extracted.financials.peer_group.peers` (if exists)
       - Calls `query_peer_sca_filings()` from Plan 01 with peer tickers (limit 6 peers)
       - Returns `{"peer_sca_records": [...], "has_peer_sca": bool, "peer_sca_summary": "N active SCAs among M peers analyzed", "peer_profiles": [...]}`
       - peer_profiles enriches existing competitive landscape peers with SCA data
       - Handles empty results gracefully per Pitfall 3: "No active SCAs detected among N peers" as a positive signal

    3. `build_concentration_assessment(state: AnalysisState) -> dict[str, Any]` (per D-09, D-11):
       - Reads from multiple state paths:
         - Customer: `state.company.customer_concentration` + `state.dossier.competitive_landscape`
         - Geographic: `state.company.geographic_footprint` + XBRL segment data
         - Product: `state.company.revenue_segments`
         - Channel: derive from business description or default to "Not Assessed"
       - Thresholds per Pitfall 4: Customer HIGH = top customer >15% revenue, Geographic HIGH = single region >80%, Product HIGH = single segment >60%
       - Returns `{"concentration_dims": [4 ConcentrationDimension dicts], "has_concentration": True, "concentration_risk_level": "HIGH"/"MEDIUM"/"LOW" (worst dimension)}`
       - Use `safe_float()` for all numeric comparisons

    4. `build_supply_chain_context(state: AnalysisState) -> dict[str, Any]` (per D-10):
       - Reads 10-K Item 1 and Item 1A text from state (raw filing text if available, else from extracted text fields)
       - Calls `extract_supply_chain()` from Plan 01
       - Returns `{"supply_chain_deps": [...], "has_supply_chain": bool, "supply_chain_risk": "HIGH"/"MEDIUM"/"LOW"}`
       - If no 10-K text available, returns empty dict with `has_supply_chain: False`

    5. `build_sector_do_concerns(state: AnalysisState) -> dict[str, Any]` (per D-07):
       - Loads `config/sector_do_concerns.json`
       - Matches company SIC code to sector via `sic_ranges`
       - Returns `{"sector_concerns": [...], "has_sector_concerns": bool, "matched_sector": "Technology"}`
       - Each concern dict: `{"concern": str, "sector_relevance": str, "company_exposure": str, "do_implication": str}`
       - company_exposure derived from signal results if available, else defaults to sector_relevance

    6. `build_regulatory_map(state: AnalysisState) -> dict[str, Any]` (per D-12, D-13):
       - Reads `state.extracted.regulatory_proceedings` (from existing regulatory_extract.py)
       - Reads environment assessment signals from `company_environment.py` context
       - Returns `{"regulatory_map": [...], "has_regulatory_map": bool, "regulatory_count": N}`
       - Each row: `{"agency": str, "jurisdiction": str, "exposure_level": str, "current_status": str, "risk_level": str, "description": str}`
       - If no regulatory proceedings found, returns `has_regulatory_map: False`

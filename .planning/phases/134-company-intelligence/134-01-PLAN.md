    1. Create `src/do_uw/stages/extract/risk_factor_classify.py` (per D-01, D-02, D-03):
       - `classify_risk_factors(factors: list[RiskFactorProfile], prior_factors: list[RiskFactorProfile] | None = None) -> list[RiskFactorProfile]`
       - Classification logic (deterministic, no LLM needed for phase 134):
         - `is_new_this_year=True` => `classification="NOVEL"`
         - `severity="HIGH"` AND (matched to a prior factor with lower severity, OR no prior match) => `classification="ELEVATED"`
         - All others => `classification="STANDARD"`
       - Uses `difflib.SequenceMatcher` (same as ten_k_yoy.py) with threshold 0.6 to match current vs prior factor titles
       - Sets `do_implication` based on category mapping: LITIGATION -> "Direct SCA trigger — company acknowledges litigation risk", REGULATORY -> "Regulatory action creates Section 10b-5 exposure if undisclosed", FINANCIAL -> "Financial deterioration triggers revenue miss SCA theory", CYBER -> "Data breach creates CCPA/privacy SCA exposure", etc.
       - Returns the same list with classification and do_implication fields populated

    2. Create `src/do_uw/stages/extract/supply_chain_extract.py` (per D-10):
       - `extract_supply_chain(item1_text: str, item1a_text: str, company_name: str = "") -> list[SupplyChainDependency]`
       - Uses regex patterns to find supplier/vendor mentions in Item 1 and Item 1A text
       - Patterns: "sole source", "single supplier", "key supplier", "depend on", "rely on.*supplier", "principal supplier"
       - For each match, extract context window (+/- 200 chars), determine dependency_type from keywords
       - switching_cost derived from language: "no alternative" / "limited alternatives" -> HIGH, "multiple sources" -> LOW
       - do_exposure derived from dependency_type: sole-source -> "Supply disruption creates revenue miss SCA if management failed to disclose concentration"
       - Returns list of SupplyChainDependency models
       - If no text provided, returns empty list

    3. Extend `supabase_litigation.py` with `query_peer_sca_filings` (per D-06, addressing Pitfall 1):

       - Returns list of PeerSCARecord models (from company_intelligence.py)
       - Logs warning (not error) when SUPABASE_KEY missing: "Supabase key not available — peer SCA contagion data will be empty"
       - Empty tickers list returns [] immediately

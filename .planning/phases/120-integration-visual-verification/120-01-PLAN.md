**Layer 2: `scripts/do_context_coverage.py`**
- Standalone script (runnable via `uv run python scripts/do_context_coverage.py`)
- Scans same templates as the pytest gate
- Prints detailed report: template path, evaluative columns found, whether each cell references a do_context variable, the variable name if found
- Prints summary: total templates scanned, covered count, uncovered count, coverage percentage
- Exit code 0 if 100% covered, 1 otherwise
- Uses `brain_unified_loader.load_signals()` to cross-reference: for each do_context variable found, trace back to whether the signal ID has a `presentation.do_context` block in YAML

Note: The 26 templates with evaluative columns are listed in the interfaces block above. The cell content patterns to look for are Jinja2 variable references like `{{ row.do_risk }}`, `{{ item.do_context }}`, `{{ do_implication }}`, etc. Some templates use macro calls like `{{ check_summary(...) }}` which embed do_context internally — those count as covered.

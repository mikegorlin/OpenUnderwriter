    **1. Add print rules for new components** in `components.css` (NOT `styles.css` — all new CSS goes in the split file). Add a `@media print` block at the end of `components.css` for component-specific print rules. The existing `@media print` block in `styles.css` stays as-is for its existing rules (page size, typography, no-print).

    **IMPORTANT — Playwright compatibility:** Use BOTH `break-before: page` (CSS Fragmentation L3) AND `page-break-before: always` (CSS 2.1 legacy) for all page break rules. Playwright's Chromium PDF renderer respects the legacy property more reliably. The `section.page-break` class already exists on sections — these rules are ADDITIVE (using both the class selector and id selectors for specificity).

    **In `components.css`**, add:

    ```css
    /* =====================================================================
       Print styles for new components (collapsible, sticky, badges, layout)
       ===================================================================== */
    @media print {
      /* Force all collapsible sections open in print */
      details.collapsible {
        display: block !important;
        border: none;
      }
      details.collapsible > summary {
        display: none !important;  /* hide toggle summary in print */
      }
      details.collapsible > .collapsible-content {
        display: block !important;
        padding: 0;
      }

      /* Force page breaks at section boundaries.
         Use BOTH break-before (modern) and page-break-before (legacy)
         for Playwright Chromium compatibility. */
      section.page-break,
      section[id="financial-health"],
      section[id="market"],
      section[id="governance"],
      section[id="litigation"],
      section[id="ai-risk"],
      section[id="scoring"] {
        break-before: page;
        page-break-before: always;
      }

      /* Never break before executive summary (keep with identity) */
      section[id="executive-summary"] {
        break-before: auto;
        page-break-before: auto;
      }

      /* Sticky headers become static in print */
      .sticky-header thead th {
        position: static;
        box-shadow: none;
      }

      /* Two-column layout stacks in print for readability */
      .two-col-profile {
        grid-template-columns: 1fr;
        gap: 0.5rem;
      }

      /* Ensure tables don't split awkwardly */
      .kv-table,
      .kv-paired,
      .data-grid-table {
        break-inside: avoid;
        page-break-inside: avoid;
      }

      /* Badge hover effects disabled in print */
      .badge-pill,
      .badge-tier {
        box-shadow: none !important;
        transform: none !important;
      }
    }
    ```

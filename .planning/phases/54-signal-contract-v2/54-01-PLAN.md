## Objective

Extend BrainSignalEntry with three optional V2 sub-model sections (`acquisition`, `evaluation`, `presentation`) plus a `schema_version` field. Add a dispatch stub in signal_engine.py that detects V2 signals and falls through to the legacy evaluation path. All 400 existing signals must load unchanged. No pipeline behavior changes.

## Context

Read these files before starting:

- `@src/do_uw/brain/brain_signal_schema.py` -- Current BrainSignalEntry model (158 lines). V2 sub-models extend this file.
- `@src/do_uw/brain/brain_unified_loader.py` -- _load_and_validate_signals() where Pydantic validation happens. No changes needed if V2 fields are Optional.
- `@src/do_uw/stages/analyze/signal_engine.py` -- execute_signals() dispatch loop (lines 82-133). schema_version check goes here before content_type dispatch.
- `@src/do_uw/brain/brain_enrichment.py` -- enrich_signal() adds backward-compat fields. V2 fields need NO enrichment (they are explicit in YAML, per user decision).

## Locked Decisions (from CONTEXT.md -- NON-NEGOTIABLE)

- V2 sub-models use `extra='forbid'` (catches YAML typos). BrainSignalEntry stays `extra='allow'`.
- Parallel threshold fields: English `threshold.red/yellow/clear` kept as-is. `evaluation.thresholds` added as structured `[{op, value, label}]`. Both coexist.
- Presentation extends, doesn't replace: `presentation` adds `detail_levels` and `context_templates` alongside existing `display` (DisplaySpec) and `facet` fields.
- snake_case naming throughout for V2 fields.
- Dispatch stub only: V2 signals fall through to legacy path. Returns None to signal fallthrough. Phase 55 fills in real V2 evaluator.
- Dispatch in signal_engine.py: In execute_signals() before content_type dispatch (not in evaluate_signal()).
- No enrichment for V2 fields -- they are explicit in YAML.

## Tasks

<task id="1" title="Define V2 Pydantic sub-models on BrainSignalEntry">

### What
Add 6 new Pydantic models to `brain_signal_schema.py` and 4 new Optional fields to `BrainSignalEntry`:

**New models (all with `extra='forbid'`):**
1. `AcquisitionSource` -- type (str), fields (list[str], dotted paths), fallback_to (str | None)
2. `AcquisitionSpec` -- sources (list[AcquisitionSource])
3. `EvaluationThreshold` -- op (str, one of `<, >, <=, >=, ==, !=, between, contains`), value (float | str | list[float]), label (str, one of `RED, YELLOW, CLEAR`)
4. `EvaluationSpec` -- formula (str | None), thresholds (list[EvaluationThreshold]), window_years (int | None)
5. `PresentationDetailLevel` -- level (str, one of `glance, standard, deep`), template (str), fields (list[str])
6. `PresentationSpec` -- detail_levels (list[PresentationDetailLevel]), context_templates (dict[str, str])

**New fields on BrainSignalEntry:**
- `schema_version: int = Field(default=1, description="1=legacy, 2=V2 declarative")`
- `acquisition: AcquisitionSpec | None = None`
- `evaluation: EvaluationSpec | None = None`
- `presentation: PresentationSpec | None = None`

### How
1. Add the 6 new model classes ABOVE BrainSignalEntry in brain_signal_schema.py (after BrainSignalProvenance, before BrainSignalEntry)
2. Add the 4 new fields to BrainSignalEntry as Optional with defaults
3. All V2 sub-models use `model_config = ConfigDict(extra="forbid")` -- this is critical for catching YAML typos
4. BrainSignalEntry keeps `model_config = {"extra": "allow"}` -- unchanged

<task id="2" title="Add schema_version dispatch stub in signal_engine.py">

### What
Add a `_evaluate_v2_stub()` function and a schema_version check in `execute_signals()` that dispatches V2 signals to the stub, which falls through to the legacy evaluation path.

### How
1. Add `_evaluate_v2_stub(sig, data)` function that logs a debug message and returns `None` (None = use legacy path)
2. In execute_signals(), after `data = map_signal_data(...)` and before the `content_type = sig.get(...)` line (around line 109), add the V2 dispatch block. **IMPORTANT:** The `content_type` variable is assigned AFTER this insertion point (line 109). The V2 branch must use `sig.get("content_type", "EVALUATIVE_CHECK")` inline to avoid a NameError when Phase 55 makes the stub return non-None:
```python
# V2 dispatch: schema_version 2 signals go through stub (Phase 55 fills in real evaluator)
if sig.get("schema_version", 1) >= 2:
    v2_result = _evaluate_v2_stub(sig, data)
    if v2_result is not None:
        # Phase 55: V2 evaluator returned a result, use it
        result = v2_result
        result.content_type = sig.get("content_type", "EVALUATIVE_CHECK")
        _determine_data_status(sig, data, result)
        results.append(result)
        continue
    # Stub returned None: fall through to legacy evaluation
```
3. The stub MUST return None in Phase 54 -- this ensures identical pipeline behavior
4. Add `_evaluate_v2_stub` to the module but NOT to `__all__` (it's internal)

### Verification
- `uv run pytest tests/brain/ -x` -- all brain tests pass
- `uv run pytest tests/stages/analyze/ -x` -- all analyze tests pass
- The stub is inert: create a test with a signal dict that has `schema_version: 2` and confirm it produces the same result as `schema_version: 1` (legacy path) by running both through evaluate_signal()

</task>

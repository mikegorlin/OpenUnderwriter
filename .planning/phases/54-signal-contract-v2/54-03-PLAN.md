**For each signal, first read the actual YAML file to find the exact signal entry.** The file names above are best guesses from the research -- verify by reading the actual files. Use `Glob` to find the right YAML file if the path doesn't exist.

**For each signal, add these V2 fields after the existing fields:**
```yaml
  # V2 additions:
  schema_version: 2
  acquisition:
    sources:
      - type: <primary_source from required_data>
        fields:
          - <dotted path from field_registry.yaml>
        fallback_to: <next source or null>
  evaluation:
    formula: <field_key from data_strategy.field_key>
    thresholds:
      - op: "<operator>"
        value: <numeric value>
        label: RED
      - op: "<operator>"
        value: <numeric value>
        label: YELLOW
    window_years: <1 or null>
  presentation:
    detail_levels:
      - level: glance
        template: "<short display string>"
      - level: standard
        template: "<standard display with context>"
      - level: deep
        fields:
          - <related field keys>
    context_templates:
      TRIGGERED: "<contextual message when triggered>"
      CLEAR: "<contextual message when clear>"
```

**Threshold conversion rules** (convert English threshold text to structured operators):
- `<1.0 current ratio` -> `{op: "<", value: 1.0, label: "RED"}`
- `<1.5 current ratio` -> `{op: "<", value: 1.5, label: "YELLOW"}`
- `>10% drop` -> `{op: ">", value: 10.0, label: "RED"}`
- `>0 filings` -> `{op: ">", value: 0, label: "RED"}`
- First-match ordering: RED thresholds listed before YELLOW (matches legacy try_numeric_compare behavior)

**IMPORTANT:** Use ruamel.yaml for in-place editing to preserve comments and formatting. If ruamel.yaml is impractical for targeted edits (it can reformat the file), it is acceptable to edit via the Edit tool by carefully adding V2 fields after the last existing field of each signal entry. Verify with git diff that only V2 fields were added, no V1 field reformatting.

<task id="2" title="Update CLI for V2 visibility">

### What
Update `brain stats` and `brain build` commands to report V2 migration progress.

### How

**brain stats** (cli_brain.py `status()` command):
After the existing "Total signals" line, add:
```python
# V2 migration progress
v2_count = sum(1 for sig in signals if sig.get("schema_version", 1) >= 2)
console.print(f"  V2 signals: [bold]{v2_count}[/bold]/{total_active} ({v2_count*100//total_active}%)")

# Field registry count

    from do_uw.brain.field_registry import load_field_registry
    registry = load_field_registry()
    console.print(f"  Field registry: [bold]{len(registry.fields)}[/bold] fields mapped")
except Exception:
    console.print("  [dim]Field registry not available[/dim]")
```

**brain build** (cli_brain.py `build()` command and brain_build_signals.py):

   - For each V2 signal, validate that `acquisition`, `evaluation`, and `presentation` are present and non-None
   - Report V2 count in the return dict: `"v2_signals": v2_count`
2. In `cli_brain.py` `build()`, after printing coverage, add:

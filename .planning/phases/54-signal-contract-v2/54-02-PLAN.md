## Objective

Create `brain/field_registry.yaml` mapping logical field names to ExtractedData/CompanyProfile dotted paths. Fields classified as DIRECT_LOOKUP (simple path traversal) or COMPUTED (named Python function). Create a Python module to load and validate the registry. The registry coexists with FIELD_FOR_CHECK -- zero deletion of legacy code in this plan.

<task id="2" title="Create field_registry.py loader module">

### What
Create `src/do_uw/brain/field_registry.py` to load, validate, and cache the field registry YAML.

### How
1. Create `src/do_uw/brain/field_registry.py` with:
   - Pydantic model `FieldRegistryEntry` with:
     - `type: Literal["DIRECT_LOOKUP", "COMPUTED"]`
     - `path: str | None = None` -- required for DIRECT_LOOKUP
     - `function: str | None = None` -- required for COMPUTED
     - `args: list[str] = Field(default_factory=list)` -- dotted paths as function args
     - `description: str = ""`
     - Use `model_config = ConfigDict(extra="forbid")`
   - Pydantic model `FieldRegistry`:
     - `version: int`
     - `fields: dict[str, FieldRegistryEntry]`
     - Use `model_config = ConfigDict(extra="forbid")`
   - Module-level cached loader function `load_field_registry() -> FieldRegistry`
     - Reads from `brain/field_registry.yaml` with CSafeLoader
     - Validates with Pydantic FieldRegistry model
     - Caches result in module-level variable
     - Returns validated FieldRegistry
   - `_reset_cache()` for testing
   - `get_field_entry(field_key: str) -> FieldRegistryEntry | None` convenience function
   - Pydantic validation: DIRECT_LOOKUP entries must have `path`, COMPUTED entries must have `function`. Use a `model_validator` on FieldRegistryEntry to enforce this.
2. File should be under 150 lines (simple loader + validation)
3. Do NOT wire this into the pipeline -- it's a standalone data layer. Phase 55 will integrate it.

### Verification
- `python -c "from do_uw.brain.field_registry import load_field_registry; r = load_field_registry(); print(f'{len(r.fields)} fields loaded')"` succeeds
- Verify `do_uw.brain.field_registry` is importable (check that `__init__.py` in `brain/` exports the module or that Python's implicit namespace packages resolve it -- the existing `brain/__init__.py` should already suffice)
- DIRECT_LOOKUP entries have path, COMPUTED entries have function (Pydantic validator enforces)
- Invalid YAML (missing required fields) raises ValidationError
- Loader caches results (second call returns same object)
- File under 150 lines

</task>

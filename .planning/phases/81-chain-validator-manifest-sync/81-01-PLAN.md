  <name>Task 1: Refactor _build_facet_signal_map to use manifest and remove NO_FIELD_ROUTING</name>
  <files>src/do_uw/brain/chain_validator.py</files>
  <action>
  1. **Refactor `_build_facet_signal_map`** to accept `OutputManifest` instead of `dict[str, Any]` (sections). Build the `{facet_id: set(signal_ids)}` map from `manifest.sections[*].facets[*].signals` instead of `sections[*].facets[*].signals`. This is the core fix -- manifest facets have 476 signals vs section YAML's 135.

  2. **Simplify render-link validation in `validate_single_chain`**: Since `_build_facet_signal_map` now reads directly from the manifest, the separate `_build_manifest_facet_ids` helper and the two-step check (signal in facet? facet in manifest?) can be collapsed. A signal found in the manifest's facet_signal_map is automatically in the manifest. Remove `_build_manifest_facet_ids`. Remove `FACET_NOT_IN_MANIFEST` gap type (since facet source IS the manifest now). The render link is now: signal in manifest facet_signal_map? Yes = complete, No = NO_FACET (broken).

  3. **Remove `NO_FIELD_ROUTING`** from `ChainGapType` enum. It is never produced by any validation logic (absorbed by MISSING_FIELD_KEY). The audit flagged this as dead code.

  4. **Update `validate_single_chain` signature**: Replace `sections: dict[str, Any]` parameter with the manifest-derived facet_signal_map (or just use manifest directly). Remove the `sections` parameter. The function should accept `manifest: Any` (already present) and build the facet_signal_map from it. Move the `_build_facet_signal_map` call OUT of validate_single_chain (it's called per-signal but the map is identical each time -- performance fix). Instead, pre-compute in validate_all_chains and pass as parameter.

  5. **Update `validate_all_chains`**: Remove `sections_dir` parameter and `load_all_sections` import/call. The function no longer needs section YAML. Pass the pre-computed `facet_signal_map` to `validate_single_chain` instead of sections. Keep `manifest_path` parameter for testing.

  6. **Update docstrings**: Fix module docstring line 7 ("signal is assigned to a facet in section YAML" -> "signal is assigned to a facet in the output manifest"). Update function docstrings to reflect manifest-based resolution. Update the Phase attribution comment to note Phase 81 update.

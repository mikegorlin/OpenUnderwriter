  <action>
Create `src/do_uw/brain/brain_migrate_v3.py` following patterns from existing `brain_migrate_yaml.py`.

The script must:

1. **Build lookup tables** (pure functions, no side effects):
   - `build_group_lookup(sections_dir) -> dict[str, str]`: Map signal_id -> group_id from section YAML facets (135 explicit mappings)
   - `build_prefix_group_map(sections_dir, manifest_path) -> dict[str, str]`: Map signal ID prefix patterns to group IDs for the 341 unmapped signals. Use facet field + signal ID prefix (e.g., FIN.PROFIT.* in financial_health facet -> look at which groups exist under financial_health in manifest -> assign to most specific match)
   - `build_field_registry(registry_path) -> dict[str, dict]`: Load field_registry.yaml as dict
   - `build_signal_index(signals_dir) -> dict[str, dict]`: Map signal_id -> raw dict for cross-referencing

2. **Compute v3 fields for each signal** (pure function `compute_v3_fields(signal: dict, lookups) -> dict`):
   - `group`: Explicit lookup first, then prefix inference, then facet-as-fallback (use facet value directly as group if no better match). Log all inferred assignments.
   - `signal_class`: From type field (foundational->foundational), work_type (infer->inference), ID pattern (COMP.*->inference, FIN.FORENSIC.*composite*->inference), else evaluative
   - `field_path`: Copy from `data_strategy.field_key` if present, else empty string. Use registry keys per Claude's discretion (minimize risk).
   - `depends_on`: For signals with field_key: trace through field_registry to find source path, then find foundational signal producing that path. Format as list of `{signal: "...", field: "..."}` dicts. Empty for foundational signals. Empty for signals without traceable field_key.
   - `provenance` expansion: Set `data_source` from `display.source_type` if available (e.g., "SEC_10K", "WEB"), else "". Set `threshold_provenance.source` = "standard" for signals with numeric thresholds, "unattributed" for others. Set `render_target` from group field value. `formula` left empty (would need signal-by-signal authoring).

3. **Remove old fields from each signal dict**:
   - Remove `type` field (replaced by `signal_class`)
   - Remove `facet` field (replaced by `group`)

4. **Write back using ruamel.yaml** (round-trip preserving comments):
   - Use `YAML(typ='rt')` with `preserve_quotes=True`, `width=120`
   - Follow pattern from `knowledge/yaml_writer.py`
   - Process each YAML file: load, modify entries, write back

5. **CLI interface**:
   - `--dry-run`: Print what would change without modifying files. Show per-signal: old facet -> new group, old type -> new signal_class, field_path, depends_on count
   - `--stats`: Print summary statistics only
   - `--verbose`: Log every signal assignment
   - No flags: execute migration

6. **Safety**: The migration script should be idempotent -- running it twice produces the same result. Check if v3 fields already exist before overwriting.

7. **Bump `schema_version` to 3** on all migrated signals.

Target: < 500 lines. Use argparse for CLI (not typer -- this is a one-shot migration tool).

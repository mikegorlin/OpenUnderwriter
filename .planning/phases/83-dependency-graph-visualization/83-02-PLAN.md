    ```python
    def generate_graph_data(
        signals: list[dict[str, Any]],
        section_filter: str = "",
        type_filter: str = "",
    ) -> dict[str, Any]:
        """Build D3-compatible nodes + links data structure.

        Returns {"nodes": [...], "links": [...], "stats": {...}}
        Each node: {id, name, signal_class, group, section, field_path, ...}
        Each link: {source, target}
        Stats: {total_nodes, total_edges, foundational, evaluative, inference, sections, groups}
        """
    ```

    Nodes include: id, name, signal_class (default "evaluative"), group, field_path, any threshold/severity info useful for sidebar display. Section derived from manifest group mapping.

    Links from depends_on: for each signal, for each dep in depends_on, create {"source": dep["signal"], "target": sig["id"]}. Only include links where both source and target exist in the filtered signal set.

    Stats: counts of nodes by signal_class, unique sections, unique groups, total edges.

    If section_filter or type_filter provided, filter signals before building nodes/links.

  <name>Task 2: Create D3.js HTML template with force-directed graph, sidebar, and filter panel</name>
  <files>src/do_uw/brain/templates/dependency_graph.html</files>
  <action>
    Create `src/do_uw/brain/templates/dependency_graph.html` -- a self-contained Jinja2 template following the audit_report.html pattern (CSS variables, dark theme, responsive layout) but using D3.js v7 CDN for the graph.

    **Layout structure:**
    - Top bar: title "Brain Signal Dependency Graph", stats summary cards (total signals, edges, per-tier counts)
    - Left sidebar (collapsible): Filter panel with checkboxes for signal_class (foundational/evaluative/inference), dropdown for section, dropdown for group. "Reset Filters" button.
    - Main area: SVG canvas with D3 force-directed graph
    - Right sidebar (hidden by default, shown on node click): Signal details panel showing name, ID, signal_class, group, field_path, depends_on list, severity/threshold info

    **D3.js graph:**
    - Load D3 v7 from CDN: `https://d3js.org/d3.v7.min.js`
    - Data injected via Jinja2: `const graphData = {{ graph_data | tojson }};`
    - Force simulation: `d3.forceSimulation(nodes)` with:
      - `d3.forceLink(links).id(d => d.id).distance(80)`
      - `d3.forceManyBody().strength(-100)`
      - `d3.forceCenter(width/2, height/2)`
      - `d3.forceCollide().radius(20)`
    - Initial node positions: y-axis by tier (foundational top, evaluative middle, inference bottom) to give force simulation a head start. x-axis random or by section hash.
    - Node styling: circles colored by signal_class (foundational=#4A90D9 blue, evaluative=#50C878 green, inference=#FF8C42 orange). Size 8px for nodes with deps, 5px for nodes without.
    - Edge styling: thin grey arrows (d3 marker-end for arrowheads) from dependency source to target.
    - Zoom/pan: `d3.zoom()` on SVG, with zoom controls (+ / - / reset buttons)
    - Hover: tooltip with signal name + ID
    - Click: populate right sidebar with full signal details

    **Filter behavior (vanilla JS):**
    - Filter checkboxes/dropdowns hide/show nodes + their connected edges
    - Filtered-out nodes get `display: none` (not removed from simulation -- preserves layout)
    - Edge hidden if either source or target is hidden
    - Stats cards update to show filtered counts

    **Performance considerations for 476 nodes:**
    - Most nodes (421) have no edges -- they form loose clusters, which is fine
    - 55 connected nodes will form the interesting structure
    - Use `simulation.alphaTarget(0).restart()` on filter change for smooth re-layout
    - Consider `simulation.stop()` after initial layout settles (alpha < 0.01) to save CPU

    **CSS:**
    - Follow audit_report.html CSS variable pattern (--bg, --text, --accent, --card-bg, etc.)
    - Dark background recommended for graph visibility
    - Responsive: sidebar collapses on narrow viewport
    - Print-friendly: hide sidebars, show graph only
  </action>

  <behavior>
    - build_dependency_graph(signals) returns a graphlib.TopologicalSorter from depends_on fields
    - build_dependency_graph validates dependency targets exist in signal set, logs warnings for dangling refs
    - detect_cycles(signals) returns None when DAG is valid (no cycles)
    - detect_cycles(signals) returns list of cycle member IDs when cycles exist (A->B->A returns ["A", "B"] or similar)
    - topological_order(signals) returns signal IDs in dependency order
    - order_signals_for_execution(signals) returns signals sorted: foundational first, evaluative second, inference third
    - order_signals_for_execution applies topological sort WITHIN each tier (only within-tier deps, cross-tier deps excluded from per-tier graph)
    - Signals with empty depends_on run in any order within their tier (stable sort)
    - Handles both dict-style depends_on ({"signal": "X"}) since loader returns raw dicts
  </behavior>
  <action>
    Create `src/do_uw/brain/dependency_graph.py` with these functions:

    1. `build_dependency_graph(signals: list[dict[str, Any]]) -> graphlib.TopologicalSorter` -- Build DAG from depends_on fields. Validate all dependency targets exist in signal set; log warnings for dangling references and exclude them. Each signal dict has `id` (str) and `depends_on` (list of dicts with `signal` key).

    2. `detect_cycles(signals: list[dict[str, Any]]) -> list[str] | None` -- Call build_dependency_graph, then ts.prepare(). If CycleError raised, return list(e.args[1]). Otherwise return None.

    3. `topological_order(signals: list[dict[str, Any]]) -> list[str]` -- Return signal IDs via ts.static_order().

    4. `order_signals_for_execution(signals: list[dict[str, Any]]) -> list[dict[str, Any]]` -- Group signals by signal_class tier (foundational/evaluative/inference, default "evaluative"). For each tier in order, build a sub-graph with ONLY within-tier dependency edges (cross-tier deps are already satisfied by tier ordering). Use TopologicalSorter.static_order() to sort within tier. Return flat list of signal dicts in final order. On CycleError within a tier, fall back to unsorted (load-time detection should have caught it).

    CRITICAL anti-pattern to avoid: Do NOT include cross-tier dependency edges in per-tier TopologicalSorter -- evaluative signal depending on foundational signal would cause "missing node" error since foundational ID is not in the evaluative sub-graph.

    - test_build_dag_no_deps: signals with empty depends_on, verify no error
    - test_dangling_reference: signal depends on non-existent ID, logs warning, excluded from graph
    - test_cycle_detection_none: acyclic graph returns None
    - test_cycle_detection_found: A->B->A cycle returns cycle members
    - test_topological_order: verify correct order for known graph
    - test_order_for_execution_tier_ordering: foundational signals appear before evaluative before inference
    - test_order_for_execution_within_tier: two evaluative signals with dep edge appear in correct order
    - test_order_for_execution_cross_tier_excluded: evaluative depending on foundational does not cause error
    - test_order_empty_list: empty input returns empty output
  </action>

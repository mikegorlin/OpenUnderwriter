  <action>
    Create `tests/stages/render/test_template_type_validation.py` per GATE-04:

    1. Template variable extractor:
       - Scan all `.j2` files under `src/do_uw/templates/html/`
       - Use regex to find Jinja2 variable references: `{{ section.field }}`, `{% if section.field %}`, `{{ section.field.subfield }}`
       - Parse into `(section_name, field_path)` tuples
       - Map section names to Pydantic models: `{"executive_summary": ExecSummaryContext, "financials": FinancialContext, "market": MarketContext, "governance": GovernanceContext, "litigation": LitigationContext}`

    2. Schema field extractor:
       - For each Pydantic model, get `model_fields` keys
       - For nested Pydantic models, recursively get sub-fields
       - Build a set of valid dot-paths per section

    3. Cross-reference test:
       ```python
       def test_template_vars_match_schema():
           template_vars = extract_template_variables()
           for section, model_cls in SECTION_MODELS.items():
               schema_fields = get_schema_fields(model_cls)
               template_refs = template_vars.get(section, set())
               unknown = template_refs - schema_fields
               assert not unknown, f"Template references unknown fields in {section}: {unknown}"
       ```

    4. Reverse check (optional, as warning not assertion):
       - Schema fields not referenced in any template -> log as "unused typed field" (informational, not blocking)

    5. Handle Jinja2 patterns:
       - `{{ executive_summary.overall_score }}` -> ("executive_summary", "overall_score")
       - `{{ executive_summary.snapshot.company_name }}` -> ("executive_summary", "snapshot.company_name")
       - `{% for item in litigation.active_cases %}` -> ("litigation", "active_cases")
       - `{{ financials.statements | length }}` -> ("financials", "statements")
       - Ignore filter chains after `|`

    6. Known exceptions list for template variables that use dynamic keys or dict access patterns (e.g., `section[key]`) -- these are documented but not validated.
  </action>

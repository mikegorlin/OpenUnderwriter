       - Collects inconsistencies into `ConsistencyReport`

    3. `check_cross_section_consistency(state, html: str, report_only: bool = True) -> ConsistencyReport`:
       - Convenience function: builds CanonicalMetrics from state, runs checker
       - Per GATE-05: `report_only=True` by default (logs warnings), set `report_only=False` to raise `ConsistencyError` on any inconsistency
       - Returns `ConsistencyReport` with list of `QACheck` items for integration with existing QA pipeline

    Key fact extraction patterns (from existing templates):
    - Revenue appears in: key_stats, executive_summary, financial section header, company profile
    - CEO appears in: executive_summary, governance board table, key_stats
    - Exchange appears in: cover, key_stats, company profile
    - Market cap appears in: key_stats, market section, executive_summary

    Use CSS class/id selectors where possible (e.g., `[data-metric="revenue"]`), fall back to text search with context.

    Do NOT modify templates in this plan -- the checker works with whatever HTML structure exists.

  <action>
    Create `src/do_uw/validation/section_completeness.py` with:

    1. `SectionCompleteness` dataclass: `section_name: str`, `total_fields: int`, `na_fields: int`, `na_ratio: float`, `suppressed: bool`, `reason: str`.

    2. `SectionCompletenessGate` class:
       - `__init__(self, threshold: float = 0.5)` -- configurable N/A ratio threshold
       - `check(self, context: dict[str, Any]) -> list[SectionCompleteness]`: walks the top-level context keys that map to section dicts (executive_summary, financials, market, governance, litigation, company, scoring), counts fields that are None, "N/A", empty string, or empty list. Returns completeness report for each section.
       - `apply_banners(self, context: dict[str, Any]) -> dict[str, Any]`: for sections exceeding threshold, replaces the section dict with a minimal dict containing `_insufficient_data: True`, `_section_name: str`, `_na_ratio: float`, `_reason: str`. Templates already guard with `{% if section_var %}` so suppressed sections will skip rendering.

    3. `check_section_completeness(context: dict, threshold: float = 0.5) -> list[SectionCompleteness]`: convenience function.

    Wire into `html_renderer.py`:
    - In `_render_html_template` (or wherever context is finalized before template.render()), call `SectionCompletenessGate().apply_banners(context)` to inject banners for broken sections.
    - Log a warning for each suppressed section: "Section '{name}' suppressed: {na_ratio:.0%} N/A ({na_fields}/{total_fields} fields)"
    - The gate does NOT raise exceptions -- it replaces broken sections with banners (GATE-02).

    N/A detection helper `_is_na(value)`:
    - None -> True
    - "" -> True
    - "N/A" (case-insensitive) -> True
    - "Not Available" -> True
    - empty list/dict -> True
    - Everything else -> False
    - Do NOT count nested dicts/lists as single N/A -- only leaf values count
  </action>

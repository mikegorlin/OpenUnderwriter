    **_forward_credibility.py** (per D-07, D-08):
    Create `build_forward_credibility(state: AnalysisState) -> dict[str, Any]` that:
    1. Reads `state.forward_looking.credibility` (CredibilityScore) for quarters list
    2. Also calls `build_earnings_trust(state)` to get `earnings_trust_summary` with beat_rate, beat_count, miss_count
    3. Classifies pattern using D-08 thresholds:
       - beat_rate > 0.75 AND avg beat magnitude < 10% -> "Consistent Beater"
       - beat_rate > 0.80 AND avg beat magnitude >= 10% -> "Sandbagging"
       - miss_rate > 0.25 -> "Unreliable"
       - Last 2+ quarters are misses AND prior 4+ were beats -> "Deteriorating"
       - Fewer than 4 quarters -> "Insufficient Data"
       - Evaluation order: Insufficient Data first (guard), then Deteriorating (most specific), Unreliable, Sandbagging, Consistent Beater, default "Mixed"
    4. Builds quarter_table from credibility.quarters: each row has quarter, guidance (guided_value), actual (actual_value), delta (actual - guided as string), magnitude (magnitude_pct formatted), beat_or_miss, row_class (CSS class: row-beat/row-miss/row-inline)
    5. Builds cumulative_pattern string like "BBBMM" from quarter beat_or_miss sequence
    6. Returns `{"credibility_available": True, "pattern": "CONSISTENT_BEATER", "pattern_label": "Consistent Beater", "pattern_color": hex, "beat_rate_pct": "78%", "quarters_assessed": N, "quarter_table": [...], "cumulative_pattern": "BBBMB", "source": str}`

    **_forward_short_sellers.py** (per D-09, D-10, D-11):
    Create two functions:

    `build_short_seller_alerts(state: AnalysisState) -> dict[str, Any]`:
    1. Define SHORT_SELLER_FIRMS = ["Citron Research", "Hindenburg Research", "Spruce Point Capital", "Muddy Waters Research", "Kerrisdale Capital"]
    2. Scan `state.acquired_data` for web search results. Check common paths: `state.acquired_data.web_search_results`, `state.acquired_data.blind_spot_results`, or iterate `state.acquired_data` dict for string content containing firm names.
    3. For each firm, check if firm name AND (company ticker OR company name) co-occur in same result. Also require "report" or "short" or "target" in proximity to reduce false positives (per Pitfall 4 in research).
    4. Each detected report: `{"firm": str, "title": str, "date": str or "Unknown", "url": str or "", "summary": str}`
    5. Returns `{"alerts_available": bool, "reports": [...], "report_count": N, "firms_checked": SHORT_SELLER_FIRMS}`

    `derive_short_conviction(state: AnalysisState) -> dict[str, Any]`:
    1. Read `state.extracted.market.short_interest` (ShortInterestProfile)
    2. If shares_short and shares_short_prior both available: compute pct_change. >10% = Rising, <-10% = Declining, else Stable.
    3. Fallback to trend_6m text (UP->Rising, DOWN->Declining, else Stable)
    4. Returns `{"conviction": "Rising"/"Stable"/"Declining", "conviction_color": hex, "conviction_rationale": str, "short_pct": float or None, "days_to_cover": float or None, "shares_short": int or None, "shares_short_prior": int or None, "pct_change": float or None}`
    5. Use safe_float() for all numeric access. Handle None/missing gracefully.

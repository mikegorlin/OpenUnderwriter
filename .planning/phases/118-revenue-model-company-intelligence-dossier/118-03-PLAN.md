Create `src/do_uw/stages/benchmark/dossier_enrichment.py` with:

1. **`enrich_dossier(state: AnalysisState) -> None`** -- main entry point, calls sub-functions:
   - `_enrich_core_do_exposure(state)` -- generates the "core D&O exposure" paragraph for DOSSIER-01
   - `_enrich_revenue_card(state)` -- adds do_risk + risk_level to each revenue card row
   - `_enrich_concentration_dimensions(state)` -- computes risk levels for 5 concentration dimensions
   - `_enrich_emerging_risks(state)` -- maps each risk to scoring factor, validates probability/impact
   - `_enrich_unit_economics(state)` -- adds benchmark comparison, assessment, do_risk, narrative
   - `_enrich_waterfall(state)` -- generates D&O insight narrative for the revenue waterfall section
   - `_enrich_asc_606(state)` -- adds do_risk to each ASC 606 element

2. **`_enrich_core_do_exposure(state)`:**
   - Reads state.scoring (tier, total_score, top factors)
   - Generates: "The core D&O exposure for {company} is {primary_risk_vector}. Scoring at {score} ({tier}), the primary driver is {top_factor.name} (F.{N} = {score}/{max}) reflecting {specific_evidence}."
   - Stores on state.dossier.core_do_exposure

3. **`_enrich_revenue_card(state)`:**
   - For each RevenueModelCardRow, generate do_risk based on attribute:
     - Revenue Quality: Tier 3 = HIGH ("One-time/project revenue creates earnings volatility -- a primary SCA trigger per F.5"), Tier 1 = LOW
     - Concentration Risk: HIGH if any dimension >30%
     - Rev-Rec Complexity: HIGH = "Complex revenue recognition is a top-3 SCA allegation category"
     - Each do_risk is company-specific with actual data values

4. **`_enrich_concentration_dimensions(state)`:**
   - Reads state.company.customer_concentration, geographic_footprint, revenue_segments
   - Creates 5 ConcentrationDimension entries: Customer, Geographic, Product, Channel, Payer
   - Risk thresholds: >30% single entity = HIGH, 15-30% = MEDIUM, <15% = LOW
   - do_implication references the litigation theory (e.g., "Loss of top customer = material misstatement if not disclosed per Item 303")

5. **`_enrich_emerging_risks(state)`:**
   - Maps each EmergingRisk.risk to a scoring factor using keyword matching:
     - "regulatory" -> F.9, "competition" -> F.5, "litigation" -> F.1, "financial" -> F.3, default -> F.5
   - Validates probability is in (High, Medium, Low) and impact is in (Very High, High, Medium, Low)

6. **`_enrich_unit_economics(state)`:**
   - Generates unit_economics_narrative: "The single most important metric for {company}'s D&O risk is {metric_name} at {value} because {reason}."
   - Selects most important metric based on: NDR < 90% for SaaS, negative growth for any, guidance miss risk

7. **`_enrich_waterfall(state)`:**
   - Reads state.dossier.waterfall_rows and state.scoring (tier, factors)
   - Generates state.dossier.waterfall_narrative: a company-specific D&O insight paragraph
   - Analyzes waterfall_rows to identify dominant growth driver (expansion vs new logo vs price)
   - Maps growth composition to D&O risk:
     - Expansion-heavy (expansion > 60% of new ARR): "Expansion-heavy growth creates earnings predictability risk if NDR compresses (F.5 narrative break indicator)"
     - New-logo-heavy (new logo > 60%): "New customer acquisition cost may pressure margins -- miss guidance if CAC payback extends (F.3 expense trajectory)"
     - Price-driven (price increases > 30% of growth): "Price-driven growth faces churn risk if competitors undercut -- disclosure risk under Item 303 if pricing power wanes"
   - Includes actual dollar amounts from waterfall_rows: e.g., "Net new ARR of $45M in FY2025 driven by expansion ($31M) vs new logo ($14M)"
   - If waterfall_rows is empty, sets waterfall_narrative to "" (graceful no-op)

8. **`_enrich_asc_606(state)`:**
   - For each ASC606Element, generate do_risk:
     - Complexity HIGH: "Revenue recognition judgment area -- 23% of SCA settlements 2019-2024 involved rev-rec allegations"
     - Complexity LOW: "Standard recognition with minimal judgment"

All enrichment reads scoring data from `state.scoring` and `state.analysis.signal_results`. If scoring not yet run (empty), enrichment still works but with generic fallback commentary noting "scoring data not yet available."

Create `src/do_uw/stages/extract/dossier_extraction.py` with:

1. **`extract_dossier(state: AnalysisState) -> None`** -- main entry point, modifies state.dossier in-place. Orchestrates 4 focused extractions:
   - `_extract_revenue_model(state)` -- uses RevenueModelExtraction on Items 1+7 text
   - `_extract_asc_606(state)` -- uses ASC606Extraction on Note 2 text
   - `_extract_emerging_risks(state)` -- uses EmergingRiskExtraction on Item 1A text
   - `_extract_unit_economics(state)` -- uses UnitEconomicsExtraction on MD&A text

2. **Each sub-function:**
   - Gets filing text from `state.acquired_data.filings` (10-K sections)
   - Builds prompt with analytical context: company name, ticker, sector, revenue, scoring summary (from state.company and state.scoring if available)
   - Calls `LLMExtractor().extract()` with the schema
   - Converts extraction result to domain models (DossierData sub-models)
   - Stores on state.dossier fields
   - Falls back gracefully if filing text is unavailable (logs warning, leaves defaults)

3. **Revenue model extraction** also:
   - CONSUMES `state.company.revenue_segments` as authoritative (per research Pitfall 2)
   - ENRICHES with growth_rate, rev_rec_method, do_exposure from LLM extraction
   - Populates revenue_flow_diagram from structured nodes/edges (converted to formatted text)
   - Populates business_description_plain from state.company.business_description (simplified by LLM if >500 chars)

4. **Each prompt includes QUAL-03 analytical context:**
   ```
   Company: {company_name} ({ticker})
   Sector: {sector}
   Revenue: {revenue}
   Revenue Model Type: {revenue_model_type}
   Scoring Context: {scoring_summary if available}
   ```

5. **Error handling:** Each sub-extraction is try/except wrapped. Failure in one does not block others. Logged at WARNING level.

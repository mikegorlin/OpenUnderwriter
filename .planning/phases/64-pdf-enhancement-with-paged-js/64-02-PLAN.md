**2. Add TOC generation + details expansion JavaScript to base.html.j2:**

At the bottom of the `<body>`, BEFORE the existing sidebar script, add a PDF preparation script that runs only when `pdf_mode` is true:

```html
{% if pdf_mode %}
<script>
(function() {
  // 1. Expand all <details> elements (PDF-06)
  document.querySelectorAll('details').forEach(function(d) {
    d.setAttribute('open', '');
  });

  // 2. Build TOC from section headings (PDF-03)
  var tocContainer = document.querySelector('.pdf-toc-entries');
  if (tocContainer) {
    var sections = document.querySelectorAll('section[id] > h2, section[id] > .section-header > h2');
    var html = '';
    sections.forEach(function(h2) {
      var section = h2.closest('section[id]');
      if (!section) return;
      var id = section.getAttribute('id');
      var title = h2.textContent.trim();
      // TOC entry: title with leader dots and page number placeholder
      html += '<div class="pdf-toc-entry">' +
        '<a href="#' + id + '">' +
        '<span class="pdf-toc-text">' + title + '</span>' +
        '<span class="pdf-toc-dots"></span>' +
        '<span class="pdf-toc-page" data-toc-for="' + id + '"></span>' +
        '</a></div>';
    });
    tocContainer.innerHTML = html;
  }
})();
</script>
{% endif %}
```

Note: Actual page numbers in the TOC cannot be determined by JavaScript alone before PDF rendering — they depend on Paged.js/Chromium pagination. The TOC will show section titles with clickable links. Page numbers are a SHOULD requirement (PDF-03) and we provide the structure for them. The links themselves are functional within the PDF (internal navigation).

**1. Optimize chart image sizes in `__init__.py`'s `_generate_chart_images()`:**

The main file size contributor for PDFs is chart images. Add DPI reduction for PDF-bound charts:

In the `_generate_chart_images()` function, the current DPI is set by the chart creation functions (typically 200). For PDF output, this is excessive — 150 DPI is sufficient for print quality at the chart sizes used. However, since the same charts serve both HTML and PDF, we do NOT change the generation DPI. Instead, we optimize at the embedding stage.

In `html_renderer.py`, add an image optimization step in `_build_pdf_html()` (or a helper function):

```python
def _optimize_chart_images_for_pdf(context: dict[str, Any]) -> None:
    """Reduce chart image sizes for PDF output.

    Re-encodes base64 PNG chart images at reduced quality for smaller PDF.
    Modifies context["chart_images"] in place.
    """
    try:
        from PIL import Image
    except ImportError:
        return  # Pillow not available, skip optimization

    import io
    optimized = {}
    for name, b64_data in context.get("chart_images", {}).items():
        if not b64_data:
            optimized[name] = b64_data
            continue
        try:
            img_bytes = base64.b64decode(b64_data)
            img = Image.open(io.BytesIO(img_bytes))
            # Resize if width > 800px (sufficient for letter-size PDF)
            if img.width > 800:
                ratio = 800 / img.width
                new_size = (800, int(img.height * ratio))
                img = img.resize(new_size, Image.LANCZOS)
            # Re-encode as optimized PNG
            buf = io.BytesIO()
            img.save(buf, format="PNG", optimize=True)
            optimized[name] = base64.b64encode(buf.getvalue()).decode("ascii")
        except Exception:
            optimized[name] = b64_data  # Keep original on error
    context["chart_images"] = optimized
```

Call this function in `_build_pdf_html()` after building the context and before rendering the template:
```python
context = build_html_context(state, chart_dir)
context["pdf_mode"] = True
_optimize_chart_images_for_pdf(context)
html_content = _render_html_template(context)
```

If Pillow is not available, the optimization is silently skipped (graceful degradation).

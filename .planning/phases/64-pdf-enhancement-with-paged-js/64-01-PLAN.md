This task adds the CSS infrastructure for Paged.js-driven PDF output. Paged.js is a CSS Paged Media polyfill — it reads standard CSS @page rules and generates paginated output in the browser DOM. The key technique: define CSS @page rules with `@top-center`, `@bottom-center` margin boxes, and use `string-set` / `content: string(...)` to flow running text into headers/footers.

**IMPORTANT architectural decision:** We do NOT bundle or load the Paged.js JavaScript library. Instead, we use CSS Paged Media properties that Playwright's Chromium already partially supports natively (via `@page`, `size`, margin boxes). For the running header/footer content, we use a simpler approach: dedicated HTML elements positioned with CSS `position: fixed` that appear on every printed page. This avoids a 500KB+ JS dependency and works with Playwright's existing print-to-PDF path.

1. **In `styles.css`**, enhance the existing `@media print` block:

   Add these rules INSIDE the existing `@media print { @page { ... } }` block:
   ```css
   @page {
     size: letter;
     margin: 1in 0.65in 0.85in 0.65in; /* top expanded for header space */
   }
   @page:first {
     margin-top: 0.65in; /* No running header on first page (cover) */
   }
   ```

   Add running header/footer fixed-position elements (Playwright prints these on every page):
   ```css
   .pdf-running-header {
     display: none; /* Hidden in browser */
   }
   .pdf-running-footer {
     display: none; /* Hidden in browser */
   }

   @media print {
     .pdf-running-header {
       display: block;
       position: fixed;
       top: 0;
       left: 0;
       right: 0;
       height: 0.4in;
       padding: 0.15in 0.65in 0;
       font-size: 7pt;
       color: #666;
       border-bottom: 0.5pt solid #ddd;
       z-index: 9999;
     }
     .pdf-running-header .header-left {
       float: left;
       font-weight: 600;
     }
     .pdf-running-header .header-right {
       float: right;
     }

     .pdf-running-footer {
       display: block;
       position: fixed;
       bottom: 0;
       left: 0;
       right: 0;
       height: 0.35in;
       padding: 0 0.65in 0.15in;
       font-size: 7pt;
       color: #666;
       border-top: 0.5pt solid #ddd;
       text-align: center;
       z-index: 9999;
     }
   }
   ```

2. **In `components.css`**, strengthen the table break-avoidance rules in the `@media print` block:

   Add/update these rules:
   ```css
   /* Prevent ALL tables from splitting across pages (PDF-04) */
   table {
     break-inside: avoid;
     page-break-inside: avoid;
   }
   /* For very long tables, allow break but keep header group */
   table.allow-break {
     break-inside: auto;
     page-break-inside: auto;
   }
   table.allow-break thead {
     display: table-header-group;
   }
   ```

   Also add rules to hide interactive chart containers and show static fallbacks:

   ```css
   /* In PDF: hide interactive JS chart containers, show static PNGs (PDF-05) */
   .chart-interactive {
     display: none !important;
   }
   .chart-static {
     display: block !important;
   }
   ```

3. **In `base.html.j2`**, add the running header and footer HTML elements:

   Right after the opening `<body>` tag (before the sticky-topbar), add:
   ```html
   {# --- PDF Running Header (visible only in @media print) --- #}
   <div class="pdf-running-header">
     <span class="header-left">CONFIDENTIAL &mdash; {{ company_name | default("Company") }} D&amp;O Worksheet</span>
     <span class="header-right">{{ generation_date | default("") }}</span>
   </div>

   {# --- PDF Running Footer (visible only in @media print) --- #}

     Angry Dolphin Underwriting System &bull; Confidential &mdash; For Internal Underwriting Use Only
   </div>
   ```

   Note: Playwright's page.pdf() has its own `display_header_footer` option with `pageNumber`/`totalPages` spans. We will use BOTH approaches: Paged.js CSS for the header content, and Playwright's native footer for page numbers (since CSS counters for page numbers require actual Paged.js JS). The existing Playwright footer_template in html_renderer.py handles "Page N of M".

4. **Do NOT add the Paged.js JavaScript library.** The approach uses CSS-only techniques that work with Playwright's Chromium print engine. The CSS `position: fixed` approach for headers/footers is well-supported in Chromium's print-to-PDF path.

IMPORTANT: Keep all changes backward-compatible with browser viewing. The `.pdf-running-header` and `.pdf-running-footer` elements are `display: none` by default and only shown in `@media print`. The browser interactive view is unaffected.

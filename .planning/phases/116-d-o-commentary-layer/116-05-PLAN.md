**Step 1: Build "Why TIER" algorithm** in `scoring_evaluative.py`:

Add function `generate_tier_explanation(scoring_result: ScoringResult) -> str`:

Algorithm (purely algorithmic, NO LLM per CONTEXT.md):
1. Get `quality_score` and `tier` from scoring_result
2. Load tier boundaries from `scoring.json` (or use the TierClassification fields)
3. Sort `factor_scores` by `points_deducted` descending to find top contributors
4. Build narrative parts:
   - Opening: "Quality score of {score:.1f} places {company} in {tier} tier ({low}-{high})."
   - Top drag: "Heaviest drag: {factor_id} {factor_name} ({deducted:.0f}/{max}), accounting for {pct:.0f}% of total deductions."
   - If tier above exists: "To reach {above_tier}, score would need +{gap:.1f} points."
   - Counterfactual: For each factor with deductions, compute hypothetical score if that factor were 0. If hypothetical reaches tier above: "If {factor_id} {factor_name} were clean (0/{max}), score would be {hypothetical:.1f} -- {above_tier} tier."
   - If tier below exists and gap is small (<5 points): "Warning: Only {gap:.1f} points above {below_tier} boundary."
5. Return joined narrative string

Add `tier_explanation` to the scoring context dict so templates can render `{{ tier_explanation }}`.

**Part A: Create src/do_uw/brain/brain_health.py**

New file (computation only, no CLI). Contains:

1. **Pydantic model:**
```python
class BrainHealthReport(BaseModel):
    total_active_signals: int
    total_all_signals: int  # including INACTIVE, DEPRECATED
    facet_coverage_pct: float  # signals in facets / total active * 100
    signals_in_facets: int
    signals_not_in_facets: int

    # Fire rate distribution (bucket -> count)
    fire_rate_distribution: dict[str, int]  # "0%", "1-10%", "10-30%", "30-50%", "50-80%", "80-100%"

    # Top problematic signals
    top_always_fire: list[dict[str, Any]]  # from effectiveness report
    top_never_fire: list[dict[str, Any]]
    top_high_skip: list[dict[str, Any]]

    # System metadata
    data_freshness: str  # last_build_at from brain_meta
    total_pipeline_runs: int
    total_backtest_runs: int
    feedback_queue_size: int

    # Run history
    tickers_analyzed: list[str]
    runs_by_ticker: dict[str, int]
```

2. **compute_brain_health(conn: duckdb.DuckDBPyConnection) -> BrainHealthReport:**
   - Total signals: `SELECT COUNT(*) FROM brain_signals_active`
   - Total all: `SELECT COUNT(DISTINCT signal_id) FROM brain_signals`
   - Facet coverage: load facets via `load_all_facets()`, collect all signal IDs, compute coverage % against active signals
   - Fire rate distribution: use `compute_effectiveness(conn)` to get per-signal fire rates, then bucket into distribution. Buckets: "0%" (exactly 0), "1-10%", "10-30%", "30-50%", "50-80%", "80-100%"
   - Top always-fire/never-fire/high-skip: take first 10 from effectiveness report lists
   - Data freshness: `SELECT meta_value FROM brain_meta WHERE meta_key = 'last_build_at'`
   - Pipeline runs: `SELECT COUNT(DISTINCT run_id) FROM brain_signal_runs WHERE is_backtest = FALSE`
   - Backtest runs: `SELECT COUNT(DISTINCT run_id) FROM brain_signal_runs WHERE is_backtest = TRUE`
   - Feedback queue: `SELECT COUNT(*) FROM brain_feedback WHERE status = 'PENDING'`
   - Tickers: `SELECT DISTINCT ticker FROM brain_signal_runs WHERE is_backtest = FALSE`
   - Runs by ticker: `SELECT ticker, COUNT(DISTINCT run_id) FROM brain_signal_runs WHERE is_backtest = FALSE GROUP BY ticker`

**Part A: Create src/do_uw/brain/brain_audit.py**

New file (computation only). Contains:

1. **Pydantic models:**
```python
class AuditFinding(BaseModel):
    category: str  # "STALENESS", "COVERAGE", "THRESHOLD", "ORPHAN"
    severity: str  # "HIGH", "MEDIUM", "LOW", "INFO"
    signal_id: str = ""
    message: str
    detail: str = ""

class BrainAuditReport(BaseModel):
    findings: list[AuditFinding]
    total_stale: int
    total_threshold_issues: int
    total_orphaned: int
    peril_coverage_available: bool
    summary: str  # one-line overall assessment
```

2. **compute_brain_audit(conn: duckdb.DuckDBPyConnection) -> BrainAuditReport:**

   **Staleness check:**
   - Query `brain_signals_active` for `last_calibrated`
   - Flag signals where `last_calibrated` is NULL or > 180 days ago as MEDIUM severity
   - Group by staleness bucket: NULL (never calibrated), >365 days, 180-365 days
   - Don't flag individual signals (too noisy with 380) -- report counts and top 10 oldest

   **Coverage imbalance by peril:**
   - Check if ANY signals have non-NULL peril_id: `SELECT COUNT(*) FROM brain_signals_active WHERE peril_id IS NOT NULL`
   - If 0 (which is the current state): report `AuditFinding(category="COVERAGE", severity="INFO", message="Peril coverage: NOT AVAILABLE (0/380 signals have peril assignments)", detail="Peril IDs not yet populated in signal YAML")`
   - If some have peril_ids: use brain_coverage_matrix view to find GAP/THIN perils

   **Threshold conflict detection:**
   - Query `brain_signals_active` for `threshold_full` JSON
   - Parse threshold_full for each signal
   - ONLY check for conflicts on numeric threshold types: `tiered_threshold`, `numeric_threshold`
   - Skip `boolean_presence`, `count_threshold`, `ratio_threshold`, `date_threshold` -- these don't have overlapping red/yellow/clear ranges
   - A conflict is: red range overlaps yellow range, or yellow range overlaps clear range
   - Example conflict: red: ">10", yellow: ">15" (the yellow should be lower threshold than red for "higher is worse")
   - Report each conflict as a finding

   **Orphaned signals:**
   - Load all facets via `load_all_facets()`
   - Collect all signal IDs across all facets
   - Query `brain_signals_active` signal IDs
   - Report signals in DB but not in any facet (already enforced by CI test, but audit provides runtime visibility)
   - Note: CI test test_every_active_signal_in_exactly_one_facet covers this statically

   **Summary generation:**
   - One-line assessment: "N findings: M high, N medium, P low" or "No structural issues found"

New file (computation only, no CLI). Contains:

1. **Pydantic models:**
```python
class SignalChange(BaseModel):
    signal_id: str
    old_status: str
    new_status: str
    old_value: str | None = None
    new_value: str | None = None
    change_direction: str  # "newly_triggered", "newly_cleared", "newly_skipped", "other"

class RunInfo(BaseModel):
    run_id: str
    run_date: str
    signal_count: int

class DeltaReport(BaseModel):
    ticker: str
    old_run: RunInfo
    new_run: RunInfo
    changes: list[SignalChange]
    newly_triggered: list[SignalChange]  # CLEAR/SKIPPED -> TRIGGERED
    newly_cleared: list[SignalChange]    # TRIGGERED/SKIPPED -> CLEAR
    newly_skipped: list[SignalChange]    # any -> SKIPPED
    other_changes: list[SignalChange]    # INFO changes, etc.
    unchanged_count: int
    error: str | None = None
```

2. **compute_delta(conn, ticker, run1_id=None, run2_id=None) -> DeltaReport:**

   **Step 1: Identify the two runs to compare.**
   If run1_id and run2_id are provided, use those directly. Otherwise:
   ```sql
   SELECT run_id, MIN(run_date) as run_date, COUNT(*) as signal_count
   FROM brain_signal_runs
   WHERE ticker = ? AND is_backtest = FALSE
   GROUP BY run_id
   ORDER BY MIN(run_date) DESC
   LIMIT 2
   ```
   If < 2 runs found, return DeltaReport with error="Need at least 2 runs for delta. Found N run(s) for {ticker}."

   The first result is the NEW run, the second is the OLD run.

   **Step 2: Join the two run snapshots and find changes.**
   ```sql
   SELECT
       COALESCE(o.signal_id, n.signal_id) as signal_id,
       o.status AS old_status,
       n.status AS new_status,
       o.value AS old_value,
       n.value AS new_value
   FROM (SELECT * FROM brain_signal_runs WHERE run_id = ?) o
   FULL OUTER JOIN (SELECT * FROM brain_signal_runs WHERE run_id = ?) n
       ON o.signal_id = n.signal_id
   WHERE o.status IS DISTINCT FROM n.status
   ORDER BY COALESCE(o.signal_id, n.signal_id)
   ```
   This catches: status changes, signals added in new run, signals removed in new run.

   **Step 3: Classify changes by direction.**
   For each change:
   - If new_status == "TRIGGERED" and old_status in ("CLEAR", "SKIPPED", "INFO", None): "newly_triggered"
   - If new_status == "CLEAR" and old_status in ("TRIGGERED", "SKIPPED", None): "newly_cleared"
   - If new_status == "SKIPPED": "newly_skipped"
   - Otherwise: "other"

   Populate the classified lists (newly_triggered, newly_cleared, newly_skipped, other_changes).

   **Step 4: Count unchanged.**
   unchanged_count = total signals in new run - len(changes)

   **Step 5: Get run metadata.**
   For each run_id, get the run_date and signal count from the join query.

3. **list_runs(conn, ticker) -> list[RunInfo]:**
   Helper to list all runs for a ticker (for --list-runs option).
   ```sql
   SELECT run_id, MIN(run_date) as run_date, COUNT(*) as signal_count
   FROM brain_signal_runs
   WHERE ticker = ? AND is_backtest = FALSE
   GROUP BY run_id
   ORDER BY MIN(run_date) DESC
   LIMIT 20
   ```

```python
@brain_app.command("delta")
def brain_delta_cmd(
    ticker: str = typer.Argument(help="Ticker to compare runs for"),
    run1: str = typer.Option("", "--run1", help="Explicit old run ID"),
    run2: str = typer.Option("", "--run2", help="Explicit new run ID"),
    list_runs: bool = typer.Option(False, "--list-runs", help="List available runs for this ticker"),
) -> None:
    """Show signal status changes between two runs for a ticker."""
```

**Implementation:**

1. **--list-runs mode:** If `--list-runs`, call `list_runs(conn, ticker)` and display a table of available runs (run_id, date, signal count), then exit. This helps the user pick explicit run IDs for comparison.

2. **Delta mode (default):**
   - Connect to brain DB
   - Call `compute_delta(conn, ticker, run1_id=run1 or None, run2_id=run2 or None)`
   - If error, print error message and exit(1)
   - Display results

**Rich output layout:**

1. **Header panel:** "Signal Delta: {TICKER}" with run comparison info:
   ```
   Old run: {run_id} ({run_date}) -- {signal_count} signals
   New run: {run_id} ({run_date}) -- {signal_count} signals
   ```

2. **Summary line:**
   ```
   {N} changes detected ({unchanged} unchanged)
   ```

3. **Newly TRIGGERED table** (if any):
   ```
   Newly Triggered (CLEAR/SKIPPED -> TRIGGERED)
   +---------------------------+-------------+----------+
   | Signal                    | Was         | Value    |
   +---------------------------+-------------+----------+
   | LIT.SCA.active_filing     | CLEAR       | 2        |
   | STOCK.PRICE.single_day    | SKIPPED     | 5        |
   +---------------------------+-------------+----------+
   ```
   Style: red/bold for the signal IDs (these are new risks)

4. **Newly CLEARED table** (if any):
   Style: green for signal IDs (risks resolved)

5. **Newly SKIPPED table** (if any):
   Style: yellow for signal IDs (data gaps)

6. **Other changes table** (if any):
   Style: dim

7. **If no changes:** Print "No signal status changes between runs."

**Error handling:**
- brain.duckdb not found: "brain.duckdb not found. Run the pipeline first."
- < 2 runs: "Need at least 2 runs for {ticker} to compute delta. Found N run(s). Run `do-uw analyze {ticker}` to create more runs."
- Invalid run IDs: "Run ID '{run_id}' not found for ticker {ticker}."

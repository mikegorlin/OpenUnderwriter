**Part A: Create src/do_uw/brain/brain_composite_schema.py**

New file. Contains schema definitions and YAML loading.

1. **CompositeDefinition (Pydantic model for YAML):**
```python
class ConclusionField(BaseModel):
    """One field in the composite's conclusion schema."""
    name: str
    description: str
    type: str = "str"  # str, list, dict, numeric

class CompositeDefinition(BaseModel):
    """Schema for a brain/composites/*.yaml file."""
    id: str  # e.g. "COMP.STOCK.drop_analysis"
    name: str  # e.g. "Stock Drop Analysis"
    description: str = ""
    member_signals: list[str]  # signal IDs that this composite reads
    conclusion_schema: dict[str, str] = Field(default_factory=dict)
    # conclusion_schema maps field_name -> description of what that field contains
    # e.g. {"events_by_pattern": "grouped by: earnings, litigation, sector, company-specific"}
    evaluator: str = "default"
    # evaluator: which evaluation function to use. "default" uses generic aggregation.
    # Named evaluators (e.g., "stock_drop_analysis") use domain-specific logic.
    severity_rules: dict[str, str] = Field(default_factory=dict)
    # severity_rules: how to determine composite severity (RED/YELLOW/CLEAR)
    # e.g. {"red": "any member TRIGGERED with company_specific=true", "yellow": "any member TRIGGERED"}
```

2. **CompositeResult (Pydantic model for evaluation output):**
```python
class CompositeResult(BaseModel):
    """Result of evaluating a composite."""
    composite_id: str
    name: str
    member_results: dict[str, dict[str, Any]]  # signal_id -> {status, value, details, ...}
    conclusion: dict[str, Any]  # structured analytical conclusion per conclusion_schema
    narrative: str  # one-paragraph summary for display
    severity: str  # "RED", "YELLOW", "CLEAR"
    member_count: int
    triggered_count: int
    skipped_count: int
```

3. **load_composite(path: Path) -> CompositeDefinition** -- load one YAML file
4. **load_all_composites(composites_dir: Path) -> dict[str, CompositeDefinition]** -- load all composites from brain/composites/

**Part B: Create src/do_uw/brain/brain_composite_engine.py**

New file. Contains the evaluation engine.

1. **evaluate_composites(composite_defs, signal_results) -> dict[str, CompositeResult]:**
   - Takes all composite definitions and all signal results (from state.analysis.signal_results)
   - For each composite definition:
     - Collect member signal results from signal_results dict
     - Call the appropriate evaluator function
     - Produce a CompositeResult
   - Return dict of composite_id -> CompositeResult

2. **Evaluator dispatch:**
   ```python
   _EVALUATORS: dict[str, Callable] = {
       "default": _evaluate_default,
       "stock_drop_analysis": _evaluate_stock_drop,
       "stock_short_analysis": _evaluate_stock_short,
       "stock_insider_analysis": _evaluate_stock_insider,
   }
   ```

3. **_evaluate_default(definition, member_results) -> CompositeResult:**
   Generic evaluator that:
   - Counts TRIGGERED/CLEAR/SKIPPED among members
   - Sets severity based on proportion triggered (>50% = RED, >0% = YELLOW, 0% = CLEAR)
   - Generates a basic narrative: "N of M member signals triggered"
   - Copies member details into conclusion

4. **_evaluate_stock_drop(definition, member_results) -> CompositeResult:**
   Domain-specific evaluator for COMP.STOCK.drop_analysis:
   - Reads details.events from STOCK.PRICE.single_day_events (populated by Plan 01)
   - Reads attribution data from STOCK.PRICE.attribution details
   - Reads peer divergence from STOCK.PATTERN.peer_divergence details
   - Reads recovery data from STOCK.PRICE.recovery details
   - Reads insider timing from STOCK.INSIDER.cluster_timing details
   - Groups events by pattern (earnings, litigation, sector, company-specific)
   - Determines per-event attribution (company vs sector)
   - Produces conclusion:
     ```python
     conclusion = {
         "events_by_pattern": {...},  # grouped events
         "attribution_summary": "2 company-specific, 1 sector-wide",
         "recovery_assessment": "no recovery from company-specific drops",
         "insider_correlation": "insider selling preceded Aug drop" or "no insider correlation",
         "total_events": N,
         "company_specific_events": M,
     }
     ```
   - Generates narrative: "AAPL experienced 3 significant drops (>5%): 2 company-specific (earnings miss, product recall) and 1 sector-wide. No recovery from company-specific events. Insider selling preceded the August drop."
   - Severity: RED if any company-specific event without recovery AND insider correlation; YELLOW if any company-specific event; CLEAR if all sector-wide
   - IMPORTANT: If details are not populated (Plan 01 not yet enriched all evaluators), fall back to the default evaluator. The composite must work even with empty details -- it just produces less rich conclusions.

5. **_evaluate_stock_short(definition, member_results) -> CompositeResult:**
   For COMP.STOCK.short_analysis:
   - Reads STOCK.SHORT.position, STOCK.SHORT.trend, STOCK.SHORT.report, STOCK.PATTERN.short_attack details
   - Produces conclusion about short interest trajectory and whether coordinated short attack is indicated
   - Severity based on short position level + trend direction + report presence

6. **_evaluate_stock_insider(definition, member_results) -> CompositeResult:**
   For COMP.STOCK.insider_analysis:
   - Reads STOCK.INSIDER.cluster_timing, STOCK.INSIDER.notable_activity, STOCK.INSIDER.summary details
   - Produces conclusion about insider trading patterns and timing relative to events
   - Severity based on cluster size and timing relative to price drops

**Key design principle:** Composites are graceful with missing data. If a member signal was SKIPPED or has no details, the composite still produces a result -- it just notes the data gap in its conclusion. Never crash on missing member data.

In `src/do_uw/stages/analyze/__init__.py`, add a `_run_composites()` function and call it in the pipeline AFTER signal execution and gap re-evaluation, but BEFORE analytical engines (since composites read signal results):

```python
def _run_composites(state: AnalysisState) -> None:
    """Evaluate signal composites after individual signals have completed."""
    from pathlib import Path
    from do_uw.brain.brain_composite_schema import load_all_composites
    from do_uw.brain.brain_composite_engine import evaluate_composites

    composites_dir = Path(__file__).parent.parent.parent / "brain" / "composites"
    if not composites_dir.exists():
        logger.info("No composites directory found, skipping composite evaluation")
        return

    composite_defs = load_all_composites(composites_dir)
    if not composite_defs:
        logger.info("No composite definitions found")
        return

    signal_results = state.analysis.signal_results if state.analysis else {}
    composite_results = evaluate_composites(composite_defs, signal_results)

    if state.analysis is not None:
        state.analysis.composite_results = {
            cid: cr.model_dump() for cid, cr in composite_results.items()
        }
    logger.info(
        "Composites: %d evaluated, severities: %s",
        len(composite_results),
        {cid: cr.severity for cid, cr in composite_results.items()},
    )
```

In `AnalyzeStage.run()`, add the composites call in the try block. Insert it AFTER the gap re-evaluation block and BEFORE `_run_analytical_engines(state)`:

```python
            # Phase 50: Signal composites (brain-level grouped analysis)
            try:
                _run_composites(state)
            except Exception:
                logger.warning("Composite evaluation failed (non-fatal)", exc_info=True)

            # Phase 26: Enhanced analytical engines (graceful degradation)
            _run_analytical_engines(state)
```

The try/except ensures composite failures don't block the pipeline (same pattern as other engines).

**Key constraints:**

- Composites run BEFORE analytical engines and section assessments
- Composite failure is non-fatal (graceful degradation)
- Follows the existing pattern in __init__.py (each engine wrapped in try/except)

**Part A: Create validation/health_summary.py**

New file (under 500 lines). Contains:

1. **Pydantic models:**
```python
class AnomalyWarning(BaseModel):
    level: str  # "WARNING" or "INFO"
    message: str
    detail: str

class HealthSummary(BaseModel):
    total_signals: int
    evaluated: int       # TRIGGERED + CLEAR + INFO
    triggered: int
    clear: int
    skipped: int
    info: int
    anomalies: list[AnomalyWarning]
    by_section: dict[str, dict[str, int]]  # prefix -> {triggered, clear, skipped, info}
```

2. **compute_health_summary(state) -> HealthSummary:**
   - Extract signal results from `state` using the same backward-compat pattern as `cli_brain_trace.py::_get_signal_results()` -- check both `signal_results` and `check_results` keys
   - Count signals by status (TRIGGERED, CLEAR, SKIPPED, INFO)
   - Group by signal prefix (first component of signal_id: BIZ, FIN, GOV, etc.) for the by_section breakdown
   - Run anomaly rules (see below)
   - Return HealthSummary

3. **Anomaly detection rules (heuristic, not ML):**
   Define as module-level constants for easy adjustment:
   ```python
   MAX_SKIPPED_THRESHOLD = 45  # Consistent with test_brain_contract.py
   ```

   Rules:
   - `_check_zero_triggered_with_litigation(state, results)`: If 0 TRIGGERED but litigation data present (state.extracted.litigation.active_cases non-empty), emit WARNING
   - `_check_high_skipped(results)`: If SKIPPED count > MAX_SKIPPED_THRESHOLD, emit WARNING
   - `_check_section_all_skipped(results)`: For each prefix section, if ALL signals are SKIPPED, emit WARNING naming the section
   - `_check_triggered_count_change(results, state)`: If `brain_signal_runs` data is available, compare triggered count with last run. If changed by >50%, emit INFO. (Optional -- skip if brain DB not accessible)

   Each rule returns `AnomalyWarning | None`. Collect non-None results into the anomalies list.

4. **print_health_summary(health: HealthSummary):**
   Rich-formatted CLI output. Use Rich Panel and Table:
   ```
   Signal Health Summary
   +-----------+-------+
   | Status    | Count |
   +-----------+-------+
   | Evaluated |   285 |
   | TRIGGERED |    42 |
   | CLEAR     |   193 |
   | INFO      |    50 |
   | SKIPPED   |    38 |
   +-----------+-------+

   By Section:
   BIZ: 12 triggered, 8 clear, 2 skipped
   FIN: 8 triggered, 15 clear, 0 skipped
   ...

   Anomalies: (0 warnings, 1 info)
     INFO: TRIGGERED count 42 is 15% higher than last run (37)
   ```

   Use color: green for CLEAR, yellow for TRIGGERED, red for WARNING anomalies, dim for INFO.

**Part A: Confidence verb mapping (NARR-03)**

Define `CONFIDENCE_VERBS` dict mapping confidence tiers to verb forms:
```python
CONFIDENCE_VERBS: dict[str, str] = {
    "HIGH": "confirms",
    "MEDIUM": "indicates",
    "LOW": "suggests",
    "INFERENCE": "pattern may indicate",
}
```

Create `calibrate_verb(confidence: str) -> str` function that returns the appropriate verb for a confidence tier. Default to "suggests" for unknown tiers. This is used by the thesis builder in narrative.py.

Create `calibrate_narrative_text(text: str, confidence: str) -> str` function that:
1. Takes a narrative text string and an overall confidence tier
2. Replaces generic assertion verbs ("shows", "has", "is") at sentence starts with the calibrated verb form
3. Only modifies the first verb per sentence to avoid over-calibration
4. Returns the modified text

Create `derive_section_confidence(state: AnalysisState, section_id: str) -> str` function that determines the dominant confidence tier for a section by:
1. Checking signal_results for the section -- count sources by confidence tier
2. Checking SourcedValue fields on relevant state objects
3. Returning the mode (most common) confidence tier, defaulting to "MEDIUM"

**Part B: Bull/Bear case extraction (NARR-02)**

Create `extract_bull_bear_cases(state: AnalysisState) -> dict[str, dict[str, Any]]` that returns a dict keyed by template section key ("executive_summary", "scoring") containing:

For executive_summary:
- `bull_case`: dict with `title` (str), `items` (list of dicts with `text`, `source`, `confidence`)
  - Source: state.executive_summary.key_findings.positives (each KeyFinding has evidence_narrative, section_origin)
  - If no positives, derive from CLEAN/FAVORABLE density sections
- `bear_case`: dict with `title` (str), `items` (list of dicts with `text`, `source`, `confidence`, `severity`)
  - Source: state.executive_summary.key_findings.negatives (each KeyFinding has evidence_narrative, scoring_impact, theory_mapping)
  - Supplement with peril_map.bear_cases if available (BearCase has theory, committee_summary, severity_estimate)
- `confidence_tier`: overall confidence tier for display

For scoring:
- `bull_case`: dict with items derived from favorable factor scores (factors scoring below threshold) and defense assessments from bear_cases
- `bear_case`: dict with items derived from high-risk factor scores, triggered red flags, and peril bear_cases
- `confidence_tier`: overall confidence tier

Each bull/bear item dict: `{"text": str, "source": str, "severity": str}` where severity is "" for bull items and "HIGH"/"MEDIUM" for bear items.

Cap at 5 items per case for readability.

**Part A: Bull/Bear Jinja2 macro**

Add `bull_bear_framing(data)` macro to `src/do_uw/templates/html/components/narratives.html.j2` (currently 142 lines, plenty of room). The macro takes a dict with `bull_case` and `bear_case` keys.

Layout: Two-column grid (bull left, bear right) at lg breakpoint, stacked on mobile.

```jinja2
{% macro bull_bear_framing(data) %}
{% if data and (data.bull_case or data.bear_case) %}
<div class="bull-bear-grid my-3">
  {% if data.bull_case %}
  <div class="bull-case">
    <h4 class="case-header bull-header">{{ data.bull_case.title|default('Bull Case') }}</h4>
    <ul class="case-items">
      {% for item in data.bull_case.items %}
      <li>{{ item.text }}{% if item.source %} <span class="source-tag">[{{ item.source }}]</span>{% endif %}</li>
      {% endfor %}
    </ul>
  </div>
  {% endif %}
  {% if data.bear_case %}
  <div class="bear-case">
    <h4 class="case-header bear-header">{{ data.bear_case.title|default('Bear Case') }}</h4>
    <ul class="case-items">
      {% for item in data.bear_case.items %}
      <li class="{{ 'severity-high' if item.severity == 'HIGH' else 'severity-medium' if item.severity == 'MEDIUM' else '' }}">{{ item.text }}{% if item.source %} <span class="source-tag">[{{ item.source }}]</span>{% endif %}</li>
      {% endfor %}
    </ul>
  </div>
  {% endif %}
</div>
{% endif %}
{% endmacro %}
```

Write a Python script that programmatically adds `facet` and `display` fields to every signal entry across all 36 YAML files. This MUST be scripted (not manual) per research anti-pattern guidance.

**Facet assignment logic (deterministic by prefix):**
- `BIZ.*` -> `business_profile`
- `EXEC.*` -> `executive_risk`
- `FIN.*` -> `financial_health`
- `FWRD.*` -> `forward_looking`
- `GOV.*` -> `governance`
- `LIT.*` -> `litigation`
- `NLP.*` -> `filing_analysis`
- `STOCK.*` -> `market_activity`

**Display spec assignment logic (by signal characteristics):**
For `value_format`, infer from the signal's threshold type and name:
- Threshold type `percentage` -> `pct_1dp`
- Threshold type `boolean` or `presence` -> `boolean`
- Threshold type `numeric` with currency-related names (revenue, debt, compensation) -> `currency`
- Threshold type `numeric` with count-related names (count, number, size) -> `count`
- Threshold type `categorical` -> `categorical`
- Default: `text`

For `source_type`, infer from the signal's `data_strategy.required_data` or `data_locations`:
- Contains "10-K" or "10_K" -> `SEC_10K`
- Contains "DEF14A" or "DEF_14A" -> `SEC_DEF14A`
- Contains "8-K" -> `SEC_8K`
- Contains "web" or "news" -> `WEB`
- Contains "stock" or "price" or "market" -> `MARKET_DATA`
- Default: `DERIVED`

For `threshold_context`: leave as empty string (auto-populated at eval time per existing behavior).

**Script approach:**
1. Load each YAML file with `yaml.safe_load()`
2. For each signal entry in the file, add/update:
   - `facet: <facet_id>` based on prefix mapping
   - `display: {value_format: <inferred>, source_type: <inferred>, threshold_context: ""}`
3. Skip signals that already have a complete `display` spec (23 signals from Phase 48)
4. Write back with `yaml.dump()` using `default_flow_style=False`, `sort_keys=False` to preserve structure
5. After script runs, verify all 400 signals have both fields

**IMPORTANT:** The `facet` field should be a simple string field. If BrainSignalEntry doesn't currently accept `facet`, add it to the schema with `facet: str = ""` alongside the existing model_config `extra="allow"`.

```python
class FacetSpec(BaseModel):
    id: str          # Unique facet ID within section
    name: str        # Display heading
    render_as: str   # Template dispatch type
    signals: list[str] = []
    columns: list[str] = []
    template: str = ""  # Explicit template path override

class SectionSpec(BaseModel):
    id: str
    name: str
    display_type: str
    signals: list[str] = []
    facets: list[FacetSpec] = []
    display_config: dict[str, Any] = {}
```

Create the `sections/forward_looking/` and `sections/executive_risk/` template directories.

Each template follows the established facet template pattern (reference: sections/scoring/temporal_signals.html.j2, sections/governance/people_risk.html.j2). Templates render content from the context dict. Since these sections' data lives within the scoring/analysis context (the forward-looking and executive risk data is extracted by context_builders/analysis.py), templates access data through the template context variables already available.

**Forward-looking templates** (5 files):

1. `early_warning.html.j2` — Renders FWRD.WARN signals as a collapsible metric table. Uses `scoring` context for forward-looking indicators. Shows signal name + assessment value. Wraps in `<h3>Early Warning Signals</h3>` with `<details>` for collapsibility.

2. `event_catalysts.html.j2` — Renders FWRD.EVENT signals as a metric table. Event dates, catalyst types, risk assessment. Collapsible subsection.

3. `macro_risks.html.j2` — Renders FWRD.MACRO signals. Macro risk factors (FX, rates, geopolitical, regulatory). Collapsible subsection.

4. `disclosure_quality.html.j2` — Renders FWRD.DISC signals as KV pairs. Disclosure quality composite, MD&A depth, non-GAAP reconciliation. Collapsible subsection.

5. `narrative_coherence.html.j2` — Renders FWRD.NARRATIVE signals as KV pairs. 10-K vs earnings alignment, analyst skepticism, auditor CAMs. Collapsible subsection.

**Executive risk templates** (4 files):

1. `executive_profiles.html.j2` — Renders EXEC.PROFILE + EXEC.CEO + EXEC.CFO + EXEC.AGGREGATE signals. Board size, independence ratio, CEO/CFO risk scores, highest risk individual. KV table format.

2. `tenure_stability.html.j2` — Renders EXEC.TENURE + EXEC.DEPARTURE signals. C-suite turnover, CEO/CFO new status, departure timing. Metric table format.

3. `insider_trading.html.j2` — Renders EXEC.INSIDER signals. CEO/CFO net selling, cluster selling, non-10b5-1 trades. Metric table with red highlighting for concerning patterns.

4. `prior_litigation.html.j2` — Renders EXEC.PRIOR_LIT signals. Prior litigation involving any officer, CEO/CFO specific. Metric table.

**Template pattern** (all templates follow this structure):
```jinja2
{# [Section] — [Facet Name] facet template.
   Renders [description]. Part of facet dispatch for [section_id] section. #}

{% set data = executive_risk or {} %}  {# or appropriate context variable #}
<div class="facet-block" id="facet-{facet_id}">
  <h3>{Facet Display Name}</h3>
  {# Render content from context #}
  {% if data %}
    <table class="w-full border-collapse text-sm my-3">
      {# ... metric rows ... #}
    </table>
  {% else %}
    <p class="text-sm text-gray-500">Data not available.</p>
  {% endif %}
</div>
```

For forward-looking templates, data comes from the scoring context (forward-looking signals are part of the scoring analysis). For executive risk templates, data comes from `executive_risk` context variable (already extracted by `extract_executive_risk()` in context_builders/analysis.py).

Important: Templates should be GRACEFUL with missing data — always use `{% if data %}` guards and `data.get('field', default)` patterns. Show "Not available" rather than crash.

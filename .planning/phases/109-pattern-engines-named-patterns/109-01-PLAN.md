       - `PeerOutlierEngine` class implementing PatternEngine Protocol
       - `engine_id = "peer_outlier"`, `engine_name = "Peer Outlier"`
       - `evaluate()` receives signal_results (not used directly) and state (AnalysisState):
         Step 1: Extract `state.benchmarks.frames_percentiles` dict. If benchmarks is None or frames_percentiles is empty, return NOT_FIRED with "Insufficient peer data."
         Step 2: For each metric in frames_percentiles, extract company_value, peer_count_sector, sector percentile. Filter to metrics with peer_count_sector >= 10 (design doc minimum).
         Step 3: Compute z-scores using MAD (median absolute deviation) instead of std for robustness. For each metric: z = |company_value - median| / (1.4826 * MAD). The 1.4826 is the consistency constant for normal distributions.
         Step 4: Identify outlier metrics: z > 3.0 for single-metric outlier (design doc threshold). Also check multi-dimensional: if 3+ metrics have z > 2.0, flag as multi-dimensional outlier.
         Step 5: If 3+ outlier metrics found (any combination of single z>3 or multi-dimensional z>2), return fired=True with findings listing each outlier metric, its z-score, company value, and direction (higher_is_better context).
       - Per user decision: reuse existing SEC Frames data only. No new ACQUIRE step.
       - For higher_is_better=False metrics (e.g., debt-to-equity), extreme HIGH values are outliers. For higher_is_better=True (e.g., ROA), extreme LOW values are outliers. The z-score is unsigned, but findings include direction context.
       - Handle edge case: if all company_values are None for eligible metrics, return NOT_FIRED.
       - Thresholds: multi_z_threshold=2.0, single_z_threshold=3.0, min_peers=10, min_outlier_metrics=3 -- all configurable via constructor kwargs.

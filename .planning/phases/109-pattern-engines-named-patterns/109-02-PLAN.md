class PatternEngine(Protocol):
    @property
    def engine_id(self) -> str: ...
    @property
    def engine_name(self) -> str: ...
    def evaluate(self, signal_results: dict[str, Any], *, state: AnalysisState | None = None) -> EngineResult: ...

class EngineResult(BaseModel):

    engine_name: str
    fired: bool = False
    confidence: float = Field(default=0.0, ge=0.0, le=1.0)
    headline: str = ""
    findings: list[dict[str, Any]] = Field(default_factory=list)
    metadata: dict[str, Any] = Field(default_factory=dict)

    1. Create `src/do_uw/stages/score/precedent_match.py` (~280 lines):
       - `PrecedentMatchEngine` class implementing PatternEngine Protocol
       - `engine_id = "precedent_match"`, `engine_name = "Precedent Match"`
       - `_load_case_library()`: Load case_library.yaml from brain/framework/. Parse via CaseLibraryEntry Pydantic model. Cache as module-level lazy singleton (same pattern as brain signal cache from Phase 104). Return list[CaseLibraryEntry].
       - `_build_company_fingerprint(signal_results)`: Convert signal evaluation results to binary fingerprint dict[str, bool]. Signal is "fired" if status in {RED, YELLOW}. CLEAR and SKIPPED are False. This matches how case library profiles are recorded.
       - `_compute_signal_weights()`: CRF-related signal IDs (from hae_crf.py patterns, or signals with "CRF" in related red_flags) get 3x weight. All others get 1.0x. This captures the design doc's emphasis on critical risk factors in similarity.
       - `weighted_jaccard(company_fingerprint, case_fingerprint, weights)`: Implement from RESEARCH.md code example. Guard against division by zero.
       - `evaluate()`:
         Step 1: Load case library. If empty or unavailable, return NOT_FIRED.
         Step 2: Build company fingerprint from signal_results.
         Step 3: Compute weighted Jaccard similarity for each case.
         Step 4: Apply confidence weighting: HIGH confidence cases get 1.0x, MEDIUM get 0.8x, LOW (POST_FILING auto-added) get 0.6x. This multiplier adjusts the similarity score.
         Step 5: Sort by adjusted similarity descending. Top 3 matches go in findings.
         Step 6: For findings, include: case_id, case name (company_name), similarity score, outcome (settlement amount or "dismissed" or "ongoing"), claim_type, key note.
         Step 7: Dismissed cases: include in results but apply 0.5x to outcome severity weight (per user decision). This means the similarity score is unchanged, but the outcome_severity field is halved.
         Step 8: Determine if engine fired: similarity > notable_threshold (0.30 per design doc) for at least 1 match. Set confidence = max(top_similarities).
       - Thresholds: notable=0.30, strong=0.50, very_strong=0.70, crf_weight=3.0 -- all configurable.

<objective>
Distinguish company-issued forward guidance from analyst consensus estimates in FIN.GUIDE.* signals so non-guiding companies (like SNA) correctly show "No" for guidance instead of being evaluated against analyst estimates they never issued.

Purpose: SNA validation audit found FIN.GUIDE.current=Yes when Snap-on does not provide forward guidance. The current code treats yfinance analyst EPS estimates as company guidance. Per user decision: determine guidance via 10-K/10-Q language patterns; gate FIN.GUIDE.current/track_record/philosophy on provides_forward_guidance; keep earnings_reaction and analyst_consensus active for all companies.

Output: New provides_forward_guidance field, guidance detection function, gated mapper logic, passing tests.
</objective>

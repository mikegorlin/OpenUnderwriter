Create the OutputSanitizer class that performs a single post-render pass on completed HTML to catch any markdown artifacts, raw Python serialization, debug strings, and system jargon that leaked through existing Jinja filters.

  <action>
    Create `src/do_uw/stages/render/output_sanitizer.py` with:

    1. **SanitizationEntry dataclass**: `category: str` (markdown|python_serial|jargon|debug), `pattern_name: str`, `matched_text: str`, `replacement: str`, `line_context: str` (surrounding ~40 chars for debugging). Per SAN-05 — builds evidence for upstream fixes.

    2. **SanitizationLog dataclass**: `entries: list[SanitizationEntry]`, `total_substitutions: int`, method `summary() -> str` that returns a human-readable log.

    3. **OutputSanitizer class** with `sanitize(html: str) -> tuple[str, SanitizationLog]`:
       - Uses BeautifulSoup to parse HTML
       - Walks only text nodes (NavigableString) — skips script, style, code, pre tags
       - Skips text inside elements with `data-raw="true"` attribute (escape hatch for intentional raw content)
       - Applies pattern categories IN ORDER (most specific first to avoid mangling):

       **Category: markdown** (SAN-02)
       - `**text**` -> `text` (bold markers in rendered HTML text)
       - `*text*` -> `text` (italic markers — but NOT inside contractions like "it's")
       - `### heading` / `## heading` / `# heading` at start of text node
       - `---` or `***` horizontal rules as standalone text nodes
       - ``` ` ``` backtick code markers

       **Category: python_serial** (SAN-03)
       - `['item1', 'item2']` -> `item1, item2` (Python list repr)
       - `["item1", "item2"]` -> `item1, item2` (Python list repr double quotes)
       - `{'key': 'value'}` -> strip entirely (Python dict repr has no user value)
       - `{"key": "value"}` -> strip entirely
       - `None` as standalone word (not in "None of the" etc.) -> `N/A`
       - `True` / `False` as standalone -> `Yes` / `No`

       **Category: jargon** (SAN-04)
       - SCREAMING_SNAKE codes from _KNOWN_CODES set -> `N/A` or strip
       - `F.N = X/Y` factor code patterns -> strip
       - `(threshold: ...)` internal context -> strip
       - `N signals triggered` prefix -> strip
       - `triggered SIGNAL_NAME (FN) —` prefix -> strip
       - `coverage=100%` / `coverage_display` internal refs -> strip
       - `evaluation_method:` / `schema_version:` debug keys -> strip

       **Category: debug** (SAN-04 catch-all)
       - `eval(...)` / `repr(...)` patterns
       - Strings starting with `<class '` or `<module '`
       - Raw exception tracebacks (`Traceback (most recent call last)`)

    4. Each pattern match creates a SanitizationEntry and appends to the log.
    5. After all text nodes processed, return `(str(soup), log)`.
    6. Log a WARNING via Python logging if `log.total_substitutions > 0`, INFO with summary.
    7. Class method `OutputSanitizer.from_defaults()` for standard config.

    IMPORTANT: Do NOT import or call existing strip_jargon/clean_narrative_text — those are pre-render Jinja filters. The OutputSanitizer re-implements the patterns it needs for post-render HTML context (operating on BeautifulSoup text nodes, not raw strings). This avoids coupling and ensures the post-render pass works independently.

    Create `tests/stages/render/test_output_sanitizer.py` with tests for every behavior listed above plus:
    - Edge case: empty string input
    - Edge case: HTML with no text nodes
    - Edge case: nested tags with mixed clean/dirty text
    - Performance: sanitize a 500KB HTML string in under 2 seconds

  <name>Task 1: Contract validation library</name>
  <files>src/do_uw/brain/contract_validator.py, tests/brain/test_contract_enforcement.py</files>
  <behavior>
    - validate_facet_template_agreement: returns violations when a manifest facet references a template that does not exist on disk
    - validate_facet_template_agreement: returns violations when a .html.j2 file exists in sections/ subdirectories but is not declared in any manifest facet
    - validate_signal_references: returns violations when a facet's signals list references a signal ID not found in brain YAML
    - validate_signal_references: facets with empty signals list are valid (data-display-only)
    - validate_all_contracts: runs all validators, returns ContractReport with pass/fail and all violations
    - Intentional breakage (tmp manifest missing a template, tmp signal YAML missing a referenced ID) produces clear ContractViolation with violation_type, facet_id, and detail message
  </behavior>
  <action>
    Create contract_validator.py with Pydantic models:
    - ContractViolation(violation_type: str, section_id: str, facet_id: str, detail: str)
    - ContractReport(valid: bool, violations: list[ContractViolation])

    Implement 3 validation functions:

    1. validate_facet_template_agreement(manifest, template_root) -> list[ContractViolation]:
       - For each facet in manifest, check template_root / facet.template exists
       - For each section in manifest, check template_root / section.template exists
       - Scan template_root/sections/ subdirs for .html.j2 files; flag any not in manifest facets or sections
       - Exclude component templates (components/), appendices (appendices/), base.html.j2, worksheet.html.j2 from orphan check
       - Also exclude top-level section wrappers that ARE in the manifest (like sections/financial.html.j2)

    2. validate_signal_references(manifest, signals_by_id) -> list[ContractViolation]:
       - For each facet with non-empty signals list, verify each signal ID exists in signals_by_id set
       - Facets with empty signals list are valid (data-display-only)

    3. validate_all_contracts(manifest_path, template_root, signals_dir) -> ContractReport:
       - Calls both validators, aggregates violations
       - valid = len(violations) == 0

    Write TDD-style tests that:
    - Create tmp manifest YAML + tmp template dirs + tmp signal YAML for isolated testing
    - Test happy path (all agree), missing template, orphaned template, broken signal ref, empty signals OK
    - Test that violation messages include the facet_id and detail for actionability
  </action>

  </verify>

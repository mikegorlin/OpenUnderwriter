**2. Create `src/do_uw/stages/analyze/credibility_engine.py`:**

Function: `compute_credibility_score(state: AnalysisState) -> CredibilityScore`

Implementation:
- Read `state.extracted.market.earnings_guidance` (EarningsGuidanceAnalysis)
- For each quarter in `earnings_guidance.quarters` where both guided and actual values exist:
  - Create `CredibilityQuarter` with quarter, metric="EPS", guided_value, actual_value
  - Compute beat_or_miss: actual > guided = "BEAT", actual < guided = "MISS", within 1% = "INLINE"
  - Compute magnitude_pct: `(actual - guided) / abs(guided) * 100`
- Compute beat_rate_pct: `beats / total_assessed * 100`
- Compute credibility_level:
  - beat_rate_pct > 80 -> "HIGH"
  - beat_rate_pct >= 50 -> "MEDIUM"
  - beat_rate_pct < 50 -> "LOW"
  - quarters_assessed == 0 -> "UNKNOWN"
- Per CONTEXT.md locked decision: yfinance provides EPS estimates vs actuals as primary data source. If LLM-extracted company-specific guidance is available (from 8-K), prefer that for the credibility assessment. Fall back to yfinance consensus with a note in `source` field.
- Return CredibilityScore with populated quarter_records

**Create `src/do_uw/stages/analyze/miss_risk.py`:**

**Function 1:** `compute_miss_risk(current_value: float | None, guidance_midpoint: float | None, credibility_level: str) -> str`

Per CONTEXT.md locked decision:
- If current_value or guidance_midpoint is None: return "UNKNOWN"
- Compute gap_pct = `abs(current - guidance) / abs(guidance) * 100`
- Base risk: gap_pct > 10 -> HIGH (2), 5-10 -> MEDIUM (1), <5 -> LOW (0)
- Credibility adjustment: "LOW" (<50% beat) -> +1 level (cap at 2), "HIGH" (>80% beat) -> -1 level (floor at 0)
- Return ["LOW", "MEDIUM", "HIGH"][adjusted_level]

**Function 2:** `map_sca_relevance(miss_risk: str, is_material: bool = False, is_financial: bool = False) -> str`

Per CONTEXT.md locked decision -- deterministic mapping, NOT LLM-generated:
- HIGH miss risk + material metric -> "10b-5: misleading forward guidance -- material metric miss creates scienter inference"
- HIGH miss risk + financial metric -> "10b-5: misleading forward guidance -- financial metric miss supports fraud-on-the-market"
- MEDIUM miss risk + financial metric -> "Potential earnings fraud theory -- moderate gap in financial metric guidance"
- MEDIUM miss risk + material non-financial -> "Potential operational misrepresentation -- moderate gap may support Section 11 claim"
- LOW or UNKNOWN -> "" (no SCA relevance)

A metric is "material" if it's revenue, EPS, net income, or operating income. A metric is "financial" if it involves dollar amounts or per-share values.

**Function 3:** `enrich_forward_statements(statements: list[ForwardStatement], credibility: CredibilityScore) -> list[ForwardStatement]`

For each ForwardStatement:
- If guidance_midpoint and current_value_numeric available: compute miss_risk
- Apply credibility_level from CredibilityScore
- Map SCA relevance based on metric_name (classify as material/financial)
- Set miss_risk_rationale with gap percentage and credibility adjustment explanation
- Return enriched list

**Function:** `compute_monitoring_triggers(state: AnalysisState) -> list[MonitoringTrigger]`

- Load `brain/config/monitoring_triggers.yaml`
- For each trigger definition, compute company-specific threshold:
  - MON-01 (SCA Filing): threshold = "Any Stanford SCAC match against {company_name}" -- always this value
  - MON-02 (Stock Below Support): read `state.extracted.market.stock_performance` for 52-week low. Threshold = f"Price below ${year_low:.2f} (52-week low)"

- Set `current_value` from state data where available (e.g., current stock price, current insider selling rate)

**2. Create `src/do_uw/stages/benchmark/quick_screen.py`:**

**Function 1:** `check_nuclear_triggers(state: AnalysisState) -> list[NuclearTriggerCheck]`

Per CONTEXT.md locked decision -- 5 deterministic checks with positive evidence:
- Load `brain/config/nuclear_triggers.yaml`
- For each nuclear trigger:
  - NUC-01 (Active SCA): Check `state.extracted.litigation` for active_cases list. `fired = len(active_cases) > 0`. Evidence = case names if fired, "Stanford SCAC clean" if not.
  - NUC-02 (SEC Enforcement): Check `state.extracted.litigation` for sec_enforcement. `fired = bool(sec_enforcement)`. Evidence = action details or "No SEC enforcement actions found".
  - NUC-03 (Financial Restatement): Check state for restatement flags (signal results or extracted data). `fired = restatement_detected`. Evidence = restatement details or "No financial restatements disclosed".
  - NUC-04 (CEO/CFO Departure Under Pressure): Check `state.extracted.governance` for recent executive departures with concerning circumstances. Evidence = departure details or "No departures under pressure identified".
  - NUC-05 (Going Concern): Check `state.extracted.financials` for going concern flag. `fired = going_concern`. Evidence = auditor opinion or "Clean audit opinion".
- Return list of NuclearTriggerCheck

**Function 2:** `build_trigger_matrix(signal_results: dict[str, Any]) -> list[TriggerMatrixRow]`

Per CONTEXT.md locked decision -- signal results aggregation:
- Import `get_signal_result`, `get_signals_by_prefix` from `_signal_consumer`
- Iterate all signal results
- Filter to signals where: status == "TRIGGERED" AND threshold_level in ("red", "yellow")
- Per Pitfall 3 from RESEARCH.md: filter to `signal_class: evaluative` or `signal_class: inference` only. Skip display-type signals.
- Group by section (map signal_id prefix to section name using PREFIX_TO_SECTION mapping):
  - SCAC/LITIG -> "Litigation", FIN/DIST/EARN -> "Financial", GOV/BOARD -> "Governance", MKT/STOCK -> "Market", FWRD -> "Forward-Looking", etc.
- Per section: sort by severity (RED first), limit to top 3 signals per section
- Map each to TriggerMatrixRow with: signal_id, signal_name (from brain metadata), flag_level, section, section_anchor (e.g., "#section-litigation"), do_context
- Sort final list: RED first, then YELLOW, then alphabetical by signal_id
- Return list of TriggerMatrixRow

**Function 3:** `compute_prospective_checks(state: AnalysisState) -> list[ProspectiveCheck]`

5 forward-looking checks per TRIGGER-02:
1. **Earnings expectations**: Check if forward EPS from yfinance shows negative revision trend. Status = GREEN if stable/up, YELLOW if declining, RED if steep decline.
2. **Major contract/deal**: Check 10-K/8-K extraction for pending M&A or major contracts. Status = GREEN if none, YELLOW if pending, RED if contentious.
3. **Regulatory decision**: Check regulatory_data from state. Status = GREEN if no pending, YELLOW if pending, RED if adverse.
4. **Competitive disruption**: Check sector classification for disruption indicators. Status = GREEN/YELLOW/RED based on findings.
5. **Macro headwinds**: Check web search blind spot results for macro concerns. Status based on findings.
Each check: populate finding with actual data, source with data source. Status = "UNKNOWN" if data unavailable.

**Function 4:** `build_quick_screen(state: AnalysisState, signal_results: dict[str, Any]) -> QuickScreenResult`

Assembles all quick screen components:
- `trigger_matrix = build_trigger_matrix(signal_results)`
- `nuclear_triggers = check_nuclear_triggers(state)`
- `nuclear_fired_count = sum(1 for nt in nuclear_triggers if nt.fired)`
- `prospective_checks = compute_prospective_checks(state)`
- `red_count = sum(1 for r in trigger_matrix if r.flag_level == "RED")`
- `yellow_count = sum(1 for r in trigger_matrix if r.flag_level == "YELLOW")`
- Return QuickScreenResult

**calibration/runner.py:**
Create `CalibrationRunner` class following the `ValidationRunner` pattern from `validation/runner.py`. Key differences:
1. After pipeline execution for each ticker, load the `state.json` from output dir
2. Parse `AnalysisState` and extract: `state.analysis.check_results` (dict of CheckResult), `state.scoring` (factor scores, quality_score, tier, patterns_detected), `state.extracted` metadata
3. Store per-ticker results in a `CalibrationTickerResult` dataclass with fields: ticker, expected_tier (from config), actual_tier, quality_score, check_results (dict[str, CheckResultSummary]), patterns_detected (list[str]), factor_scores (dict[str, float]), duration_seconds, error (str | None)
4. `CheckResultSummary` dataclass: check_id, status (TRIGGERED/CLEAR/SKIPPED/INFO), value (Any), threshold_level (str | None), evidence (str), factors (list[str])
5. `CalibrationReport` dataclass: tickers (dict[str, CalibrationTickerResult]), run_date, total_duration, errors (list[str])
6. Support checkpointing (write `.calibration_checkpoint.json` after each ticker)
7. Support `--fresh` and `--no-llm` flags passed through to Pipeline
8. Use `record_analysis_run()` from `knowledge.learning` for each completed ticker to feed the learning infrastructure

Use Pydantic models for CalibrationTickerResult and CalibrationReport (project standard). Import AnalysisState from do_uw.models.state. The runner reads state.json with `json.loads()` + `AnalysisState.model_validate()` after pipeline completes.

**calibration/__init__.py:**
Export CalibrationRunner, CalibrationReport, CalibrationTickerResult from runner.py. Export get_calibration_tickers from config.py.

**impact_ranker.py:**
Create `ImpactRanker` class that ranks all checks by impact score = weight x fire_rate x severity_contribution.

**Algorithm (per CONTEXT.md decision -- auto-ranked, NOT manual):**
1. Load `scoring.json` to get factor max_points: for each factor in `scoring_config["factors"]`, extract `factor_id` and `max_points`
2. For each CheckMetrics:
   - weight = max(factor_weights[f] for f in check.factors) or 0 if no factors mapped
   - fire_rate = check.fire_rate
   - severity = average of threshold_level values across triggered tickers: red=3.0, yellow=2.0, clear=0.0. If no triggered tickers, severity=0.
   - impact_score = weight * fire_rate * severity
3. Sort descending by impact_score
4. Return top N as `list[RankedCheck]`

**RankedCheck** Pydantic model: check_id, check_name, impact_score, weight, fire_rate, severity, factors, rank (1-based)

**ImpactRanker methods:**
1. `__init__(scoring_config_path: Path | None = None)` -- load scoring.json from default config path if not provided
2. `rank(metrics: list[CheckMetrics], top_n: int = 20) -> list[RankedCheck]` -- rank and return top N
3. `get_unmapped_checks(metrics: list[CheckMetrics]) -> list[CheckMetrics]` -- checks with no factors (impact_score = 0)
4. `get_factor_distribution(metrics: list[CheckMetrics]) -> dict[str, int]` -- count of checks per factor (to identify the F10=102 vs F8=2 anomaly)

**tests/test_calibration_analyzer.py:**
1. Test CheckAnalyzer with synthetic CalibrationReport (3 tickers, 5 checks each):
   - One check that fires for all 3 tickers (100% fire rate)
   - One check that fires for 0 tickers (0% fire rate)
   - One check skipped for 2/3 tickers (high skip rate)
   - One check with generic evidence (LOW quality)
   - One check with good evidence (HIGH quality)
2. Test `get_dead_checks()` returns the 0% fire rate check
3. Test `get_always_fire_checks()` returns the 100% fire rate check
4. Test `get_low_evidence_checks()` returns the LOW quality check
5. Test ImpactRanker with synthetic metrics:
   - Check mapped to F1 (20 points) with 50% fire rate, severity 3.0 -> impact 30.0
   - Check mapped to F10 (2 points) with 100% fire rate, severity 2.0 -> impact 4.0
   - Verify F1 check ranks higher despite lower fire rate
6. Test `get_unmapped_checks()` returns checks with empty factors list
7. Test `get_factor_distribution()` returns correct counts
8. Test `rank(top_n=2)` returns exactly 2 results

Build synthetic CalibrationReport/CalibrationTickerResult/CheckResultSummary fixtures that don't require Plan 01 -- construct them directly as dicts or minimal dataclass instances if the Pydantic models aren't available yet (use `if TYPE_CHECKING` imports or construct test-local equivalents).

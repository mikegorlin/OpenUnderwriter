  <action>
Create `PlaybookValidator` class that validates each industry playbook against calibration results.

**PlaybookValidationResult** Pydantic model:
- playbook_id: str (e.g., "TECH_SAAS")
- playbook_name: str
- validation_depth: str ("deep" or "light")
- tickers_tested: list[str]
- total_industry_checks: int (how many checks the playbook defines)
- industry_checks_fired: int (how many actually triggered for at least 1 ticker)
- industry_checks_skipped: int (always SKIPPED across all tickers)
- industry_checks_clear: int (never triggered but ran successfully)
- differentiation_score: float (0.0-1.0: ratio of fired to total)
- differentiation_grade: str ("STRONG" >= 0.5, "PARTIAL" >= 0.2, "WEAK" >= 0.1, "NONE" < 0.1)
- non_functional_checks: list[str] (ALL SKIPPED across all tickers)
- claim_theories_validated: list[str] (from playbook data, checked against check firings)
- claim_theories_unvalidated: list[str] (theories with no supporting check firings)
- findings: list[str] (human-readable observations)

**PlaybookValidator methods:**

1. `__init__(report: CalibrationReport, config: list[CalibrationTicker])` -- store inputs

2. `validate_playbook(playbook_id: str, tickers: list[str], depth: str = "light") -> PlaybookValidationResult` -- validate one playbook:
   a. Load playbook definition from playbook_data.py using the existing playbook infrastructure
   b. Get industry-specific check IDs from the playbook (these are checks with IDs matching the playbook's section pattern, e.g., checks tagged with the playbook's industry)
   c. For each ticker: look at that ticker's check_results in CalibrationReport. Count how many industry checks are TRIGGERED, CLEAR, SKIPPED
   d. Compute differentiation_score = industry_checks_fired / total_industry_checks
   e. Identify non-functional checks (SKIPPED for ALL tickers)
   f. For deep validation: also check claim_theories from playbook data. A theory is "validated" if at least one check related to it fired. A theory is "unvalidated" if none of its related checks fired.
   g. Generate findings strings describing what was found

3. `validate_all() -> list[PlaybookValidationResult]` -- validate all 10 playbooks:
   - Use DEEP_VALIDATION_VERTICALS from config.py for deep validation
   - Use LIGHT_VALIDATION_VERTICALS for light validation
   - For playbooks not in either set, use any available ticker that matches the industry

4. `generate_report(results: list[PlaybookValidationResult]) -> str` -- Markdown report:
   ```
   # Industry Playbook Validation Report

   ## Summary
   | Playbook | Depth | Tickers | Fired/Total | Differentiation |
   ...

   ## Deep Validation Details
   ### TECH_SAAS (AAPL, SMCI, NVDA)
   #### Check Results
   | Check | AAPL | SMCI | NVDA |
   ...
   #### Claim Theory Validation
   - [VALIDATED] "Patent trolls target tech companies..." -- check SECT6.PATENT fires for 2/3
   - [UNVALIDATED] "Cloud revenue concentration risk..." -- no related checks fire
   #### Findings
   - ...

   ## Non-Functional Industry Checks
   | Playbook | Check ID | Status |
   ...
   ```

5. `get_weak_playbooks() -> list[PlaybookValidationResult]` -- differentiation_grade is "WEAK" or "NONE"

**Claim theory to check mapping:**
Each playbook in playbook_data.py has `claim_theories` (list of theory objects). Each theory has keywords and risk areas. Map these to checks by matching check_id prefixes or factor assignments. This mapping is heuristic -- a theory about "revenue recognition" maps to checks containing "REV" or "RESTATE" in their ID. Document the mapping approach in code comments.

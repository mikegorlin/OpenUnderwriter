  <action>
Create `EnrichmentPipeline` class that processes calibration results and enriches the knowledge store.

**EnrichmentPipeline methods:**

1. `__init__(store: KnowledgeStore, report: CalibrationReport, metrics: list[CheckMetrics], anomalies: list[CalibrationAnomaly])` -- store all inputs

2. `record_analysis_runs() -> int` -- for each ticker in report, call `learning.record_analysis_run()` with the AnalysisOutcome (ticker, run_date, checks_fired, checks_clear, quality_score, tier). Return count of runs recorded.

3. `create_incubating_observations() -> int` -- for each CalibrationAnomaly, create an INCUBATING note in the knowledge store using `store.add_note()`. Tag with "calibration,incubating,{anomaly_type}". Content is JSON with anomaly details. Return count of observations created. Per user decision: all observations start as INCUBATING.

4. `discover_new_patterns() -> list[dict[str, Any]]` -- co-firing analysis:
   - From learning data, get all check pairs that co-fire >70% of the time (Jaccard similarity)
   - Compare against existing 17 composite patterns in patterns.json
   - Co-firing clusters NOT covered by existing patterns are "new pattern candidates"
   - Create INCUBATING notes for each new candidate with tags "calibration,incubating,new_pattern"
   - Return list of new pattern descriptions

5. `find_redundant_pairs() -> list[tuple[str, str, float]]` -- call `learning.find_redundant_pairs(threshold=0.85)`. Per user decision: flag for human review, do NOT auto-deprecate. Create notes tagged "calibration,redundancy_review" for each pair. Return list of (check_a, check_b, similarity).

6. `auto_promote(n_threshold: int = 3) -> int` -- query INCUBATING observations. If an observation type (by check_id or pattern) is confirmed across n_threshold or more tickers (check the tickers_affected in content JSON), promote from INCUBATING to CANDIDATE by updating the note tag. Per user decision: N=3 is the default threshold. Return count promoted.

7. `run_full_enrichment(auto_promote: bool = False) -> EnrichmentSummary` -- orchestrate steps 2-6. EnrichmentSummary Pydantic model with: runs_recorded, observations_created, new_patterns_discovered, redundant_pairs_found, promotions (if auto_promote).

**EnrichmentSummary** Pydantic model:
- runs_recorded: int
- observations_created: int
- new_patterns_discovered: int
- redundant_pairs_found: int
- promotions: int

**weight_adjuster.py:**
Create `WeightAdjuster` class that proposes scoring weight adjustments within +/-10% bounds per user decision.

**WeightAdjustment** Pydantic model:
- factor_id: str
- factor_name: str
- current_weight: float (max_points)
- proposed_weight: float
- change_pct: float (positive = increase, negative = decrease)
- reason: str (evidence-based justification)
- requires_approval: bool (True if change > 10%)

**Adjustment logic:**
1. Load current weights from scoring.json
2. For each factor, analyze calibration data:
   - If factor has very few checks mapped (like F8 with 2) but high expected importance, propose weight reduction or flag for check additions
   - If checks in a factor consistently mismatch expected tiers (clean companies trigger red), propose weight reduction
   - If checks in a factor show strong correlation with tier accuracy, propose weight increase
3. Cap all adjustments at +/-10% of current max_points
4. Mark anything requiring >10% as requires_approval=True

**WeightAdjuster methods:**
1. `__init__(scoring_config_path: Path | None = None)` -- load scoring.json
2. `analyze(metrics: list[CheckMetrics], report: CalibrationReport) -> list[WeightAdjustment]` -- compute proposed adjustments
3. `apply_adjustments(adjustments: list[WeightAdjustment], dry_run: bool = True) -> dict[str, Any]` -- if dry_run=True, return what would change. If False, write updated scoring.json. NEVER apply requires_approval=True changes without explicit flag.
4. `generate_adjustment_report(adjustments: list[WeightAdjustment]) -> str` -- Markdown table of proposed changes

**cli_calibrate.py updates:**
Update the `calibrate enrich` command to:
1. Load CalibrationReport from JSON
2. Run CheckAnalyzer + AnomalyDetector
3. Create EnrichmentPipeline with KnowledgeStore
4. Call `run_full_enrichment(auto_promote=auto_promote_flag)`
5. Print EnrichmentSummary with Rich
6. Optionally run WeightAdjuster and print proposed adjustments

    1. Create `src/do_uw/stages/resolve/ticker_resolver.py`:
       - `resolve_ticker(input_str: str, cache: AnalysisCache | None = None) -> ResolvedTicker`: Main entry point. Detects if input is a ticker (all uppercase, <= 5 chars, no spaces) or a company name.
       - If ticker: Call SEC's `company_tickers.json` (via `sec_get` from rate_limiter, cached in AnalysisCache with 30-day TTL). Look up ticker to get CIK and company name. Return immediately if found.
       - If company name: Download and cache `company_tickers.json`, use `rapidfuzz.process.extractOne()` with `fuzz.WRatio` scorer against all company names. Threshold of 80 for auto-match. If best score >= 90 AND second-best < 80, auto-proceed. If ambiguous (multiple matches > 80), return all matches with scores for user selection.
       - `ResolvedTicker` is a simple dataclass: `ticker: str, cik: int, company_name: str, confidence: float` (0-100 fuzzy match score, 100 for exact ticker match)
       - Parent entity resolution: Group entries from company_tickers.json by CIK. If multiple tickers share the same CIK (e.g., GOOG/GOOGL -> CIK 1652044), use the first ticker alphabetically as canonical but store all in `all_tickers`.
       - Keep under 200 lines.

    2. Create `src/do_uw/stages/resolve/sec_identity.py`:
       - `resolve_company_identity(cik: int, ticker: str, cache: AnalysisCache | None = None) -> CompanyIdentity`: Given a resolved CIK, fetch full company identity from SEC.
       - Primary path: Try edgartools MCP first. Call `Company(ticker)` via MCP -- the exact MCP tool name must be discovered at runtime. If MCP is not available (no MCP context), fall back to direct API.
       - Fallback path: Call SEC submissions API `https://data.sec.gov/submissions/CIK{zero_padded_cik}.json` via `sec_get()`. Parse response to extract: name, sic, sicDescription, stateOfIncorporation, fiscalYearEnd, exchanges, tickers, entityType.
       - FPI detection: Check `entityType` field for "foreign-private-issuer" or check if filing history contains 20-F filings. Set `is_fpi` flag accordingly.
       - Map SIC code to sector: Load `sectors.json` from brain/ config using ConfigLoader, map SIC code to sector. If no exact match, use SIC division (first digit ranges).
       - Wrap every field in `SourcedValue` with `source="SEC EDGAR submissions"`, `confidence=Confidence.HIGH`, `as_of=datetime.now(UTC)`.
       - Cache the full submissions response with key `sec:{ticker}:identity:{date}` and 30-day TTL.
       - IMPORTANT: Since this code runs inside the pipeline (not in a context where MCP tools are directly available via Claude), the edgartools MCP path should be designed as a callable function parameter or feature flag. For Phase 2, use the direct SEC EDGAR API as the primary implementation. The MCP integration can be added when the actual MCP tool invocation pattern is established. Do NOT import or hardcode MCP tool calls.
       - Keep under 250 lines.

    3. Replace `src/do_uw/stages/resolve/__init__.py` stub with real implementation:
       - `ResolveStage.run(state)`:

         c. Populate `state.company = CompanyProfile(identity=resolved_identity)` -- set identity fields with SourcedValue wrappers

         e. Fetch employee_count from yfinance similarly
         f. Log what was resolved: company name, CIK, exchange, FPI status
         g. Handle errors gracefully: if SEC resolution fails entirely, raise with clear error message. If yfinance fails for market_cap, log warning but continue (not a hard requirement for resolve stage).
       - Keep `validate_input` checking that ticker is non-empty.
       - Import AnalysisCache and instantiate it for cache usage.
       - Keep under 150 lines.

<decisions_made>

Brain architecture decisions from this session's conversation:

- **Work type is the missing fundamental dimension**: extract | evaluate | infer
  (not content_type/category/hazard_or_signal — those were overlapping proxies)

- **3 axes, not 1**: every check has work_type (what it does) + risk position (where in framework)
  + acquisition_tier (how hard to get data). These are orthogonal.

- **Presentation in checks = just 2 fields**: worksheet_section (semantic grouping) +
  display_when (filtering logic). NOT layout/HTML/template details — renderer owns those
  via work_type → component mapping.

- **DuckDB = pure cache**: rebuilt from YAML via brain build. Nothing enters DuckDB that
  isn't first declared in YAML. brain_loader still reads DuckDB for performance.

- **Split checks by domain prefix**: brain/checks/biz.yaml, fin.yaml, gov.yaml, exec.yaml,
  lit.yaml, stock.yaml, fwrd.yaml, nlp.yaml (8 files, matches existing ID prefixes)

- **117 checks already cross-referenced in causal_chains.yaml**: can auto-populate
  chain_roles for these during migration. 283 checks have no chain — correctly flagged
  as unlinked (human backlog), not silently wrong.

- **Deprecation list after migration**:
  checks.json, patterns.json, red_flags.json, brain_migrate_config.py,
  brain_migrate_framework.py, brain_migrate_scoring.py (simplified),
  enrichment_data.py, enrichment_data_ext.py, BackwardCompatLoader

- **Article → brain flow**: read → identify claim → new check OR update existing OR
  extend chain → express in YAML with provenance → brain build → in next analysis

<context>
The brain currently has a fundamental coherence problem: framework YAML (good) and checks JSON
(disconnected) are two separate systems. 0/400 checks know their peril or causal chain.
Multiple overlapping classification fields (content_type, category, hazard_or_signal, signal_type)
with no governing rule. The user correctly diagnosed this as "necessary but insufficient."

Phase 44 fixes this by making every check a self-describing knowledge unit that knows its
position in the risk framework, how it got its data, when to display, and where the knowledge
came from (provenance). DuckDB becomes a pure cache. The brain becomes live: read an article,
write a YAML entry with provenance, run brain build, used in next analysis.

The design is fully captured in the ROADMAP.md Phase 44 entry — the planner has everything
needed without additional research. Consider --skip-research flag to go straight to planning,
or use the rich ROADMAP entry as the CONTEXT.md equivalent.

<objective>
Clean up the Phase 3 vs Phase 4 dual model duplication in governance and market models. Phase 3 created skeleton fields (e.g., `GovernanceData.executives: list[ExecutiveProfile]`) that are never populated by any extractor. Phase 4 added forensic sub-models (e.g., `GovernanceData.leadership: LeadershipStability`) with the actual data. The skeleton fields create confusion: code that reads `executives` gets an empty list, while the real data is at `leadership.executive_profiles`.

Purpose: Eliminate confusion about which field path to use for any given data concept. This makes Phase 18+ extraction work cleaner -- there's ONE path for each piece of data, not two competing empty-vs-populated paths.

Output: Clean models with no duplicate field paths for the same concept
</objective>

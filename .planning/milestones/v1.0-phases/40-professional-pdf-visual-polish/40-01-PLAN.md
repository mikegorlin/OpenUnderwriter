<objective>
Replace the Tailwind CSS CDN and Google Fonts CDN dependencies with pre-compiled static CSS and self-hosted fonts, creating a zero-network-dependency PDF rendering pipeline.

Purpose: The Tailwind CDN `<script>` tag requires network access and JavaScript execution in Playwright. This is fragile for CI/offline environments and adds latency. Static CSS is deterministic and instant.

Output: compiled.css (committed artifact), self-hosted font files, updated base template, build script.
</objective>

<task type="auto">
  <name>Task 1: Install Tailwind standalone CLI and build compiled CSS</name>
  <files>
    scripts/build-css.sh
    src/do_uw/templates/html/input.css
    src/do_uw/templates/html/compiled.css
  </files>
  <action>
  1. Install pytailwindcss as a dev dependency: `uv add --dev pytailwindcss`

  2. Create `src/do_uw/templates/html/input.css` with:
     - `@import "tailwindcss";` (Tailwind v4 import syntax)
     - `@theme` block defining all custom colors used in templates: navy (#0B1D3A), navy-light (#1A2D4A), gold (#D4A843), gold-light (#F5ECD8), risk-red (#B91C1C), risk-amber (#D97706), risk-blue (#1D4ED8), risk-green (#16A34A), neutral-gray (#6B7280), bg-alt (#F8FAFC)
     - `@theme` block defining custom font families: heading (Inter, system-ui, sans-serif), body (Georgia, "Times New Roman", serif), mono ("JetBrains Mono", Consolas, monospace)
     - NOTE: Tailwind v4 uses `@theme` blocks not `tailwind.config`. Check context7 for exact v4 syntax if needed.

  3. Download Inter (400, 500, 600, 700 weights) and JetBrains Mono (400) WOFF2 files:
     - Use `curl` to fetch from Google Fonts API or fontsource CDN
     - Save to `src/do_uw/templates/html/fonts/` directory
     - Add `@font-face` declarations in input.css referencing relative paths to the WOFF2 files

  4. Create `scripts/build-css.sh`:
     ```bash
     #!/usr/bin/env bash
     set -euo pipefail
     cd "$(dirname "$0")/.."
     uv run tailwindcss \
       -i src/do_uw/templates/html/input.css \
       -o src/do_uw/templates/html/compiled.css \
       --content 'src/do_uw/templates/html/**/*.j2' \
       --minify
     echo "CSS compiled successfully."
     ```
     Make it executable: `chmod +x scripts/build-css.sh`

  5. Run the build script and verify compiled.css is generated and non-empty.
     Verify it contains the custom color classes (bg-navy, text-gold, etc.) by grepping.

  NOTE on Tailwind v4: The standalone CLI for v4 may use slightly different syntax than v3. Check `uv run tailwindcss --help` for the actual flags. The `--content` flag scans files for class names. If pytailwindcss installs v4, `@import "tailwindcss"` is the correct import (not `@tailwind base/components/utilities`). If it installs v3, use `@tailwind base; @tailwind components; @tailwind utilities;` instead.

  IMPORTANT: The compiled CSS must include ALL Tailwind utility classes used across the 17 template files (flex, grid, text-*, bg-*, px-*, py-*, etc.). The build script scans .j2 files to detect which classes are needed.
  </action>
  <verify>
  - `ls src/do_uw/templates/html/compiled.css` exists and is > 10KB
  - `grep "bg-navy" src/do_uw/templates/html/compiled.css` returns matches (custom color compiled)
  - `grep "grid-cols-2" src/do_uw/templates/html/compiled.css` returns matches (layout class compiled)
  - `ls src/do_uw/templates/html/fonts/*.woff2` shows Inter and JetBrains Mono files
  - `bash scripts/build-css.sh` runs without error
  </verify>
  <done>compiled.css contains all Tailwind utility classes needed by templates, font files are present, build script is repeatable</done>

  <files>src/do_uw/stages/render/word_renderer.py, tests/stages/render/test_word_density.py</files>
  <action>
Modify `word_renderer.py` to:

1. **Use pre-computed narratives:**
   - At the top of section rendering, check if `state.analysis.pre_computed_narratives` has a narrative for the current section
   - If available, render it as the section's opening paragraph (OUT-03 compliance)
   - If not available, fall back to existing narrative generation (backward compat)
   - Add "AI Assessment" label as a small italic prefix when using LLM-generated narratives (DATA-14)

2. **Add density-tier indicators:**
   - Simple colored text at the start of each section: CLEAN = no indicator (clean is normal), ELEVATED = amber text "Elevated Concern", CRITICAL = red text "Critical Risk"
   - Use `docx_helpers` for colored text (same pattern as existing risk styling)
   - Read density from `state.analysis.section_densities.get(section_id)`
   - No layout changes — Word stays clean and functional per user decision

3. **Helper function `_add_density_indicator(doc, section_id, state)`:**
   - Reads section density
   - CLEAN: nothing
   - ELEVATED: Adds paragraph with amber-colored "ELEVATED CONCERN" text
   - CRITICAL: Adds paragraph with red-colored "CRITICAL RISK" text
   - Uses existing `_oxml()` pattern for colored text runs

4. **Helper function `_add_section_narrative(doc, section_id, state)`:**
   - Checks pre_computed_narratives for the section
   - If available: adds paragraph with the narrative, italic prefix "[AI Assessment]"
   - If not: does nothing (section renderers still generate their own content)

5. **Call both helpers at the start of each section render** (after section heading, before data tables).

6. **"Not Available" rendering:**
   - When a data field is None, render "Not Available — [reason]" per DATA-12
   - This is already partially implemented; ensure consistency across all sections

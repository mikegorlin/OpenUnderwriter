  <action>
Create `src/do_uw/stages/benchmark/narrative_generator.py` with:

1. **`generate_section_narrative(section_id, section_data, density, company_name, client?) -> str`**
   - Builds a prompt from section data and density level
   - Density controls length: CLEAN = "2-3 concise sentences", ELEVATED = "4-5 sentences covering concerns and D&O implications", CRITICAL = "6-8 sentences with forensic detail and cross-references"
   - Uses claude-sonnet model (not opus) per research cost guidance
   - max_tokens: 400 (CLEAN), 600 (ELEVATED), 800 (CRITICAL)
   - Prompt tone: "Bloomberg analyst report — factual, specific, professional"
   - Prompt includes: company name, section ID, density level, serialized section data (truncated to 3000 chars), rules (cite numbers, explain D&O relevance, no hedging, no generic statements)
   - Label output as "AI Assessment" per DATA-14

2. **`generate_executive_thesis(state_summary, density, company_name, client?) -> str`**
   - Tiered: CLEAN 3-4 sentences, ELEVATED/CRITICAL 6-8 sentences per user decision
   - Synthesizes key findings, risk tier, claim probability into a coherent thesis

3. **`generate_meeting_prep_questions(findings, density, company_name, client?) -> list[str]`**
   - Per user decision: "tied to specific findings only"
   - Returns 3-5 questions for CLEAN, 5-8 for ELEVATED, 8-12 for CRITICAL

4. **In-memory cache:**
   - `_narrative_cache: dict[str, str] = {}` module-level
   - Cache key = `hashlib.sha256(json.dumps({section_id, density, data}, sort_keys=True, default=str).encode()).hexdigest()[:16]`
   - Cache hit skips LLM call
   - Per-run cache (not persisted) per research recommendation

5. **`generate_all_narratives(state) -> PreComputedNarratives`**
   - Main entry point called from BenchmarkStage
   - Iterates sections: company, financial, market, governance, litigation, scoring, ai_risk
   - Extracts section-relevant data from state for each section's prompt
   - Gets density from state.analysis.section_densities (defaulting to CLEAN if missing)
   - Returns populated PreComputedNarratives model
   - Wraps in try/except per section — partial failure doesn't crash

6. **Lazy import pattern:**
   - `anthropic` and `instructor` imported inside functions (matching existing LLMExtractor pattern)
   - If anthropic unavailable, fall back to existing rule-based narratives from `md_narrative.py` and `md_narrative_sections.py`
   - Log warning when falling back

7. **Fallback implementation:**
   - Import existing `build_company_narrative`, `build_governance_narrative`, `build_litigation_narrative`, `build_scoring_narrative` from `md_narrative_sections.py`
   - These produce rule-based narratives that serve as "good enough" when LLM is unavailable
   - Fallback narratives are NOT labeled "AI Assessment"

**Tests in `tests/stages/benchmark/test_narrative_generator.py`:**
- test_cache_key_deterministic: Same input -> same key
- test_cache_key_differs_on_density: CLEAN vs ELEVATED -> different keys
- test_narrative_fallback: When anthropic is mocked to raise ImportError, falls back to rule-based
- test_generate_all_narratives_structure: Returns PreComputedNarratives with expected fields
- test_density_controls_max_tokens: Verify prompt construction includes correct length guidance per density
- Mock anthropic client for tests (do NOT make real API calls in tests)

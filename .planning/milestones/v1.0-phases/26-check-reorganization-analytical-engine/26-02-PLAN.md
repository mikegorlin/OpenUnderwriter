<objective>
Build the temporal change detection engine that identifies quarter-over-quarter and year-over-year changes in key financial metrics -- the primary trigger for securities class actions. Simple consecutive-period counting provides 80% of the value without complex time-series models. Add ~10 new FIN.TEMPORAL.* checks to checks.json.

Purpose: SCAs are triggered by CHANGES (revenue deceleration, margin compression), not static levels. The current system evaluates point-in-time metrics but misses the trajectory that plaintiffs' attorneys target. This engine fills that critical gap.

Output: temporal_engine.py + temporal_metrics.py (split for 500-line compliance), ~10 new checks in checks.json, unit tests.
</objective>

  <action>
    **1. Create temporal_engine.py** (~300 lines):

    `TemporalAnalyzer` class:
    - `__init__(self, config: dict)`: Load thresholds from temporal_thresholds.json config
    - `classify_temporal_trend(values: list[float], direction: Literal["higher_is_worse", "lower_is_worse"], consecutive_threshold: int = 3) -> TemporalClassification`:
      - Count consecutive adverse moves (direction-aware)
      - Count consecutive favorable moves
      - CRITICAL: adverse >= consecutive_threshold + 1
      - DETERIORATING: adverse >= consecutive_threshold
      - IMPROVING: favorable >= consecutive_threshold
      - STABLE: default
    - `analyze_metric(metric_name: str, values: list[float], periods: list[str], direction: str) -> TemporalSignal`:
      - Call classify_temporal_trend
      - Compute total_change_pct = (last - first) / abs(first) * 100 (handle zero division)
      - Build evidence string: "Revenue declined 3 consecutive quarters (-15.2% total: Q1'24 $X -> Q4'24 $Y)"
      - Return TemporalSignal model from models/temporal.py
    - `analyze_all_temporal(extracted: ExtractedData, company: CompanyProfile | None) -> TemporalAnalysisResult`:
      - Use temporal_metrics.py to extract multi-period values
      - Run each metric through analyze_metric
      - Return TemporalAnalysisResult with all signals + summary narrative

    Import TemporalSignal, TemporalClassification, TemporalAnalysisResult from do_uw.models.temporal.

    **2. Create temporal_metrics.py** (~250 lines):

    Helper functions to extract multi-period financial data from ExtractedData:
    - `extract_temporal_metrics(extracted: ExtractedData) -> dict[str, list[tuple[str, float]]]`:
      Returns dict mapping metric name to list of (period_label, value) tuples.

    Individual metric extractors (each returns list[tuple[str, float]] or empty list):
    - `_extract_revenue_trend(extracted)`: Revenue from financial_statements across available periods
    - `_extract_gross_margin_trend(extracted)`: Gross margin % across periods
    - `_extract_operating_margin_trend(extracted)`: Operating margin % across periods
    - `_extract_dso_trend(extracted)`: Days sales outstanding (AR / Revenue * 365)
    - `_extract_operating_cash_flow_trend(extracted)`: Operating cash flow across periods
    - `_extract_net_income_cfo_divergence(extracted)`: NI/CFO ratio across periods (higher_is_worse when diverging)
    - `_extract_working_capital_trend(extracted)`: Current assets - current liabilities
    - `_extract_debt_ratio_trend(extracted)`: Total debt / total assets

    Each extractor accesses financial_statements on ExtractedData, iterates through available periods (FY Prior, FY Latest, potentially YTD), and extracts the relevant line items. Use safe access patterns (None checks) since data may not be available for all periods. If fewer than 2 data points available, return empty list (temporal analysis not possible).

    Direction map (config-driven from temporal_thresholds.json or hardcoded defaults):
    ```python
    METRIC_DIRECTIONS: dict[str, str] = {
        "revenue_growth": "lower_is_worse",
        "gross_margin": "lower_is_worse",
        "operating_margin": "lower_is_worse",
        "dso_days": "higher_is_worse",
        "operating_cash_flow": "lower_is_worse",
        "net_income_cfo_divergence": "higher_is_worse",
        "working_capital": "lower_is_worse",
        "debt_ratio": "higher_is_worse",
    }
    ```

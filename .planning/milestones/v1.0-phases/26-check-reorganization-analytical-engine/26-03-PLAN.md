<objective>
Build financial forensics composite scores (FIS, RQS, CFQS) that combine multiple academic fraud detection models into unified, actionable metrics. Add Dechow F-Score and Montier C-Score alongside the existing Beneish M-Score for multi-perspective manipulation detection. Add new FIN.FORENSIC.* and FIN.QUALITY.* checks.

Purpose: Single models (Beneish alone) have ~50% false positive rates. Combining multiple models dramatically reduces false positives. The unified composites also give underwriters a single "financial integrity" number rather than requiring them to interpret 5 different academic scores. Per CONTEXT.md: "always display as score gauges/indicators, with prominent callouts/alerts only when thresholds are breached."

Output: forensic_composites.py + forensic_models.py (split for 500-line compliance), ~13 new checks in checks.json, unit tests.
</objective>

    **1. Create forensic_models.py** (~350 lines):

    Individual forensic model computation functions. These are academic fraud/manipulation detection models:

    - `compute_dechow_f_score(extracted: ExtractedData) -> tuple[float, str]`:
      Dechow F-Score (simplified, per research "don't hand-roll" — no cross-sectional peer regression).
      Uses: change in receivables, change in inventories, change in cash sales, change in earnings.
      Returns (score, evidence_narrative). Score > 1.85 suggests high manipulation probability.
      If required financial data unavailable, return (0.0, "Insufficient data for Dechow F-Score").

    - `compute_montier_c_score(extracted: ExtractedData) -> tuple[float, str]`:
      Montier C-Score: 6 binary indicators (0 or 1):
      1. Increasing difference between net income and cash flow from operations
      2. Increasing days sales in receivables
      3. Increasing days sales in inventory
      4. Increasing other current assets vs revenue
      5. Declining depreciation relative to net PP&E
      6. High total asset growth
      Sum 0-6. Score >= 4 = high manipulation risk.
      Returns (score, evidence_narrative).

    - `compute_enhanced_sloan_ratio(extracted: ExtractedData) -> tuple[float, str]`:
      Enhanced Sloan Accrual Ratio = (Net Income - Operating Cash Flow) / Total Assets.
      High accruals (>0.10) = lower earnings persistence = higher manipulation risk.
      Returns (ratio, evidence_narrative).

    - `compute_accrual_intensity(extracted: ExtractedData) -> tuple[float, str]`:
      Accrual intensity = |Working Capital Accruals| / |Operating Cash Flow|.
      High intensity (>0.50) indicates earnings heavily driven by accruals.
      Returns (ratio, evidence_narrative).

    Each function:
    - Accesses financial data from extracted.financial_statements, extracted.financials
    - Uses safe None checks throughout (data may be absent)
    - Returns tuple of (score, evidence_narrative_string)
    - Evidence narrative explains what was computed and which data was used

    **2. Create forensic_composites.py** (~400 lines):

    Composite score computation that combines individual models:

    - `compute_financial_integrity_score(extracted: ExtractedData, config: dict) -> FinancialIntegrityScore`:
      Combines 5 sub-dimensions with configurable weights (from forensic_models.json):
      1. **Manipulation detection** (30%): Beneish M-Score (existing) + Dechow F-Score + Montier C-Score. Each normalized to 0-100 where 100=no manipulation detected. Composite of available models (skip if data insufficient).
      2. **Accrual quality** (20%): Enhanced Sloan ratio + accrual intensity. High accruals = low score.
      3. **Revenue quality** (20%): DSO trend + AR/revenue divergence + Q4 revenue concentration + deferred revenue changes. Use extracted financial line items.
      4. **Cash flow quality** (15%): Quality of earnings ratio (CFO/NI), cash conversion efficiency, capex adequacy.
      5. **Audit risk** (15%): Material weaknesses, auditor changes, restatements, going concern.
      Zone classification from config: HIGH_INTEGRITY (80-100), ADEQUATE (60-80), CONCERNING (40-60), WEAK (20-40), CRITICAL (0-20).
      Import FinancialIntegrityScore from do_uw.models.forensic.

    - `compute_revenue_quality_score(extracted: ExtractedData, config: dict) -> RevenueQualityScore`:
      Sub-scores: DSO trend (30%), AR divergence (25%), Q4 concentration (25%), deferred revenue (20%).
      DSO trend: compare current vs prior year DSO.
      AR divergence: AR growing faster than revenue is adverse.
      Q4 concentration: >30% of annual revenue in Q4 is yellow, >40% is red.
      Deferred revenue: declining deferred revenue with growing recognized revenue is adverse.

    - `compute_cash_flow_quality_score(extracted: ExtractedData, config: dict) -> CashFlowQualityScore`:
      Sub-scores: Quality of earnings (CFO/NI ratio — near 1.0 is healthy), cash conversion, capex adequacy.

    Helper functions:
    - `_normalize_to_score(raw_value: float, good_threshold: float, bad_threshold: float) -> float`: Map raw values to 0-100 scale
    - `_weighted_composite(scores: dict[str, float], weights: dict[str, float]) -> float`: Weighted average, skipping None/unavailable
    - `_classify_zone(score: float, zones: dict[str, list[float]]) -> ForensicZone`: Map score to zone using config thresholds

    Load forensic_models.json config for weights and zone thresholds. Use ConfigLoader if available, or load JSON directly.

    All forensic composite functions must handle missing data gracefully — if a sub-dimension has no data, compute the composite from available sub-dimensions only (reweight).
  </action>

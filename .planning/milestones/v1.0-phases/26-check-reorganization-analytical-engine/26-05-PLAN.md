    ```python
    # Phase 26: Enhanced analytical engines (all wrapped in try/except for graceful degradation)

    # Layer 3a: Temporal change detection
    try:
        from do_uw.stages.analyze.temporal_engine import TemporalAnalyzer
        from do_uw.stages.analyze.temporal_metrics import extract_temporal_metrics
        from do_uw.config.loader import ConfigLoader

        temporal_config = ConfigLoader().load_config("temporal_thresholds")
        analyzer = TemporalAnalyzer(temporal_config)
        temporal_result = analyzer.analyze_all_temporal(state.extracted, state.company)
        state.analysis.temporal_signals = temporal_result.model_dump()
        logger.info("Temporal: %d signals, %d deteriorating",
                     len(temporal_result.signals),
                     sum(1 for s in temporal_result.signals if s.classification == "DETERIORATING"))
    except Exception:
        logger.warning("Temporal analysis failed; continuing", exc_info=True)

    # Layer 3b: Financial forensics composites
    try:
        from do_uw.stages.analyze.forensic_composites import (
            compute_financial_integrity_score,
            compute_revenue_quality_score,
            compute_cash_flow_quality_score,
        )
        forensic_config = ConfigLoader().load_config("forensic_models")
        fis = compute_financial_integrity_score(state.extracted, forensic_config)
        rqs = compute_revenue_quality_score(state.extracted, forensic_config)
        cfqs = compute_cash_flow_quality_score(state.extracted, forensic_config)
        state.analysis.forensic_composites = {
            "financial_integrity_score": fis.model_dump(),
            "revenue_quality_score": rqs.model_dump(),
            "cash_flow_quality_score": cfqs.model_dump(),
        }
        logger.info("Forensics: FIS=%.0f (%s), RQS=%.0f, CFQS=%.0f",
                     fis.overall_score, fis.zone, rqs.overall_score, cfqs.overall_score)
    except Exception:
        logger.warning("Forensic composites failed; continuing", exc_info=True)

    # Layer 3c: Executive forensics
    try:
        from do_uw.stages.analyze.executive_forensics import run_executive_forensics
        board_risk = run_executive_forensics(state)
        if board_risk is not None:
            state.analysis.executive_risk = board_risk.model_dump()
            logger.info("Executive forensics: aggregate=%.0f, highest=%s",
                         board_risk.weighted_score, board_risk.highest_risk_individual)
    except Exception:
        logger.warning("Executive forensics failed; continuing", exc_info=True)

    # Layer 3d: NLP signals
    try:
        from do_uw.stages.analyze.nlp_signals import analyze_nlp_signals
        # Prior year text from acquired data if available
        prior_text = None  # TODO: populate from acquired prior-year 10-K when available
        nlp_results = analyze_nlp_signals(state.extracted, prior_text)
        state.analysis.nlp_signals = nlp_results
        logger.info("NLP: %d signals detected", sum(1 for v in nlp_results.values() if isinstance(v, dict) and v.get("detected")))
    except Exception:
        logger.warning("NLP signals failed; continuing", exc_info=True)
    ```

    **5. Implement IES-aware check amplification** (in ScoreStage or factor_scoring):
    Per CONTEXT.md: "Checks should be IES-aware: a deteriorating signal in a high-IES company is amplified vs. the same signal in a low-IES company."
    Implementation: In the factor scoring step, if IES is available (state.hazard_profile.ies_score > 0), apply a multiplier to behavioral signal contributions:
    - IES > 60: behavioral signal points * 1.25 (elevated inherent exposure amplifies behavioral signals)
    - IES > 75: behavioral signal points * 1.50
    - IES < 40: behavioral signal points * 0.85 (low inherent exposure dampens behavioral signals)
    Cap: amplified points cannot exceed the factor's max_points.

    **6. Implement combined contribution caps** (in ScoreStage):
    Per CONTEXT.md: "Weight normalization via combined contribution cap: set a ceiling per domain for hazard + check combined."
    Implementation: After factor scoring, check if both hazard profile (IES) and factor scoring evaluated the same domain. If hazard already accounts for X% of the cap, factor scoring can only add up to (100-X)%.

    Practically: IES adjusts the claim probability via Factor 0 (already in Phase 25). The contribution cap ensures that structural checks reclassified as CONTEXT_DISPLAY don't also score in factors. Since Plan 01 already reclassifies structural checks as CONTEXT_DISPLAY (not scored), the primary mechanism is: CONTEXT_DISPLAY checks have empty factors -> they don't contribute to factor scoring. The cap is a safety net for any remaining overlap.

    Add a _apply_contribution_caps() function in score/__init__.py that logs warnings if both IES and factor scores are high for the same domain.

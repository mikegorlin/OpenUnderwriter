    **2. Create nlp_signals.py** (~350 lines):

    NLP signal detection that compares current and prior year 10-K text:

    - `analyze_nlp_signals(extracted: ExtractedData, prior_year_text: dict[str, str] | None) -> dict[str, Any]`:
      Orchestrator. Returns dict with signal results for all NLP dimensions.
      If prior_year_text is None, gracefully return results with "prior year unavailable" notes.

    - `compute_readability_change(current_text: str, prior_text: str | None) -> dict`:
      Uses textstat library:
      - `textstat.gunning_fog(text)` for Fog Index
      - `textstat.flesch_reading_ease(text)` for Flesch score
      Compare current vs prior. Increasing Fog Index (harder to read) is adverse — research shows companies obfuscate before bad news.
      Returns: `{"current_fog": float, "prior_fog": float | None, "fog_change": float | None, "classification": str, "evidence": str}`
      Classification: "INCREASING_COMPLEXITY" (fog increased >2 points), "STABLE", "IMPROVING_CLARITY"

    - `detect_tone_shift(current_mda: str, prior_mda: str | None) -> dict`:
      Simple keyword-based tone analysis (not NLP sentiment model):
      - Count negative keywords: "decline", "loss", "adverse", "uncertain", "risk", "challenge", "impairment", "restructuring", "litigation", "investigation"
      - Count positive keywords: "growth", "improve", "strong", "exceed", "record", "innovation", "opportunity"
      - Compute negative_ratio = neg_count / total_count
      Compare current vs prior year ratio. Increasing negative tone is adverse.
      Returns: `{"current_negative_ratio": float, "prior_negative_ratio": float | None, "shift": float | None, "classification": str, "evidence": str}`

    - `track_risk_factor_evolution(current_risk_factors: list[str], prior_risk_factors: list[str] | None) -> dict`:
      Compare risk factor section between years:
      - Count total risk factors (current vs prior)
      - Identify NEW risk factors (in current but not prior — fuzzy match by first 50 chars)
      - Identify REMOVED risk factors
      - New risk factors are potentially significant disclosures
      Returns: `{"current_count": int, "prior_count": int | None, "new_factors": list[str], "removed_factors": list[str], "net_change": int | None, "evidence": str}`
      If prior_risk_factors is None, report current state only (INFO).

    - `detect_whistleblower_language(text: str) -> dict`:
      Scan for whistleblower/qui tam related language in 10-K text:
      Keywords: "whistleblower", "qui tam", "False Claims Act", "relator", "retaliation", "hotline complaint", "internal investigation triggered by"
      Returns: `{"detected": bool, "matches": list[str], "evidence": str}`

    Note: Per CONTEXT.md, prior-year 10-K acquisition is a prerequisite. If the current ACQUIRE stage does not fetch prior-year 10-K, the NLP checks should gracefully degrade by passing `prior_year_text=None`. The plan for extending ACQUIRE can be handled within this task if straightforward (just fetching one additional filing), or flagged for a future phase if it requires significant ACQUIRE stage changes.

    Check if the current ACQUIRE stage already stores prior-year filings or if the filing_documents dict can be extended. If not straightforward, have NLP checks work in "current year only" mode with INFO status noting prior year unavailable.

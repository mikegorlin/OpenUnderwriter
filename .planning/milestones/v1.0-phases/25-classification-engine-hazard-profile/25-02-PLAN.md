    **Tiered fallback pattern for every dimension (NON-NEGOTIABLE):**
    Each dimension mapping MUST follow this 3-tier pattern:
      1. **Primary data source**: Try the direct structured field (e.g., `company.identity.sector`)
      2. **Proxy signal fallback**: If primary is None/empty, try alternative data sources that provide indirect evidence (e.g., risk factor text mentions, related financial ratios, config-driven defaults)
      3. **Neutral default**: If both primary and proxy are unavailable, return empty dict `{}` (triggers neutral score with `data_available=False` in the scorer)

    The return dict MUST include a `"_data_tier"` key with value `"primary"`, `"proxy"`, or `"unavailable"` so scorers can adjust confidence.

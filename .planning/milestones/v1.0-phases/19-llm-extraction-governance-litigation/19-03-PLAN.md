    **llm_litigation.py** -- Create converter module. Must stay under 500 lines.

    **Constants:**
    - `_LLM_SOURCE = "10-K (LLM)"`

    **Functions to implement:**

    1. `convert_legal_proceedings(extraction: TenKExtraction) -> list[CaseDetail]`
       - Iterate `extraction.legal_proceedings`, create `CaseDetail` for each:
         - `case_name`: `sourced_str(proc.case_name, _LLM_SOURCE, Confidence.HIGH)` if proc.case_name
         - `court`: `sourced_str(proc.court, ...)` if proc.court
         - `filing_date`: parse proc.filing_date (YYYY-MM-DD string) to `SourcedValue[date]` using `_parse_date()`
         - `allegations`: convert proc.allegations (single str) to `[sourced_str(proc.allegations, ...)]`
         - `legal_theories`: Use existing `detect_legal_theories` from sca_extractor.py if available, OR implement a simple keyword matcher: check for "10b-5", "Section 11", "Section 14", "ERISA", "antitrust", "FCPA" in allegations and case_name text. Produce `list[SourcedValue[str]]` with LegalTheory enum values.
         - `status`: map proc.status ("ACTIVE", "SETTLED", "DISMISSED", "APPEAL") to sourced_str
         - `settlement_amount`: sourced_float if proc.settlement_amount not None
         - `class_period_start`, `class_period_end`: parse date strings
         - `named_defendants`: convert proc.named_defendants (list[str], from 19-01 expansion) to `[sourced_str(d, _LLM_SOURCE, Confidence.HIGH) for d in proc.named_defendants]`
         - `coverage_type`: infer from legal theories -- if "10b-5" or "Section 11" present, use CoverageType.SCA_SIDE_C; if "ERISA", use EMPLOYMENT_ENTITY; etc. Use a simple mapping dict.
       - Skip proceedings with empty case_name
       - Return list of CaseDetail

    2. `convert_contingencies(extraction: TenKExtraction) -> list[ContingentLiability]`
       - Iterate `extraction.contingent_liabilities` (now `list[ExtractedContingency]` after 19-01)
       - For each ExtractedContingency:
         - `description`: sourced_str(c.description, _LLM_SOURCE, Confidence.HIGH)
         - `asc_450_classification`: sourced_str(c.classification, ...) if c.classification
         - `accrued_amount`: sourced_float(c.accrued_amount, ...) if not None
         - `range_low`: sourced_float(c.range_low, ...) if not None
         - `range_high`: sourced_float(c.range_high, ...) if not None
         - `source_note`: sourced_str(c.source_passage, _LLM_SOURCE, Confidence.HIGH) if c.source_passage
       - Skip entries with empty description
       - Return list of ContingentLiability

    3. `convert_risk_factors(extraction: TenKExtraction) -> list[RiskFactorProfile]`
       - Iterate `extraction.risk_factors` (list[ExtractedRiskFactor])
       - For each:
         - Map directly to RiskFactorProfile fields: title, category, severity, is_new_this_year, source_passage
         - `source`: set to _LLM_SOURCE
         - `do_relevance`: infer from category -- LITIGATION/REGULATORY -> HIGH, FINANCIAL/CYBER -> MEDIUM, OPERATIONAL/ESG/AI/OTHER -> LOW. Simple mapping dict.
       - Skip entries with empty title
       - Return list of RiskFactorProfile

    4. `convert_forum_provisions(extraction: DEF14AExtraction) -> ForumProvisions`
       - This is CROSS-DOMAIN: DEF 14A data populating a litigation defense model
       - Import DEF14AExtraction (already imported via llm_helpers)
       - `has_exclusive_forum`: from extraction.exclusive_forum_provision
       - `exclusive_forum_details`: from extraction.forum_selection_clause
       - Source: "DEF 14A (LLM)"

    5. `_parse_date(date_str: str | None) -> date | None`
       - Parse "YYYY-MM-DD" to `datetime.date`
       - Return None on parse failure
       - Use `datetime.strptime(date_str, "%Y-%m-%d").date()` with try/except

    6. `_infer_legal_theories(text: str) -> list[str]`
       - Simple keyword matching against allegation text
       - Check for: "10b-5" or "10(b)" -> "RULE_10B5", "section 11" -> "SECTION_11", "section 14" -> "SECTION_14A", "erisa" -> "ERISA", "antitrust" -> "ANTITRUST", "fcpa" or "foreign corrupt" -> "FCPA", "product liability" -> "PRODUCT_LIABILITY", "cyber" or "data breach" or "privacy" -> "CYBER_PRIVACY"
       - Case-insensitive matching
       - Return list of LegalTheory string values

    7. `_infer_coverage_type(theories: list[str]) -> str`
       - If any securities theory (RULE_10B5, SECTION_11, SECTION_14A): return "SCA_SIDE_C"
       - If DERIVATIVE_DUTY: return "DERIVATIVE_SIDE_A"
       - If ERISA or EMPLOYMENT_DISCRIMINATION: return "EMPLOYMENT_ENTITY"
       - If PRODUCT_LIABILITY: return "PRODUCT_ENTITY"
       - Default: "REGULATORY_ENTITY"

    **Import pattern:** Same as llm_governance.py. Import `from do_uw.stages.extract.sourced import sourced_str, sourced_float, now`. Import ForumProvisions from litigation_details. Import DEF14AExtraction for convert_forum_provisions.

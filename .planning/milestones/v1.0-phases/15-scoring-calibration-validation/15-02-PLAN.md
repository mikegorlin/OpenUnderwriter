Create `tests/test_scoring_validation.py` with parameterized tests that validate the scoring engine against known-outcome company archetypes. These tests construct AnalysisState fixtures with synthetic data representing real-world company risk profiles, then verify the scoring engine produces expected tier ranges.

Follow the existing test patterns from `tests/test_score_stage.py`:
- Use `_sv()` helper for SourcedValue construction
- Use `_load_scoring_config()` and `_load_sectors_config()` for config loading
- Use `_make_company()` for CompanyProfile construction
- Import from `do_uw.stages.score.factor_scoring`, `red_flag_gates`, `tier_classification`
- Use `from __future__ import annotations` for type compatibility

**Company Archetypes to Test (8 scenarios):**

1. **PRISTINE_BLUE_CHIP** - Stable mega-cap, no litigation history, strong governance, low volatility
   - Expected: WIN or WANT tier (quality_score >= 71)
   - Profile: $200B market cap, TECH sector, no SCA, no restatement, <20% decline, strong governance, stable officers, low SI, low vol, no distress, 0 guidance misses, no IPO/SPAC

2. **SOLID_MID_CAP** - Healthy mid-cap, clean record, normal metrics
   - Expected: WANT or WRITE tier (quality_score 51-85)
   - Profile: $8B market cap, INDU sector, no active litigation, settled suit 5-10yr ago, 25% decline, 1 guidance miss, normal SI, normal vol, adequate leverage

3. **GROWTH_DARLING_STRESSED** - High-growth tech with elevated risk signals
   - Expected: WRITE tier (quality_score 51-70)
   - Profile: $15B market cap, TECH sector, no SCA, 35% stock decline, 2 guidance misses, elevated SI (2x sector), high volatility, heavy insider selling, IPO 30 months ago

4. **DISTRESSED_COMPANY** - Financial distress with going concern
   - Expected: WATCH or WALK tier (quality_score <= 50) AND CRF-004 ceiling applied
   - Profile: $500M market cap, BIOT sector, going concern opinion, <6 months cash runway, 55% stock decline, 3+ guidance misses, material weakness

5. **ACTIVE_SCA_DEFENDANT** - Company with active securities class action
   - Expected: WALK tier (quality_score <= 30) due to CRF-001 ceiling
   - Profile: $10B market cap, FINS sector, active SCA, 40% stock decline, SEC investigation, 2 guidance misses
   - CRITICAL: Regardless of how good other factors are, CRF-001 must cap at WALK/30

6. **SPAC_PENNY_STOCK** - Post-SPAC merger with stock below $5
   - Expected: WATCH or WALK (quality_score <= 50) due to CRF-006
   - Profile: $200M market cap, TECH sector, SPAC merger 12 months ago, stock at $3, 70% decline, elevated SI, high vol

7. **SHORT_SELLER_TARGET** - Named in recent activist short report
   - Expected: WATCH or worse (quality_score <= 50) due to CRF-007
   - Profile: $5B market cap, HLTH sector, Hindenburg report 3 months ago, 45% decline, rising SI

8. **RESTATEMENT_CRISIS** - Recent Big R restatement with audit issues
   - Expected: WATCH or worse (quality_score <= 50) due to CRF-005
   - Profile: $3B market cap, CONS sector, restatement 6 months ago, auditor fired, material weakness, 30% decline

For each archetype:
- Construct an `AnalysisState` with all relevant fields populated
- Call `score_all_factors()` to get factor scores
- Call `evaluate_red_flag_gates()` to check CRF triggers
- Call `apply_crf_ceilings()` to apply ceilings
- Call `classify_tier()` to get tier assignment
- Assert the tier is within expected range
- Assert specific factor scores are non-zero where expected (e.g., F1 > 0 for active SCA)
- Assert CRF gates trigger where expected

Use `@pytest.mark.parametrize` where appropriate to reduce boilerplate. Group related assertions logically.

**Monotonicity Tests:**
Add tests that verify scoring monotonicity:
- More litigation history = higher F1 score
- Larger stock decline = higher F2 score
- More guidance misses = higher F5 score
- Higher short interest = higher F6 score
- These ensure the scoring engine never produces counterintuitive orderings

IMPORTANT: Use the _sv() helper and existing model constructors. For fields that require populated sub-models (e.g., LitigationLandscape, StockPerformance), construct minimal but realistic instances. Do NOT use mock.patch -- construct real objects with the scoring engine running against them.

<task type="auto">
  <name>Task 2: Create cross-profile tier differentiation tests</name>
  <files>tests/test_tier_differentiation.py</files>
  <action>
Create `tests/test_tier_differentiation.py` that validates the scoring engine produces sensible RELATIVE ordering across company archetypes -- not just that each archetype lands in the right tier individually, but that the full spectrum produces a coherent risk ranking.

Follow the same test patterns as Task 1.

**Differentiation Tests:**

1. **Risk Ordering Test**: Run all 8 archetypes through the scoring engine and verify the quality scores are monotonically ordered:
   - PRISTINE_BLUE_CHIP > SOLID_MID_CAP > GROWTH_DARLING_STRESSED > others
   - All CRF-triggered profiles score <= 50 (or 30 for WALK-ceiling CRFs)
   - Store the archetype fixture builders in module-level functions so both test files can reference them (or duplicate -- test files are allowed to be independent)

2. **Sector Differentiation Test**: Same risk profile across different sectors should produce DIFFERENT scores due to sector baselines:
   - Construct identical risk profiles for TECH, FINS, BIOT, UTIL sectors
   - Verify that sector baseline differences (short interest norms, volatility norms, leverage norms) cause score variation
   - BIOT with 8% SI should score lower (better) than TECH with 8% SI (because BIOT baseline is 6% vs TECH 4%)
   - Verify the variation is meaningful (not just rounding differences) -- at least 3+ quality score points difference between most divergent sectors

3. **Red Flag Dominance Test**: Verify that CRF gates dominate factor scores:
   - Construct an otherwise-pristine company with ONE red flag (e.g., active SCA)
   - Verify the CRF ceiling overrides the factor-based score
   - The quality score should be at or below the ceiling, NOT the factor-calculated score
   - Test this for CRF-001 (WALK/30), CRF-004 (WATCH/50), CRF-005 (WATCH/50)

4. **Cumulative Risk Test**: Verify that adding risk factors to a baseline profile progressively lowers the score:
   - Start with PRISTINE_BLUE_CHIP (high score)
   - Add 30% stock decline -> score decreases
   - Add 2 guidance misses -> score decreases further
   - Add elevated short interest -> score decreases further
   - Add material weakness -> score decreases further
   - Each addition should produce a strictly lower quality score

5. **Edge Case Tests**:
   - Zero-risk profile: all factors at 0 points -> score = 100, tier = WIN
   - Maximum-risk profile: all factors at max points -> score = 0, tier = NO_TOUCH
   - Exactly-at-boundary: score = 86 (WIN/WANT boundary), score = 51 (WRITE/WATCH boundary)

6. **Cross-Ticker Archetype Names**: Name the test scenarios after real company archetypes (without using actual ticker data) to make the test output readable:
   - "apple_like" for PRISTINE_BLUE_CHIP
   - "boeing_like" for a company with litigation + stock decline + governance issues
   - "smci_like" for RESTATEMENT_CRISIS (accounting issues + short seller attention)
   - "lucid_like" for SPAC_PENNY_STOCK

After all tests pass, run the full test suite to verify no regressions:

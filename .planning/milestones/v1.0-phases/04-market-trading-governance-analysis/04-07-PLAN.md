<task type="auto">
  <name>Task 1: Create leadership_profiles.py extractor</name>
  <files>src/do_uw/stages/extract/leadership_profiles.py</files>
  <action>
Create extractor combining DEF 14A proxy text (bios, titles, tenure) with 8-K Item 5.02 text (departures/appointments) and litigation data.

**Main functions:**

1. `extract_leadership_profiles(state: AnalysisState) -> tuple[LeadershipStability, ExtractionReport]`
   - Orchestrates profile extraction and stability assessment
   - Returns LeadershipStability with full executive roster and departure analysis

**Helpers:**

2. `_extract_executives_from_proxy(proxy_text: dict[str, str]) -> list[LeadershipForensicProfile]`
   - Parse proxy_compensation and proxy_governance sections
   - Extract officer names and titles via regex patterns:
     - "Chief Executive Officer", "Chief Financial Officer", "Chief Operating Officer", "Chief Legal Officer", "Chief Accounting Officer", "General Counsel"
     - Pattern: name in context of title mention, age, bio paragraph
   - Extract bio_summary: paragraph following name/title
   - Infer tenure from bio text ("appointed in 2020", "has served since", "joined in")
   - Set confidence = MEDIUM (DEF 14A text extraction is unreliable)
   - If proxy text unavailable, fall back to existing state.extracted.governance.executives (from Phase 3 if populated)

3. `_extract_departures_from_8k(eight_k_texts: list[dict[str, Any]]) -> list[LeadershipForensicProfile]`
   - Parse 8-K Item 5.02 text for departure/appointment events
   - Patterns: "departure", "resignation", "termination", "retirement", "appointment"
   - Classify departure_type: "PLANNED" (retirement, personal reasons), "UNPLANNED" (other)
   - Extract departure_date from filing date
   - departure_context: relevant 8-K text snippet

4. `_assess_stability(executives: list[LeadershipForensicProfile], departures: list[LeadershipForensicProfile]) -> LeadershipStability`
   - Compute avg_tenure_years, longest/shortest tenured
   - Identify red_flags:
     - Sudden CFO departure (departure_type=UNPLANNED, title contains CFO)
     - CAO departure after filing season (Jan-Mar departure)
     - 3+ executive departures within 12 months
     - CFO + another C-suite within 6 months
     - Mid-term board resignation
     - General Counsel departure during active litigation
   - stability_score: start at 100, deduct points per red flag (configurable)

5. `_search_prior_litigation(name: str, state: AnalysisState) -> list[str]`
   - **Primary source:** Search state.acquired_data.litigation_data (already acquired by litigation_client.py in Phase 2) for executive name matches. This includes Stanford SCAC data, SEC enforcement actions, and court records that were fetched during ACQUIRE.
   - **Secondary source:** Search state.acquired_data.web_search_results and state.acquired_data.blind_spot_results for "name + securities class action", "name + SEC enforcement", "name + fraud" mentions. The blind spot discovery pattern in ACQUIRE already runs exploratory queries with risk terms.
   - **Matching strategy:** Case-insensitive substring match on executive last name against defendant/respondent fields in litigation records. For web results, search for full name in result snippets.
   - Return list of prior litigation findings as SourcedValue[str] with source citation
   - LOW confidence (name matching is inherently unreliable, cross-company attribution uncertain)
   - NOTE: This does NOT require new data acquisition -- it leverages data already in state from Phase 2 ACQUIRE

**Confidence levels:**
- DEF 14A extracted bios: MEDIUM
- 8-K departure analysis: MEDIUM
- Prior litigation search: LOW
- Stability score: MEDIUM (derived)

ExtractionReport expected: ["executives_found", "departures_18mo", "red_flags", "stability_score", "prior_litigation_searched"]
Source: "DEF 14A proxy text, 8-K Item 5.02"

**compensation_analysis.py:**

1. `extract_compensation(state: AnalysisState) -> tuple[CompensationAnalysis, ExtractionReport]`
   - Parse proxy_compensation section from DEF 14A text
   - Extract CEO compensation from Summary Compensation Table (SCT)
   - Regex for dollar amounts near "Chief Executive Officer" or CEO name
   - Extract salary, bonus/non-equity incentive, stock awards, option awards, all other comp
   - Compute comp_mix: percentage breakdown of each component
   - Extract ceo_pay_ratio from "CEO Pay Ratio" section (look for "X:1" or "X to 1" pattern)
   - Extract say_on_pay_pct from proxy proposal results (look for "say-on-pay" vote percentage)
   - Determine say_on_pay_trend from current vs prior year if available
   - has_clawback: search for "clawback" in proxy text
   - clawback_scope: "DODD_FRANK_MINIMUM" if minimal language, "BROADER" if expanded coverage
   - related_party_transactions: search "Related Party" or "Certain Relationships" section
   - notable_perquisites: search "All Other Compensation" detail or perquisite disclosure

2. `_extract_sct_values(text: str) -> dict[str, float | None]`
   - Parse Summary Compensation Table structure
   - This is the HARDEST parsing task -- SCT format varies enormously across companies
   - Strategy: look for dollar amounts ($X,XXX,XXX) in proximity to compensation category keywords
   - Accept partial extraction with LOW confidence
   - Fall back to total comp from yfinance info dict if available

**Confidence:** All DEF 14A extracted values are MEDIUM at best, LOW for SCT parsing. say_on_pay_pct is MEDIUM (more standardized format).

ExtractionReport expected: ["ceo_total_comp", "comp_mix", "pay_ratio", "say_on_pay", "clawback", "related_party"]

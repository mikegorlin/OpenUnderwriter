**earnings_guidance.py:**

Two public functions:

1. `extract_earnings_guidance(state: AnalysisState) -> tuple[EarningsGuidanceAnalysis, ExtractionReport]`
   - Read earnings_dates from state.acquired_data.market_data (added in 04-01)
   - For each quarter row: extract EPS Estimate, Reported EPS, Surprise(%)
   - Create EarningsQuarterRecord:
     - quarter: derive from earnings date (e.g., "Q1 2024")
     - guidance_midpoint: EPS Estimate value
     - actual_eps: Reported EPS value
     - result: "BEAT" if actual > estimate, "MISS" if actual < estimate, "MEET" if within 1%
     - miss_magnitude_pct: (actual - estimate) / abs(estimate) * 100 if estimate != 0
     - stock_reaction_pct: look up stock return on earnings date from history_1y
   - Compute aggregates: beat_rate, consecutive_miss_count (current streak), guidance_withdrawals
   - Determine philosophy: "CONSERVATIVE" if beat_rate > 75%, "AGGRESSIVE" if beat_rate < 50%, "NO_GUIDANCE" if no guidance data
   - ExtractionReport expected: ["quarters", "beat_rate", "miss_count", "philosophy"]
   - Source: "yfinance earnings_dates"
   - Handle yfinance earnings_dates bugs: KeyError, NaN values, missing columns

2. `extract_analyst_sentiment(state: AnalysisState) -> tuple[AnalystSentimentProfile, ExtractionReport]`
   - Read from market_data["info"] dict and market_data["recommendations"] and market_data["upgrades_downgrades"]
   - coverage_count from info["numberOfAnalystOpinions"]
   - consensus from info["recommendationKey"]
   - recommendation_mean from info["recommendationMean"]
   - Price targets from info dict or analyst_price_targets
   - Recent upgrades/downgrades from upgrades_downgrades (filter to 90 days)
   - Count upgrades vs downgrades in 90-day window
   - Estimate revisions: from recommendations changes (current vs prior month)
   - ExtractionReport expected: ["coverage_count", "consensus", "price_targets", "upgrades_downgrades", "estimate_revisions"]

**capital_markets.py:**

1. `extract_capital_markets(state: AnalysisState) -> tuple[CapitalMarketsActivity, ExtractionReport]`
   - Read S-3, S-1, 424B filing metadata from state.acquired_data.filings
   - For each offering filing: create CapitalMarketsOffering with type, date, filing_type
   - Compute Section 11 exposure window: filing_date + 3 years = section_11_window_end
   - Count active_section_11_windows (windows that haven't expired as of analysis date)
   - has_atm_program: check for "at-the-market" or "ATM" in 8-K descriptions or S-3 filings
   - convertible_securities: check info dict for convertible debt mentions
   - ExtractionReport expected: ["shelf_registrations", "offerings_3yr", "section_11_windows", "atm_program", "convertible_securities"]
   - Source: "SEC filing metadata"

**adverse_events.py:**

1. `compute_adverse_event_score(state: AnalysisState) -> tuple[AdverseEventScore, ExtractionReport]`
   - Load severity weights from config/adverse_events.json via ConfigLoader
   - Read all SECT4 extraction results from state.extracted.market:
     - stock_drops: count by severity tier
     - insider_analysis: cluster events, 10b5-1 modifications
     - short_interest: spike detection, short seller reports
     - earnings_guidance: misses, consecutive misses, withdrawals
     - analyst: downgrades in 90d
     - capital_markets: active Section 11 windows
   - For each event type: multiply count by severity weight
   - total_score = sum of all weighted events
   - event_count = total number of events
   - severity_breakdown: dict mapping event type to weighted score contribution
   - ExtractionReport expected: ["total_score", "event_count", "severity_breakdown"]
   - NOTE: This extractor MUST run AFTER all other SECT4 extractors (dependency order)

**sentiment_analysis.py:**

Three public functions:

1. `extract_sentiment(state: AnalysisState) -> tuple[SentimentProfile, ExtractionReport]`
   - Run L-M analysis on 10-K MD&A text (Item 7) as proxy for earnings call sentiment
   - Extract broader sentiment signals from web search results
   - Combine into SentimentProfile

2. `assess_narrative_coherence(state: AnalysisState) -> tuple[NarrativeCoherence, ExtractionReport]`
   - Check alignment between management claims and observable data
   - Rule-based checks (not LLM) per research recommendation

**L-M Sentiment helpers:**

3. `_analyze_lm_sentiment(text: str) -> dict[str, Any]`
   - import pysentiment2 as ps (local import to avoid import issues if not installed)
   - lm = ps.LM()
   - tokens = lm.tokenize(text)
   - score = lm.get_score(tokens)
   - Returns: {"positive": score["Positive"], "negative": score["Negative"], "polarity": score["Polarity"], "subjectivity": score["Subjectivity"]}
   - Also compute custom counts for uncertainty, litigious, and constraining words if available from pysentiment2 API
   - Wrap in try/except -- pysentiment2 may have import or runtime issues

4. `_compute_sentiment_trends(current_score: dict, filing_texts: dict) -> SentimentProfile`
   - Analyze current 10-K Item 7 text
   - If prior year Item 7 available (from 10-K filing history), analyze that too
   - Compare negative/positive/uncertainty word counts between periods
   - Set management_tone_trajectory:
     - "IMPROVING" if negative count decreased and/or positive increased
     - "DETERIORATING" if negative count increased and/or positive decreased
     - "STABLE" otherwise
   - hedging_language_trend from uncertainty word count comparison
   - Note: CEO vs CFO divergence (SECT5-04c) deferred until earnings call transcripts acquired -- set ceo_cfo_divergence to None
   - Note: Q&A evasion (SECT5-04d) deferred -- set qa_evasion_score to None

**Broader sentiment helpers:**

5. `_extract_broader_signals(web_results: dict, market_data: dict) -> None`
   - Sets SentimentProfile fields from web search results:
   - glassdoor_rating: search web_search_results for "glassdoor" mentions with rating numbers
   - news_sentiment: overall tone from news results (simple positive/negative keyword counting)
   - social_media_sentiment: Reddit/social mentions from blind_spot_results
   - employee_sentiment: derived from glassdoor + LinkedIn indicators
   - All with LOW confidence (web-sourced)

**Narrative coherence helpers:**

6. `_check_strategy_vs_results(state: AnalysisState) -> str`
   - Compare stated strategy (from 10-K Item 1) against financial results
   - If company claims "growth" but revenue declining -> "MISALIGNED"
   - Simple keyword + data comparison

7. `_check_insider_vs_confidence(state: AnalysisState) -> str`
   - If management states optimism in filings but insiders are net selling -> "MISALIGNED"
   - Requires insider_analysis data (SECT4-04) if available, else None

8. `_check_tone_vs_financials(state: AnalysisState) -> str`
   - If L-M sentiment is positive but financial trajectory is deteriorating -> "MISALIGNED"
   - Compare polarity score against financial health narrative

9. `_check_employee_vs_management(state: AnalysisState) -> str`
   - If Glassdoor rating declining but management messaging positive -> "MISALIGNED"
   - Requires broader sentiment data

Each coherence check produces a SourcedValue[str] with LOW confidence and "AI Assessment" label per DATA-14 requirement. The methodology is disclosed: "Rule-based comparison of [X] against [Y]."

ExtractionReport expected: ["lm_sentiment", "tone_trajectory", "glassdoor", "news_sentiment", "coherence_flags"]

Create extractor with two data sources: (1) Form 4 XML from filing_text (primary, HIGH confidence), (2) yfinance insider_transactions (fallback, MEDIUM confidence).

**Main function:**
```python
def extract_insider_trading(
    state: AnalysisState,
) -> tuple[InsiderTradingAnalysis, ExtractionReport]:
```

**Form 4 XML parsing helpers:**

1. `_parse_form4_xml(xml_text: str) -> list[InsiderTransaction]`
   - Use xml.etree.ElementTree to parse Form 4 XML
   - Extract: reportingOwner name/title, transactionDate, transactionCode (S/P/A/M/F/G/D), shares, pricePerShare, acquiredDisposedCode
   - Check for AFF10B5ONE element for 10b5-1 plan indicator (post-April 2023)
   - Set is_10b5_1 based on AFF10B5ONE, is_discretionary = (code in ("S", "P") and not is_10b5_1)
   - Map transactionCode to human-readable transaction_type ("SALE", "PURCHASE", "GRANT", etc.)
   - Handle missing elements gracefully (return partial transaction)

2. `_extract_from_form4s(form4_list: list[dict[str, Any]]) -> list[InsiderTransaction]`
   - Iterate over Form 4 XML text fetched by filing_text.fetch_form4_xml()
   - Call _parse_form4_xml for each
   - Sort by date, filter to trailing 18 months

3. `_extract_from_yfinance(insider_data: dict[str, Any]) -> list[InsiderTransaction]`
   - Fallback: parse yfinance insider_transactions dict
   - Column names may vary ("Insider Trading" or "Insider", "Transaction" or "Type")
   - Use .get() for all column access
   - Set is_10b5_1 = None (yfinance doesn't provide this)
   - MEDIUM confidence

**Cluster detection:**

4. `_detect_cluster_selling(transactions: list[InsiderTransaction], window_days: int = 30, min_insiders: int = 3) -> list[InsiderClusterEvent]`
   - Filter to sales (transaction_code "S" or transaction_type "SALE")
   - Sliding window: for each sale, check how many unique insiders sold within next 30 days
   - If >= 3 unique insiders: create InsiderClusterEvent
   - Deduplicate overlapping clusters (merge if window overlaps)

**Aggregation:**

5. `_compute_aggregates(transactions: list[InsiderTransaction]) -> dict`
   - total_sold_value: sum of value for sales
   - total_bought_value: sum of value for purchases
   - net_buying_selling: "NET_BUYING" if bought > sold, "NET_SELLING" if sold > bought, "NEUTRAL"
   - pct_10b5_1: percentage of sale transactions with is_10b5_1 == True
   - Filter M/F/G/A transactions out of buying/selling calculation

**ExtractionReport:**
Expected: ["transactions", "cluster_events", "net_direction", "10b5_1_classification", "aggregate_ownership"]
Source: "SEC Form 4 XML" (primary) or "yfinance insider_transactions" (fallback)

**Main function:**

```python
    state: AnalysisState,

```

**Helpers:**

1. `_extract_current_short_interest(info: dict[str, Any]) -> ShortInterestProfile`
   - short_pct_float from info["shortPercentOfFloat"] (convert from 0-1 to 0-100%)
   - days_to_cover from info["shortRatio"]
   - Compute trend_6m from current vs prior month: info["sharesShort"] vs info["sharesShortPriorMonth"]
     - If current > prior * 1.1: "RISING"
     - If current < prior * 0.9: "DECLINING"
     - Else: "STABLE"

2. `_compare_vs_peers(company_si: float, peer_data: dict[str, Any]) -> float`
   - If state.extracted.financials.peer_group exists, get peer tickers
   - For v1, use vs_sector_ratio as ratio of company SI to average SI from info dict
   - Set vs_sector_ratio field

3. `_identify_short_seller_reports(web_results: dict[str, Any], company_name: str) -> list`

   - Known short sellers: Hindenburg Research, Muddy Waters, Citron Research, Spruce Point Capital, Kerrisdale Capital, Iceberg Research, Grizzly Research, Blue Orca Capital, Gotham City Research, Bonitas Research
   - Store as short_seller_reports with date, source, key allegations

<objective>
Build the traceability chain validation and provenance tracking layers on top of the knowledge store -- ensuring every check has explicit data-to-output linkage and every modification is auditable.

Purpose: Traceability ensures that no check can be activated without all pipeline links being verified (data source exists, extraction logic exists, output mapping exists, scoring rule exists). Provenance ensures that knowledge evolution is auditable -- no silent changes or lost history.

Output: Traceability validator that reports missing links for any check, and provenance utilities for querying modification history.
</objective>

  <name>Task 1: Create traceability chain validation</name>
  <files>
    src/do_uw/knowledge/traceability.py
    tests/knowledge/test_traceability.py
  </files>
  <action>
    Create `src/do_uw/knowledge/traceability.py`:

    1. **TraceabilityLink** dataclass:
       - link_type: str (DATA_SOURCE, EXTRACTION, OUTPUT, SCORING)
       - expected: str (what should exist)
       - found: bool
       - detail: str (description of what was found or missing)

    2. **TraceabilityReport** dataclass:
       - check_id: str
       - check_name: str
       - status: str (COMPLETE, INCOMPLETE, BROKEN)
       - links: list[TraceabilityLink]
       - missing_count: int
       - summary: str

    3. **KNOWN_DATA_SOURCES**: set of valid data source IDs from checks.json data_sources section (SEC_10K, SEC_10Q, SEC_DEF14A, SEC_8K, SEC_FORM4, SEC_13DG, MARKET_PRICE, LITIGATION_DB, NEWS_SEARCH, etc.)

    4. **KNOWN_SCORING_FACTORS**: set of valid factor IDs (F1_prior_litigation through F10_deal_structure)

    5. **EXTRACTOR_STATE_PATHS**: set of state paths that are actually populated by EXTRACT stage extractors. Build by scanning `src/do_uw/stages/extract/` files for assignments to `state.extracted.*` and `state.company.*` patterns. Hardcode the known set at module level (e.g., "extracted.financials.statements", "extracted.market.insider_trading", "extracted.governance.board", "extracted.litigation.securities_class_actions", etc.). This set is the ground truth for whether an extractor actually populates a given path.

    6. **validate_traceability(check: dict[str, Any]) -> TraceabilityReport**:
       - Check DATA_SOURCE: each entry in required_data is in KNOWN_DATA_SOURCES
       - Check EXTRACTION: data_locations is non-empty and has valid state paths (basic format validation: starts with "extracted." or "company.")
       - Check EVALUATION: for each state path in data_locations, verify the path (or its parent) exists in EXTRACTOR_STATE_PATHS. If a check references a data_location that no extractor populates, mark as INCOMPLETE with detail "No extractor populates {path}". This catches checks that have data_locations defined but no actual extraction code to fill them.
       - Check OUTPUT: output_section is set and valid (SECT1-SECT7)
       - Check SCORING: scoring_factor is in KNOWN_SCORING_FACTORS (or None for info-only checks)
       - Return TraceabilityReport with all links assessed

    7. **validate_all_checks(store: KnowledgeStore) -> list[TraceabilityReport]**:
       - Query all checks from store
       - Run validate_traceability on each
       - Return sorted by missing_count descending (most broken first)

    8. **get_activation_readiness(store: KnowledgeStore, check_id: str) -> TraceabilityReport**:
       - Run traceability validation for a single check
       - Used when transitioning DEVELOPING -> ACTIVE to verify all links

    Create `tests/knowledge/test_traceability.py`:
    - Test fully traceable check returns COMPLETE status
    - Test check missing data source returns INCOMPLETE with correct link
    - Test check missing scoring factor returns INCOMPLETE
    - Test check missing output section returns INCOMPLETE
    - Test validate_all_checks processes multiple checks
    - Test get_activation_readiness with store interaction
    - Test info-only checks (no scoring factor) are valid
    - Test check with data_location path not in EXTRACTOR_STATE_PATHS returns INCOMPLETE with "No extractor populates" detail
    - Test check with data_location path whose parent IS in EXTRACTOR_STATE_PATHS returns COMPLETE (parent match allowed)
  </action>

<task type="auto">
  <name>Task 2: Create provenance tracking and audit trail utilities</name>
  <files>
    src/do_uw/knowledge/provenance.py
    tests/knowledge/test_provenance.py
  </files>
  <action>
    Create `src/do_uw/knowledge/provenance.py`:

    1. **ProvenanceEntry** dataclass:
       - version: int
       - field_name: str
       - old_value: str | None
       - new_value: str | None
       - changed_at: datetime
       - changed_by: str
       - reason: str | None

    2. **ProvenanceSummary** dataclass:
       - check_id: str
       - check_name: str
       - origin: str (BRAIN_MIGRATION, USER_ADDED, AI_GENERATED)
       - created_at: datetime
       - current_version: int
       - total_modifications: int
       - status_transitions: list[ProvenanceEntry] (filtered to status changes only)
       - recent_changes: list[ProvenanceEntry] (last 10 changes, any field)

    3. **get_check_history(store: KnowledgeStore, check_id: str) -> list[ProvenanceEntry]**:
       - Query CheckHistory table for all entries matching check_id
       - Return sorted by version ascending (oldest first)

    4. **get_provenance_summary(store: KnowledgeStore, check_id: str) -> ProvenanceSummary**:
       - Get check metadata + full history
       - Compute status_transitions (filter history for field_name="status")
       - Get recent_changes (last 10 entries)
       - Return summary

    5. **get_migration_stats(store: KnowledgeStore) -> dict[str, Any]**:
       - Count checks by origin (BRAIN_MIGRATION, USER_ADDED, AI_GENERATED)
       - Count checks by status (ACTIVE, DEPRECATED, etc.)
       - Count total history entries
       - Return summary dict

    6. **get_deprecation_log(store: KnowledgeStore) -> list[dict[str, Any]]**:
       - Find all DEPRECATED checks
       - Include the history entry for when status changed to DEPRECATED (reason)
       - Return list sorted by deprecation date

    Create `tests/knowledge/test_provenance.py`:
    - Test get_check_history returns entries in version order
    - Test get_provenance_summary includes status transitions
    - Test get_migration_stats counts correctly by origin
    - Test get_deprecation_log includes deprecation reason
    - Test check with no history returns empty lists
    - Use in-memory SQLite with pre-populated test data including history entries
  </action>

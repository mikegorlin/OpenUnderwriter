  <action>
    Create `src/do_uw/knowledge/lifecycle.py` with:

    1. **CheckStatus** enum (StrEnum for consistency with codebase pattern):
       - INCUBATING = "INCUBATING" -- Raw idea captured
       - DEVELOPING = "DEVELOPING" -- Building data/eval/output chain
       - ACTIVE = "ACTIVE" -- Production-ready, executes in pipeline
       - DEPRECATED = "DEPRECATED" -- Preserved but inactive

    2. **VALID_TRANSITIONS** dict mapping current status to allowed next statuses:
       - INCUBATING -> [DEVELOPING, DEPRECATED]
       - DEVELOPING -> [ACTIVE, INCUBATING, DEPRECATED]
       - ACTIVE -> [DEPRECATED]
       - DEPRECATED -> [DEVELOPING] (reactivation path)

    3. **validate_transition(from_status, to_status)** -> bool:
       - Returns True if transition is valid per VALID_TRANSITIONS
       - Returns False otherwise

    4. **transition_check(session, check_id, to_status, changed_by, reason)** -> None:
       - Load check from session by ID
       - Validate transition is allowed
       - Raise ValueError if invalid transition
       - Update check.status, check.modified_at
       - Increment check.version
       - Insert CheckHistory record with field_name="status", old_value, new_value, changed_by, reason
       - session.flush() (let caller commit)

    5. **record_field_change(session, check_id, field_name, old_value, new_value, changed_by, reason)** -> None:
       - Generic field change recording
       - Insert CheckHistory record
       - Update check.modified_at and check.version
       - session.flush()

    Create `tests/knowledge/__init__.py` (empty).
    Create `tests/knowledge/test_lifecycle.py` with tests:
    - Test all valid transitions succeed
    - Test invalid transitions raise ValueError (e.g., INCUBATING -> ACTIVE directly)
    - Test history record created on transition
    - Test record_field_change creates history entry
    - Test DEPRECATED -> DEVELOPING reactivation works
    - Use in-memory SQLite for tests: `create_engine("sqlite://", echo=False)` with `Base.metadata.create_all(engine)`
  </action>

    1. Create `src/do_uw/knowledge/ingestion.py`:

       **IngestionResult** dataclass:
       - document_name: str
       - doc_type: str
       - checks_created: int
       - notes_added: int
       - errors: list[str]

       **DocumentType** StrEnum:
       - SHORT_SELLER_REPORT = "SHORT_SELLER_REPORT"
       - CLAIMS_STUDY = "CLAIMS_STUDY"
       - UNDERWRITER_NOTES = "UNDERWRITER_NOTES"
       - INDUSTRY_ANALYSIS = "INDUSTRY_ANALYSIS"
       - REGULATORY_GUIDANCE = "REGULATORY_GUIDANCE"
       - GENERAL = "GENERAL"

       **extract_knowledge_items(text: str, doc_type: DocumentType, extraction_fn: Callable[[str, DocumentType], list[dict[str, Any]]] | None = None) -> list[dict[str, Any]]**:
       - If extraction_fn is provided, delegate to it (hook for future LLM-based extraction in Phase 13). This enables pluggable extraction strategies without modifying ingestion.py.
       - Default (extraction_fn=None): Rule-based extraction (v1 is rule-based for structured docs):
       - Parse text for structured patterns:
         - Lines starting with "RISK:" or "CHECK:" -> potential check ideas
         - Lines starting with "NOTE:" or "OBSERVATION:" -> underwriting notes
         - Bullet points under "KEY FINDINGS" / "CONCLUSIONS" headers -> notes
         - Numbered lists -> potential check items
       - Each extracted item has: type (check_idea/note), title, content, tags (inferred from doc_type)
       - Returns list of extracted items
       - Add a TODO comment in code: "# TODO(Phase 13): Implement LLM-based extraction_fn for unstructured industry reports"

       **ingest_document(store: KnowledgeStore, document_path: Path, doc_type: DocumentType) -> IngestionResult**:
       - Read document text (supports .txt and .md files)
       - Call extract_knowledge_items
       - For each check_idea: create incubating check via store (status=INCUBATING, origin=AI_GENERATED, name from title, metadata with source doc)
       - For each note: add note via store (tag with doc_type, source=document_path.name)
       - Return IngestionResult

       **ingest_text(store: KnowledgeStore, text: str, source_name: str, doc_type: DocumentType) -> IngestionResult**:
       - Same as ingest_document but from raw text (for CLI piping)

    2. Update `src/do_uw/cli.py` -- Add a `knowledge` command group (Typer sub-app):

       IMPORTANT: Check current cli.py line count. If close to 500 lines, create a separate `src/do_uw/cli_knowledge.py` and register as a sub-app.

       Commands:
       - `do-uw knowledge migrate` -- Run JSON-to-knowledge-store migration
         - Calls migrate_from_json with brain_dir
         - Prints migration result (counts for each type)
       - `do-uw knowledge stats` -- Show knowledge store statistics
         - Calls get_migration_stats

       - `do-uw knowledge ingest <filepath> --type <doc_type>` -- Ingest a document
         - Calls ingest_document
         - Prints ingestion result
       - `do-uw knowledge search <query>` -- Full-text search
         - Calls store.search_checks
         - Prints matching checks with scores

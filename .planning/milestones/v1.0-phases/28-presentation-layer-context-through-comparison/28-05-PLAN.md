  **Create tier_helpers.py:**
  Create src/do_uw/stages/render/tier_helpers.py with rendering helpers for the four-tier display:

  1. `render_objective_signal(doc: Any, ds: DesignSystem, signal_text: str, risk_level: str, evidence: str | None = None) -> None`
     - Renders a visually highlighted callout for objective risk findings
     - Uses a light background shading (risk-level-dependent color from design_system.py)
     - Bold signal text, followed by evidence in normal weight
     - Example: "[ELEVATED] Beneish M-Score = -1.42 (LIKELY MANIPULATOR)" with yellow background

  2. `render_scenario_context(doc: Any, ds: DesignSystem, scenario_text: str) -> None`
     - Renders Level 2 "What COULD BE" context in italic/indented style
     - Example: "Companies with M-Score below -1.78 experience SCA filings at 2.3x the base rate."
     - Uses DOCaption or similar lighter style to distinguish from facts

  3. `add_meeting_prep_ref(doc: Any, ds: DesignSystem, question_ref: str) -> None`
     - Renders Level 3 "What to ASK" cross-reference
     - Example: "See Meeting Prep Q3: Earnings quality clarification" in accent color, italic
     - Placed after the signal it relates to

  4. `render_customary_block(doc: Any, ds: DesignSystem, text: str) -> None`
     - Standard body text rendering for Customary tier data
     - Just adds a paragraph with DOBody style -- exists for consistency and readability in caller code

  **Wire tier helpers into sections with elevated signals:**
  In sect3_financial.py (only the NOT-clean path):
  - When distress model shows DISTRESS or GREY zone, use render_objective_signal() instead of plain text
  - Add render_scenario_context() with claim rate data: "Companies entering the distress zone experience claim frequency 3.2x industry baseline."
  - These scenario texts can be hardcoded constants (derived from brain/ domain knowledge, not dynamic lookups) -- they're educational context, not data

  In sect4_market.py (only the NOT-clean path):
  - When insider cluster selling detected, use render_objective_signal()
  - Add render_scenario_context(): "Cluster insider selling within 90 days of a stock drop correlates with securities class actions in 42% of cases."

  In sect5_governance.py (only the NOT-clean path):
  - When CEO/Chair duality or overboarding flagged, use render_objective_signal()
  - Add scenario context where applicable

  In sect6_litigation.py (only the NOT-clean path):
  - When active SCA or SEC enforcement exists, use render_objective_signal()
  - Add scenario context for defense assessment: "Cases with PSLRA safe harbor and forum selection provisions settle at 40% lower median amounts."

  Note: Meeting prep cross-references (add_meeting_prep_ref) require knowing the question number. Since questions are generated separately in the meeting prep section, cross-references should use descriptive text like "See Meeting Prep: Bear Case Scenarios" or "See Meeting Prep: Earnings Quality" rather than specific Q numbers (which may vary). This avoids a dependency between section rendering and question numbering.

<objective>
Wire the LLM extraction engine into the ExtractStage pipeline and CLI. LLM extraction runs as a pre-step before existing regex extractors, iterating over all filing documents in `acquired.filing_documents`, looking up the appropriate schema from the registry, and calling `LLMExtractor.extract()`. Results are stored alongside (not replacing) regex extraction results for downstream consumers to use.

Purpose: This plan connects the LLM engine (Plan 01) and schemas (Plan 02) to the actual pipeline, making LLM extraction a working feature that runs on every `do-uw analyze` command.
Output: Working end-to-end pipeline with LLM extraction enabled by default when ANTHROPIC_API_KEY is set.
</objective>

         ```
         # Phase 0: LLM extraction (pre-step)
         llm_results = _run_llm_extraction(state, self._use_llm)
         ```
       - Create module-level function `_run_llm_extraction(state: AnalysisState, use_llm: bool) -> dict[str, Any]`:
         1. If not `use_llm`, return empty dict.
         2. Import `LLMExtractor` and `ExtractionCache` from `do_uw.stages.extract.llm` (lazy import to avoid import-time failures if deps missing).
         3. Import `get_schema_for_filing` and `get_prompt` from schema/prompt modules.
         4. Try to create `ExtractionCache(db_path=Path(".cache/analysis.db"))`.
         5. Try to create `LLMExtractor(cache=cache)`. If instructor is not installed or API key missing, this will be handled inside extract() returning None.
         6. Get filing_documents from `state.acquired_data.filing_documents` (the dict of form_type -> list of FilingDocument dicts, per Phase 17 fix).
         7. Iterate over filing_documents:
            ```
            for form_type, docs in filing_documents.items():
                entry = get_schema_for_filing(form_type)
                if entry is None:
                    continue  # No schema for this filing type (e.g., Form 4)
                prompt = get_prompt(entry.prompt_key)
                for doc in docs:
                    accession = doc.get("accession", "")
                    full_text = doc.get("full_text", "")
                    if not full_text or not accession:
                        continue
                    result = extractor.extract(
                        filing_text=full_text,
                        schema=entry.schema,
                        accession=accession,
                        form_type=form_type,
                        system_prompt=prompt,
                        max_tokens=entry.max_tokens,
                    )
                    if result is not None:
                        all_results[f"{form_type}:{accession}"] = result
            ```
         8. After all extractions, log cost summary: `extractor.cost_summary`.
         9. Return `all_results` dict.
         10. Wrap entire function body in try/except. On any exception, log warning "LLM extraction failed; continuing with regex only" and return empty dict.

       - Store `llm_results` on `state.acquired_data` (or a new field). For Phase 18, store as `state.acquired_data.llm_extractions: dict[str, Any]` -- add this field to `AcquiredData` model in state.py.

       IMPORTANT: The LLM extraction results are stored but NOT yet consumed by downstream extractors. Phase 19-20 will wire the LLM results into the governance, litigation, and other extractors. Phase 18 is about building the engine and verifying it works. The existing regex extractors continue to run exactly as before.

       - Keep the file under 500 lines. The existing __init__.py is ~397 lines. Adding ~60 lines for the new function and wiring keeps it under 460. If it would exceed, split `_run_llm_extraction` into a separate file `llm_orchestrator.py`.

       NOTE: Store as serialized dicts (not Pydantic models) since the specific schema type varies by filing type. Downstream consumers will know which schema to deserialize with.

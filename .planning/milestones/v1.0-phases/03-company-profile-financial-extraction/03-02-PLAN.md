  <action>
  Create `company_profile.py` (~400 lines, split if approaching 500) implementing full SECT2 extraction.

  **Main function:**
  ```python
  def extract_company_profile(
      state: AnalysisState,
  ) -> tuple[CompanyProfile, ExtractionReport]:
  ```
  Takes the analysis state (with acquired_data populated) and returns an enriched CompanyProfile plus a validation report.

  **Sub-extractors (private functions):**

  1. **`_enrich_identity(state) -> list[ExtractionReport]`** (SECT2-01):
  Populate remaining identity fields not set by RESOLVE stage. Use Company Facts API data to confirm SIC/NAICS. Use yfinance `info` dict from `acquired_data.market_data` for exchange, industry classification, employee count, market cap. Map industry using SECT2-08 classification.

  2. **`_extract_business_description(state) -> tuple[SourcedValue[str] | None, ExtractionReport]`** (SECT2-02):
  Extract business description from 10-K Item 1. Use edgartools or filing text from acquired data. If filing text available, extract first ~2000 chars of Item 1 narrative. Create a SourcedValue with filing reference.

  3. **`_extract_revenue_segments(state) -> tuple[list, ExtractionReport]`** (SECT2-02):
  Extract revenue breakdown by segment from the Company Facts XBRL data or 10-K text. Look for dimensional XBRL data under `srt:ConsolidationItemsAxis` or `us-gaap:StatementBusinessSegmentsAxis`. If no segment data in XBRL, note as "Not Available" with reason "not_disclosed" or "single_segment_filer". CRITICAL: Never impute segments that don't exist.

  4. **`_extract_geographic_footprint(state) -> tuple[list, ExtractionReport]`** (SECT2-03):
  Parse Exhibit 21 (subsidiaries list) for geographic presence. Multi-strategy parser:
  - Strategy 1: HTML table extraction (look for `<table>` elements)
  - Strategy 2: Pattern matching for "Name | Jurisdiction" pairs
  - Strategy 3: Line-by-line parsing with jurisdiction keyword detection
  Count subsidiaries per jurisdiction. Flag tax haven jurisdictions by cross-referencing against tax_havens.json config.

  5. **`_extract_concentration(state) -> tuple[list, list, ExtractionReport]`** (SECT2-04):
  Extract customer and supplier concentration from 10-K text. Look for phrases like "represented X% of revenue" or "major customer" in Item 1/Item 7. Parse percentage values. Return lists of customer and supplier concentration entries.

  6. **`_extract_operational_complexity(state) -> tuple[dict, ExtractionReport]`** (SECT2-05):
  Detect complexity indicators: subsidiary count (from Exhibit 21), VIEs (search for "variable interest" in filing text), dual-class stock (search for "Class A", "Class B" in proxy/10-K), special purpose entities. Return dict with boolean flags and counts.

  7. **`_extract_business_changes(state) -> tuple[list, ExtractionReport]`** (SECT2-06):
  Detect acquisitions, divestitures, restructurings from 8-K filings and 10-K text. Search for keywords: "acquisition", "merger", "divestiture", "restructuring", "discontinued operations" in filing text. Extract with dates.

  8. **`_map_do_exposure(profile) -> list`** (SECT2-07):
  Map business operations to D&O exposure factors. Use the populated profile fields to determine exposure types:
  - Revenue concentration > 20% single customer -> CUSTOMER_CONCENTRATION_RISK
  - International operations (>30% non-US revenue) -> REGULATORY_MULTI_JURISDICTION
  - High employee count (>10K) -> EMPLOYMENT_LITIGATION_RISK
  - Financial sector SIC -> REGULATORY_SENSITIVE
  - Recent M&A activity -> TRANSACTION_LITIGATION_RISK
  - Technology sector -> IP_LITIGATION_RISK
  This is DERIVED (not from filings directly) so mark as LOW confidence.

  9. **`_build_event_timeline(state) -> list`** (SECT2-10):
  Build chronological event timeline from 8-K filings, 10-K significant events, and market data. Include: IPO date, major acquisitions, CEO changes, restatements, material events from 8-K. Sort chronologically.

  10. **`_generate_section_summary(profile) -> SourcedValue[str]`** (SECT2-11):
  Generate a 2-3 sentence summary paragraph describing the company, its business, key risk factors for D&O. Mark as DERIVED/LOW confidence since it's synthesized from extracted data, not directly from a filing.

  **Important implementation details:**
  - Every sub-extractor returns an ExtractionReport listing expected vs. found fields
  - Use `create_report()` from validation.py for consistent report creation
  - When data is not found, use "Not Available" with reason code, never fabricate
  - Source every value: filing type + date + accession number or "yfinance" or "derived"
  - Filing text should be accessed from `state.acquired_data.filings` (populated by SEC client in ACQUIRE)
  - For Exhibit 21, if raw text not in acquired_data, use `sec_get_text()` to fetch it (mark as ACQUIRE expansion in code comment)

  <action>
  Create `financial_statements.py` (~450 lines, split if approaching 500) implementing three-statement XBRL extraction.

  **Main function:**
  ```python
  def extract_financial_statements(
      state: AnalysisState,
  ) -> tuple[FinancialStatements, list[ExtractionReport]]:
  ```
  Returns populated FinancialStatements model plus extraction reports for each statement.

  **Tier 1: Company Facts API extraction** (primary path):

  1. Load Company Facts data from `state.acquired_data.filings["company_facts"]`
  2. Load XBRL mapping via `load_xbrl_mapping()`
  3. For each statement type (income, balance_sheet, cash_flow):
     a. Get the list of expected concepts from xbrl_concepts.json filtered by `statement` field
     b. For each concept, call `resolve_concept()` to get historical values
     c. `get_period_values()` to extract latest 3 annual periods
     d. Build `FinancialLineItem` for each concept with values keyed by period label ("FY2024", "FY2023", etc.)
     e. Compute `yoy_change` as `(FY_latest - FY_prior) / abs(FY_prior) * 100` where both exist

  **Income statement concepts** (SECT3-02, ~15 line items):
  revenue, cost_of_revenue, gross_profit, research_development, sga_expense, operating_income, interest_expense, other_income, pretax_income, income_tax_expense, net_income, eps_basic, eps_diluted, weighted_avg_shares_basic, weighted_avg_shares_diluted

  **Balance sheet concepts** (SECT3-03, ~15 line items):
  cash_and_equivalents, accounts_receivable, inventory, current_assets, property_plant_equipment, goodwill, intangible_assets, total_assets, accounts_payable, current_liabilities, long_term_debt, total_liabilities, stockholders_equity, retained_earnings, shares_outstanding

  **Cash flow concepts** (SECT3-04, ~10 line items):
  operating_cash_flow, depreciation_amortization, capital_expenditures, investing_cash_flow, financing_cash_flow, dividends_paid, share_repurchases

  **Period labeling:**
  - Determine fiscal year from the `fy` field in XBRL facts
  - Label as "FY{year}" for annual (e.g., "FY2024")
  - For quarterly: "Q{quarter} {year}" (e.g., "Q3 2024")
  - The `periods` list on FinancialStatement should be in chronological order

  **YTD column:**
  If the most recent quarterly filing (10-Q) has year-to-date figures, include them as a separate column labeled "YTD {year}".

  **Deduplication logic** (CRITICAL -- see Research pitfall #2):
  - Company Facts API returns facts from ALL filings (10-K, 10-Q, amendments)
  - For annual values: filter to `form="10-K"`, then deduplicate by `end` date, keeping the most recently `filed` entry
  - For quarterly values: filter to `form="10-Q"`, apply same dedup
  - For instant facts (balance sheet): use `end` date only
  - For duration facts (income/cash flow): use `end`+`fy`+`fp` combination

  **Tier 2: edgartools fallback** (when Company Facts has gaps):
  If a statement has < 50% coverage from Company Facts API, attempt edgartools fallback:
  ```python
  from edgar import Company  # type: ignore[import-untyped]
  company = Company(ticker)
  filings = company.get_filings(form="10-K").head(3)
  # Extract from filing-level XBRL
  ```
  Note: edgartools makes network calls, so this is technically an ACQUIRE expansion. Mark with comment `# ACQUIRE expansion: edgartools filing fetch`. Import locally to avoid import-time side effects.

  **ExtractionReport per statement:**
  For each of the 3 statements, produce an ExtractionReport:
  - `expected_fields`: All concepts for that statement from xbrl_concepts.json
  - `found_fields`: Concepts that resolved to data
  - `missing_fields`: Concepts that returned None
  - `source_filing`: "Company Facts API" or specific filing accession

  **Edge cases:**
  - Newly public companies: May only have 1-2 periods. Extract what's available, set `periods_available` accordingly.
  - Foreign filers: May use IFRS taxonomy instead of US-GAAP. Check `facts.get("ifrs-full", {})` as fallback.
  - Non-standard fiscal year: FY ending in March means "FY2024" is April 2023 - March 2024. Derive from the `end` date.

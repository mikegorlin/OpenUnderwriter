  **2a. Create validation.py** (`stages/extract/validation.py`, ~200 lines):

  Build the extraction validation framework that prevents silent incompleteness and imputation. This is the CRITICAL trust mechanism per CONTEXT.md.

  ```python
  @dataclass
  class ExtractionReport:
      extractor_name: str          # e.g., "income_statement"
      expected_fields: list[str]   # What we expected to find
      found_fields: list[str]      # What we actually found
      missing_fields: list[str]    # Expected but not found
      unexpected_fields: list[str] # Found but not expected (informational)
      coverage_pct: float          # found / expected
      confidence: Confidence       # Overall extraction confidence
      source_filing: str           # Filing type + date + accession
      fallbacks_used: list[str]    # Any fallback strategies employed
      warnings: list[str]          # Issues detected during extraction
  ```

  Add helper functions:
  - `create_report(extractor_name, expected, found, source_filing, ...)` -- factory that computes derived fields
  - `validate_no_imputation(extracted_values, source_values)` -- verifies extracted data exists in source
  - `merge_reports(reports: list[ExtractionReport]) -> ExtractionReport` -- combine sub-reports
  - `log_report(report: ExtractionReport)` -- log extraction coverage at appropriate level (WARNING if coverage < 50%, INFO otherwise)

  Use `Confidence` enum from `do_uw.models.common`. Coverage thresholds:
  - >= 80%: HIGH confidence
  - 50-80%: MEDIUM confidence
  - < 50%: LOW confidence

  **2b. Create xbrl_mapping.py** (`stages/extract/xbrl_mapping.py`, ~200 lines):

  Build the XBRL concept resolver that maps canonical names to actual XBRL data using the mapping table.

  Functions:
  - `load_xbrl_mapping() -> dict[str, XBRLConcept]` -- load and parse xbrl_concepts.json
  - `resolve_concept(facts: dict, mapping: dict, concept_name: str, form_type: str = "10-K") -> list[dict] | None` -- try each tag in priority order
  - `extract_concept_value(facts: dict, concept: str, form_type: str, unit: str) -> list[dict]` -- filter and deduplicate facts by form type, deduplicate by end+fy+fp
  - `get_latest_value(entries: list[dict]) -> dict | None` -- get most recent entry
  - `get_period_values(entries: list[dict], periods: int = 3) -> list[dict]` -- get N most recent annual periods

  Create `XBRLConcept` typed dict or dataclass:
  ```python
  class XBRLConcept(TypedDict):
      canonical_name: str
      xbrl_tags: list[str]
      unit: str
      period_type: str  # "instant" or "duration"
      statement: str
      description: str
  ```

  Key deduplication logic: when Company Facts API returns multiple entries for same concept+end date (because 10-K and 10-Q both report it), prefer the entry from the `form_type` parameter, and among those pick the most recently `filed` entry.

  **1a. Create audit_risk.py** (~300 lines):

  ```python
  def extract_audit_risk(
      state: AnalysisState,
  ) -> tuple[AuditProfile, ExtractionReport]:
  ```

  Extract audit risk data from 10-K filing text and XBRL (SECT3-12):

  1. **Auditor Identity**: Search 10-K for PCAOB auditor report section. Look for Big 4 names ("Deloitte", "Ernst & Young", "EY", "KPMG", "PricewaterhouseCoopers", "PwC") and other known firms. Also check Company Facts for `dei:AuditorName` XBRL element.

  2. **Auditor Tenure**: Count consecutive years with the same auditor by checking prior 10-K filings' auditor names. If prior filing data not available, check for "We have served as the Company's auditor since {year}" in auditor report text.

  3. **Opinion Type**: Parse auditor report for opinion language:
     - "unqualified opinion" or "presents fairly, in all material respects" -> "unqualified"
     - "qualified opinion" or "except for" -> "qualified"
     - "adverse opinion" -> "adverse"
     - "disclaimer of opinion" -> "disclaimer"
     Default to "unqualified" if standard language found (most common).

  4. **Going Concern**: Search for "going concern", "substantial doubt about", "ability to continue as a going concern" in auditor report and 10-K text.

  5. **Material Weaknesses**: Search Item 9A / SOX 404 section for "material weakness", "material weakness in internal control". Extract description of each weakness found.

  6. **Restatements**: Search for "restatement", "restated", "corrected and restated" in 10-K. Classify as Big R (filed restatement, 10-K/A) or Little r (revision in subsequent filing). Extract date and scope if mentioned.

  7. **Late Filings**: Check if filing date (from acquired metadata) is after the SEC deadline. Large accelerated filers: 60 days after FY end for 10-K. Accelerated: 75 days. Non-accelerated: 90 days.

  8. **SEC Comment Letters**: If acquired_data has SEC EFTS search results mentioning "CORRESP" filings, count them. Comment letters indicate SEC scrutiny.

  9. **Critical Audit Matters (CAMs)**: Search PCAOB auditor report for "Critical Audit Matter" sections. Extract title/description of each CAM. CAMs required since 2019 for large accelerated filers.

  For each field, create SourcedValue with specific filing reference.
  Expected fields for ExtractionReport: [auditor_name, is_big4, tenure_years, opinion_type, going_concern, material_weaknesses, restatements, late_filings, comment_letters, critical_audit_matters] (10 fields).

  **1b. Create tax_indicators.py** (~200 lines):

  ```python
  def extract_tax_indicators(
      state: AnalysisState,
  ) -> tuple[SourcedValue[dict[str, Any]] | None, ExtractionReport]:
  ```

  Compute tax indicators from financial data and filing text (SECT3-13):

  1. **Effective Tax Rate (ETR)**: Income Tax Expense / Pretax Income. Compare to statutory rate (21% for US corporations). Flag if ETR < 15% (potentially aggressive) or ETR > 30% (unusual, may indicate one-time items).

  2. **ETR Trend**: Compare current year ETR to prior 2 years. Decreasing trend may indicate aggressive tax planning.

  3. **Deferred Tax Analysis**: Extract deferred tax assets and liabilities from XBRL (concepts: DeferredTaxAssetsNet, DeferredTaxLiabilitiesNet). Large deferred tax assets with valuation allowance = red flag.

  4. **Tax Haven Subsidiaries**: Cross-reference Exhibit 21 subsidiaries (from company profile extraction) against tax_havens.json. Count subsidiaries in each tax haven category (zero_tax, low_tax, preferential_regime). Flag if > 20% of subsidiaries are in tax havens.

  5. **Unrecognized Tax Benefits**: Search XBRL for `UnrecognizedTaxBenefits`. Large UTB balance suggests aggressive tax positions. Check if increasing over time.

  6. **Transfer Pricing Risk**: Flag if company has significant international operations (from geographic footprint) AND declining ETR. This combination suggests potential transfer pricing scrutiny.

  Return dict with keys: effective_tax_rate, etr_trend, deferred_tax_net, tax_haven_subsidiary_count, tax_haven_pct, unrecognized_tax_benefits, transfer_pricing_risk_flag.
  ExtractionReport expected fields: [etr, etr_trend, deferred_tax, tax_havens, utb, transfer_pricing].

  **2a. Create peer_group.py** (~350 lines):

  ```python
  def construct_peer_group(
      state: AnalysisState,
      override_peers: list[str] | None = None,
  ) -> tuple[PeerGroup, ExtractionReport]:
  ```

  Build multi-signal peer group (SECT2-09, SECT3-05 foundation):

  **Step 1: Build candidate universe** using `financedatabase`:
  ```python
  import financedatabase as fd  # type: ignore[import-untyped]
  equities = fd.Equities()
  # Filter by country="United States" and broad sector
  candidates = equities.search(country="United States", sector=target_sector)
  ```
  If financedatabase is unavailable or fails, fall back to a simpler approach using yfinance sector peers.

  **Step 2: Filter by market cap band** (0.5x to 2x target):
  For each candidate, check market cap. Reject if outside band. This requires market data -- use yfinance info dict.

  **Step 3: Score each candidate** with composite peer scoring:
  - SIC match (25%): 4-digit match = 100, 3-digit = 75, 2-digit = 50, no match = 0
  - Industry match (20%): yfinance `industryKey` exact match = 100, `sectorKey` match = 50, no match = 0
  - Market cap proximity (25%): Score based on how close to target (1.0x = 100, 0.5x/2.0x = 50, edge of band = 0)
  - Revenue similarity (15%): Ratio of smaller/larger revenue * 100 (identical = 100, 10x difference = 10)
  - Description overlap (15%): Jaccard similarity of top business keywords from `longBusinessSummary`

  Normalize each signal to 0-100 before applying weights.

  **Step 4: Select top peers:**
  - Sort by composite score descending
  - Take top 10 candidates
  - Ensure minimum 5 peers (if fewer candidates, expand market cap band to 0.3x-3x)
  - If `override_peers` provided, include those first, then auto-fill remaining slots

  **Step 5: Assign peer tiers:**
  - "primary_sic": 4-digit SIC match within market cap band
  - "sector_etf": Get sector ETF ticker from `sectors.json` config
  - "market_cap_cohort": Similar market cap regardless of industry

  **Rate limiting note:** yfinance calls for peer candidates should be batched and rate-limited. Don't call yfinance.Ticker().info for more than ~20 candidates.

  **Caching:** Peer group data should be cached per ticker with 7-day TTL (peers don't change frequently).

  Return `PeerGroup` model populated with `PeerCompany` entries.
  ExtractionReport expected=[at least 5 peers], found=[actual count].

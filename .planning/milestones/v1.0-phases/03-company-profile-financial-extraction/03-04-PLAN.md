  <action>
  Create `distress_models.py` (~400 lines) implementing all four distress models with sector-appropriate variants.

  **Main function:**
  ```python
  def compute_distress_indicators(
      statements: FinancialStatements,
      sector: str,
      market_cap: float | None = None,
  ) -> tuple[DistressIndicators, list[ExtractionReport]]:
  ```
  Reads extracted financial statements and computes all four distress scores. Returns populated DistressIndicators plus validation reports.

  **Helper: Extract inputs from FinancialStatements:**
  Create `_extract_inputs(statements, concept_name) -> float | None` that safely navigates the FinancialStatement model to get the latest value for a given concept. Returns None if not found.

  **Model 1: Altman Z-Score** (SECT3-07):
  - `_compute_z_score(inputs, sector, market_cap) -> DistressResult`
  - Sector routing:
    - FINS, REIT, INSUR sectors -> `_altman_z_double_prime()` (Z''-Score for non-manufacturing)
    - Pre-revenue (revenue == 0 or None) -> `_early_stage_metrics()` (cash runway, burn rate)
    - All others -> `_altman_z_original()` (standard 5-factor)
  - Original Z-Score: `Z = 1.2*(WC/TA) + 1.4*(RE/TA) + 3.3*(EBIT/TA) + 0.6*(MktCap/TL) + 1.0*(Sales/TA)`
    - Zones: <1.81 DISTRESS, 1.81-2.99 GREY, >2.99 SAFE
  - Z''-Score: `Z'' = 6.56*(WC/TA) + 3.26*(RE/TA) + 6.72*(EBIT/TA) + 1.05*(BV_Equity/TL)`
    - Zones: <1.1 DISTRESS, 1.1-2.6 GREY, >2.6 SAFE
  - Early stage: Return DistressResult with `model_variant="early_stage"`, score=None, zone=NOT_APPLICABLE, and `alternative_metrics` dict containing: cash_runway_months, monthly_burn_rate, cash_to_debt_ratio

  **Model 2: Beneish M-Score** (SECT3-07):
  - `_compute_m_score(inputs) -> DistressResult`
  - Requires current AND prior year data for all 8 input ratios:
    - DSRI = (Receivables_t/Revenue_t) / (Receivables_t-1/Revenue_t-1)
    - GMI = GrossMargin_t-1 / GrossMargin_t
    - AQI = (1 - (CA_t + PPE_t)/TA_t) / (1 - (CA_t-1 + PPE_t-1)/TA_t-1)
    - SGI = Revenue_t / Revenue_t-1
    - DEPI = DepRate_t-1 / DepRate_t
    - SGAI = (SGA_t/Revenue_t) / (SGA_t-1/Revenue_t-1)
    - TATA = (NI_t - OCF_t) / TA_t
    - LVGI = (TL_t/TA_t) / (TL_t-1/TA_t-1)
  - M = -4.84 + 0.920*DSRI + 0.528*GMI + 0.404*AQI + 0.892*SGI + 0.115*DEPI - 0.172*SGAI + 4.679*TATA - 0.327*LVGI
  - Zones: > -1.78 DISTRESS (manipulation likely), <= -1.78 SAFE
  - If any input ratio is incalculable (division by zero or missing data), mark `is_partial=True`, list missing inputs, compute with available inputs only

  **Model 3: Ohlson O-Score** (SECT3-07):
  - `_compute_o_score(inputs) -> DistressResult`
  - O = -1.32 - 0.407*log(TA/GNP_deflator) + 6.03*(TL/TA) - 1.43*(WC/TA) + 0.076*(CL/CA) - 1.72*X + -2.37*(NI/TA) - 1.83*(FFO/TL) + 0.285*OENEG - 0.521*CHIN
  - For GNP deflator: Use a constant approximation (GDP deflator ~130 for 2024) stored in the function. This is a simplification that affects absolute score but not relative ranking.
  - X = 1 if TL > TA, else 0
  - OENEG = 1 if NI < 0 in both current and prior year, else 0
  - CHIN = (NI_t - NI_t-1) / (|NI_t| + |NI_t-1|)
  - Probability = exp(O) / (1 + exp(O))
  - Zones: probability > 0.5 DISTRESS, 0.25-0.5 GREY, < 0.25 SAFE

  **Model 4: Piotroski F-Score** (SECT3-07):
  - `_compute_f_score(inputs) -> DistressResult`
  - 9 binary criteria (0 or 1 each, total 0-9):
    - Profitability: positive NI, improving ROA, positive OCF, OCF > NI (accruals quality)
    - Leverage: decreasing long-term debt, improving current ratio, no share dilution
    - Efficiency: improving gross margin, improving asset turnover
  - Zones: 8-9 SAFE (strong), 3-7 GREY, 0-2 DISTRESS (weak)
  - Each of the 9 criteria must be individually trackable (for transparency)

  **4-Quarter Trajectory:**
  For each model, attempt to compute the score for the last 4 quarters (using quarterly financial data if available). Store as list of `{period: str, score: float, zone: str}` in `DistressResult.trajectory`. If quarterly data insufficient, store only annual scores.

  **Missing input handling (CRITICAL per CONTEXT.md):**
  - Never silently skip a missing input
  - Set `is_partial = True` when any required input is None
  - List all missing inputs in `missing_inputs: list[str]`
  - Log a WARNING for each missing input
  - Still compute the score if > 50% of inputs available; set to None if < 50%
  - Record `model_variant` to indicate which formula was used

  ```python
  def compute_earnings_quality(

  ```

  Compute earnings quality forensic analysis ratios (SECT3-06):

  1. **Accruals Ratio** = (Net Income - Operating Cash Flow) / Total Assets
     - Positive accruals = less earnings quality (cash lags income)
     - Flag if > 0.10 (high accruals)

  2. **Operating Cash Flow to Net Income** = OCF / Net Income
     - Healthy range: 0.8 to 1.5
     - Below 0.5 = poor earnings quality (income not backed by cash)
     - Negative when NI and OCF have different signs = major red flag

  3. **Revenue Quality (Days Sales Outstanding trend):**
     - DSO = (Accounts Receivable / Revenue) * 365
     - Increasing DSO trend = possible channel stuffing or aggressive revenue recognition
     - Calculate for current and prior year, report delta

  4. **Asset Quality (Non-current asset growth vs revenue growth):**
     - If non-current assets grow faster than revenue = possible capitalization of expenses

  5. **Cash Flow Adequacy** = OCF / (CapEx + Debt Payments + Dividends)
     - Below 1.0 = company cannot fund obligations from operations

  6. **Quality Score Summary:**
     - Aggregate findings into an overall quality assessment: STRONG, ADEQUATE, WEAK, RED_FLAG
     - Based on how many individual metrics are outside normal ranges

  Return as `SourcedValue[dict[str, float | None]]` with source="Derived from XBRL financial statements" and confidence=MEDIUM (derived, not audited directly).

  ExtractionReport lists expected ratios vs. computed ratios (some may be incalculable if inputs missing).

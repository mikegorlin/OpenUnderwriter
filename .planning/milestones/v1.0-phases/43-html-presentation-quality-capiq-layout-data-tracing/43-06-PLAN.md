  <action>
    Make three targeted changes to `sidebar.css`. Do NOT touch styles.css or base.html.j2.

    **Measure topbar height:** The `.sticky-topbar-inner` has `padding: 0.5rem 1.5rem` (from styles.css). The company name inside is `font-size: 0.875rem`. The rendered topbar is approximately 44px tall. Use `44px` as the offset value throughout.

    **Change 1 — Fix sidebar top offset:**

    In the `.sidebar-toc` rule, change `top: 0` to `top: 44px`. Also set height to `calc(100vh - 44px)` so the sidebar fills only the available viewport below the topbar.

    Before:
    ```css
    .sidebar-toc {
      position: sticky;
      top: 0;
      height: 100vh;
      ...
    }
    ```

    After:
    ```css
    .sidebar-toc {
      position: sticky;
      top: 44px;          /* offset by topbar height so sidebar sticks below topbar */
      height: calc(100vh - 44px);
      ...
    }
    ```

    **Change 2 — Fix content clipping (overflow on .worksheet-main):**

    In the `.worksheet-main` rule, add `overflow-x: auto`:

    Before:
    ```css
    .worksheet-main {
      min-width: 0;
      padding: 0 1.5rem;
    }
    ```

    After:
    ```css
    .worksheet-main {
      min-width: 0;
      overflow-x: auto;   /* allows wide tables to scroll horizontally within the column */
      padding: 0 1.5rem;
    }
    ```

    **Change 3 — Fix layout offset so content starts below topbar:**

    Add a new rule for `.worksheet-layout` that offsets the grid below the topbar. The topbar is `position: sticky; top: 0; z-index: 100` so it occupies the top 44px. Without a `margin-top` or `padding-top` on the layout wrapper, the first section starts at y=0 and is initially hidden behind the topbar.

    Append this rule after the existing `.worksheet-layout` rule:
    ```css
    body {
      padding-top: 44px;  /* reserve space for the sticky topbar so content is never hidden behind it */
    }
    ```

    Note: `padding-top` on `body` works because the topbar is `position: sticky` within the normal flow — the body just needs enough padding to push the content start below the topbar height. This is simpler and more reliable than `margin-top` on the layout div.

    **Update @media print block:**

    In the existing `@media print` block at the bottom of sidebar.css, ensure `.sidebar-toc` print rule is preserved as-is (already `display: none !important`). No changes needed to print block.

    The final sidebar.css should look like:
    ```css
    /* Two-column CapIQ layout ... */
    .worksheet-layout {
      display: grid;
      grid-template-columns: 180px 1fr;
      align-items: start;
    }

    body {
      padding-top: 44px;
    }

    .sidebar-toc {
      position: sticky;
      top: 44px;
      height: calc(100vh - 44px);
      overflow-y: auto;
      background: #f8fafc;
      border-right: 1px solid #e2e8f0;
      padding: 0.75rem 0;
      font-size: 11px;
      z-index: 10;
    }

    /* ... rest of sidebar rules unchanged ... */

    .worksheet-main {
      min-width: 0;
      overflow-x: auto;
      padding: 0 1.5rem;
    }

    /* ... topbar meta styles unchanged ... */

    @media print { ... unchanged ... }
    ```

    Write the complete updated sidebar.css (preserving all existing rules, only changing the three items above).

    **D — html_renderer.py: Inject ipo_date into template context**

    In `build_html_context()`, after the `context["generation_date"]` line, add:

    ```python
    # IPO date for identity block — derived from firstTradeDateMilliseconds in yfinance data
    # stored as years_public in company profile; we need to re-derive the actual date here
    ipo_date_str = ""
    if state.company and state.company.years_public:
        yp = state.company.years_public.value if hasattr(state.company.years_public, 'value') else state.company.years_public
        if yp is not None:
            from datetime import UTC, datetime, timedelta
            approx_ipo = datetime.now(tz=UTC) - timedelta(days=int(yp) * 365)
            ipo_date_str = str(approx_ipo.year)  # show year only since exact date lost
    context["ipo_date"] = ipo_date_str
    ```

    Note: the exact IPO date timestamp from yfinance (`firstTradeDateMilliseconds`) is computed during EXTRACT but only stored as `years_public` (count). The render context can only show the approximate year from that count. If the project later stores the actual date string, `context["ipo_date"]` will carry it directly. For now, showing the year is accurate enough and more informative than a raw count. If `years_public` is None, `ipo_date_str` remains empty and the template falls through to the `elif` branch.

  <action>
    Create `src/do_uw/templates/html/appendices/qa_audit.html.j2`.

    This appendix renders a transparent audit trail of every check that ran, grouped by section. Context variable `check_results_by_section` is already available in the template context (built by `_group_checks_by_section()` in html_renderer.py and passed as `check_results_by_section`).

    Each check result in `check_results_by_section[section_name]` is a dict with typical keys:
    - `check_id` — unique check identifier (e.g., "GOV-01")
    - `name` — human-readable check name (e.g., "Board Independence")
    - `status` or `ceiling` — CLEAR / ELEVATED / TRIGGERED / SKIPPED
    - `description` or `finding` — what was found
    - `evidence` — list of source citations (strings)
    - `confidence` — HIGH / MEDIUM / LOW
    - `source` — fallback source string if evidence list is empty

    Template content:
    ```jinja2
    {# QA / Audit Appendix — check-level audit trail grouped by section.
       Shows: check ID, check name, what was verified, source used, confidence.
       Section id="qa-audit" for sidebar TOC anchor.
       Uses check_results_by_section from html_renderer._group_checks_by_section(). #}

    {% if check_results_by_section %}
    <section id="qa-audit" class="page-break">
      <h2>QA / Audit Trail</h2>
      <p class="text-xs text-gray-500 mb-3">
        Audit record of all checks executed in this analysis. For each check:
        what was verified, the data source used, and confidence level
        (HIGH = audited/official, MEDIUM = unaudited/estimated, LOW = web/derived).
      </p>

      {% for section_name, checks in check_results_by_section.items() %}
      {% if checks %}
      <div class="qa-section-group">
        <h3 class="qa-section-title">{{ section_name | replace('_', ' ') | title }}</h3>
        <table class="w-full border-collapse text-xs qa-table">
          <thead>
            <tr class="bg-navy text-white">
              <th class="text-left px-2 py-1 w-20">Check ID</th>
              <th class="text-left px-2 py-1 w-40">Check</th>
              <th class="text-left px-2 py-1">Finding / Verified</th>
              <th class="text-left px-2 py-1 w-28">Source</th>
              <th class="text-left px-2 py-1 w-16">Conf.</th>
              <th class="text-left px-2 py-1 w-20">Status</th>
            </tr>
          </thead>
          <tbody>
            {% for check in checks %}
            {% set status = (check.get('status') or check.get('ceiling') or 'UNKNOWN') | upper %}
            {% set conf = (check.get('confidence') or 'MEDIUM') | upper %}
            {% set evidence = check.get('evidence', []) %}
            {% set src_text = evidence[0] if evidence else check.get('source', '—') %}
            <tr class="{{ 'bg-red-50' if status == 'TRIGGERED' else ('bg-amber-50' if status == 'ELEVATED' else ('bg-gray-50' if status == 'SKIPPED' else '')) }}">
              <td class="px-2 py-1 font-mono font-semibold">{{ check.get('check_id', '—') }}</td>
              <td class="px-2 py-1 font-semibold">{{ check.get('name', check.get('check_id', '—')) }}</td>
              <td class="px-2 py-1">{{ check.get('description', check.get('finding', '—')) | truncate(120) }}</td>
              <td class="px-2 py-1 text-gray-500">{{ src_text | truncate(40) }}</td>
              <td class="px-2 py-1">
                {% if conf == 'HIGH' %}
                  <span class="qa-conf qa-conf-high">HIGH</span>
                {% elif conf == 'MEDIUM' %}
                  <span class="qa-conf qa-conf-med">MED</span>
                {% else %}
                  <span class="qa-conf qa-conf-low">LOW</span>
                {% endif %}
              </td>
              <td class="px-2 py-1">{{ traffic_light(status, status) }}</td>
            </tr>
            {% endfor %}
          </tbody>
        </table>
      </div>
      {% endif %}
      {% endfor %}

      <p class="text-xs text-gray-400 mt-4">
        Total checks: {{ check_results_by_section.values() | sum(attribute='__len__') }}
        &bull; Generated: {{ generation_date }}
      </p>
    </section>
    {% endif %}
    ```

    - 0.89 < 1.0 -> RED (correct)
    - 1.2 < 1.5 -> YELLOW

    Check how the evaluator handles string values in check_evaluators.py. The issue is likely that the evaluator sees a non-null string value and returns INFO (informational) instead of recognizing "NONE" as a negative/clear result.

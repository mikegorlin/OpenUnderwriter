  <action>
Fix each false trigger identified in the QUESTION-AUDIT:

**1. BIZ.DEPEND.labor (Concentration Risk Composite)**
- Problem: `field_key=employee_count` but threshold expects `>2 labor risk flags`
- Fix: Change field routing so this check receives the count of labor risk flags (from extracted labor dependency data), NOT the raw employee count. If no labor flag count data exists in extracted data, route to a field that returns None (so check goes INFO/SKIPPED rather than false RED at 150,000).
- In check_field_routing.py, find the BIZ.DEPEND routing and fix the field_key for `labor`.

**2. BIZ.DEPEND.key_person (Customer Concentration Risk)**
- Problem: `field_key=employee_count` but name is "Customer Concentration Risk"
- Fix: Route to customer concentration data (e.g., top_customer_pct, customer_concentration_index) from extracted data. If no customer concentration data exists, route to None.

**3. GOV.BOARD.ceo_chair (CEO Chair Separation)**
- Problem: Evaluating boolean (1.0 = True = combined) against numeric threshold `red < 50.0` (board independence %)
- Fix: This check should evaluate boolean CEO-chair duality. Either: (a) change the threshold to a boolean check (`combined = red`, `separated = clear`), or (b) change the field routing to point to board independence % if that's what the threshold intends. Based on the check name "CEO Chair Separation", option (a) is correct -- fix the threshold in checks.json to be a boolean/classification check.

**4. GOV.PAY.peer_comparison (Peer Comparison)**
- Problem: Value 533.0 against threshold > 75.0 -- unclear units
- Fix: Examine what data the mapper returns for this check. If it's CEO total comp in millions and threshold should be a percentile rank, the data needs a different field. If it's a peer comparison ratio and 533% is legitimate (CEO pay is 5.33x peer median), then the threshold might need recalibration to a reasonable range. Investigate the mapper, determine correct units, and fix either the routing or the threshold.

**5. FIN.LIQ.position (Liquidity Position)**
- Problem: Current ratio 0.89 flagged RED against threshold `red < 6.0`
- Fix: The threshold `red < 6.0` is nonsensical for a current ratio (6.0 would flag nearly every company as RED). A reasonable threshold: `red < 1.0`, `yellow < 1.5`, `clear >= 1.5`. Update the threshold in checks.json.

For each fix, document what was wrong and what was changed in a code comment.

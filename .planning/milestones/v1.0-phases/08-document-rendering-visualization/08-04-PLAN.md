  <action>
1. Create `meeting_prep.py` (~400 lines):
   - `render_meeting_prep(doc: Document, state: AnalysisState, ds: DesignSystem) -> None`:
     - Add page break before appendix (this is the ONE exception to "no forced page breaks" -- the appendix is structurally separate from analysis)
     - Add heading "Appendix: Meeting Prep Companion"
     - Add introductory paragraph explaining this section's purpose
     - **Generate questions by walking the state model:**

   - Question generation functions (internal):
     - `_generate_clarification_questions(state) -> list[MeetingQuestion]`:
       - Scan for LOW confidence SourcedValue fields across all extracted data
       - Scan for data points with conflicting sources
       - Each produces a question asking the broker/client to clarify the data
       - Example: "Revenue segments show 45% international exposure (LOW confidence, web source). Can you confirm the current international revenue breakdown?"

     - `_generate_forward_indicator_questions(state) -> list[MeetingQuestion]`:
       - Scan for upcoming events: refinancing windows, SOL expiry dates, earnings dates, major contract expirations
       - Scan for trend signals: deteriorating distress scores, increasing short interest
       - Example: "Altman Z-Score has declined from 2.8 to 1.9 over 4 quarters. What is the company's plan to address leverage concerns?"

     - `_generate_gap_filler_questions(state) -> list[MeetingQuestion]`:
       - Scan for None fields in the state model that SHOULD have data
       - Focus on fields that matter for D&O: missing insider trading data, unavailable governance metrics
       - Example: "Board independence ratio could not be determined from proxy filings. What is the current board composition?"

     - `_generate_credibility_test_questions(state) -> list[MeetingQuestion]`:
       - Scan for narrative coherence mismatches (governance sentiment vs financial reality)
       - Scan for patterns that suggest undisclosed issues
       - Example: "Management guidance projects 15% revenue growth, but the peer group average is 3% and the company's trailing 3-year CAGR is 1%. What supports the growth projection?"

   - `MeetingQuestion` dataclass:
     - `question: str` -- the question to ask
     - `category: str` -- CLARIFICATION, FORWARD_INDICATOR, GAP_FILLER, CREDIBILITY_TEST
     - `priority: float` -- numeric priority for sorting (higher = more important)
     - `context: str` -- why this matters for D&O
     - `good_answer: str` -- what a reassuring answer looks like
     - `bad_answer: str` -- what a concerning answer looks like
     - `follow_up: str` -- if the answer is concerning, what to ask next / what to document / escalation triggers

   - **Rendering:**
     - Collect all questions from 4 generators
     - Sort by priority descending (most important first, regardless of category)
     - Render each question as a mini-section:
       - Question number + category tag in brackets (e.g., "[CLARIFICATION]")
       - The question itself (bold)
       - Context paragraph (why this matters for D&O)
       - "Expected answers" sub-section: good answer vs concerning answer
       - "If concerning" sub-section: follow-up questions, documentation guidance, escalation triggers
     - Add a footer note: "Questions generated from analysis data as of {date}. Priority reflects scoring impact and data confidence gaps."

2. Wire meeting prep into word_renderer.py: call render_meeting_prep(doc, state, ds) after all 7 sections.

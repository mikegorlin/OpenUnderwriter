1. **Create src/do_uw/cli_brain.py**: New Typer sub-app with 7 commands. Follow the pattern from cli_knowledge.py (Rich tables, console output, Typer options).

   a. **`brain status`**: Show brain.duckdb summary:
      - Use BrainDBLoader to connect and query
      - Total checks (active vs total), by lifecycle_state, by content_type, by report_section
      - Taxonomy counts: risk_questions, factors, hazards, report_sections
      - Backlog: open items count
      - Brain DB file size and location
      - Display using Rich tables

   b. **`brain gaps`**: Run pipeline gap detection:
      - Load checks via BrainDBLoader
      - Call `build_manifest()` then `detect_gaps()`
      - Display GapReport using Rich tables with severity coloring:
        * CRITICAL = red, WARNING = yellow, INFO = dim
      - Show summary: total checks, fully supported, gap counts by type and severity
      - Option: `--severity` filter (CRITICAL, WARNING, INFO, ALL)
      - Option: `--type` filter (SOURCE_NOT_ACQUIRED, NO_FIELD_KEY, etc.)

   c. **`brain effectiveness`**: Show check effectiveness metrics:
      - Call `compute_effectiveness()` from brain_effectiveness.py
      - Display EffectivenessReport tables:
        * Always-fire checks (red — too sensitive?)
        * Never-fire checks (yellow — miscalibrated?)
        * High-skip checks (yellow — data gap?)
        * Consistent checks (green — reliable)
      - Show confidence note based on N
      - Option: `--min-runs` (default 1)

   d. **`brain changelog`**: Show recent changelog:
      - Use BrainWriter.get_changelog()
      - Display as Rich table: timestamp, check_id, change_type, description, changed_by
      - Option: `--check` to filter by check_id
      - Option: `--limit` (default 50)

   e. **`brain backlog`**: Show open backlog items:
      - Use BrainDBLoader.load_backlog()
      - Display as Rich table: ID, title, priority, risk_questions, status, effort
      - Color by priority: CRITICAL=red, HIGH=yellow, MEDIUM=default, LOW=dim

   f. **`brain export-docs`**: Export brain to readable Markdown:
      - Query brain_checks_active, organized by report_section
      - For each section, list checks with: check_id, name, question, threshold_red/yellow/clear, risk_questions, lifecycle_state
      - Write to specified output path (default: brain_export.md)
      - This makes the brain readable without code (Decision D9)
      - Option: `--output` path (default stdout or brain_export.md)

   g. **`brain backtest`**: Run backtesting (delegates to BacktestRunner — built in Task 2):
      - Argument: state file path (e.g., `output/AAPL/state.json`)
      - Call `run_backtest(state_path)` from knowledge/backtest.py
      - Display BacktestResult: ticker, checks executed, triggered/clear/skipped/info counts
      - Option: `--record` flag to store results in brain_check_runs (default True)
      - Option: `--compare` flag to compare with previous backtest of same state

2. **Register in cli.py**: Add `from do_uw.cli_brain import brain_app` and `app.add_typer(brain_app, name="brain")` in the same pattern as knowledge_app.

1. **Create src/do_uw/knowledge/backtest.py**: Build the BacktestRunner following 32-RESEARCH.md pattern:

   a. `BacktestResult` Pydantic model:
      - `ticker: str`
      - `state_path: str`
      - `state_date: str` — extracted from state file metadata or created_at
      - `checks_executed: int`
      - `triggered: int`
      - `clear: int`
      - `skipped: int`
      - `info: int`
      - `results_by_id: dict[str, str]` — check_id -> status string

   b. `BacktestComparison` Pydantic model:
      - `ticker: str`
      - `run_a: BacktestResult`
      - `run_b: BacktestResult`
      - `changed: list[dict]` — checks that changed status between runs (check_id, old_status, new_status)
      - `new_checks: list[str]` — checks in B but not A (newly added)
      - `removed_checks: list[str]` — checks in A but not B (retired)

   c. `run_backtest(state_path: Path, record: bool = True) -> BacktestResult`:
      - Load state file using standard JSON deserialization (import from models)
      - Load current check definitions via BrainDBLoader
      - Call execute_checks() from check_engine with the loaded state's ExtractedData and CompanyProfile
      - Count results by status (TRIGGERED, CLEAR, SKIPPED, INFO)
      - If record=True, insert results into brain_check_runs with is_backtest=TRUE using record_check_runs_batch()
      - Return BacktestResult

   d. `compare_backtests(result_a: BacktestResult, result_b: BacktestResult) -> BacktestComparison`:
      - Compare results_by_id between two runs
      - Identify checks that changed status, new checks, removed checks
      - Return structured comparison

   e. **State loading approach**: The state files (output/AAPL/state.json, output/TSLA/state.json) contain the full AnalysisState. Use the existing deserialization:
      ```python
      from do_uw.models.state import AnalysisState
      import json
      with open(state_path) as f:
          data = json.load(f)
      state = AnalysisState.model_validate(data)
      ```
      Then access `state.extracted` (ExtractedData) and `state.company` (CompanyProfile).
      If the state file format doesn't match current AnalysisState (schema drift), catch ValidationError and log which fields failed.

   f. **Generate unique run_id**: Use format `backtest_{ticker}_{timestamp}` for brain_check_runs entries.

2. **Tests** (tests/knowledge/test_backtest.py):
   - Create a minimal synthetic state file (or use fixture with ExtractedData + CompanyProfile) for testing
   - Test run_backtest() with synthetic state -> produces BacktestResult with correct counts
   - Test compare_backtests() with two different results -> identifies changes
   - Test that record=True inserts rows into brain_check_runs (use in-memory DuckDB)
   - Test that record=False does NOT insert rows
   - Test with non-existent state file -> raises appropriate error
   - Do NOT depend on real output/AAPL/state.json (test environment may not have it)

3. **Integration with Brain CLI**: The `brain backtest` command in Task 1 delegates to `run_backtest()`. Ensure the interface matches what Task 1 expects.

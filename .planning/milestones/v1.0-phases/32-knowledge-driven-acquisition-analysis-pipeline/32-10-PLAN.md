    Create `src/do_uw/stages/analyze/inference_evaluator.py` (~250 lines) with:

    1. `evaluate_inference_pattern(check: dict, data: dict) -> CheckResult` — the main entry point:
       - Reads `pattern_ref` from the check definition
       - Dispatches to pattern-specific evaluation functions via a registry dict
       - Falls back to generic multi-signal evaluation if no specific handler exists

    2. `_evaluate_multi_signal(check: dict, data: dict) -> CheckResult` — generic multi-signal evaluator:
       - Collects ALL data values from the mapped data dict (not just first_data_value)
       - Counts how many signals are present (non-None values)
       - Counts how many signals are "active" (truthy, above zero, or match a condition)
       - Determines status based on signal density:
         - TRIGGERED: majority of expected signals are active (pattern detected)
         - INFO: some signals present but pattern not fully formed
         - CLEAR: all signals present, none active (pattern explicitly not detected)
         - SKIPPED: insufficient data (most signals None)
       - Records evidence as: "N of M signals active: [signal_names_that_fired]"
       - Records which specific signals contributed via evidence string

    3. Pattern-specific handlers (registered in PATTERN_HANDLERS dict):

       `_evaluate_stock_pattern(check: dict, data: dict) -> CheckResult`:
       - For STOCK.PATTERN.* checks. Reads threshold.detection text to understand what signals to look for
       - Examines all mapped data keys for: drop magnitude, trigger type, peer comparison, short interest, timeframe
       - Applies temporal logic: looks for co-occurrence within the detection window
       - Returns TRIGGERED when the pattern's multi-signal condition is met

       `_evaluate_governance_effectiveness(check: dict, data: dict) -> CheckResult`:
       - For GOV.EFFECT.* checks. Cross-references governance data points
       - Looks for combinations: e.g., material weakness + audit committee composition + auditor change = elevated risk
       - Uses pattern_ref to determine which governance signals to combine
       - Returns severity based on how many governance signals are active

       `_evaluate_executive_pattern(check: dict, data: dict) -> CheckResult`:
       - For EXEC.INSIDER.* and EXEC.TENURE.* checks
       - cluster_selling: looks for 3+ insiders selling within 30-day window
       - non_10b51: looks for discretionary (non-plan) insider sales
       - c_suite_turnover: looks for multiple C-suite departures within 12 months

    4. `PATTERN_HANDLERS: dict[str, Callable]` registry:

<objective>
Wire the ACQUIRE stage to read check declarations from brain.duckdb at runtime, deriving required data sources and filing sections from data_strategy fields. Currently, ACQUIRE fetches a fixed set of sources regardless of what checks need. After this plan, ACQUIRE consults the brain at startup, builds an AcquisitionManifest, logs the brain-derived requirements, and validates after acquisition that all declared needs were met.

Purpose: Close SC-1 gap. The brain already declares what each check needs (247 of 388 checks have data_strategy with field_key and primary_source). The gap_detector already knows how to compare requirements to capabilities. But the ACQUIRE orchestrator has zero brain references. This plan wires them together.

Output: Brain-aware ACQUIRE stage with requirement derivation, structured logging, and post-acquisition validation.
</objective>

    Create `src/do_uw/stages/acquire/brain_requirements.py` (~120 lines) with:

    1. `derive_brain_requirements()` function that:
       - Imports and instantiates BrainDBLoader (lazy, with graceful fallback if brain.duckdb missing)
       - Calls `loader.load_checks()` to get active checks — note: this returns a dict with a `'checks'` key, so extract the list via `checks_data = loader.load_checks(); checks_list = checks_data.get('checks', [])`
       - Calls `build_manifest(checks_list)` from `do_uw.knowledge.requirements`
       - Returns an `AcquisitionManifest` (or None if brain unavailable)
       - Logs a structured summary: "Brain requires N sources: SEC_10K (229 checks), MARKET_PRICE (29 checks), ..."

    2. `validate_acquisition_coverage(manifest: AcquisitionManifest, acquired_sources: set[str])` function that:
       - Compares manifest.required_sources against what was actually acquired
       - Returns a dict with: satisfied (list[str]), missing (list[str]), coverage_pct (float)
       - Logs WARNING for any missing sources: "Brain requires source X (N checks depend on it) but ACQUIRE did not fetch it"

    3. `log_section_requirements(manifest: AcquisitionManifest)` function that:
       - For each source with required_sections, logs the specific sections needed
       - Example: "SEC_10K requires sections: item_1_business, item_7_mda, item_9a_controls (from 229 checks)"

    Design constraints:
    - BrainDBLoader import must be lazy (inside function body) to avoid circular imports
    - If brain.duckdb doesn't exist, log INFO and return None — never break ACQUIRE
    - Keep under 150 lines. This is a thin adapter, not a new subsystem
    - All functions are pure (testable without DuckDB) except derive_brain_requirements()

    Create `tests/stages/acquire/test_brain_requirements.py` with:
    - Test validate_acquisition_coverage with full coverage (all sources satisfied)

    - Test log_section_requirements with mocked manifest
    - Test derive_brain_requirements returns None gracefully when brain.duckdb missing
    - Use AcquisitionManifest directly (no DuckDB needed for validation tests)

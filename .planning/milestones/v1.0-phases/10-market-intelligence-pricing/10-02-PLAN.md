**`src/do_uw/knowledge/pricing_analytics.py`** -- Pure computation module plus a MarketPositionEngine class that wraps PricingStore for segment queries.

**Data classes (use dataclasses, not Pydantic -- these are internal analytics results, not API models):**

`MarketPosition`:
- peer_count: int
- confidence_level: str  -- HIGH (50+), MEDIUM (10-49), LOW (3-9), INSUFFICIENT (<3)
- median_rate_on_line: float | None
- mean_rate_on_line: float | None
- ci_low: float | None  -- 95% CI lower bound
- ci_high: float | None  -- 95% CI upper bound
- percentile_25: float | None
- percentile_75: float | None
- min_rate: float | None
- max_rate: float | None
- trend_direction: str  -- HARDENING, SOFTENING, STABLE
- trend_magnitude_pct: float | None  -- period-over-period change
- data_window: str  -- e.g., "2024-03 to 2026-02"

`TrendPoint`:
- period: str  -- "YYYY-H1" or "YYYY-H2" (half-year buckets)
- median_rate: float
- count: int
- mean_rate: float

`MarketTrends`:
- segment_label: str  -- e.g., "LARGE / TECH"
- points: list[TrendPoint]
- overall_direction: str  -- HARDENING, SOFTENING, STABLE
- overall_magnitude_pct: float | None
- total_quotes: int

**Pure functions (no class needed -- testable independently):**

`_t_value(n: int) -> float`: Lookup table for t-distribution critical values at 95% CI. Include values for n=2..5, 10, 15, 20, 30, 50, 100. Return 1.96 for n>100 (normal approximation). Return 0.0 for n<2.

`compute_market_position(rates: list[float], dates: list[datetime] | None = None) -> MarketPosition`:
- If len(rates) < 3: return INSUFFICIENT with all stats as None
- Compute: median (statistics.median), mean (statistics.mean), stdev (statistics.stdev), quantiles (statistics.quantiles with n=4)
- CI: margin = t_value(n) * stdev / sqrt(n); ci_low = mean - margin; ci_high = mean + margin
- Confidence: HIGH if n>=50, MEDIUM if n>=10, LOW if n>=3
- Trend: if dates provided, split into first/second half by date, compare medians. HARDENING if later > earlier by >5%, SOFTENING if later < earlier by >5%, else STABLE. Magnitude = (later_median - earlier_median) / earlier_median * 100.
- data_window: min/max of dates formatted as "YYYY-MM"

`compute_trends(rates_with_dates: list[tuple[float, datetime]], period_type: str = "half") -> list[TrendPoint]`:
- Group rates by half-year period (H1 = Jan-Jun, H2 = Jul-Dec)
- For each period with data: compute median_rate, mean_rate, count
- Return sorted by period

`_classify_confidence(n: int) -> str`: HIGH/MEDIUM/LOW/INSUFFICIENT based on thresholds.

**MarketPositionEngine class:**

Constructor: `__init__(self, store: PricingStore)` -- receives store instance (dependency injection for testability).

Methods:
- `get_market_position(self, market_cap_tier: str | None = None, sector: str | None = None, layer_position: str | None = None, score_range: tuple[float, float] | None = None, months_back: int = 24) -> MarketPosition`: Query store.get_rates_for_segment() with filters, then call compute_market_position(). If score_range provided, also filter by quality_score between range bounds (requires a new query method or post-filter).
- `get_trends(self, market_cap_tier: str | None = None, sector: str | None = None, layer_position: str | None = None, months_back: int = 48) -> MarketTrends`: Get rates with dates from store, compute_trends(), determine overall direction.
- `get_position_for_analysis(self, ticker: str, quality_score: float, market_cap_tier: str, sector: str | None) -> MarketPosition`: Convenience method that queries by the company's profile. Returns market position for peer pricing comparison.

Keep under 400 lines. Use `from __future__ import annotations`.

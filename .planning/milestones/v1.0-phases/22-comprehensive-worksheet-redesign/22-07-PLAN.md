  <action>
Redesign worksheet.md.j2 (currently 374 lines) from scratch for complete data presentation.

The v2 template renders rich Markdown tables and narratives matching the Word document content. Each section should feel like a professional analyst report in Markdown format.

**Template structure:**

1. **Header**: Company name, ticker, generation date, tier badge
2. **Section 1 -- Executive Summary**:
   - Thesis narrative paragraph
   - Company snapshot table (Markdown table)
   - Tier classification with score breakdown
   - Key findings tables (negatives + positives) with severity, impact, source columns
   - Claim probability band
   - Tower recommendation

3. **Section 2 -- Company Profile**:
   - Identity table
   - Business description
   - Revenue segments table
   - Geographic footprint table
   - D&O exposure mapping table

4. **Section 3 -- Financial Health**:
   - Financial narrative
   - Income statement table with YoY changes
   - Balance sheet + cash flow tables
   - Distress indicators table with zones
   - Audit risk table
   - Debt structure table

5. **Section 4 -- Market & Trading**:
   - Market narrative
   - Stock drop events table
   - Insider trading table
   - Short interest summary
   - Earnings guidance track record table

6. **Section 5 -- Governance & Leadership**:
   - Governance narrative
   - Board composition table
   - Compensation table
   - Ownership table
   - Anti-takeover provisions list

7. **Section 6 -- Litigation & Regulatory**:
   - Litigation narrative
   - Securities class actions table
   - SEC enforcement pipeline
   - Regulatory proceedings table
   - Defense assessment table
   - Contingent liabilities table
   - SOL windows table

8. **Section 7 -- Risk Scoring Synthesis**:
   - Scoring narrative
   - Composite score breakdown table (10 factors)
   - Red flag gates table
   - Active patterns table
   - Allegation mapping table

9. **Section 8 -- AI Transformation Risk**:
   - AI risk overview
   - Sub-dimension scores table
   - Peer comparison table

10. **Appendix -- Meeting Prep**:
    - Questions organized by category
    - Each with source finding reference

**Template patterns:**
- Use `{% if section.has_data %}` not `{% if section %}` for clarity
- Iterate over lists with proper empty-state handling: "No securities class actions identified" not "N/A"
- Source citations in italics after data points: *[SEC 10-K, 2024-02-15, HIGH]*
- Risk level indicators in bold: **CRITICAL**, **ELEVATED**, **MODERATE**
- Use horizontal rules between sections
- Use blockquotes for D&O context paragraphs

Keep under 500 lines. This is achievable because Jinja2 iterates over data rather than hardcoding each field.

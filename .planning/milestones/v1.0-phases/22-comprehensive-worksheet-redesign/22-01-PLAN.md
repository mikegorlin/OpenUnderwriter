   - Stock performance with sector-relative context

4. **insider_narrative(state)**: Insider trading pattern analysis:

   - D&O underwriting conclusion on governance quality

6. **litigation_narrative(state)** -> move to md_narrative_sections.py:

8. **company_narrative(state)** -> NEW in md_narrative_sections.py:
   - Business description with revenue concentration
   - Geographic risk (international exposure, tax haven subsidiaries)

Enhance formatters.py (currently 168 lines) with utilities needed by v2 renderers:

1. **format_source_trail(sv: SourcedValue) -> str**: Produce a full source attribution string: "[SEC 10-K, filed 2024-02-15, Item 7, HIGH confidence]". More detailed than existing format_citation which only shows "[SEC 10-K, 2024-12-31, HIGH]". Include filing section when available from SourcedValue.source.

2. **format_risk_level(level: str) -> str**: Standardize risk level display with consistent casing and emoji-free indicators: "ELEVATED", "CRITICAL", "MODERATE", "LOW". Used across all sections for consistent risk communication.

3. **format_date_range(start: str | None, end: str | None) -> str**: Format date ranges for class periods, SOL windows, etc. "Jan 2023 - Mar 2024" format.

4. **format_compact_table_value(value: float | int | None, is_currency: bool = False, is_pct: bool = False) -> str**: Single function for table cell formatting. Handles None, negative values, compact notation for currencies, percentage formatting. Reduces boilerplate in section renderers.

5. **sv_val(sv: SourcedValue[T] | None, default: T | str = "N/A") -> T | str**: Generic helper to safely extract .value from a SourcedValue or return default. Already partially exists as na_if_none but this is typed and more ergonomic.

6. **format_change_indicator(current: float, prior: float) -> str**: Format YoY change with direction arrow: "+12.3%" or "-5.7%". No green (per branding). Use blue for improvement, red for deterioration based on metric direction.

Add unit tests for all new formatters in test_render_framework.py (they already test formatters there).

Do NOT modify existing formatter function signatures -- they are called by existing section renderers.

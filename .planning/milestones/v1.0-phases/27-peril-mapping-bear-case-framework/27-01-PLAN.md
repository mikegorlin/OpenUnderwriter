  In check_results.py, add a DataStatus StrEnum with three values:
  - EVALUATED: Data acquired, check ran, result produced
  - DATA_UNAVAILABLE: Check exists but required data was not acquired (with specific reason)
  - NOT_APPLICABLE: Check genuinely doesn't apply to this company type

  Add two fields to CheckResult with backward-compatible defaults:
  - data_status: str = Field(default="EVALUATED", description="Data pipeline status: EVALUATED, DATA_UNAVAILABLE, NOT_APPLICABLE")
  - data_status_reason: str = Field(default="", description="Why data is unavailable or not applicable")

  IMPORTANT: Use str type (not DataStatus enum) for the field to ensure JSON serialization compatibility with the existing dict[str, Any] storage pattern. The DataStatus enum is for code-side convenience only.

  In check_engine.py, update the evaluate_check() function:
  - When a check gets SKIPPED status (data missing), set data_status="DATA_UNAVAILABLE" and data_status_reason to describe what data was missing (e.g., "Required field '{field}' not available in extracted data")
  - When status is TRIGGERED, CLEAR, or INFO, set data_status="EVALUATED" (the default, so no change needed)
  - Add a helper _determine_data_status() that takes the check config, mapped data dict, and the result, and sets data_status appropriately

  Also update the _make_skipped() helper (or wherever SKIPPED results are created) to always set data_status="DATA_UNAVAILABLE".

  For NOT_APPLICABLE: Add a simple heuristic in execute_checks() -- if the check ID prefix suggests sector-specificity (e.g., check has a "sector_filter" field in config) and the company's SIC sector doesn't match, mark NOT_APPLICABLE. This is a best-effort first pass; most checks will be EVALUATED or DATA_UNAVAILABLE.

  <action>
  Create pipeline_audit.py with functions to programmatically audit the data pipeline for all checks:

  1. audit_check_pipeline(check_id, check_config, extracted, company) -> dict:
     - Calls map_check_data() for the check
     - Returns audit result dict with:
       - check_id: str
       - has_mapper: bool (whether map_check_data returned a non-empty dict)
       - all_values_none: bool (all mapped values are None)
       - mapped_fields: list[str] (fields the mapper provides)
       - non_none_fields: list[str] (fields with actual data)
       - data_status: "NO_MAPPER" | "ALL_NONE" | "HAS_DATA"
       - check_category: str (from check_classification.json)
       - plaintiff_lenses: list[str]

  2. audit_all_checks(checks, extracted, company) -> dict:
     - Runs audit_check_pipeline for every AUTO check
     - Returns summary:
       - total_checks: int
       - has_data: int (checks with at least some non-None data)
       - no_mapper: int (checks with empty mapper result)
       - all_none: int (checks with mapper but all None values)
       - by_category: dict[str, dict] (breakdown by DECISION_DRIVING / CONTEXT_DISPLAY)
       - by_section: dict[str, dict] (breakdown by section prefix)
       - unwired_checks: list[str] (check IDs that are NO_MAPPER or ALL_NONE)
       - details: dict[str, dict] (per-check audit results)

  3. format_audit_report(audit_result) -> str:
     - Produces a human-readable text summary of the audit
     - Shows totals, by-category breakdown, and lists the unwired checks

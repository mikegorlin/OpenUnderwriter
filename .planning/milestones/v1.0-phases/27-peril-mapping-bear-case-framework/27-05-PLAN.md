  Create src/do_uw/stages/render/sections/sect7_peril_map.py with Word document rendering for the peril map section.

  1. render_peril_map(doc, state, ds) -> None:
     - Main entry point following the existing section renderer pattern (doc: Document, state: AnalysisState, ds: DesignSystem)
     - Reads peril_map from state.analysis.peril_map (dict -> deserialize using PerilMap model)
     - If peril_map is None, render a "Peril assessment not available" placeholder
     - Calls _render_plaintiff_heat_map() for the 7x2 grid
     - Calls _render_bear_cases() for each bear case
     - Calls _render_settlement_summary() for settlement prediction
     - Calls _render_tower_characterization() for layer-by-layer risk

  2. _render_plaintiff_heat_map(doc, assessments, ds) -> None:
     - Creates a Word table: 7 rows (one per plaintiff lens) x 3 columns (Lens, Probability, Severity)
     - Header row: "Plaintiff Type | Probability | Severity"
     - Each cell styled with background color based on band:
       - VERY_LOW/NUISANCE: light green
       - LOW/MINOR: light yellow
       - MODERATE/MODERATE: yellow/orange
       - ELEVATED/SIGNIFICANT: orange
       - HIGH/SEVERE: red
     - Use DesignSystem colors where possible, fall back to RGB shading
     - Below the table, add a one-line note for each lens that has probability >= MODERATE:
       "[Lens]: [N] triggered checks. Key: [top finding]"
     - Per user decision: heat map style -- 7x2 grid. Visual, scannable.

  3. _render_bear_cases(doc, bear_cases, ds) -> None:
     - If no bear cases: render "No bear cases identified — analysis did not find sufficient evidence to construct litigation scenarios." (Silence means clean.)
     - For each bear case:
       - Heading: "Bear Case: [theory_label]" (styled as subsection)
       - Committee summary paragraph (bold, 2-3 sentences) -- the quick read
       - If defense_assessment exists: "Defense: [assessment]" in italics
       - "Evidence Chain" sub-heading
       - Numbered list of evidence items (check_id, description, source)
       - "Probability: [band] | Severity: [severity_estimate]" footer line

  4. _render_settlement_summary(doc, state, ds) -> None:
     - Reads severity_scenarios from state.scoring.severity_scenarios
     - Reads settlement_prediction from state.analysis.settlement_prediction (if available)
     - Renders a table with 4 scenarios: Favorable (25th) | Median (50th) | Adverse (75th) | Catastrophic (95th)
     - Columns: Scenario | DDL | Settlement | Defense Costs | Total Exposure
     - Per user decision: BOTH percentile ranges for actuarial view AND named scenarios for narrative view
     - Below table: brief methodology note

  5. _render_tower_characterization(doc, state, ds) -> None:
     - If settlement_prediction dict has tower_risk_characterization:
       - Render per-layer risk table: Layer | Expected Loss Share | Risk Characterization
       - Per user decision: "Primary layer carries X% of expected loss exposure" -- analytical
     - If not available: "Tower characterization not available"

  Color band mapping helper:
  - _band_color(band_value) -> RGBColor: Maps probability/severity band strings to colors

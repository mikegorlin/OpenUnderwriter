  <action>
  Create src/do_uw/stages/score/bear_case_builder.py using structured analytical templates (per Claude's discretion in research -- structured templates are reproducible, testable, and map directly to structured data).

  The evidence gate: A bear case is ONLY constructed when its allegation theory has exposure_level of MODERATE or HIGH in the AllegationMapping. This uses the existing map_allegations() output.

  1. build_bear_cases(allegation_mapping, check_results, peril_assessments, extracted, company) -> list[BearCase]:
     - Main entry point
     - Iterates through allegation_mapping.theories
     - For each theory with exposure_level MODERATE or HIGH:
       - Calls _construct_bear_case() for that theory
     - Returns list of BearCase objects
     - Clean company (all LOW): returns empty list
     - Troubled company: returns up to 5-6 bear cases

  2. _construct_bear_case(theory_exposure, check_results, peril_assessments, extracted, company) -> BearCase:
     - Determines plaintiff_type from theory-to-lens mapping:
       A_DISCLOSURE -> SHAREHOLDERS, B_GUIDANCE -> SHAREHOLDERS, C_PRODUCT_OPS -> CUSTOMERS/REGULATORS,
       D_GOVERNANCE -> SHAREHOLDERS, E_MA -> SHAREHOLDERS
     - Builds evidence_chain from triggered checks that map to this theory's factor_sources
     - Generates committee_summary: 2-3 sentences using structured template:
       "[Company] faces [probability_band] [theory_label] exposure based on [N] findings including [top_finding]. [Severity context sentence]."
     - Generates the evidence chain as ordered EvidenceItem list (check_id, description, source, severity)
     - Calls _assess_defense() for company-specific defenses

  3. _assess_defense(theory, extracted, company) -> str | None:
     - Returns defense assessment ONLY when company-specific measures exist
     - Checks for:
       - Forum selection clause (from governance/proxy data -- if extracted.governance has forum_selection_clause)
       - PSLRA safe harbor language (from forward-looking statement disclaimers)
       - Exculpation provisions (from charter/bylaws)
       - Section 11 defense for IPO claims
     - Returns None if no company-specific defense found (not generic defenses)
     - Format: "Defense: [clause type] — [specific company provision]"

  4. _build_committee_summary(theory, evidence_chain, probability_band, severity_band, company_name) -> str:
     - Structured template producing 2-3 sentences:
       Sentence 1: "[Company] faces [probability_band] exposure to [theory description] claims."
       Sentence 2: "Key evidence: [top 2 findings from evidence chain]."
       Sentence 3 (if severity >= MODERATE): "[Severity context: estimated exposure range or impact]."

  5. _theory_label(theory) -> str:
     - Maps AllegationTheory enum to human-readable labels:
       A_DISCLOSURE -> "securities disclosure fraud"
       B_GUIDANCE -> "guidance manipulation"
       C_PRODUCT_OPS -> "product/operational failure"
       D_GOVERNANCE -> "governance breach"
       E_MA -> "M&A-related"

  6. _get_evidence_for_theory(theory, check_results, theory_factors) -> list[EvidenceItem]:
     - Filters check_results to those triggered under the theory's factor_sources
     - Also includes checks whose plaintiff_lenses overlap with the theory's primary lens
     - Returns EvidenceItem list sorted by severity (highest first)
     - Maximum 10 evidence items per bear case

  Write tests:
  - Test with MODERATE exposure -> bear case constructed with committee summary
  - Test with LOW exposure -> no bear case constructed (evidence gate works)
  - Test defense assessment returns None when no company-specific defense
  - Test defense assessment returns string when forum selection clause present
  - Test committee summary is 2-3 sentences
  - Test evidence chain ordering (highest severity first)
  - Test clean company (all theories LOW) -> empty bear case list
  - Test troubled company (3 theories MODERATE/HIGH) -> 3 bear cases

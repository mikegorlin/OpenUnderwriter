<objective>
Replace Phase 12's tier-based severity model with a DDL-based settlement prediction model, and refactor tower positioning to characterize risk by layer.

Purpose: The current severity model uses market-cap-tier lookup tables that produce generic ranges. The new model computes Dollar-Loss-Damages (DDL) from actual stock drops, applies case characteristic multipliers (accounting fraud, insider selling, etc.), and produces settlement estimates grounded in company-specific evidence. This is the critical upgrade from "here are ranges" to "here's what this company's exposure actually looks like." Tower positioning shifts from prescriptive attachment points to analytical risk characterization by layer.

Output: New settlement_prediction.py, refactored severity_model.py, updated ScoreStage, tests.
</objective>

  <action>
  Create src/do_uw/stages/score/settlement_prediction.py implementing the 5-step settlement prediction framework:

  1. predict_settlement(market_cap, stock_drops, case_characteristics, calibration_config, tier) -> SeverityScenarios:
     - Main entry point that replaces model_severity()
     - Step 1: Compute DDL from stock drops (market_cap * max_drop_magnitude)
     - Step 2: Apply base settlement percentage (~1% of DDL from config)
     - Step 3: Apply case characteristic multipliers (from config)
     - Step 4: Compute percentile scenarios (25th/50th/75th/95th) using spread factors from config
     - Step 5: Add defense cost estimates per scenario
     - Returns SeverityScenarios with populated ddl_amount on each scenario
     - Falls back to market-cap-tier ranges (existing logic) when stock_drops is empty

  2. compute_ddl(market_cap, stock_drops) -> float:
     - DDL = market_cap * maximum single-drop magnitude
     - If multiple drops, also compute cumulative DDL = market_cap * sum(drops)
     - Return the maximum of single-event and cumulative
     - If no drops, return 0.0

  3. detect_case_characteristics(state: AnalysisState) -> dict[str, bool]:
     - Inspects existing analysis results to detect case characteristics:
       - accounting_fraud: Check if CRF-01 (restatement) or CRF-02 (SEC investigation) triggered
       - restatement: Check CRF-01 or relevant check results for restatement signals
       - insider_selling: Check if F5 (insider trading) factor has high score
       - institutional_lead_plaintiff: Check litigation data for institutional investors
       - top_tier_counsel: Check litigation data for tier-1 firms (from plaintiff_firms.json)
       - sec_investigation: Check CRF-02 or litigation data for SEC enforcement
       - class_period_over_1yr: Check if stock drops span >1 year
       - multiple_corrective_disclosures: Check if >1 significant stock drop event
       - going_concern: Check financial data for going concern opinion
       - officer_termination: Check governance data for recent departures
     - Each characteristic maps to a boolean; all default False if data unavailable

  4. _build_scenario_from_settlement(base_settlement, calibration_config) -> list[SeverityScenario]:
     - Uses scenario_spread factors from config:
       favorable: base * 0.5, median: base * 1.0, adverse: base * 2.0, catastrophic: base * 4.0
     - Adds defense costs: favorable 15%, median 20%, adverse 25%, catastrophic 30%
     - Populates ddl_amount on each scenario
     - Returns 4 SeverityScenario objects

  5. characterize_tower_risk(severity_scenarios, actuarial_config) -> dict[str, Any]:
     - NEW: Characterizes risk by layer instead of prescribing attachment points
     - Uses ILF (Increased Limit Factor) from actuarial.json to compute expected loss share per layer
     - Output: {"primary": {"expected_loss_share_pct": 65.0, "risk_characterization": "Primary layer absorbs majority..."}, "low_excess": {...}, ...}
     - Per user decision: "Primary layer carries X% of expected loss exposure" -- analytical, not prescriptive
     - Returns dict that will be stored on the peril map

  - Test scenario spread matches config factors
  - Test characterize_tower_risk produces per-layer assessments
  - Test that output SeverityScenarios is compatible with actuarial_model.compute_expected_loss()

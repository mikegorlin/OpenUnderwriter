  Create src/do_uw/stages/score/peril_mapping.py (keep under 400 lines -- bear case construction will be in a separate file in Plan 04).

  The engine aggregates existing check results by plaintiff lens to build PlaintiffAssessments:

  1. build_peril_map(state: AnalysisState) -> PerilMap:
     - Main entry point called from ScoreStage
     - Extracts check_results from state.analysis
     - Loads check_classification.json for plaintiff_lens_defaults
     - Loads plaintiff_firms.json for firm matching
     - Calls assess_lens() for each of the 7 PlaintiffLens values
     - Calls match_plaintiff_firms() against litigation data
     - Determines overall_peril_rating (highest probability band across assessments)
     - Collects coverage_gaps from DATA_UNAVAILABLE checks
     - Returns PerilMap (bear_cases left empty -- populated by Plan 04)

  2. assess_lens(lens, check_results, allegation_mapping, claim_probability, severity_scenarios) -> PlaintiffAssessment:
     - Filters check_results to those with this lens in their plaintiff_lenses
     - Counts triggered, clear, skipped, total, and evaluated (data_status == EVALUATED)
     - For SHAREHOLDERS and REGULATORS (full modeling):
       - Maps triggered check count + claim_probability + severity_scenarios to probability_band
       - Maps severity_scenarios to severity_band
       - Extracts top findings from triggered checks
     - For other 5 lenses (proportional treatment):
       - Triggered > 0 means the lens is active
       - Severity is estimated from the highest-severity triggered check
       - Probability band is simpler: 0 triggered = VERY_LOW, 1-2 = LOW, 3-5 = MODERATE, 6+ = ELEVATED
     - Returns PlaintiffAssessment with modeling_depth flag

  3. _map_to_probability_band(triggered_count, total_count, claim_probability, is_full_model) -> str:
     - For full model: Uses claim_probability range + triggered count ratio
     - For proportional: Simple count-based mapping

  4. _map_to_severity_band(severity_scenarios, triggered_checks, is_full_model) -> str:
     - For full model: Maps 50th percentile settlement to severity bands
       (< $5M = NUISANCE, $5-25M = MINOR, $25-100M = MODERATE, $100-500M = SIGNIFICANT, $500M+ = SEVERE)
     - For proportional: Maps triggered check count to a simpler band

  5. match_plaintiff_firms(litigation_data, firms_config) -> list[PlaintiffFirmMatch]:
     - Checks litigation.active_cases and litigation.historical_cases for lead counsel mentions
     - Matches against plaintiff_firms.json tiers using substring matching
     - Returns list of PlaintiffFirmMatch with tier, multiplier, source

  6. _extract_key_findings(triggered_checks, max_findings=5) -> list[str]:
     - Pulls the most significant triggered checks as human-readable strings
     - Prioritizes DECISION_DRIVING over CONTEXT_DISPLAY

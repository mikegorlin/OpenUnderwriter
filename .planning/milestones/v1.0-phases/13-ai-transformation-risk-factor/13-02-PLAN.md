**A. Create `src/do_uw/stages/extract/ai_disclosure_extract.py`:**

Function `extract_ai_disclosures(state: AnalysisState) -> tuple[AIDisclosureData, ExtractionReport]`:

1. Get Item 1A text from the most recent 10-K in `state.acquired_data.filing_documents`:
   - Look for "10-K" key in filing_documents
   - If available, take the most recent document's `full_text`
   - Use `extract_section(full_text, "Item 1A")` to isolate risk factors section (import from `do_uw.stages.extract.filing_sections`)
   - If no 10-K or no Item 1A, return default AIDisclosureData with ExtractionReport showing 0% coverage

2. AI keyword matching (case-insensitive, word-boundary):
   - Core AI terms: "artificial intelligence", "machine learning", "generative AI", "large language model", "deep learning", "neural network", "AI model", "AI system", "AI technology", "AI-powered", "AI-driven", "AI-enabled"
   - Broader automation terms (weighted lower): "automation", "algorithmic", "natural language processing", "computer vision", "robotic process automation"
   - Use `re.compile(rf"\b{re.escape(kw)}\b", re.IGNORECASE)` for matching
   - Count total mentions (mention_count)

3. Sentiment classification:
   - Scan for threat language near AI keywords: "risk", "threat", "disrupt", "displace", "compete", "challenge", "vulnerable", "obsolete", "replace"
   - Scan for opportunity language: "opportunity", "leverage", "invest", "adopt", "enhance", "improve", "transform", "benefit", "advantage"
   - Count threat_mentions and opportunity_mentions
   - Classify sentiment: THREAT if threat > 2 * opportunity, OPPORTUNITY if opportunity > 2 * threat, BALANCED otherwise, UNKNOWN if total < 3

4. Risk factor extraction:
   - For each AI keyword match, extract a 200-char window around the match as context
   - Deduplicate overlapping windows
   - Store as risk_factors list (capped at 10 most relevant)

5. YoY trend: If state has a prior 10-K (check filing_documents for multiple 10-Ks sorted by date), compare current vs prior mention counts. INCREASING if current > prior * 1.2, DECREASING if current < prior * 0.8, STABLE otherwise.

6. Build ExtractionReport with expected_fields=["mention_count", "sentiment", "risk_factors", "yoy_trend"] and found_fields for each that was successfully populated.

**B. Create `src/do_uw/stages/extract/ai_patent_extract.py`:**

Function `extract_patent_activity(state: AnalysisState) -> tuple[AIPatentActivity, ExtractionReport]`:

1. Get company legal name from `state.company.identity.legal_name.value`
2. Build USPTO API query URL: `https://developer.uspto.gov/ibd-api/v1/patent/application`
   - Parameters: `searchText={company_name} AND (artificial intelligence OR machine learning)`, `rows=50`
3. Make HTTP request using `httpx.get()` with 10-second timeout
   - Wrap in try/except: on any error (network, JSON parse, API error), return default AIPatentActivity with ExtractionReport showing 0% coverage
   - Respect rate limiting: use `time.sleep(0.5)` before request (USPTO rate limits are generous but be polite)
4. Parse response JSON:
   - Extract `response.results` (list of patent records)
   - Count total `ai_patent_count`
   - Build `recent_filings` list with patent_number, filing_date, title (cap at 20 most recent)
   - Determine `filing_trend`: if filings in last 2 years > filings in 2-4 years ago, "INCREASING"; if less, "DECREASING"; else "STABLE"
5. If company name not available, return default with ExtractionReport noting "company name unavailable"
6. Build ExtractionReport with expected_fields=["patent_count", "filings", "trend"] and appropriate found_fields.

IMPORTANT: The USPTO API may change or be unavailable. The extractor MUST gracefully degrade to default values on any API error. Mark patent data as MEDIUM confidence (public patent database, but company name matching may have false positives).

**A. Create `src/do_uw/stages/extract/ai_competitive_extract.py`:**

Function `assess_competitive_position(state: AnalysisState, company_disclosure: AIDisclosureData) -> tuple[AICompetitivePosition, ExtractionReport]`:

1. Get peer companies from `state.extracted.financials.peer_group` (if available). These are PeerCompany objects with `ticker` and `name` fields.
2. For each peer (cap at 5 peers for performance), do a lightweight AI mention count:
   - Check if peer's 10-K is in state.acquired_data.filing_documents (it won't be unless --peers was used)
   - If not available, check if there's any cached peer data in acquired_data (web_search_results may have peer info)
   - If no peer data available at all, return AICompetitivePosition with adoption_stance="UNKNOWN" and peer_comparison_available=False
3. If peer AI mention counts are available:
   - Compute `peer_avg_mentions` = mean of peer counts
   - Compute `percentile_rank` = (count of peers with fewer mentions / total peers) * 100
   - Classify `adoption_stance`:
     - LEADING if company mentions > peer_avg * 1.5 AND company has patent activity
     - INLINE if company mentions within 0.5x-1.5x of peer_avg
     - LAGGING if company mentions < peer_avg * 0.5
     - UNKNOWN if insufficient data
4. Build ExtractionReport with expected_fields=["peer_comparison", "adoption_stance"] and appropriate found_fields.

Note: Peer comparison will often return UNKNOWN in single-ticker analysis. This is expected and documented. The extractor should NOT fail or log errors when peer data is unavailable -- this is the normal case.

  <name>Task 1: Litigation sub-orchestrator + summary narrative + timeline events</name>
  <files>src/do_uw/stages/extract/extract_litigation.py, tests/test_extract_litigation.py</files>
  <action>
  Create `extract_litigation.py` following the exact pattern of `extract_market.py` and `extract_governance.py`.

  **Function signature:**
  ```python
  def run_litigation_extractors(
      state: AnalysisState,
      reports: list[ExtractionReport],
  ) -> LitigationLandscape:
  ```

  **Structure (follow extract_governance.py pattern):**
  1. Create `landscape = LitigationLandscape()`.
  2. Call each extractor in dependency order, each wrapped in try/except with logger.warning on failure, returning default empty model on failure:

  ```
  # 1. Securities class actions (SECT6-03) -- foundational
  landscape.securities_class_actions = _run_sca_extractor(state, reports)

  # 2. SEC enforcement pipeline (SECT6-04)
  landscape.sec_enforcement = _run_sec_enforcement(state, reports)

  # 3. Derivative suits (SECT6-05)
  landscape.derivative_suits = _run_derivative_suits(state, reports)

  # 4. Regulatory proceedings (SECT6-06)
  landscape.regulatory_proceedings = _run_regulatory_proceedings(state, reports)

  # 5. Deal litigation (SECT6-07)
  landscape.deal_litigation = _run_deal_litigation(state, reports)

  # 6. Workforce/product/environmental + whistleblower (SECT6-08)
  #    The extractor returns 3-tuple (WPE, whistleblowers, report).
  #    The wrapper unpacks all 3, appends report to reports list,
  #    and returns only (WPE, whistleblowers) as 2-tuple to caller.
  wpe, whistleblowers = _run_workforce_product(state, reports)
  landscape.workforce_product_environmental = wpe
  landscape.whistleblower_indicators = whistleblowers

  # 7. Defense assessment (SECT6-09)
  landscape.defense = _run_defense_assessment(state, reports)

  # 8. Industry claim patterns (SECT6-10) -- reads peer data
  landscape.industry_patterns = _run_industry_claims(state, reports)

  # 9. SOL map (SECT6-11) -- reads extracted litigation data
  #    Write intermediate litigation to state first so SOL mapper
  #    can read active cases for trigger dates
  _ensure_extracted_litigation(state, landscape)
  landscape.sol_map = _run_sol_mapper(state, reports)

  # 10. Contingent liabilities (SECT6-12)
  #    Same pattern as workforce_product: extractor returns 3-tuple,
  #    wrapper unpacks all 3, appends report to reports list,
  #    returns only (liabilities, total_reserve) as 2-tuple to caller.
  liabilities, total_reserve = _run_contingent_liabilities(state, reports)
  landscape.contingent_liabilities = liabilities
  landscape.total_litigation_reserve = total_reserve

  # SECT6-01: Generate litigation summary narrative (MUST be last)
  landscape.litigation_summary = _generate_litigation_summary(landscape)

  # SECT6-02: Build timeline events
  landscape.litigation_timeline_events = _build_timeline_events(landscape)

  # Compute matter counts
  landscape.active_matter_count = _count_active_matters(landscape)
  landscape.historical_matter_count = _count_historical_matters(landscape)
  ```

  3. Each `_run_*` wrapper function uses lazy imports (inside try block) and returns default empty model on failure -- exactly like extract_market.py pattern. Two wrappers handle 3-tuple extractors:
     - `_run_workforce_product`: calls `extract_workforce_product_environmental()` which returns `(WPE, list[WhistleblowerIndicator], ExtractionReport)`. The wrapper unpacks all 3, appends the report to `reports`, and returns only `(WPE, list[WhistleblowerIndicator])` as a 2-tuple.
     - `_run_contingent_liabilities`: calls the contingent liabilities extractor which returns `(liabilities, total_reserve, ExtractionReport)`. The wrapper unpacks all 3, appends the report to `reports`, and returns only `(liabilities, total_reserve)` as a 2-tuple.
     All other `_run_*` wrappers handle standard 2-tuple `(model, ExtractionReport)` extractors -- they unpack, append report, return model.

  4. `_ensure_extracted_litigation(state, landscape)` -- Write intermediate litigation to state so SOL mapper can read active cases. Same pattern as `_ensure_extracted_market` in extract_market.py.

  **Summary narrative generation (`_generate_litigation_summary`):**
  Rule-based, mirrors `_generate_governance_summary` pattern. Returns SourcedValue[str] with LOW confidence.

  Synthesizes 5 dimensions:
  1. Active matters count and severity: "X active securities class actions, Y regulatory proceedings..."
  2. Historical pattern: "Historical litigation activity shows..." (based on case count, types, and outcomes)
  3. Regulatory pipeline: "SEC enforcement pipeline position: [stage]. Industry sweep [detected/not detected]."
  4. Defense posture: "Defense strength assessed as [STRONG/MODERATE/WEAK] with [forum/PSLRA details]."
  5. Emerging exposure: "Industry claim patterns suggest exposure to [theories]. [N] open SOL windows."

  **Timeline event construction (`_build_timeline_events`):**
  Collect events from all extracted data into a chronological list:
  - SCA filing dates -> event_type="case_filing"
  - Settlement dates -> event_type="settlement"
  - SEC enforcement action dates -> event_type="enforcement_action"
  - Regulatory proceeding dates -> event_type="regulatory"
  Sort by date descending (most recent first).

  **Matter counting:**
  - `_count_active_matters`: Count cases/proceedings with status ACTIVE across all categories.
  - `_count_historical_matters`: Count cases/proceedings with status SETTLED or DISMISSED.
  Both return SourcedValue[int] with MEDIUM confidence.

  Keep file under 500 lines. The governance sub-orchestrator is 425 lines -- this should be similar. Use lazy imports inside try/except to keep import block small.

  **Tests** (`tests/test_extract_litigation.py`):
  - Test run_litigation_extractors calls all 10 wrapper functions (mock each individual extractor at its source module)
  - Test failure isolation: one extractor failing doesn't crash the sub-orchestrator
  - Test litigation summary is generated as SourcedValue[str] with LOW confidence
  - Test timeline event construction sorts by date
  - Test active/historical matter counting
  - Test intermediate state write before SOL mapper
  </action>

  <files>src/do_uw/stages/extract/regulatory_extract.py, tests/test_regulatory_extract.py</files>
  <action>
  Create `regulatory_extract.py` for non-SEC regulatory proceedings.

  **Function signature:**
  ```python
  def extract_regulatory_proceedings(
      state: AnalysisState,
  ) -> tuple[list[RegulatoryProceeding], ExtractionReport]:
  ```

  **Agency detection patterns** (each maps to a RegulatoryProceeding with agency field):
  ```python
  AGENCY_PATTERNS: list[tuple[str, str]] = [
      (r"(?:DOJ|Department\s+of\s+Justice)\s+(?:investigation|inquiry|prosecution|indictment|settlement)", "DOJ"),
      (r"(?:FCPA|Foreign\s+Corrupt\s+Practices)", "DOJ_FCPA"),
      (r"(?:FTC|Federal\s+Trade\s+Commission)\s+(?:investigation|enforcement|consent|order|complaint)", "FTC"),
      (r"(?:FDA|Food\s+and\s+Drug\s+Administration)\s+(?:warning|inspection|483|consent\s+decree|recall)", "FDA"),
      (r"(?:EPA|Environmental\s+Protection\s+Agency)\s+(?:enforcement|violation|consent|remediation|Superfund)", "EPA"),
      (r"(?:CFPB|Consumer\s+Financial\s+Protection)\s+(?:enforcement|consent|investigation|examination)", "CFPB"),
      (r"(?:OCC|Office\s+of\s+the\s+Comptroller)\s+(?:enforcement|consent|examination|order)", "OCC"),
      (r"(?:OSHA|Occupational\s+Safety)\s+(?:citation|violation|inspection|penalty)", "OSHA"),
      (r"(?:state\s+attorney|attorney\s+general)\s+(?:investigation|action|suit|enforcement|settlement)", "STATE_AG"),
      (r"(?:EEOC|Equal\s+Employment\s+Opportunity)\s+(?:charge|complaint|investigation|consent)", "EEOC"),
  ]
  ```

  **Extraction logic:**
  1. Get 10-K text via `get_filing_document_text(state, "10-K")`. Extract Item 3 section.
  2. Also get Item 1A (Risk Factors) for "under investigation" language.
  3. Split Item 3 text into paragraphs/sentences. For each agency pattern match:
     - Create RegulatoryProceeding with agency, description (sentence context, max 500 chars), status, coverage_type (REGULATORY_ENTITY), do_implications.
  4. Search 8-K filings for regulatory event disclosures (Item 8.01 containing agency names).
  5. Cross-reference web search results from `state.acquired_data.web_search_results` and `blind_spot_results` for regulatory keywords.
  6. Extract FCPA-specific signals (Stanford FCPA Clearinghouse mentions in web search).
  7. Time horizon: 5 years for regulatory proceedings.
  8. Assign proceeding_type: "investigation", "enforcement", "penalty", "consent_decree" based on keyword context.
  9. Extract penalty amounts if `\$[\d,]+(?:\.\d+)?\s*(?:million|billion)` appears near agency keywords.

  ExtractionReport expected: ["doj", "ftc", "fda", "epa", "cfpb", "occ", "state_ag", "fcpa", "eeoc", "osha"]

  **Tests** (`tests/test_regulatory_extract.py`):
  - Test each agency pattern matches correctly
  - Test multi-agency detection in single filing
  - Test penalty amount extraction
  - Test FCPA-specific detection
  - Test empty state returns empty list
  - Test time horizon filtering
  - Test proceeding_type classification
  </action>

  <action>
  **deal_litigation.py** -- M&A and deal litigation.

  Function signature:
  ```python
  def extract_deal_litigation(
      state: AnalysisState,
  ) -> tuple[list[DealLitigation], ExtractionReport]:
  ```

  Detection patterns:
  ```python
  DEAL_PATTERNS: list[tuple[str, str]] = [
      (r"merger\s+(?:objection|challenge|litigation|lawsuit)", "merger_objection"),
      (r"(?:appraisal|Section\s+262)\s+(?:action|proceeding|petition|demand)", "appraisal"),
      (r"disclosure[\-\s]+only\s+settlement", "disclosure_only"),
      (r"[Rr]evlon\s+(?:duty|claim|standard|breach)", "revlon"),
      (r"fiduciary\s+duty\s+(?:challenge|claim|breach).*(?:merger|acquisition|deal|transaction)", "fiduciary"),
  ]
  ```

  Logic:
  1. Search 10-K Item 3 for deal litigation patterns.
  2. Search 8-K filings for M&A-related disclosures with litigation.
  3. Cross-reference web search for deal litigation mentions.
  4. Create DealLitigation for each match with: deal_name, litigation_type, status, court, description, settlement_amount.
  5. Time horizon: 5 years.

  ExtractionReport expected: ["merger_objection", "appraisal", "disclosure_only", "revlon", "fiduciary", "settlement_amount"]

  **workforce_product.py** -- Workforce, product, and environmental claims.

  Function signature (3-tuple: model, whistleblower indicators, report):
  ```python
  def extract_workforce_product_environmental(
      state: AnalysisState,
  ) -> tuple[WorkforceProductEnvironmental, list[WhistleblowerIndicator], ExtractionReport]:
  ```

  Detection patterns organized by category:
  ```python
  CATEGORY_PATTERNS: dict[str, list[tuple[str, str]]] = {
      "employment_matters": [
          (r"employment\s+(?:litigation|lawsuit|claim|action)", "employment"),
          (r"wage\s+(?:and\s+hour|theft)|(?:Fair\s+Labor\s+Standards|FLSA)", "wage_hour"),
          (r"discrimination\s+(?:suit|claim|action|complaint)", "discrimination"),
      ],
      "eeoc_charges": [
          (r"(?:EEOC|Equal\s+Employment)\s+(?:charge|complaint|investigation)", "eeoc"),
      ],
      "whistleblower_complaints": [
          (r"whistleblower\s+(?:complaint|claim|suit|action|award)", "whistleblower"),
          (r"qui\s+tam|False\s+Claims\s+Act", "qui_tam"),
          (r"SEC\s+whistleblower\s+(?:award|bounty|program)", "sec_whistleblower"),
          (r"Dodd-Frank\s+(?:whistleblower|retaliation)", "dodd_frank_whistleblower"),
      ],
      "warn_notices": [
          (r"WARN\s+(?:Act|notice)|Worker\s+Adjustment", "warn"),
          (r"mass\s+layoff|plant\s+clos(?:ure|ing)", "mass_layoff"),
      ],
      "product_recalls": [
          (r"product\s+recall|recall\s+(?:of|affecting)", "recall"),
          (r"CPSC|Consumer\s+Product\s+Safety", "cpsc"),
      ],
      "mass_tort_exposure": [
          (r"mass\s+tort|multi-?district\s+litigation|MDL", "mass_tort"),
          (r"personal\s+injury\s+(?:claim|suit|action)", "personal_injury"),
      ],
      "environmental_actions": [
          (r"(?:EPA|environmental)\s+(?:enforcement|remediation|cleanup|violation)", "environmental"),
          (r"Superfund|CERCLA|Clean\s+(?:Air|Water)\s+Act", "superfund"),
          (r"climate[\-\s]related\s+(?:litigation|lawsuit|claim)", "climate"),
      ],
      "cybersecurity_incidents": [
          (r"data\s+breach|cybersecurity\s+incident|unauthorized\s+access", "data_breach"),
          (r"(?:GDPR|CCPA|privacy)\s+(?:violation|penalty|fine)", "privacy"),
          (r"ransomware|cyber[\-\s]?attack", "cyber_attack"),
      ],
  }
  ```

  Logic:
  1. Search 10-K Item 3, Item 1A, and 8-K filings for each category's patterns.
  2. Cross-reference web search and blind spot results.
  3. For each match, create a SourcedValue[str] containing the matched sentence context (max 500 chars) with source and confidence.
  4. Populate the appropriate list field on WorkforceProductEnvironmental.
  5. Whistleblower findings are ALSO added to `state`'s WhistleblowerIndicator list (via the return value -- the sub-orchestrator will handle this).
  6. Time horizon: 3 years for employment/product/environmental.

  ExtractionReport expected: ["employment_matters", "eeoc_charges", "whistleblower_complaints", "warn_notices", "product_recalls", "mass_tort_exposure", "environmental_actions", "cybersecurity_incidents"]

  NOTE: The 3-tuple return (model, whistleblower list, report) allows the sub-orchestrator to populate both `workforce_product_environmental` and `whistleblower_indicators` on LitigationLandscape. The sub-orchestrator's wrapper function unpacks all 3 values internally, appends the report to the reports list, and returns only (WPE, whistleblowers) as a 2-tuple to the caller.

  **Tests:**
  - `test_deal_litigation.py`: Test each deal pattern, test settlement extraction, test empty returns empty, test 5-year horizon.
  - `test_workforce_product.py`: Test each category pattern, test multi-category detection, test whistleblower indicator extraction, test 3-year horizon, test cybersecurity detection.

  Keep each file under 500 lines.

  **defense_assessment.py** -- Defense strength analysis.

  Function signature:
  ```python
  def extract_defense_assessment(
      state: AnalysisState,
  ) -> tuple[DefenseAssessment, ExtractionReport]:
  ```

  Logic:
  1. **Forum selection provisions** -- Parse DEF 14A text via `get_filing_document_text(state, "DEF 14A")`:
     - Federal forum provision pattern: `r"(?i)(?:federal\s+forum|Securities\s+Act).*?(?:United\s+States\s+federal\s+district\s+court|federal\s+court)"` (multiline, check within 500 chars window)
     - Exclusive forum provision pattern: `r"(?i)(?:exclusive\s+forum|forum\s+selection).*?(?:Court\s+of\s+Chancery|Delaware|state\s+of\s+incorporation)"` (multiline, check within 500 chars window)
     - Set ForumProvisions fields with matched text context as details.

  2. **PSLRA safe harbor usage** -- Parse 10-K full text:
     - Look for `r"(?i)(?:safe\s+harbor|Private\s+Securities\s+Litigation\s+Reform\s+Act|PSLRA|forward[\-\s]looking\s+statement)"` patterns
     - Classify usage: STRONG (explicit PSLRA citation + meaningful cautionary language), MODERATE (generic forward-looking disclaimer), WEAK (minimal or boilerplate), NONE (no safe harbor language found)
     - Check quality indicators: Does the company use specific cautionary language vs. generic boilerplate? Count specific risk factors mentioned in the safe harbor statement.

  3. **Truth on the Market defense viability** -- Check if public information about alleged misrepresentation was already in the market:
     - Look for `r"(?i)(?:truth[\-\s]on[\-\s]the[\-\s]market|already\s+known|public\s+information|prior\s+disclosure|analyst\s+reports\s+noted)"` in Item 3 or web search
     - Rate as VIABLE, PARTIAL, or NOT_APPLICABLE based on evidence.

  4. **Judge track record** -- If active SCA cases exist (from state.extracted.litigation if available):
     - Extract judge name from case details if present
     - Search web results for judge + "securities" + "dismissal" for plaintiff/defense-friendly indicators
     - Mark as LOW confidence

  5. **Prior dismissal success** -- Check Item 3 for `r"(?i)(?:dismissed|motion\s+to\s+dismiss\s+granted|judgment\s+in\s+favor\s+of)"` patterns.

  6. **Overall defense strength** -- Rule-based: STRONG if federal forum + PSLRA strong + prior dismissals; WEAK if no forum provisions + no safe harbor; MODERATE otherwise.

  7. **Defense narrative** -- 2-3 sentence summary.

  ExtractionReport expected: ["forum_provisions", "pslra_safe_harbor", "truth_on_market", "judge_track_record", "prior_dismissals", "overall_strength", "defense_narrative"]

  **industry_claims.py** -- Industry claim pattern analysis.

  Function signature:
  ```python
  def extract_industry_claim_patterns(
      state: AnalysisState,
  ) -> tuple[list[IndustryClaimPattern], ExtractionReport]:
  ```

  Logic:
  1. Get company's SIC code from `state.company.identity.sic` (SourcedValue).
  2. Load `config/industry_theories.json` via ConfigLoader.
  3. Find matching SIC range: iterate through industry_theories keys (format "XXXX-YYYY"), check if company SIC falls within any range.
  4. For each matching theory in the matched industry:
     - Create IndustryClaimPattern with: legal_theory, description, sic_range, this_company_exposed (default True for same industry).
     - Search web results for peer companies + theory keywords to find peer_examples.
     - Check if contagion_risk applies: if a peer company in the same SIC range has been sued under this theory, set contagion_risk = True with LOW confidence.
  5. Cross-reference with state.extracted.financials.peer_group (if available) to name specific peer companies.
  6. Compute exposure_rationale based on company's business description matching theory keywords.
  7. If no SIC range matches, return empty list with LOW coverage report and a warning.

  ExtractionReport expected: ["industry_match", "theories_mapped", "peer_examples", "contagion_risks", "exposure_assessments"]

  **sol_mapper.py** -- Statute of limitations window computation.

  Function signature:
  ```python
  def compute_sol_map(
      state: AnalysisState,
  ) -> tuple[list[SOLWindow], ExtractionReport]:
  ```

  Logic:
  1. Load `config/claim_types.json` via ConfigLoader. Parse the `claim_types` dict.
  2. For each claim type in config:
     - Create a SOLWindow with: claim_type, sol_years, repose_years (from config).
     - Determine trigger_date:
       - If active SCA cases exist (from extracted litigation data on state), use the earliest filing date as trigger for 10b-5/Section_11/Section_14a.
       - If SEC enforcement actions exist, use earliest enforcement date as trigger for FCPA/antitrust.
       - If no specific trigger event found, use the most recent 10-K filing date as a proxy trigger (with LOW confidence) and note trigger_description as "Most recent annual report date (proxy)".
     - Compute `sol_expiry = trigger_date + timedelta(days=sol_years * 365)`.
     - Compute `repose_expiry = trigger_date + timedelta(days=repose_years * 365)`.
     - Set `sol_open = today < sol_expiry`.
     - Set `repose_open = today < repose_expiry`.
     - Set `window_open = sol_open and repose_open` (both must be open for claim to be viable).
  3. Sort SOLWindows: open windows first, then by repose_expiry ascending (soonest-closing first).
  4. Only include claim types where trigger_date is available (don't create windows with None dates).

  ExtractionReport expected: ["claim_types_evaluated", "windows_open", "windows_closed", "trigger_dates_found", "trigger_dates_proxy"]

  **contingent_liab.py** -- Known matters and contingent liabilities from 10-K footnotes.

  Function signature:
  ```python
  def extract_contingent_liabilities(
      state: AnalysisState,
  ) -> tuple[list[ContingentLiability], ExtractionReport]:
  ```

  Logic:
  1. Get 10-K full text via `get_filing_document_text(state, "10-K")`.
  2. Search full text (not just Item 3) for ASC 450 / contingent liability language:
     - `r"(?i)(?:contingent|contingency|loss\s+contingenc(?:y|ies))"` -- identifies footnote sections
     - `r"(?i)(?:probable|reasonably\s+possible|remote)\s+(?:that|loss|liability)"` -- ASC 450 classification
     - `r"(?i)(?:accrued|accrual|reserve[ds]?)\s+(?:for|of)?\s*(?:litigation|legal|contingent)"` -- accrued amounts
     - `r"(?i)(?:range\s+of\s+(?:possible|potential)\s+loss|(?:possible|potential)\s+loss.*?range)"` -- range disclosures
  3. For each contingent liability mention:
     - Extract ASC 450 classification (probable, reasonably_possible, remote) from surrounding text.
     - Extract accrued_amount if `\$[\d,]+(?:\.\d+)?\s*(?:million|billion)` appears near "accrued" or "reserve" keywords.
     - Extract range_low and range_high if range disclosure pattern found.
     - Description: sentence context (max 500 chars).
     - source_note: the specific footnote reference if identifiable.
  4. Also search Item 3 for pending litigation matters to create contingent liability entries for each disclosed active case.
  5. Check state.extracted.litigation.total_litigation_reserve if already populated (from financial statements).
  6. Deduplicate by description similarity.

  ExtractionReport expected: ["probable_matters", "reasonably_possible_matters", "remote_matters", "accrued_amounts", "range_disclosures", "total_reserve"]

  Also extract the total_litigation_reserve amount if found -- return it as a third value:
  ```python
  def extract_contingent_liabilities(
      state: AnalysisState,
  ) -> tuple[list[ContingentLiability], SourcedValue[float] | None, ExtractionReport]:
  ```
  The sub-orchestrator will use this to populate `LitigationLandscape.total_litigation_reserve`.

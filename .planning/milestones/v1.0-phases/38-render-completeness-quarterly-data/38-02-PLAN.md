<feature>
  <name>Render Coverage Test Framework</name>
  <files>src/do_uw/stages/render/coverage.py, tests/test_render_coverage.py</files>
  <behavior>
    **State field walker** (`walk_state_values`):
    - Input: state dict from `model_dump(mode='python')`
    - Output: list of (field_path, value, value_type) tuples for all non-null leaf values
    - Handles SourcedValue dicts (has 'value' + 'source' keys) — extracts the .value
    - Handles nested dicts, lists, enums, dates, booleans
    - Skips None values, empty lists, empty dicts, empty strings
    - Skips fields in the explicit EXCLUSION_PATHS set (acquired_data.*, stages.*, version, created_at, updated_at, active_playbook_id, and other internal metadata)

    **Format-aware value matcher** (`check_value_rendered`):
    - Input: (field_path, value, rendered_text)
    - Output: bool (whether value appears in rendered text)
    - For float values: check both raw and formatted versions (format_currency, format_percentage)
    - For bool values: check "Yes"/"No" or the stringified value
    - For StrEnum values: check .value (the string representation)
    - For str values: check substring match (case-insensitive for names/labels)
    - For int values: check the string form and comma-formatted form
    - For date/datetime values: check common date formats (YYYY-MM-DD, MM/DD/YYYY)

    **Exclusion list** (fields NOT expected in render output):
    - `acquired_data.*` — raw acquisition data, internal only
    - `stages.*` — pipeline progress metadata
    - `version`, `created_at`, `updated_at` — schema metadata
    - `active_playbook_id` — internal routing
    - `analysis.check_results.*` — individual check results rendered as summaries, not raw
    - `analysis.governance_clean`, `analysis.litigation_clean`, etc. — deprecated booleans
    - `*.source`, `*.confidence`, `*.as_of` — SourcedValue metadata (rendered as badges, not raw values)
    - Any field where the value is a dict with only internal keys

    **Coverage calculation:**
    - total_fields = count of walked non-null, non-excluded leaf values
    - covered = count where check_value_rendered returns True
    - coverage_pct = covered / total_fields * 100

    Test cases:
    - walk_state_values({}) -> []
    - walk_state_values({"ticker": "AAPL"}) -> [("ticker", "AAPL", str)]
    - walk_state_values({"company": {"identity": {"legal_name": {"value": "Apple", "source": "SEC", "confidence": "HIGH"}}}}) -> [("company.identity.legal_name", "Apple", str)]
    - check_value_rendered("financials.revenue", 391035000000.0, "Revenue: $391.0B") -> True
    - check_value_rendered("company.ticker", "AAPL", "**AAPL** — Apple Inc.") -> True
    - check_value_rendered("financials.z_score", 5.23, "Altman Z: 5.23") -> True
  </behavior>
  <implementation>

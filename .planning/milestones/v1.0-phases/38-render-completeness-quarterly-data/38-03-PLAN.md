    **Model addition** in `src/do_uw/models/financials.py`:
    Add QuarterlyUpdate model:
    ```python
    class QuarterlyUpdate(BaseModel):
        """Single quarterly report data extracted from 10-Q."""
        model_config = ConfigDict(frozen=False)

        quarter: str  # e.g., "Q1 FY2026"
        period_end: str  # e.g., "2025-12-28"
        filing_date: str  # e.g., "2026-01-30"
        accession: str = ""  # Source filing accession number
        revenue: SourcedValue[float] | None = None
        net_income: SourcedValue[float] | None = None
        eps: SourcedValue[float] | None = None
        prior_year_revenue: float | None = None  # Same YTD period, prior year
        prior_year_net_income: float | None = None
        prior_year_eps: float | None = None
        new_legal_proceedings: list[str] = Field(default_factory=list)
        legal_proceedings_updates: list[str] = Field(default_factory=list)
        going_concern: bool = False
        going_concern_detail: str | None = None
        material_weaknesses: list[str] = Field(default_factory=list)
        new_risk_factors: list[str] = Field(default_factory=list)
        md_a_highlights: list[str] = Field(default_factory=list)
        subsequent_events: list[str] = Field(default_factory=list)
    ```

    Add `quarterly_updates` field to ExtractedFinancials:
    ```python
    quarterly_updates: list[QuarterlyUpdate] = Field(
        default_factory=list,
        description="Post-annual 10-Q quarterly updates, most recent first",
    )
    ```

    **Extraction integration** in new file `src/do_uw/stages/extract/quarterly_integration.py`:

    Function `aggregate_quarterly_updates(state: AnalysisState) -> list[QuarterlyUpdate]`:
    1. Get most recent 10-K filing date from `acquired_data.filing_documents["10-K"]`
    2. Iterate `acquired_data.llm_extractions` for keys starting with `10-Q:` or `6-K:`
    3. For each, check if the filing_date is AFTER the most recent 10-K
    4. Convert TenQExtraction fields to QuarterlyUpdate with SourcedValue wrapping
    5. For new_legal_proceedings: extract title/description from ExtractedLegalProceeding objects into strings
    6. For new_risk_factors: extract title from ExtractedRiskFactor objects into strings
    7. Sort by period_end descending (most recent first)
    8. Return the list

    **Also investigate** the 10-Q extraction gap noted in research: AAPL has 9 acquired 10-Q filings but 0 LLM extractions with `10-Q:` prefix. Check `llm_extraction.py` to see if 10-Qs are being skipped or stored differently. If 10-Q LLM extraction isn't being triggered, add it to the extraction pipeline (the schema and prompt already exist in `schemas/ten_q.py`). The `_filter_superseded_quarterlies` function already handles filtering post-annual 10-Qs.

    **YTD comparison approach** (SC-3): The QuarterlyUpdate model stores `prior_year_revenue`, `prior_year_net_income`, `prior_year_eps` for the same YTD period from the prior year. If a Q2 10-Q is filed, this would be 6-month YTD current vs 6-month YTD prior year. The TenQExtraction schema gives us the current quarter's figures; the prior year comparison comes from comparing against the same quarter's filing from the prior year (if available in llm_extractions) or from the 10-Q filing itself which typically includes a YTD comparison column.

    For now, store the prior_year fields as None if not available from the LLM extraction. The renderer will show "N/A" for unavailable prior-year data. Future enhancement could parse the 10-Q's comparison tables.

    **Wire into pipeline** in `src/do_uw/stages/extract/llm_extraction.py` (or the extract stage runner if extraction is orchestrated elsewhere):
    After LLM extraction completes for all filings, call `aggregate_quarterly_updates(state)` and assign the result to `state.extracted.financials.quarterly_updates`. Without this step, the function exists but is never invoked, and `quarterly_updates` remains empty at render time. Find the location where LLM extraction results are written to state and add the call there.

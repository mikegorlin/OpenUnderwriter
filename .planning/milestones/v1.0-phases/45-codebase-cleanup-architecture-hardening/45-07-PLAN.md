<objective>
Eliminate the dual-write to both `brain.duckdb` and `knowledge.db`. Port knowledge CLI commands that read check run history to use `brain.duckdb` instead. Add `knowledge.db` to `.gitignore`.

Purpose: Every pipeline run currently writes check results to both stores. `knowledge.db` is 24MB, tracked in git, and grows unboundedly. The dual-write violates the "brain.duckdb is the single source of truth" invariant and doubles write I/O. This plan makes brain.duckdb the exclusive store for check run telemetry.

Dependencies:
- Plan 03 (brain/legacy/ reorganization): Hardens the brain.duckdb migration path, ensuring brain.duckdb is reliably available before we remove the knowledge.db fallback.
- Plan 06 (500-line splits): cli_knowledge.py is split in Plan 06 into cli_knowledge.py + cli_knowledge_checks.py. This plan modifies cli_knowledge.py after that split, so Plan 06 must complete first to avoid editing a pre-split file.

Output: Single-write pipeline; knowledge CLI reads brain.duckdb; knowledge.db gitignored.
</objective>

<objective>
Address two CONCERNS.md data-integrity items not covered by Plans 01-07:

1. **ConfigLoader silent fallback**: `BrainDBLoader.load_scoring()`, `load_patterns()`, `load_red_flags()`, `load_sectors()` silently fall back to JSON files via `ConfigLoader` when DuckDB tables are empty. This violates the "brain.duckdb is the single source of truth" invariant. Replace the silent fallback with a hard error that tells the user to run `brain build`.

2. **brain_migrate_yaml.py sync gap**: `brain_migrate_yaml.py` is a standalone script that converts `checks.json` → domain YAML files. It is not called by `brain build`, so the YAML files can silently diverge from `checks.json`. Add a sync-check assertion in `brain_build_checks.py` that compares check IDs between `checks.json` and the YAML files, failing clearly if they diverge.

Purpose: Both issues allow silent data integrity failures — the worst kind. The ConfigLoader fallback means the pipeline can run on stale JSON data without warning. The YAML sync gap means a YAML edit doesn't propagate to DuckDB unless someone remembers to run a standalone script. Both produce incorrect underwriting output with no error.

Output: Hard-fail in BrainDBLoader when DuckDB is empty; sync-check in brain build that catches YAML/JSON divergence.

Note: Depends on Plan 03 because Plan 03 moves the brain migrate files to brain/legacy/ and adds the post-migration hard-fail guard to brain_loader._get_conn(). This plan adds the query-time hard-fail (different location — the load_* methods, not _get_conn).
</objective>

  <files>

  <action>
    Read `src/do_uw/brain/brain_build_checks.py` fully.

    The `brain build` command calls this file to populate `brain_checks` in DuckDB from the YAML files in `brain/checks/`. Currently, if someone edits `checks.json` directly (bypassing YAML), the YAML files become stale — but `brain build` succeeds silently with the old YAML data.

    Add a sync-check function that:
    1. Reads all check IDs from `checks.json` (the JSON source)
    2. Reads all check IDs from the YAML files in `brain/checks/**/*.yaml`
    3. Compares the two sets
    4. If they differ, raises a clear error before DuckDB is populated

    Add this function before the main build function:
    ```python
    def _validate_yaml_json_sync(brain_dir: Path) -> None:
        """Assert that checks.json and brain/checks YAML files define the same check IDs.

        Raises RuntimeError if they diverge. This prevents silent data integrity failures
        where a checks.json edit bypasses the YAML layer (or vice versa).

        Run angry-dolphin brain migrate-yaml to regenerate YAML from checks.json,
        or edit YAML directly and do not edit checks.json.
        """
        import json
        import glob

        checks_json_path = brain_dir / "checks.json"
        if not checks_json_path.exists():
            return  # checks.json may not exist in YAML-only mode — skip check

        with open(checks_json_path) as f:
            json_data = json.load(f)

        # checks.json is either a list or a dict with a "checks" key
        if isinstance(json_data, list):
            json_ids = {c["check_id"] for c in json_data if "check_id" in c}
        else:
            json_ids = {c["check_id"] for c in json_data.get("checks", []) if "check_id" in c}

        yaml_ids: set[str] = set()
        yaml_pattern = str(brain_dir / "checks" / "**" / "*.yaml")
        for yaml_path in glob.glob(yaml_pattern, recursive=True):
            try:
                import yaml  # PyYAML is already a dependency
                with open(yaml_path) as f:
                    data = yaml.safe_load(f)
                if isinstance(data, dict) and "checks" in data:
                    for check in data["checks"]:
                        if "check_id" in check:
                            yaml_ids.add(check["check_id"])
                elif isinstance(data, list):
                    for check in data:
                        if isinstance(check, dict) and "check_id" in check:
                            yaml_ids.add(check["check_id"])
            except Exception:
                pass  # malformed YAML — let the build fail later with a better error

        if not yaml_ids:
            return  # YAML files not yet generated — skip (first-time setup)

        json_only = json_ids - yaml_ids
        yaml_only = yaml_ids - json_ids

        if json_only or yaml_only:
            msg_parts = [
                "YAML and checks.json are out of sync — brain build aborted.",
                "This prevents silent data integrity failures.",
            ]
            if json_only:
                msg_parts.append(f"  In checks.json but NOT in YAML ({len(json_only)}): {sorted(json_only)[:5]}{'...' if len(json_only) > 5 else ''}")
            if yaml_only:
                msg_parts.append(f"  In YAML but NOT in checks.json ({len(yaml_only)}): {sorted(yaml_only)[:5]}{'...' if len(yaml_only) > 5 else ''}")
            msg_parts.append("Fix: Run 'angry-dolphin brain migrate-yaml' to regenerate YAML from checks.json.")
            msg_parts.append("  OR: Manually align the YAML and checks.json IDs.")
            raise RuntimeError("\n".join(msg_parts))
    ```

    Then call `_validate_yaml_json_sync(brain_dir)` at the start of the main build function, before any DuckDB writes begin.

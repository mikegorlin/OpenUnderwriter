<objective>
Remove the deprecated `*_clean` boolean fields (`governance_clean`, `market_clean`, `financial_clean`, `litigation_clean`) and the local fallback recompute functions (`_is_market_clean()`, `_is_governance_clean()`) from render sections. Force render to read only from `state.analysis.section_densities`.

Purpose: Phase 29's goal was "render reads pre-computed values." Two parallel decision systems for the same density question violate ARCH-03 (no scoring logic outside score/; no recomputation in render). The fallback functions recompute logic that was meant to be pre-computed in ANALYZE. Removing them eliminates the dual-system and enforces the correct data flow.

Output: Clean render sections that read from section_densities only; deprecated boolean fields removed from state model.

Note: This plan depends on Plans 01-03 completing first (Wave 1) because Plan 02 renames BackwardCompatLoader → BrainKnowledgeLoader in several render files, and Plan 03 hardens brain_loader.py. Editing render sections after those renames avoids merge conflicts.
</objective>

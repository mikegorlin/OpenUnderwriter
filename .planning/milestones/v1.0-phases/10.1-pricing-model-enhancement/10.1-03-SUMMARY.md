# Phase 10.1 Plan 03: Rate Inference and Document Ingestion Summary

**One-liner:** Configurable rate decay inference engine fills missing D&O tower layer pricing from known rate relationships, and instructor-based document ingestion pipeline converts Excel/PDF/text to structured pricing data via LLM extraction.

## What Was Done

### Task 1: Rate Inference Engine with Configurable Decay Factors

Created `pricing_inference.py` (292 lines) with five public functions and `rate_decay.json` config:

1. **`load_rate_decay_config()`** -- Loads decay factors from `src/do_uw/config/rate_decay.json`. Falls back to hardcoded defaults if file is missing/malformed (defensive). Filters out non-numeric keys (description, notes).

2. **`infer_layer_rate()`** -- Given primary rate + layer position, returns inferred rate. PRIMARY returns unchanged; EXCESS uses configured decay factor (50% for 1st excess, 35% for 2nd, etc.); SIDE_A uses side_a_factor (25%). Fallback formula `max(0.10, 0.60 - 0.08*n)` for layers beyond config range.

3. **`infer_primary_from_excess()`** -- Back-calculates primary rate from known excess layer rate by dividing by the decay factor. Safe division guard for zero factor.

4. **`infer_missing_layers()`** -- Main inference function. 4-step process: (a) compute rate from premium/limit where both known (COMPUTED), (b) find primary rate or back-calculate from lowest known excess, (c) infer missing rates from primary, (d) compute premium from inferred rate * limit. All inferred values marked INFERRED.

5. **`compute_data_completeness()`** -- Classifies as COMPLETE (2+ verified with full data), PARTIAL (2+ verified but some gaps), or FRAGMENT (0-1 verified data points).

Config file `rate_decay.json` with default factors: excess_1=0.50, excess_2=0.35, ..., excess_10=0.10, side_a_factor=0.25.

### Task 2: Document Ingestion Pipeline with LLM Extraction

Installed 3 new dependencies: `instructor` (1.14.5), `pdfplumber` (0.11.9), `anthropic` (0.79.0).

Created `pricing_ingestion.py` (461 lines) with:

1. **Pydantic models** -- `ExtractedLayer` (10 optional fields) and `ExtractedProgram` (12 fields including layers list) define the LLM output schema with Field descriptions to guide extraction.

2. **`extract_text_from_excel()`** -- Uses openpyxl to read .xlsx workbooks sheet-by-sheet, converting to tab-separated text with sheet headers. Handles read-only mode and data-only (computed cell values).

3. **`extract_text_from_pdf()`** -- Uses pdfplumber for both text extraction and table detection. Formats tables as tab-separated text. Page-by-page processing.

4. **`extract_text_from_file()`** -- Extension-based dispatcher supporting .xlsx, .xls, .pdf, .txt, .md, .csv. Raises ValueError with supported formats list for unknown types.

5. **`extract_pricing_from_text()`** -- Uses instructor.from_provider("anthropic/claude-sonnet-4-20250514") with system prompt explaining D&O tower semantics (primary/excess/side_a, "10xs10" notation, retention rules). Requires ANTHROPIC_API_KEY env var.

6. **`map_extracted_to_inputs()`** -- Maps ExtractedProgram to ProgramInput + PolicyYearInput with EnhancedLayerInput layers. All layers marked DataSource.AI_EXTRACTED. User-provided ticker overrides LLM-extracted ticker.

7. **`ingest_document()`** -- End-to-end pipeline: extract text -> LLM parse -> map to inputs -> find/create program -> add policy year. Returns summary dict with program_id, policy_year_id, layers_extracted, data_completeness.

System prompt includes comprehensive D&O tower semantics: primary retention rules, excess stacking notation, Side A on-top-of-ABC semantics, commission tracking, rate calculations.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed compute_data_completeness single-layer edge case**
- **Found during:** Task 1 test verification
- **Issue:** A single verified layer returned "COMPLETE" because verified_count == len(layers). A single data point is not meaningful for analysis.
- **Fix:** Changed condition to require `verified_count >= 2 AND verified_count == len(layers)` for COMPLETE.
- **Files modified:** src/do_uw/knowledge/pricing_inference.py
- **Commit:** 386b17a

**2. [Rule 3 - Blocking] Module-level instructor import for test patchability**
- **Found during:** Task 2 test execution
- **Issue:** Function-level `import instructor` inside try/except was not patchable via `unittest.mock.patch` (module had no `instructor` attribute).
- **Fix:** Moved to module-level import with try/except fallback to None, enabling `patch("do_uw.knowledge.pricing_ingestion.instructor")`.
- **Files modified:** src/do_uw/knowledge/pricing_ingestion.py
- **Commit:** 52c260c

**3. [Rule 3 - Blocking] Added anthropic package dependency**
- **Found during:** Task 2 dependency installation
- **Issue:** `instructor.from_provider("anthropic/...")` requires the `anthropic` package installed separately from instructor.
- **Fix:** Added `uv add anthropic` alongside instructor and pdfplumber.
- **Commit:** 52c260c

**4. [Rule 1 - Bug] Pyright strict compliance fixes**
- **Found during:** Task 2 pyright check
- **Issue:** Two pyright errors: (a) `default_factory=list` gives `list[Unknown]`, (b) instructor `from_provider` return type seen as possibly async coroutine.
- **Fix:** (a) lambda closure pattern for default_factory, (b) cast() for instructor return value.
- **Commit:** 52c260c

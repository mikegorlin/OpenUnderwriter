  Create stages/score/pattern_detection.py (~450 lines):
  - `def detect_all_patterns(patterns_config: dict[str, Any], extracted: ExtractedData, company: CompanyProfile | None) -> list[PatternMatch]`: Iterate over all patterns in patterns_config["patterns"]. For each, call `_detect_pattern()`. Return list of PatternMatch (both detected and not-detected for completeness).
  - `def _detect_pattern(pattern_config: dict[str, Any], extracted: ExtractedData, company: CompanyProfile | None) -> PatternMatch`: Evaluate a single pattern:
    1. Get trigger_conditions from pattern_config
    2. For each trigger condition, evaluate using `_evaluate_trigger()`
    3. Count matched triggers. A pattern is detected when enough triggers match. Default threshold: majority (>50%) of defined triggers must match. If a trigger has "any_of" sub-conditions, any sub-condition matching counts the trigger as matched.
    4. If detected, compute severity from severity_modifiers
    5. Compute score_impact dict from pattern_config["score_impact"] + severity modifier points
    6. Return PatternMatch with all fields populated
  - `def _evaluate_trigger(trigger: dict[str, Any], extracted: ExtractedData, company: CompanyProfile | None) -> bool`: Evaluate a single trigger condition. Steps:
    1. Get the field name from trigger["field"]
    2. Look up the actual value using `_get_pattern_field_value(field_name, extracted, company)`
    3. If value is None, return False (missing data = trigger not matched, not skipped)
    4. Apply operator comparison: gt, lt, gte, lte, eq, ne, in, not_in
  - `def _get_pattern_field_value(field_name: str, extracted: ExtractedData, company: CompanyProfile | None) -> Any | None`: Map pattern field names to actual ExtractedData paths. This is the pattern-specific data mapper. Key mappings:
    - Stock fields: single_day_drop_pct, decline_from_high, peer_avg_drop_pct, volatility_90d, stock_price
    - Market fields: short_interest_pct, short_interest_trend, insider_cluster_selling, si_rising
    - Financial fields: accruals_ratio, beneish_m_score_zone, debt_to_ebitda, current_ratio, cash_runway_months, revenue_yoy
    - Governance fields: ceo_tenure_months, cfo_tenure_months, board_independence, executive_departures, say_on_pay_result
    - Litigation fields: active_sca_count, sec_enforcement_active, derivative_count
    - Business fields: customer_concentration, supplier_concentration, business_description_mentions_ai
    - Forward fields: guidance_misses_8q, disclosure_quality_trend, md_a_hedging_score
    Return None for fields that don't map to available data yet. Log unmapped fields.
  - `def _compute_severity(pattern_config: dict[str, Any], matched_triggers: list[str], extracted: ExtractedData, company: CompanyProfile | None) -> str`: Evaluate severity_modifiers, compute total additional points. Map to severity level: 0 additional = "BASELINE", 1-2 = "ELEVATED", 3-4 = "HIGH", 5+ = "SEVERE".
  - `def _compute_score_impact(pattern_config: dict[str, Any], severity: str, severity_points: int) -> dict[str, float]`: Compute the factor score impact from pattern_config["score_impact"]. Each entry is like {"F2": {"base": 2, "max": 5}}. Total impact = min(base + severity_points, max).

  **Allegation Theory Mapping (SECT7-05):**
  - `def map_allegations(factor_scores: list[FactorScore], patterns: list[PatternMatch], red_flags: list[RedFlagResult], extracted: ExtractedData) -> AllegationMapping`: Map all findings to 5 theories.
    1. For each theory A-E, evaluate which findings contribute:
       - Theory A (Disclosure): F1 litigation, F3 restatement, F5 guidance misses, disclosure-quality patterns, earnings quality patterns. Check factor scores for non-zero points in F1, F3, F5.
       - Theory B (Guidance): F5 guidance misses, F2 stock decline (if earnings-triggered), guidance erosion pattern. Check for guidance-related evidence in factor scores.
       - Theory C (Product/Ops): Regulatory proceedings, product litigation, workforce claims. Check extracted.litigation for regulatory/product matters. F7/F8 if operational.
       - Theory D (Governance): F9 governance, F10 officer stability, derivative suits, insider trading. Check governance-related patterns (TURNOVER_STRESS, PROXY_ADVISOR_RISK).
       - Theory E (M&A): F4 IPO/SPAC/MA, deal litigation, appraisal rights. Check F4 factor score.
    2. For each theory, determine exposure_level:
       - HIGH: Factor scores total >= 8 points for that theory's factors OR critical pattern detected
       - MODERATE: Factor scores total 3-7 points
       - LOW: Factor scores total 0-2 points
    3. Collect specific findings (evidence strings from factor scores and pattern triggers)
    4. Determine primary_exposure = theory with highest total factor points
    5. Generate concentration_analysis string summarizing which theories are dominant

  **Risk Type Classification (SECT7-04):**
  - `def classify_risk_type(extracted: ExtractedData, company: CompanyProfile | None, factor_scores: list[FactorScore], patterns: list[PatternMatch]) -> RiskTypeClassification`: Classify into 7 archetypes. Evaluation logic (rule-based, NOT LLM):
    - DISTRESSED: going_concern opinion OR Altman Z in distress zone OR F8 >= 6 points
    - BINARY_EVENT: Has pending material regulatory decision OR active clinical trial OR pending merger vote (check capital_markets, regulatory)
    - GROWTH_DARLING: Revenue growth > 20% YoY AND market_cap / revenue > 10x (high multiple) AND < 5 years public
    - GUIDANCE_DEPENDENT: F5 > 0 AND company issues quarterly guidance AND analyst consensus spread is narrow
    - REGULATORY_SENSITIVE: SIC in regulated industry (pharma 2830-2836, banking 6020-6029, insurance 6311-6399, utilities 4911-4941, energy 1311-1382) OR F1 has regulatory enforcement
    - TRANSFORMATION: Recent major M&A > 25% market cap OR CEO < 12 months (new leadership pivot)
    - STABLE_MATURE: None of the above triggered AND F8 == 0 (no distress) AND revenue growth < 10%
    - Default to STABLE_MATURE if none match
    - Secondary overlay: If primary is one type but another type also triggers, set secondary
  - Evidence: list what factors/data triggered the classification
  - needs_calibration = True always (per SECT7-11)
  - Helper: `_is_regulated_industry(sic_code: int | None) -> bool` using SIC ranges

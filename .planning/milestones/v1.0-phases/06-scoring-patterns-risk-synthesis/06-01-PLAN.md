  Create check_results.py (~200 lines):
  - `CheckStatus(StrEnum)`: TRIGGERED, CLEAR, SKIPPED, INFO (three-state evaluation + info-only)
  - `CheckResult(BaseModel)`: check_id, check_name, status (CheckStatus), value (str | float | None), threshold_level (str: red/yellow/clear/""), evidence (str), source (str), factors (list[str] for F1-F10 mapping), section (int 1-6), needs_calibration (bool per SECT7-11)
  - `aggregate_results(results: list[CheckResult]) -> dict[str, int]`: Return counts of executed, passed (CLEAR), failed (TRIGGERED), skipped (SKIPPED), info (INFO)

  Create check_engine.py (~400 lines):
  - `CHUNK_SIZE = 50` constant for progress reporting
  - `execute_checks(checks: list[dict[str, Any]], extracted: ExtractedData, company: CompanyProfile | None) -> list[CheckResult]`: Main entry point. Filters to AUTO execution_mode checks. Iterates in CHUNK_SIZE batches, calling `map_check_data()` then `evaluate_check()` for each. Logs chunk progress. Returns list of CheckResult.
  - `evaluate_check(check: dict[str, Any], data: dict[str, Any]) -> CheckResult`: Dispatcher based on threshold.type field. Handles 7 threshold types:
    - `tiered` (309 checks): Compare data value against threshold.red/yellow/clear levels. For qualitative tiered checks where data value is a string or proxy, try float comparison if possible, else report as INFO with data value.
    - `info` (19 checks): Always INFO status, report the value found.
    - `percentage` (10 checks): Numeric comparison against percentage thresholds.
    - `boolean` (2 checks): True/false evaluation.
    - `count` (2 checks): Count-based comparison.
    - `value` (4 checks): Simple value comparison.
    - `pattern` (6 checks): Mark as INFO -- pattern detection happens in SCORE stage.
    - `search` (1 check): Mark as INFO -- search-based.
    - `multi_period` (1 check): Mark as INFO -- complex multi-quarter.
    - `classification` (5 checks): Report classification value as INFO.
  - For ALL evaluations: if required data is None/missing in the data dict, return SKIPPED status with evidence "Required data unavailable: {field_name}". NEVER evaluate None as passing.
  - Extract factor mapping from check config: factors = check.get("factors", []) or check.get("scoring_impact", {}).get("factors", [])
  - Set needs_calibration = True if check has a "threshold" with numeric values (all configurable params per SECT7-11)

  - Test CheckResult creation and serialization
  - Test aggregate_results counts
  - Test evaluate_check with tiered threshold (TRIGGERED case)
  - Test evaluate_check with tiered threshold (CLEAR case)
  - Test evaluate_check with missing data (SKIPPED case)
  - Test evaluate_check with info type
  - Test evaluate_check with boolean type
  - Test execute_checks with a batch of 3 mock checks

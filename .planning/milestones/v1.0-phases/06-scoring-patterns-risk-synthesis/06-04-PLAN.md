  **Loss Severity Modeling (SECT7-08):**
  - `def model_severity(market_cap: float | None, tier: TierClassification, scoring_config: dict[str, Any]) -> SeverityScenarios | None`: If market_cap is None, return None (insufficient data). Steps:
    1. Compute DDL (Dollar Damages Line) for 3 decline scenarios:
       - 10% decline: market_cap * 0.10
       - 20% decline: market_cap * 0.20
       - 30% decline: market_cap * 0.30
    2. Look up base settlement range from scoring_config["severity_ranges"]["by_market_cap"]. Find the tier matching the market_cap. Base range is [low_m, high_m] in millions.
    3. Look up tier_multipliers from scoring_config["severity_ranges"]["tier_multipliers"]. Find entry matching tier.tier value. Multiplier is [low_mult, high_mult].
    4. Compute 4 scenarios:
       - 25th percentile (favorable): base_range[0] * tier_multiplier[0] (lowest settlement * lowest multiplier). Defense cost: 15% of settlement estimate (config-driven, NEEDS CALIBRATION).
       - 50th percentile (median): average of low and high base * average multiplier. Defense cost: 20%.
       - 75th percentile (adverse): base_range[1] * tier_multiplier[1] (highest base * highest multiplier). Defense cost: 25%.
       - 95th percentile (catastrophic): 75th * 2.0 (tail risk). Defense cost: 30%.
    5. Total exposure for each: settlement + defense cost
    6. Return SeverityScenarios with all scenarios, market_cap, decline_scenarios dict, needs_calibration=True

  **Tower Position Recommendation (SECT7-09):**
  - `def recommend_tower(tier: TierClassification, severity: SeverityScenarios | None, extracted: ExtractedData, scoring_config: dict[str, Any]) -> TowerRecommendation`: Steps:
    1. Look up tower position guidance from scoring_config["tower_positions"]["positions"]. Match based on tier:
       - WIN -> PRIMARY
       - WANT -> PRIMARY or LOW_EXCESS
       - WRITE -> LOW_EXCESS or MID_EXCESS
       - WATCH -> MID_EXCESS or HIGH_EXCESS
       - WALK -> HIGH_EXCESS or DECLINE
       - NO_TOUCH -> DECLINE
    2. Determine recommended_position (the better of the two options for the tier)
    3. Compute minimum_attachment based on severity 25th percentile (if available): minimum attachment = 50th percentile settlement estimate (provides adequate buffer)
    4. Build layer assessments for each tower position (PRIMARY through HIGH_EXCESS) using config guidance
    5. Side A assessment: evaluate indemnification capacity:
       - If going_concern or Altman Z distress: "HIGH Side A value -- company may not be able to indemnify"
       - If cash < 12 months runway: "ELEVATED Side A value -- limited indemnification capacity"
       - Otherwise: "Standard Side A considerations"
    6. Return TowerRecommendation with needs_calibration=True

  **Red Flag Summary (SECT7-10):**
  - `def compile_red_flag_summary(factor_scores: list[FactorScore], red_flags: list[RedFlagResult], patterns: list[PatternMatch], allegation_mapping: AllegationMapping | None) -> RedFlagSummary`: Consolidate all flagged items:
    1. From red_flags: Any triggered CRF -> CRITICAL severity FlaggedItem
    2. From factor_scores: Any factor with points_deducted >= 80% of max_points -> HIGH severity; >= 50% -> MODERATE; any points > 0 -> LOW
    3. From patterns: Any detected pattern -> severity maps to FlagSeverity (SEVERE/HIGH -> HIGH, ELEVATED -> MODERATE, BASELINE -> LOW)
    4. Each FlaggedItem includes: description, source (from evidence), severity, scoring_impact (e.g., "F1: 20 pts"), allegation_theory (from mapping if available), trajectory (default "NEW" -- trajectory analysis deferred to future)
    5. Sort items by severity (CRITICAL first, then HIGH, MODERATE, LOW)
    6. Count items per severity level
    7. Return RedFlagSummary

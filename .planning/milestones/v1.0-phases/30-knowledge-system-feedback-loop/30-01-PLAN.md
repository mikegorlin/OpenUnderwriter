<objective>
Make the persistent knowledge.db the primary source of check definitions. BackwardCompatLoader detects whether knowledge.db has been seeded and reads from it directly, falling back to in-memory migration only when the persistent store is empty. Migration uses idempotent upsert so it can be run multiple times safely.

Purpose: Eliminates re-reading brain/checks.json on every pipeline run. The knowledge store becomes the single source of truth for check definitions, completing the Phase 9 vision.

Output: Modified compat_loader.py (persistent-first), migrate.py (idempotent), store.py (check_count helper), regression tests.
</objective>

    **compat_loader.py** -- Modify `__init__` to try the persistent store first:
    ```python
    def __init__(self, store=None, playbook_id=None):
        if store is not None:
            self._store = store
        else:
            self._store = self._create_default_store()
        self._playbook_id = playbook_id

    @staticmethod
    def _create_default_store() -> KnowledgeStore:
        # Try persistent store first
        persistent = KnowledgeStore()  # default knowledge.db
        if persistent.check_count() > 0:
            logger.debug("Using persistent knowledge store (%d checks)", persistent.check_count())
            return persistent
        # Fall back to in-memory migration
        from do_uw.knowledge.migrate import migrate_from_json
        store = KnowledgeStore(db_path=None)
        if _BRAIN_DIR.exists():
            result = migrate_from_json(_BRAIN_DIR, store)
            logger.debug("Auto-migrated: %d checks ...", result.checks_migrated)
        return store
    ```

    **analyze/__init__.py** -- After `execute_checks()` returns results, add a call to record them:
    ```python
    def _record_check_results(state, results):
        """Write per-check results to knowledge store for feedback analysis."""
        try:
            from do_uw.knowledge.models import CheckRun
            from do_uw.knowledge.store import KnowledgeStore
            store = KnowledgeStore()
            run_id = f"{state.ticker}_{state.created_at.strftime('%Y%m%d_%H%M%S')}"
            runs = []
            for r in results:
                runs.append(CheckRun(
                    run_id=run_id,
                    ticker=state.ticker,
                    run_date=state.created_at,
                    check_id=r.check_id,
                    status=r.status.value,  # Use enum value string
                    value=str(r.value) if r.value is not None else None,
                    data_status=r.data_status,
                ))
            store.write_check_runs(runs)
            logger.info("Recorded %d check results for run %s", len(runs), run_id)
        except Exception:
            logger.warning("Failed to record check results (non-fatal)", exc_info=True)
    ```
    Call this at the end of the ANALYZE stage's run method, AFTER check results are stored on state. Wrap in try/except so feedback recording failure does NOT break the pipeline. This is non-critical telemetry.

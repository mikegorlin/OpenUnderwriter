  <action>
Create `src/do_uw/scripts/enrich_checks.py` -- a standalone script that enriches brain/checks.json in-place.

The script must be idempotent (safe to run multiple times) and must NOT remove or rename any existing fields.

**Content Type Classification** (`classify_content_type(check)`):
- `signal_type == "PATTERN"` -> `INFERENCE_PATTERN`
- `category == "CONTEXT_DISPLAY"` AND `factors` is empty list -> `MANAGEMENT_DISPLAY`
- `category == "CONTEXT_DISPLAY"` AND `factors` is non-empty -> `EVALUATIVE_CHECK` (conservative: checks with factors contribute to scoring, so they ARE evaluative per research recommendation)
- `category == "DECISION_DRIVING"` -> `EVALUATIVE_CHECK`
- Default: `EVALUATIVE_CHECK`

**Depth Classification** (`classify_depth(check)`):
- `threshold.type` in `("info", "display", "classification")` AND no factors -> `1` (DISPLAY)
- `signal_type` in `("PATTERN", "FORENSIC")` -> `3` (INFER)
- Any `required_data` contains `"SCAC_SEARCH"`, `"SEC_ENFORCEMENT"`, `"COURTLISTENER"` -> `4` (HUNT)
- `threshold.type` in `("percentage", "count", "value", "boolean")` -> `2` (COMPUTE)
- `threshold.type == "temporal"` -> `3` (INFER -- temporal analysis requires multi-period inference)
- Default: `2` (COMPUTE)

**Field Key Migration** (`migrate_field_key(check, field_for_check_dict)`):
- Import FIELD_FOR_CHECK from `check_field_routing.py` (use sys.path manipulation or direct dict copy)
- If `check["id"]` is in FIELD_FOR_CHECK, set `check["data_strategy"]["field_key"]` to the value
- Also set `check["data_strategy"]["primary_source"]` from `check["required_data"][0]` if available
- Do NOT set `extraction_path` yet -- that requires more analysis and will be done in a future phase

**Pattern Reference** (`set_pattern_ref(check)`):
- For INFERENCE_PATTERN checks, derive pattern_ref from check ID patterns:
  - `STOCK.PATTERN.event_collapse` -> `EVENT_COLLAPSE`
  - `STOCK.PATTERN.informed_trading` -> `INFORMED_TRADING`
  - `STOCK.PATTERN.cascade` -> `PRICE_CASCADE`
  - `STOCK.PATTERN.death_spiral` -> `DEATH_SPIRAL`
  - `STOCK.PATTERN.short_attack` -> `SHORT_ATTACK`
  - `STOCK.PATTERN.peer_divergence` -> `PEER_DIVERGENCE`
  - For others, derive by converting the last part of the ID to UPPER_SNAKE_CASE

**Script main():**
1. Load brain/checks.json
2. Copy FIELD_FOR_CHECK dict (hardcode it in the script to avoid import path issues -- copy all 236 entries from check_field_routing.py)
3. For each check: classify content_type, classify depth, migrate field_key, set pattern_ref if applicable
4. Update checks.json version to "9.0.0" and schema to "BRAIN_CHECKS_V8"
5. Write enriched checks.json with `indent=2` and `ensure_ascii=False`
6. Print summary: counts by content_type, counts by depth, field_key coverage count

Add `if __name__ == "__main__":` block so it runs standalone with `python src/do_uw/scripts/enrich_checks.py`.

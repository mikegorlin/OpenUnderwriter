  <action>
Create `src/do_uw/knowledge/check_definition.py` with these models:

1. **ContentType** enum (StrEnum): `MANAGEMENT_DISPLAY`, `EVALUATIVE_CHECK`, `INFERENCE_PATTERN`

2. **DepthLevel** IntEnum: `DISPLAY = 1` (extract & display), `COMPUTE = 2` (extract & compute), `INFER = 3` (extract & infer), `HUNT = 4` (hunt & analyze)

3. **DataStrategy** (BaseModel):
   - `primary_source: str` (e.g., "SEC_10K") -- required
   - `extraction_path: str | None = None` (e.g., "financials.liquidity.current_ratio") -- dotted path into ExtractedData
   - `field_key: str | None = None` (e.g., "current_ratio") -- the field key the evaluator sees
   - `fallback_sources: list[str] = Field(default_factory=list)`
   - `computation: str | None = None` -- name of computation function for complex derivations (e.g., "_compute_ceo_cfo_selling_pct")

4. **EvaluationCriteria** (BaseModel):
   - `type: str` (e.g., "tiered", "boolean", "percentage") -- maps to existing threshold.type
   - `metric: str | None = None` -- what is being measured
   - `direction: str | None = None` -- "lower_is_worse", "higher_is_worse", "presence_is_bad"
   - `thresholds: dict[str, Any] | None = None` -- structured thresholds (red/yellow/clear)

5. **PresentationHint** (BaseModel):
   - `display_format: str | None = None` (e.g., "ratio", "percentage", "currency", "boolean", "narrative")
   - `worksheet_label: str | None = None` -- human-readable label override
   - `section_placement: str | None = None` -- human-readable section name

6. **CheckDefinition** (BaseModel):
   - All existing fields preserved with same names and types (id, name, section, pillar, factors, required_data, data_locations, threshold, execution_mode, claims_correlation, tier, category, signal_type, hazard_or_signal, plaintiff_lenses)
   - New fields, ALL OPTIONAL with backward-compatible defaults:
     - `content_type: ContentType = ContentType.EVALUATIVE_CHECK` -- default to EVALUATIVE_CHECK so existing checks don't break
     - `depth: DepthLevel = DepthLevel.COMPUTE` -- default level 2
     - `rationale: str | None = None` -- why this check matters for D&O underwriting
     - `data_strategy: DataStrategy | None = None` -- how to get the data
     - `evaluation_criteria: EvaluationCriteria | None = None` -- structured evaluation metadata
     - `presentation: PresentationHint | None = None` -- display hints
     - `pattern_ref: str | None = None` -- for INFERENCE_PATTERN, reference to patterns.json ID
   - Use `model_config = ConfigDict(extra="allow")` so unknown fields in checks.json don't cause validation errors
   - Add a class method `from_check_dict(cls, data: dict[str, Any]) -> CheckDefinition` that calls `model_validate(data)`
   - Add a method `to_check_dict(self) -> dict[str, Any]` that returns `model_dump(exclude_none=True)`

**Step 1: Update `migrate.py` to persist enriched fields.**

Find the function that creates Check ORM objects from brain/checks.json dicts (likely `_migrate_checks` or similar). Read the file first to find the exact function name and structure.

When creating a Check ORM object from a check dict, add population of the new columns:
```python
check_orm.content_type = check_dict.get("content_type")
check_orm.depth = check_dict.get("depth")
check_orm.rationale = check_dict.get("rationale")
# field_key from data_strategy sub-dict
ds = check_dict.get("data_strategy")
if isinstance(ds, dict):
    check_orm.field_key = ds.get("field_key")
    check_orm.extraction_path = ds.get("extraction_path")
check_orm.pattern_ref = check_dict.get("pattern_ref")
```

This is additive -- checks without enriched fields will have None values, which is fine since the ORM columns are nullable.

**Step 2: Update `store.py` query_checks to support filtering by new fields.**

Find the `query_checks` method in KnowledgeStore. Add optional filter parameters:
- `content_type: str | None = None` -- filter by content type
- `depth: int | None = None` -- filter by depth level

Apply these as WHERE clauses when provided:
```python
if content_type is not None:
    query = query.where(Check.content_type == content_type)
if depth is not None:
    query = query.where(Check.depth == depth)
```

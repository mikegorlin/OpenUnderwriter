  <action>
Run the full validation suite using the CLI, starting with a pilot to verify infrastructure:

1. **Clear all output and cache for fresh runs:**
   ```bash
   rm -rf output/*/state.json
   rm -rf .cache/validation_checkpoint.json
   ```

2. **PILOT RUN (3 tickers from different categories):**
   Create a file `pilot_tickers.txt` with 3 tickers: one standard (NVDA), one known-outcome (SMCI), one edge-case (TSM).
   ```bash
   angry-dolphin validate run --fresh --conservative-rate --tickers-file pilot_tickers.txt --output output
   ```
   Verify: all 3 complete (pass or documented fail), checkpoint works, reports generated, costs within budget.
   If pilot fails systematically (e.g., rate limiting breaks, checkpoint doesn't work): fix infrastructure before proceeding.

3. **FULL RUN (remaining 22 tickers):**
   ```bash
   angry-dolphin validate run --fresh --conservative-rate --output output
   ```
   The checkpoint will skip the 3 pilot tickers. This runs the remaining 22 sequentially at 5 req/sec.

3. **Generate cost report after completion:**
   ```bash
   angry-dolphin validate cost-report --output output
   ```

4. **Capture the results:**
   - Save both Rich console output and JSON reports
   - Document which tickers passed and which failed
   - For each failure: note the stage, error message, and root cause

5. **Triage failures into categories:**
   - **Fix in Plan 06**: Extraction bugs, converter issues, scoring problems (code changes needed)
   - **Tolerance adjustment**: Ground truth values slightly off due to fiscal year changes (adjust tolerance)
   - **Expected**: Known-incomplete tickers (e.g., RIDE if delisted with missing filings)
   - **Infrastructure**: Rate limiting, API errors (should be fixed by Plan 02 retry logic)

**IMPORTANT:** If the validation run is interrupted (timeout, connection error), it will resume from the checkpoint. Re-run the same command to continue.

**IMPORTANT:** This task may take 2-4 hours (25 tickers x ~7 min each). Monitor the first 3-4 tickers to ensure the infrastructure works, then let it run.

After the run completes, create a summary document at `output/validation_run_notes.md` with:
- Total tickers attempted / passed / failed
- Failure categorization
- Average cost per company
- Average duration per company
- Industry coverage confirmation
- Known-outcome company results (did SMCI, RIDE, COIN, LCID, PLUG score high risk?)

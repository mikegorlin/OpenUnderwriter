**batch.py** -- BatchExtractor class (~300 lines max):
- Separate utility, NOT integrated into LLMExtractor (per RESEARCH anti-pattern guidance)
- `BatchExtractor.__init__(self, model: str = "claude-haiku-4-5-20250315")`
- `prepare_request(self, filing_text: str, form_type: str, accession: str, system_prompt: str, tool_schema: dict[str, Any]) -> dict[str, Any]`:
  - Returns a batch request dict with `custom_id=f"{form_type}:{accession}"` and `params` including model, max_tokens=16384, messages, and tools (tool_use schema matching what instructor generates)
  - The tool definition should match the extraction schema for the given form_type (get schema from `src/do_uw/stages/extract/llm/schemas/`)
- `submit_batch(self, requests: list[dict[str, Any]]) -> str`:
  - Calls `anthropic.Anthropic().messages.batches.create(requests=requests)`
  - Returns batch_id
  - Logs batch submission with request count
- `poll_batch(self, batch_id: str, poll_interval: float = 60.0) -> list[dict[str, Any]]`:
  - Polls `client.messages.batches.retrieve(batch_id)` until status is terminal (ended, errored, canceled, expired)
  - Returns list of result dicts via `client.messages.batches.results(batch_id)`
  - Logs progress periodically
- `parse_results(self, results: list[dict[str, Any]], schema_map: dict[str, type]) -> dict[str, Any]`:
  - For each result, extract the tool_use content, validate against the appropriate Pydantic schema
  - Returns dict keyed by custom_id with parsed extraction data
  - On validation failure: log warning, store None for that result

Import anthropic at top with lazy try/except pattern (per 18-03 decision). All errors logged, never raised.

**cost_report.py** -- Cost report generation (~200 lines):
- `CostReportEntry` dataclass: `ticker` (str), `total_cost_usd` (float), `duration_seconds` (float), `by_filing_type` (dict[str, float])
- `CostReport` dataclass: `entries` (list[CostReportEntry]), `grand_total_usd` (float), `by_filing_type_total` (dict[str, float])
- `generate_cost_report(output_dir: Path) -> CostReport`:
  - Scan output_dir for ticker subdirectories containing state.json
  - For each ticker: load state, get accession numbers from acquired_data.filing_documents, query ExtractionCache for per-accession costs
  - Group costs by filing type (10-K, DEF 14A, 8-K, etc.)
  - Compute per-company totals and grand total
- `print_cost_report(report: CostReport) -> None`:
  - Rich table: `# | Ticker | Duration | Total Cost | 10-K | DEF14A | 8-K | Other`
  - Footer row with totals
- `save_cost_report(report: CostReport, path: Path) -> None`:
  - JSON serialization of the full report

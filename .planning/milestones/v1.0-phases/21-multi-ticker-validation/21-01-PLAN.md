  <action>
Create `src/do_uw/validation/` package with four modules:

**config.py** -- Ticker configuration:
- Define `TickerEntry` as a TypedDict with fields: `ticker` (str), `industry` (str), `category` (str -- "standard", "known_outcome", "edge_case")
- `VALIDATION_TICKERS: list[TickerEntry]` constant with 24 tickers:
  - Tech: NVDA, CRM (TECH_SAAS playbook)
  - Biotech: MRNA, AMGN (BIOTECH_PHARMA playbook)
  - Energy: XOM, NEE (ENERGY_UTILITIES playbook)
  - Healthcare: UNH, HCA (HEALTHCARE playbook)
  - CPG: PG, KO (CPG_CONSUMER playbook)
  - Media: DIS, NFLX (MEDIA_ENTERTAINMENT playbook)
  - Industrials: CAT, HON (INDUSTRIALS playbook)
  - REITs: PLD, AMT (REITS playbook)
  - Transportation: UNP, FDX (TRANSPORTATION playbook)
  - Known-outcome: SMCI, RIDE, COIN, LCID, PLUG
  - Edge-case: TSM (Foreign Private Issuer, files 20-F instead of 10-K, CIK 1046179, SIC 3674)
- `get_tickers(category: str | None = None) -> list[str]` filter function

**runner.py** -- ValidationRunner class (~250 lines max):
- `__init__(self, tickers: list[str], output_dir: Path, fresh: bool = True, use_llm: bool = True)`
- `run(self) -> ValidationReport` -- iterates tickers sequentially, runs Pipeline per ticker
- Checkpoint: after each ticker completes, write `output_dir / ".validation_checkpoint.json"` with `{"completed": {"NVDA": {"status": "PASS", "duration": 420.5, "cost_usd": 0.85}, ...}, "failed": {"CRM": {"error": "...", "stage": "extract"}}}`
- On restart: read checkpoint, skip completed/failed tickers
- Continue on failure: wrap `pipeline.run()` in try/except PipelineError, record failure, move to next
- Clear ticker output before running if fresh=True (delete `output_dir/TICKER/` directory)
- Pass `pipeline_config={"no_llm": not use_llm}` to Pipeline constructor
- Track per-ticker timing with `time.monotonic()`
- Import Pipeline, PipelineError, AnalysisState at top level

**report.py** -- Report generation:
- `ValidationReport` as a dataclass with fields: `run_date` (str), `results` (dict[str, TickerResult]), `summary` (ReportSummary)
- `TickerResult` dataclass: `status` (str -- "PASS"/"FAIL"), `duration_seconds` (float), `cost_usd` (float), `error` (str | None), `failed_stage` (str | None)
- `ReportSummary` dataclass: `total` (int), `passed` (int), `failed` (int), `total_cost_usd` (float), `avg_duration_seconds` (float)
- `print_report(report: ValidationReport) -> None` -- Rich table display
- `save_report(report: ValidationReport, path: Path) -> None` -- JSON serialization

**__init__.py** -- Re-exports: `ValidationRunner`, `VALIDATION_TICKERS`, `ValidationReport`

Use pyright strict throughout. Type hints on all functions. No file over 500 lines.

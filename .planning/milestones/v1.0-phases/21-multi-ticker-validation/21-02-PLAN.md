Enhance `rate_limiter.py` (currently 113 lines) with two new capabilities:

**1. Configurable rate limit -- `set_max_rps(rps: int) -> None`:**
- Validates rps in range [1, 10], clamps to bounds
- Updates module-level `_SEC_MAX_RPS` and `_SEC_INTERVAL = 1.0 / rps`
- Thread-safe (acquires `_lock` before modifying)
- Also add `get_max_rps() -> int` for testability

**2. Retry with exponential backoff -- modify `sec_get()` and `sec_get_text()`:**
- Add `max_retries: int = 5` parameter to both functions
- On `httpx.HTTPStatusError`:
  - Status 403 (rate limited): wait 10.0 seconds before retry
  - Status >= 500 (server error): wait `2.0 * (2 ** attempt)` seconds (2s, 4s, 8s, 16s, 32s)
  - Other 4xx: raise immediately (don't retry client errors)
- On `httpx.RequestError` (connection failures): use exponential backoff same as 5xx
- Log each retry: `logger.warning("SEC GET %s failed (%s), retrying in %.0fs (attempt %d/%d)", url, error_type, wait, attempt+1, max_retries)`
- After final retry failure: raise the original exception
- Keep the `_rate_limit()` call BEFORE each attempt (rate limiter still applies on retries)

The file should stay under 200 lines total after these changes.

  <name>Task 1: Diagnose and fix all validation failures</name>
  <files>

  <action>
For EACH failure documented in the Plan 05 validation report:

1. **Load the failed ticker's state.json** (if it exists) from `output/<TICKER>/state.json`
2. **Identify the failure stage and root cause:**
   - RESOLVE failures: SEC EDGAR identity lookup issues (CIK not found, SIC mapping)
   - ACQUIRE failures: Data source unavailable, rate limiting, timeout
   - EXTRACT failures: LLM extraction returned None for too many filings, converter crash, missing data mapping
   - ANALYZE failures: Check execution error (data field missing that check assumes exists)
   - SCORE failures: Scoring engine crash (missing factor data, unexpected None)
   - BENCHMARK failures: Peer comparison error (no peer data, market cap missing)
   - RENDER failures: Template rendering crash (None field not handled)

3. **Apply targeted fix** for each failure category:
   - **Converter crash**: Add None guard / default value in the affected converter function
   - **Missing data mapping**: Add fallback path or graceful None return
   - **None field in template**: Add None check in the specific section renderer
   - **Scoring assumption**: Add defensive check for optional data
   - **Edge case company**: Add special handling (e.g., FPI, REIT, pre-revenue)
   - **Ground truth mismatch**: Adjust tolerance or correct ground truth value if filing changed

4. **Pattern-based fixes** (fix the pattern, not just the symptom):
   - If 3+ tickers fail in the same spot: the code has a systemic issue, fix the root cause
   - If only 1 ticker fails in a unique spot: targeted fix with defensive coding
   - If a known-outcome company doesn't show high risk: check that relevant extractors are finding the signals

5. **After all fixes**: Run the full test suite to verify no regressions:
   ```bash
   uv run pytest -x --tb=short
   ```

**Common fix patterns from prior phases:**
- `if value is None: return None` for graceful degradation
- `cast(dict[str, Any], ...)` for pyright strict compliance on new code
- `try/except` wrapper in sub-orchestrators for isolation
- Default values in SourcedValue constructors

**CRITICAL:** Every fix must be accompanied by a test that verifies the fix. If you add a None guard, add a test that passes None and expects the guarded behavior.

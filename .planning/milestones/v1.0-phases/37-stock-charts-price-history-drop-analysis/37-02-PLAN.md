**Part A: Create stock_chart_data.py (~200 lines)**

Create a new module `src/do_uw/stages/render/charts/stock_chart_data.py` that extracts and prepares data for chart rendering. This separates data extraction from matplotlib rendering to keep each file under 500 lines.

The module provides:

1. `ChartData` dataclass:
```python
@dataclass
class ChartData:
    dates: list[datetime]
    prices: list[float]          # Company close prices (dollar values)
    etf_dates: list[datetime] | None
    etf_prices: list[float] | None
    etf_ticker: str
    spy_dates: list[datetime] | None
    spy_prices: list[float] | None
    drops: list[StockDropEvent]
    ticker: str
    period: str                   # "1Y" or "5Y"
```

2. `extract_chart_data(state: AnalysisState, period: str) -> ChartData | None`:
   - Reads `acquired_data.market_data` using the CORRECT keys:
     - Company: `history_1y` (period="1Y") or `history_5y` (period="5Y")
     - Sector ETF: `sector_history_1y` / `sector_history_5y`
     - SPY: `spy_history_1y` / `spy_history_5y`
     - ETF ticker: `sector_etf`
   - Uses `get_close_prices()` and `get_dates()` from `stock_drops.py` to extract data
   - Parses date strings to `datetime` objects
   - For 5Y period: aggregates to weekly data (keep Friday/last-trading-day-of-week close price)
   - Returns None if company price data is empty or too short (<5 points)
   - Gets drop events from `state.extracted.market.stock_drops` (filters by period: for 1Y all 5%+ drops, for 5Y only >10% single-day or >15% cumulative per user decision)

3. `compute_chart_stats(data: ChartData) -> dict[str, str | float | None]`:
   - Computes: current_price, high_52w, low_52w, total_return_pct, sector_return_pct, alpha_pct
   - Returns dict with formatted values for the stats header
   - Returns "N/A" for unavailable metrics

4. `index_to_base(prices: list[float], base_value: float = 100.0) -> list[float]`:
   - Index a price series to a base value (for ETF/SPY overlay on right axis)

5. `aggregate_weekly(dates: list[datetime], prices: list[float]) -> tuple[list[datetime], list[float]]`:
   - Group by ISO week, take last entry per week
   - Used for 5Y chart density reduction

The new stock_charts.py must implement per the user's locked decisions:

1. **`create_stock_chart(state: AnalysisState, period: str = "1Y", ds: DesignSystem | None = None) -> io.BytesIO | None`**:
   Main entry point. Returns PNG BytesIO or None.

   Implementation steps:
   a. Call `extract_chart_data(state, period)` -- return None if no data
   b. Call `compute_chart_stats(data)` for header metrics
   c. Set up matplotlib: `matplotlib.use("Agg")`, use `plt.style.use('dark_background')` then override with BLOOMBERG_DARK colors
   d. Create figure: `fig = plt.figure(figsize=(10, 5.5), dpi=200, facecolor=BLOOMBERG_DARK["bg"])`
   e. Create stats header axes (top 10% of figure) -- render key metrics
   f. Create main chart axes with `fig.add_axes(...)` for the chart body
   g. Create right Y-axis with `ax.twinx()`
   h. Render all chart elements (see below)
   i. Return `save_chart_to_bytes(fig, dpi=200)`

2. **Stats header bar** (`_render_stats_header(ax_header, stats, ticker)`):
   - Axes positioned at top of figure (no frame, just text)
   - Background: BLOOMBERG_DARK["header_bg"]
   - Display: Ticker name, Current Price, 52W H/L, Total Return, vs Sector, Alpha
   - Muted labels above, bold values below
   - Color-code returns: green if positive, red if negative

3. **Company price area** (`_render_price_area(ax, dates, prices)`):
   - Plot company price line (BLOOMBERG_DARK["price_up"] color)
   - `ax.fill_between()` with `where=` clause:
     - Green fill (alpha=0.2) where price >= starting price
     - Red fill (alpha=0.2) where price < starting price
   - Use `interpolate=True` for smooth intersection handling
   - Left Y-axis label: "$" with ticker name

4. **Sector ETF + SPY overlays** (`_render_overlays(ax2, data)`):
   - Index ETF and SPY prices to 100
   - ETF: dashed gold line (BLOOMBERG_DARK["etf_line"])
   - SPY: dotted light blue line (BLOOMBERG_DARK["spy_line"])
   - Right Y-axis label: "Indexed (100 = start)"

5. **Divergence bands** (`_render_divergence_bands(ax2, data)`):
   - Compute indexed company price for comparison
   - Where abs(company_indexed - etf_indexed) > 10 points, shade the gap
   - Use `ax2.fill_between()` with BLOOMBERG_DARK["divergence_alpha"] color
   - Only render if ETF data is available

6. **Drop markers** (`_render_drop_markers(ax, data)`):
   - For each drop event in data.drops:
     - Find the y-position (company price) on the drop date
     - 5-10% drops: yellow dot (BLOOMBERG_DARK["drop_yellow"]), markersize=6
     - 10%+ drops: red dot (BLOOMBERG_DARK["drop_red"]), markersize=8
     - Add small annotation text below marker with drop % (e.g., "-8.3%")
   - Use zorder=5 for markers to be on top

7. **Legend** (`_render_legend(ax, ax2, ticker, etf_ticker)`):
   - Combine handles from both axes
   - Position: upper left (since stats are at top)
   - Semi-transparent dark background for readability
   - Include: company line, ETF line, SPY line
   - Small font (7pt) to not overwhelm

8. **Backward-compatible exports**:
   - Keep `create_stock_performance_chart(state, period, ds)` as alias calling `create_stock_chart()`
   - Keep `create_stock_performance_chart_5y(state, ds)` as alias
   - Update `__all__` to export all three

**Part C: Add recovery time, event grouping, market-wide tagging (stock_drops.py)**

Add three new functions to stock_drops.py:

1. `compute_recovery_days(prices: list[float], drop_idx: int, period_days: int = 1) -> int | None`:
   - Pre-drop index is `max(0, drop_idx - period_days)`
   - Pre-drop price is `prices[pre_drop_idx]`
   - Iterate from `drop_idx + 1` to end of prices, return first index where `prices[i] >= pre_drop_price`
   - Return `i - drop_idx` (trading days to recover)
   - Return None if never recovered in analysis period

2. `group_consecutive_drops(drops: list[StockDropEvent], prices: list[float], dates: list[str]) -> list[StockDropEvent]`:
   - Sort drops by date
   - Walk through sorted drops. If the next drop's date is within 2 trading days (consecutive trading day or one day gap) of the current event's end, merge them
   - Merged event: date = first drop's date, cumulative_pct = computed from pre-first-drop price to post-last-drop price, period_days = sum of period_days, drop_type = MULTI_DAY
   - Keep the trigger_event from the drop with the worst single-day pct
   - Return the grouped list

3. `tag_market_wide_events(drops: list[StockDropEvent], spy_history: dict[str, Any]) -> list[StockDropEvent]`:
   - Get SPY close prices and dates
   - Compute SPY daily returns
   - For each drop, find SPY return on same date. If SPY return <= -3.0, set `drop.is_market_wide = True`
   - Return updated drops

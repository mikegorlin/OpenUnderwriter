  <action>
    1. Create `src/do_uw/dashboard/charts.py` (~200-300L):
       - All Plotly imports: `import plotly.graph_objects as go  # type: ignore[import-untyped]`
       - All figure-returning functions take AnalysisState and return `Any` (plotly Figure typed as Any for pyright strict)

       Functions to implement:
       a) `build_risk_radar(state: AnalysisState) -> Any`:
          - Scatterpolar chart with 10 scoring factors
          - r values = points_deducted / max_points (0-1 fraction scale)
          - theta = factor names
          - fill="toself", marker color = navy #1A1446
          - Layout: polar radialaxis visible with range [0,1], no legend
          - Return empty figure if state.scoring is None

       b) `build_risk_heatmap(state: AnalysisState) -> Any`:
          - go.Heatmap with single row of factor risk fractions
          - x = factor names, y = ["Risk Level"]
          - Custom colorscale: blue(0) -> amber(0.5) -> orange(0.75) -> red(1.0)
            Using design.py risk colors: #4A90D9 -> #FFB800 -> #E67300 -> #CC0000
          - hovertemplate showing factor name and percentage
          - Compact layout: height=120, minimal margins
          - Return empty figure if state.scoring is None

       c) `build_factor_bar_chart(state: AnalysisState) -> Any`:
          - go.Bar horizontal chart showing points deducted per factor
          - x = points_deducted, y = factor_names
          - Color each bar based on deduction severity:
            >= 75% of max -> risk_critical, >= 50% -> risk_high, >= 25% -> risk_elevated, else -> risk_moderate
          - Layout: horizontal orientation, factor names on y-axis
          - Return empty figure if state.scoring is None

       d) `build_score_gauge(score: float, max_score: float, title: str) -> Any`:
          - go.Indicator in gauge mode
          - Value = score, range = [0, max_score]
          - Steps matching risk spectrum colors at thresholds (0-30 blue, 30-50 amber, 50-75 orange, 75-100 red)
          - Title displayed below gauge
          - Compact layout for embedding in cards

       e) `build_tier_gauge(quality_score: float) -> Any`:
          - Specialized gauge for overall quality score (0-100)
          - Steps: 0-20 (critical/red), 20-40 (high/orange), 40-60 (elevated/amber), 60-80 (moderate/blue), 80-100 (neutral/gray)
          - Threshold bar at current score
          - Title: "Quality Score"

    2. Create `src/do_uw/dashboard/charts_financial.py` (~150-250L):
       - Same Plotly import pattern

       Functions to implement:
       a) `build_distress_gauges(state: AnalysisState) -> dict[str, Any]`:
          - Returns dict of gauge figures keyed by model name: "z_score", "o_score", "m_score", "f_score"
          - Each gauge uses model-appropriate ranges and zone coloring:
            - Z-Score: >2.99 safe (blue), 1.81-2.99 gray (gray zone), <1.81 distress (red)
            - O-Score: <0.5 safe (blue), >0.5 distress (red) (probability of bankruptcy)
            - M-Score: <-1.78 unlikely (blue), >-1.78 possible (red) (earnings manipulation)
            - F-Score: >=7 strong (blue), 4-6 moderate (amber), <4 weak (red)
          - Return empty figures if state.extracted.financials.distress is None

       b) `build_peer_comparison_bars(state: AnalysisState, metric_key: str) -> Any`:
          - Horizontal bar chart comparing company metric against peer group
          - Company bar in navy (#1A1446), peer bars in gray (#999999)
          - metric_key selects which metric to compare (market_cap, revenue, z_score, quality_score, etc.)
          - Return empty figure if no peer data available

       c) `build_red_flag_summary(state: AnalysisState) -> Any`:
          - Horizontal bar chart of triggered red flags
          - Each bar shows flag name and severity
          - Color-coded by severity: critical=red, high=orange, elevated=amber
          - Return empty figure if no red flags triggered
  </action>

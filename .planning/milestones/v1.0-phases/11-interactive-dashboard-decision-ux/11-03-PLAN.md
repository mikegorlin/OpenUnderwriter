  <action>
    1. Update `src/do_uw/dashboard/state_api.py` -- add section detail extraction functions:
       - `extract_section_detail(state: AnalysisState, section_id: str) -> dict[str, Any]`
         Maps section_id to appropriate state data:
         - "company" -> company profile details (identity, business desc, subsidiaries, D&O exposure)
         - "financials" -> financial statements, distress scores, debt analysis, audit data
         - "market" -> stock performance, insider trades, short interest, earnings, analyst sentiment
         - "governance" -> board forensics, leadership profiles, compensation, ownership
         - "litigation" -> active/historical matters, SEC enforcement, regulatory proceedings, defense assessment
         - "scoring" -> factor scores with contributing checks, red flags, patterns, tier classification
         Returns dict with: title, section_id, items (list of findings), charts (list of chart URLs applicable to this section)

       - `extract_finding_detail(state: AnalysisState, section_id: str, finding_idx: int) -> dict[str, Any]`
         Drills into a specific finding: evidence narrative, source citation (filing type, date, URL), confidence level, D&O underwriting context, scoring impact

       - `extract_meeting_questions(state: AnalysisState, category: str | None = None) -> list[dict[str, Any]]`
         Reuses the meeting question generation from md_renderer._extract_meeting_questions but adds category filtering
         If category is provided, filter to only that category (CLARIFICATION, FORWARD_INDICATOR, GAP_FILLER, CREDIBILITY_TEST)

       - `extract_peer_metrics(state: AnalysisState) -> dict[str, Any]`
         Returns available peer comparison metrics: list of {key, label, company_value, peer_values: list}
         Sources from benchmark data in state

       IMPORTANT: Keep state_api.py under 500 lines. If approaching limit, split into state_api.py + state_api_sections.py

    2. Update `src/do_uw/dashboard/app.py` -- add drill-down and meeting prep routes:
       - GET `/section/{section_id}` (update existing placeholder):
         Returns HTMLResponse rendering section.html with section detail data from extract_section_detail()
         This is the htmx target when clicking a section card on the index page

       - GET `/section/{section_id}/finding/{finding_idx}`:
         Returns HTMLResponse rendering _finding_detail.html partial
         This is the htmx target when clicking a finding within a section

       - GET `/meeting-prep`:
         Returns HTMLResponse rendering _meeting_prep.html with all questions
         Query param `?category=CLARIFICATION` for filtering

       - GET `/api/peer-comparison`:
         Returns HTMLResponse rendering _peer_comparison.html partial
         Query param `?metric=market_cap` selects which metric to compare
         Includes the chart container for the peer comparison Plotly chart

       IMPORTANT: Keep app.py under 500 lines. If approaching limit, split routes into app.py (core + static) + routes_sections.py (section/finding/meeting routes) using FastAPI APIRouter.

    3. Create `src/do_uw/templates/dashboard/section.html`:
       - Extends base.html
       - Back button to index (plain link, not htmx)
       - Section title with risk badge
       - Section-specific charts (from section detail "charts" key)
       - Findings list: each finding is a DaisyUI collapse/accordion component
         - Summary line with confidence badge (HIGH=blue, MEDIUM=amber, LOW=gray)
         - Expandable detail loaded via hx-get="/section/{id}/finding/{idx}" hx-target="this" hx-swap="innerHTML"
       - Scoring impact summary (if applicable): which factors this section affects

    4. Create Jinja2 partial templates:
       a) `partials/_summary_card.html` -- reusable card macro:
          `{% macro summary_card(title, section_id, badge_class, metrics) %}`
          - DaisyUI card with hover shadow, cursor-pointer
          - hx-get="/section/{{ section_id }}" hx-target="#detail-panel" hx-swap="innerHTML"
          - Renders 2-3 key metrics from metrics dict
          - Badge with risk level class

       b) `partials/_risk_gauge.html` -- inline risk gauge macro:
          `{% macro risk_gauge(label, value, max_val, level_class) %}`
          - DaisyUI progress bar styled with risk colors
          - Text overlay with value/max

       c) `partials/_factor_detail.html` -- scoring factor drill-down:
          - Factor name, points deducted / max points
          - Contributing checks list with status badges
          - Evidence narratives
          - Source citations with filing type and date

       d) `partials/_peer_comparison.html` -- interactive peer comparison:
          - DaisyUI select dropdown for metric selection
          - hx-get="/api/peer-comparison?metric={{ selected }}" hx-target="#peer-chart-area" on change
          - Chart container for peer comparison bar chart
          - Table below with exact values for company vs each peer

       e) `partials/_finding_detail.html` -- individual finding deep-dive:
          - Evidence narrative (full text)
          - Source citation: filing type, filing date, URL/CIK reference
          - Confidence level badge
          - D&O underwriting context (why this matters for D&O insurance)
          - Scoring impact (which factor, how many points)

       f) `partials/_meeting_prep.html` -- meeting prep questions:
          - Category filter tabs using DaisyUI tabs:
            ALL | CLARIFICATION | FORWARD_INDICATOR | GAP_FILLER | CREDIBILITY_TEST
          - Each tab uses hx-get="/meeting-prep?category=X" hx-target="#questions-list"
          - Question list with:
            - Category badge
            - Question text (bold)
            - Context paragraph
            - "Good answer" and "Bad answer" in green/red (or blue/red per no-green rule) callout boxes
            - Follow-up question in smaller text

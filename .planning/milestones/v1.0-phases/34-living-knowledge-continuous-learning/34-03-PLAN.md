<task type="auto">
  <name>Task 1: Build feedback recording and querying module</name>
  <files>
    src/do_uw/knowledge/feedback.py
  </files>
  <action>
    Create src/do_uw/knowledge/feedback.py (~250-350 lines) with:

    1. `record_feedback(conn: DuckDBPyConnection, entry: FeedbackEntry) -> int`:
       - Inserts into brain_feedback table
       - Returns feedback_id
       - If feedback_type is "MISSING_COVERAGE", also call _auto_propose_check()

    2. `_auto_propose_check(conn: DuckDBPyConnection, entry: FeedbackEntry) -> int | None`:
       - When feedback says "you missed X", auto-generate a ProposalRecord:
         - source_type = "FEEDBACK"
         - source_ref = f"feedback_{entry.feedback_id}"
         - proposal_type = "NEW_CHECK"
         - proposed_check = a skeleton check dict with:
           - check_id derived from note keywords (e.g., "ING.FEED.{sanitized_keyword}")
           - name from the note text (first 80 chars)
           - content_type = "EVALUATIVE_CHECK"
           - lifecycle_state = "INCUBATING"
           - threshold_type = "boolean"
           - question = entry.note
           - rationale = f"Proposed from underwriter feedback by {entry.reviewer}"
         - Insert into brain_proposals table
         - Also insert the INCUBATING check via BrainWriter.insert_check
       - Return proposal_id or None if auto-proposal fails

    3. `get_feedback_summary(conn: DuckDBPyConnection) -> FeedbackSummary`:
       - Query brain_feedback for counts by type and status:
         - pending_accuracy = count where feedback_type='ACCURACY' AND status='PENDING'
         - pending_threshold = count where feedback_type='THRESHOLD' AND status='PENDING'
         - pending_coverage_gaps = count where feedback_type='MISSING_COVERAGE' AND status='PENDING'
       - Query brain_proposals for pending_proposals = count where status='PENDING'
       - recent_feedback = last 10 feedback entries (ordered by created_at DESC)
       - recent_proposals = last 10 proposals (ordered by created_at DESC)
       - Return FeedbackSummary

    4. `get_feedback_for_check(conn: DuckDBPyConnection, check_id: str) -> list[FeedbackEntry]`:
       - Query brain_feedback where check_id matches, ordered by created_at DESC
       - Return list of FeedbackEntry

    5. `mark_feedback_applied(conn: DuckDBPyConnection, feedback_id: int, change_id: int) -> None`:
       - Update brain_feedback SET status='APPLIED', applied_at=current_timestamp, applied_change_id=change_id
       - Used by the calibration workflow (Plan 04) after applying a change

    Use lazy import for BrainWriter (inside _auto_propose_check) to avoid circular imports. Use `from __future__ import annotations` throughout.

    ```python
    feedback_app = typer.Typer(
        name="feedback",
        help="Underwriter feedback: record, review, summarize",
        invoke_without_command=True,
    )
    ```

    Use Typer's callback pattern so that `do-uw feedback AAPL --note ...` is the default
    action (matching locked decision: `do-uw feedback AAPL --check X --note Y`), while
    `do-uw feedback summary` is a named subcommand.

    a. `@feedback_app.callback(invoke_without_command=True)` as the default handler:
       - Argument: ticker (str, default=None) -- when provided without a subcommand, this is the feedback recording flow
       - Options:
         - --check/-c (str|None): Check ID this feedback applies to
         - --note/-n (str|None): Feedback text (required when ticker is provided)
         - --reviewer/-r (str, default "anonymous"): Named reviewer
         - --type/-t (str, default "ACCURACY"): ACCURACY, THRESHOLD, or MISSING_COVERAGE
         - --direction/-d (str|None): FALSE_POSITIVE, FALSE_NEGATIVE, TOO_SENSITIVE, TOO_LOOSE
         - --run-id (str|None): Optional run ID for traceability
       - If ticker is provided but no note: error "Note is required when recording feedback"
       - If ticker is None and no subcommand invoked: show help
       - Validates: feedback_type must be one of the three allowed values; direction only valid for ACCURACY/THRESHOLD types
       - Connects to brain.duckdb, calls record_feedback()
       - Prints confirmation: "Feedback recorded (ID: {id})"
       - If MISSING_COVERAGE, also prints "Auto-proposed INCUBATING check: {check_id}"

    b. `@feedback_app.command("summary")`:
       - Options: --verbose/-v (bool, default False)
       - Connects to brain.duckdb, calls get_feedback_summary()
       - Default output (compact):
         ```
         Pending Feedback Summary
         ========================
         Accuracy flags:    3
         Threshold tuning:  1
         Coverage gaps:     2
         Pending proposals: 4
         ```
       - If --verbose: also show recent_feedback and recent_proposals as Rich tables
         - Feedback table: ID | Ticker | Check | Type | Direction | Reviewer | Note (truncated 60 chars)
         - Proposals table: ID | Source | Check | Type | Status | Rationale (truncated 60 chars)

    c. `@feedback_app.command("list")`:
       - Options: --check/-c (str|None) to filter by check_id, --ticker/-t (str|None) to filter by ticker
       - Shows all feedback entries matching filters as Rich table
       - If --check provided, calls get_feedback_for_check()

    2. Register in cli.py:
       - Add `from do_uw.cli_feedback import feedback_app` import
       - Add `app.add_typer(feedback_app, name="feedback")` alongside existing sub-apps
       - NOTE: cli.py already has cli_ingest import from Plan 02 -- add feedback alongside it

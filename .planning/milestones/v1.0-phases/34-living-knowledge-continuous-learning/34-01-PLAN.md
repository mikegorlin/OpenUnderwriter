    1. In brain_schema.py, add two new tables to _TABLES_DDL:

    ```sql
    CREATE SEQUENCE IF NOT EXISTS feedback_seq START 1;
    CREATE TABLE IF NOT EXISTS brain_feedback (
        feedback_id INTEGER PRIMARY KEY DEFAULT nextval('feedback_seq'),
        ticker VARCHAR,
        check_id VARCHAR,
        run_id VARCHAR,
        feedback_type VARCHAR NOT NULL,  -- ACCURACY, THRESHOLD, MISSING_COVERAGE
        direction VARCHAR,  -- FALSE_POSITIVE, FALSE_NEGATIVE, TOO_SENSITIVE, TOO_LOOSE
        note TEXT NOT NULL,
        reviewer VARCHAR NOT NULL,
        status VARCHAR NOT NULL DEFAULT 'PENDING',  -- PENDING, APPLIED, REJECTED
        applied_at TIMESTAMP,
        applied_change_id INTEGER,
        created_at TIMESTAMP NOT NULL DEFAULT current_timestamp
    );

    CREATE SEQUENCE IF NOT EXISTS proposal_seq START 1;
    CREATE TABLE IF NOT EXISTS brain_proposals (
        proposal_id INTEGER PRIMARY KEY DEFAULT nextval('proposal_seq'),
        source_type VARCHAR NOT NULL,  -- INGESTION, FEEDBACK, PATTERN
        source_ref VARCHAR,
        check_id VARCHAR,
        proposal_type VARCHAR NOT NULL,  -- NEW_CHECK, THRESHOLD_CHANGE, DEACTIVATION
        proposed_check JSON,
        proposed_changes JSON,
        backtest_results JSON,
        rationale TEXT NOT NULL,
        status VARCHAR NOT NULL DEFAULT 'PENDING',  -- PENDING, APPROVED, REJECTED, APPLIED
        reviewed_by VARCHAR,
        reviewed_at TIMESTAMP,
        created_at TIMESTAMP NOT NULL DEFAULT current_timestamp
    );
    ```

    2. Update brain_checks_active view in _VIEWS_DDL to exclude INCUBATING and INACTIVE:
    ```sql
    CREATE OR REPLACE VIEW brain_checks_active AS
    SELECT * FROM brain_checks_current
    WHERE lifecycle_state NOT IN ('RETIRED', 'INCUBATING', 'INACTIVE');
    ```
    NOTE: The locked lifecycle is INCUBATING → ACTIVE → INACTIVE. ACTIVE checks appear in
    brain_checks_active (same visibility as existing SCORING/MONITORING/INVESTIGATION checks).
    INCUBATING and INACTIVE checks are excluded. This means ACTIVE is the Phase 34 equivalent
    of the existing SCORING state for visibility purposes.

    3. Add indexes for the new tables to _INDEXES_DDL:
    ```sql
    CREATE INDEX IF NOT EXISTS idx_feedback_ticker ON brain_feedback(ticker);
    CREATE INDEX IF NOT EXISTS idx_feedback_check ON brain_feedback(check_id);
    CREATE INDEX IF NOT EXISTS idx_feedback_status ON brain_feedback(status);
    CREATE INDEX IF NOT EXISTS idx_proposals_status ON brain_proposals(status);
    CREATE INDEX IF NOT EXISTS idx_proposals_check ON brain_proposals(check_id);
    ```

    4. In brain_writer.py, verify that insert_check already supports lifecycle_state='INCUBATING' in check_data (it should -- the field is passed through). No changes needed if so. Add a docstring clarification that INCUBATING checks are excluded from brain_checks_active.

    5. In brain_loader.py, verify that the brain_checks_active query already filters INCUBATING via the updated view. No code changes needed in loader -- the view update in schema handles it.

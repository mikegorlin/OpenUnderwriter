  <action>
    1. Create src/do_uw/knowledge/discovery.py (~150-200 lines) with:

    a. `process_blind_spot_discoveries(blind_spot_results: dict[str, Any], ticker: str) -> list[dict[str, Any]]`:
       - Takes the blind_spot_results dict from state.acquired_data.blind_spot_results
       - Filters for results with high D&O relevance:
         - Look for URLs in results that contain keywords: litigation, lawsuit, SEC, enforcement, fraud, settlement, investigation, short seller, activist, whistleblower
         - Score each result 0-10 based on keyword density in title/snippet
         - Only process results scoring >= 5
       - For each high-relevance result:
         - Try to fetch URL content via ingestion_llm.fetch_url_content (with try/except for failures)
         - If content fetched, run extract_document_intelligence on it
         - Store any proposals via store_proposals (into brain_proposals as INCUBATING)
       - Return list of discovery summaries: [{url, title, event_type, proposals_generated}]
       - All operations are non-blocking: failures for individual URLs logged and skipped
       - This does NOT auto-change anything -- only creates proposals in brain_proposals

    b. `_score_relevance(title: str, snippet: str) -> int`:
       - Keyword-based scoring for D&O relevance
       - Returns 0-10 score

    c. `get_discovery_summary(discoveries: list[dict[str, Any]]) -> str`:
       - Formats discoveries into a summary string for worksheet display
       - "N documents analyzed, M proposals generated from blind spot search"

    2. In web_search.py, add a post-search hook:
       - After blind spot results are collected (look for where blind_spot_results is populated in the search client)
       - Add a try/except block that calls process_blind_spot_discoveries
       - Store the discovery summary in blind_spot_results dict under key 'discovery_findings'
       - This is non-blocking: if discovery fails, blind spot search results are still intact
       - Import discovery.py lazily inside the try block to avoid import overhead when not needed
       - IMPORTANT: Only run discovery if blind_spot_results has actual results (not empty/None)
  </action>

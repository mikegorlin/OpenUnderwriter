  <action>
    1. Create src/do_uw/knowledge/ingestion_llm.py (~200-300 lines) with:

    a. `extract_document_intelligence(text: str, doc_type: str, model: str = "claude-haiku-4-5-20250514") -> DocumentIngestionResult`:
       - Uses instructor + anthropic pattern matching existing LLMExtractor in stages/extract/llm/extractor.py
       - System prompt: "You are a D&O liability insurance expert analyzing documents for underwriting relevance. Extract: what company is affected, what happened, what are the D&O implications, which existing check categories might cover this, and what new checks might be needed."
       - Returns DocumentIngestionResult (from ingestion_models.py)
       - Handles errors gracefully (returns a minimal result with confidence="LOW" on failure)
       - Lazy import anthropic and instructor inside the function (matching existing pattern)

    b. `fetch_url_content(url: str) -> str`:
       - Uses httpx.get() with timeout=30, follow_redirects=True
       - Basic HTML-to-text conversion: strip tags, decode entities, collapse whitespace
       - Returns plain text content (LLM handles messy text fine)
       - Truncate to 50,000 chars to stay within token budget

    c. `generate_impact_report(ingestion_result: DocumentIngestionResult, doc_name: str, doc_type: str) -> IngestionImpactReport`:
       - Computes checks_affected from len(affected_checks)
       - Computes gaps_found from non-empty gap_analysis
       - Computes proposals_generated from len(proposed_new_checks)
       - Generates summary string for CLI display

    d. `store_proposals(writer: BrainWriter, result: DocumentIngestionResult, doc_name: str) -> int`:
       - For each proposed_new_check, insert into brain_proposals table (NOT brain_checks -- proposals only)
       - Also insert proposed checks as INCUBATING checks via writer.insert_check if the proposal has enough detail (check_id, threshold, question)
       - Return count of proposals stored
       - Uses conn.execute() to insert into brain_proposals with source_type='INGESTION', source_ref=doc_name

    2. In ingestion.py, update extract_knowledge_items to wire the LLM extraction_fn:
       - The existing TODO on ~line 212 says "Phase 13: Implement LLM-based extraction_fn"
       - Add a function `llm_extraction_fn(text: str, doc_type: DocumentType) -> list[dict[str, Any]]` that:
         - Calls extract_document_intelligence from ingestion_llm.py
         - Converts DocumentIngestionResult to the list[dict] format expected by extract_knowledge_items
         - Handles import errors gracefully (if anthropic not available, return empty list)
       - Wire it as the default extraction_fn when no custom one is provided

    Keep ingestion_llm.py under 300 lines. Use `from __future__ import annotations` throughout.

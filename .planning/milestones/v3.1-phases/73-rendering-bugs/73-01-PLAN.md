    **financials_quarterly.py** (~150 lines):
    - `build_quarterly_trend_context(state: AnalysisState) -> dict[str, Any]`
    - Returns `{"has_data": bool, "periods": list[str], "summary_strip": list[dict], "income_metrics": list[dict], "balance_metrics": list[dict], "cashflow_metrics": list[dict]}`
    - Summary strip: 4 cross-statement metrics (Revenue, Net Income, Free Cash Flow, Total Assets) with latest value + YoY change
    - Each metric dict: `{"label": str, "cells": list[str], "sparkline": str (SVG), "yoy_pct": float|None, "yoy_direction": "up"|"down"|"flat"}`
    - Uses `render_sparkline()` from `charts/sparklines.py` for inline sparklines
    - Graceful fallback: if `quarterly_xbrl` absent, try `yfinance_quarterly` data
    - Income tab metrics: Revenue, COGS, Gross Profit, Operating Income, Net Income, EPS
    - Balance tab metrics: Total Assets, Total Liabilities, Stockholders Equity, Cash, Total Debt, Working Capital
    - Cash flow tab metrics: Operating CF, Capital Expenditures, Free CF, Investing CF, Financing CF

    **financials_forensic.py** (~200 lines):
    - `build_forensic_dashboard_context(state: AnalysisState) -> dict[str, Any]`
    - Returns `{"has_data": bool, "bands": list[dict], "beneish": dict}`
    - Each band: `{"severity": "critical"|"warning"|"normal", "color": str (CSS var), "modules": list[dict]}`
    - Each module: `{"name": str, "composite_score": float, "zone": str, "worst_finding": str, "findings": list[dict]}`
    - Sort modules by severity (worst-first), group into bands, hide empty bands
    - Beneish dict: `{"has_data": bool, "m_score": float, "probability": float, "zone": str, "components": list[dict]}`
    - Each Beneish component: `{"code": str, "name": str, "value": float, "threshold": float, "pass": bool}`
    - 8 components: DSRI, GMI, AQI, SGI, DEPI, SGAI, LVGI, TATA with standard thresholds
    - Color mapping: use CSS variables `var(--do-risk-red)`, `var(--do-risk-amber)`, `var(--do-risk-green)`

    **financials.py** changes (minimal, ~10 lines added):
    - Import both new builders
    - Call them inside `extract_financials()` and merge results into context dict under `quarterly_trends` and `forensics` keys

    **Tests** (2 files, ~80 lines each):
    - Test that builders return `has_data: False` when state data is missing
    - Test that builders produce correct structure when mock data is provided
    - Test sparkline generation is called for each metric with values
    - Test Beneish component pass/fail logic against known thresholds
    - Test severity sorting (critical modules sorted before warning before normal)

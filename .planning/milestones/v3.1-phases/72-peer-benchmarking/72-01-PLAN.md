  <action>
Create `src/do_uw/stages/acquire/clients/sec_client_frames.py` with:

1. **FramesMetricDef dataclass**: `metric_name`, `xbrl_tag`, `unit` (default "USD"), `period_type` ("duration" or "instant"), `higher_is_better`.

2. **FRAMES_METRICS registry** (list[FramesMetricDef]): All concepts from research -- 8 direct metrics (Revenues, NetIncomeLoss, Assets, StockholdersEquity, Liabilities, OperatingIncomeLoss, NetCashProvidedByOperatingActivities, ResearchAndDevelopmentExpense) plus 2 component-only metrics needed for derived ratios (AssetsCurrent, LiabilitiesCurrent). Total: 10 unique XBRL tags.

3. **`_build_period_string(period_type: str, year: int) -> str`**: Returns "CY{year}" for duration, "CY{year}I" for instant.

4. **`_determine_best_period(company_10k_year: int | None) -> int`**: Uses company's most recent 10-K fiscal year. Falls back to current_year - 1.

5. **`acquire_frames(company_cik: int, company_10k_year: int | None, cache: AnalysisCache | None) -> dict[str, list[dict]]`**:
   - For each metric in FRAMES_METRICS, build URL: `https://data.sec.gov/api/xbrl/frames/us-gaap/{tag}/{unit}/{period}.json`
   - Check cache first (key: `sec:frames:{tag}:{unit}:{period}`)
   - If cache miss, call `sec_get()`, extract `data` list from response
   - Cache with TTL: 180 days for completed periods (year < current_year), 1 day for current
   - Store only the `data` list (list of {accn, cik, entityName, loc, end, val}), NOT the full response (memory optimization per research pitfall 3)
   - Return dict keyed by metric_name -> list of entity dicts
   - On API error for any single metric, log warning and continue (partial results OK)

6. **`acquire_sic_mapping(ciks: set[int], cache: AnalysisCache | None) -> dict[int, str]`**:
   - For each CIK, check cache (key: `sec:sic:{cik}`)
   - For uncached CIKs, batch-fetch `data.sec.gov/submissions/CIK{cik:010d}.json`, extract `sic` field
   - Cache each CIK's SIC with 90-day TTL
   - If a CIK's submissions fetch fails, skip it (log warning)
   - Return dict of cik -> sic_code (4-digit string)
   - Limit batch to 500 CIKs per run to avoid excessive API calls on first run (~50 seconds at 10 req/sec). Log how many CIKs were skipped. This is incremental -- subsequent runs fill in more.

Create `tests/test_sec_client_frames.py` with comprehensive unit tests mocking `sec_get` and `AnalysisCache`. Do NOT make real API calls.

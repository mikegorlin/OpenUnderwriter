<objective>
Extend the Form 4 parser with post-transaction ownership tracking, relationship flags, deduplication, and gift/estate filtering. Add ownership concentration analysis with tiered alerts per user decisions.

Purpose: The existing parser extracts basic transaction data but misses critical D&O underwriting signals -- how much of their holdings insiders are selling, whether transactions are gifts vs real sales, and whether filings are original vs amended. This plan adds the data foundation that pattern detection (Plan 02) builds on.

Output: Extended InsiderTransaction model, enhanced parser, dedup logic, ownership concentration computation, comprehensive tests.
</objective>

    class OwnershipConcentrationAlert(BaseModel):
        insider_name: str
        role: str  # CEO, CFO, Director, etc.
        severity: str  # RED_FLAG, WARNING, INFORMATIONAL, POSITIVE
        personal_pct_sold: float  # % of own holdings sold in window
        outstanding_pct: float | None  # % of shares outstanding (if available)
        shares_sold: float

    class OwnershipTrajectoryPoint(BaseModel):

    Add `compute_ownership_concentration()`:
    1. Group non-derivative transactions by insider_name
    2. For each insider, build trajectory from shares_owned_following over 36 months
    3. Compute % of holdings sold in trailing 6 months: `shares_sold / (shares_remaining + shares_sold)`

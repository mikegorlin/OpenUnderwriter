<objective>
Build semantic content QA framework that validates rendered output against source data.

Purpose: The system renders data from state.json through Jinja2 templates into HTML. Formatting errors, template bugs, or data path changes can cause the rendered values to diverge from source data. Semantic QA catches these by parsing the HTML and comparing values back to state. Research identified this as the one truly novel piece of work in Phase 75 (no existing infrastructure).

Output: Semantic QA module with validation functions, CI-integrated pytest tests.
</objective>

  <name>Task 1: Create semantic QA validation module</name>
  <files>src/do_uw/stages/render/semantic_qa.py</files>
  <action>
Create `src/do_uw/stages/render/semantic_qa.py` with these functions:

1. `_parse_financial_value(text: str) -> float | None`: Parse formatted financial values back to numbers. Handle: "$1.2B" -> 1.2e9, "$450M" -> 4.5e8, "$12.5K" -> 12500, commas, negative values in parens "(1.2B)" -> -1.2e9, plain numbers. Return None if unparsable.

2. `_extract_state_revenue(state_data: dict) -> float | None`: Navigate state dict to find revenue from XBRL statements. Path: `extracted.financials.statements` -> find `statement_type == "income_statement"` -> find `line_items` where `concept == "revenue"` -> return `value`. Return None if not found.

3. `_extract_state_board_size(state_data: dict) -> int | None`: Navigate: `extracted.governance.board_composition.size`. Return None if missing.

4. `_extract_state_overall_score(state_data: dict) -> float | None`: Navigate: `scored.overall_score`. Return None if missing.

5. `_extract_html_value(soup: BeautifulSoup, label_pattern: str, section_hint: str | None = None) -> str | None`: Find a value in HTML by looking for a label matching `label_pattern` (regex) and extracting the adjacent value cell/span. Search within section_hint class/id if provided. Handle both KV-table format (label in one td, value in next) and dl/dt/dd format.

6. `validate_revenue(state_data: dict, soup: BeautifulSoup) -> tuple[bool, str]`: Compare state revenue vs HTML revenue. Use 1% tolerance for rounding. Return (pass, message).

7. `validate_board_size(state_data: dict, soup: BeautifulSoup) -> tuple[bool, str]`: Compare state board size vs HTML board size. Exact integer match. Return (pass, message).

8. `validate_overall_score(state_data: dict, soup: BeautifulSoup) -> tuple[bool, str]`: Compare state overall score vs HTML score. 0.1 absolute tolerance. Return (pass, message).

9. `validate_output(state_path: Path, html_path: Path) -> list[tuple[str, bool, str]]`: Run all validations. Load state JSON and HTML. Return list of (check_name, passed, message). Skip checks where source data is None (not available for this ticker).

Use BeautifulSoup with "lxml" parser. Keep module under 250 lines.

1. Create `src/do_uw/brain/post_pipeline.py`:

```python
def run_post_pipeline_learning(ticker: str) -> dict[str, Any]:
```

This function:
- Gets brain DB connection via `get_brain_db_path()` + `duckdb.connect()`
- Calls `compute_calibration_report(conn)` -- generates THRESHOLD_CALIBRATION proposals for drifting signals
- Extracts `fire_rate_alerts` from the calibration report -- logs each alert at WARNING level with signal_id, fire_rate, alert_type, recommended_action
- Calls `compute_lifecycle_proposals(conn)` -- generates LIFECYCLE_TRANSITION proposals
- Logs summary: N drift proposals, M fire-rate alerts, K lifecycle proposals
- Returns dict with counts: `{"drift_proposals": N, "fire_rate_alerts": M, "lifecycle_proposals": K, "total_proposals": N+K}`
- Entire function wrapped in try/except -- failure in learning loop must NEVER fail the pipeline. Log error and return empty dict.
- Close DB connection in finally block.

2. Wire into `pipeline.py`:
- After the pipeline metadata logging (line ~235, before `return state`), add:
```python
# Post-pipeline learning: generate calibration + lifecycle proposals
try:
    from do_uw.brain.post_pipeline import run_post_pipeline_learning
    learning = run_post_pipeline_learning(state.company.ticker or "UNKNOWN")
    if learning.get("total_proposals", 0) > 0:
        logger.info(
            "Post-pipeline learning: %d proposals generated "
            "(run 'brain apply-proposal' to review)",
            learning["total_proposals"],
        )
except Exception:
    logger.debug("Post-pipeline learning skipped", exc_info=True)
```
- Use lazy import (inside try block) to avoid import-time dependency on brain modules
- Use logger.debug for skip (not warning) -- this is non-critical

Keep post_pipeline.py under 100 lines. Keep pipeline.py changes to ~10 lines.

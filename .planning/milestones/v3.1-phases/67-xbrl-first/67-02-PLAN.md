<objective>
Add sign normalization layer and coverage validation tooling to the XBRL extraction pipeline. Sign normalization fixes the ~12% of filings that report values with incorrect signs. Coverage validation provides visibility into extraction quality per ticker.

Purpose: Without sign normalization, ratios computed from XBRL data can be wildly wrong (negative current ratio, inverted margins). Without coverage validation, extraction gaps go undetected until downstream failures.

Output: normalize_sign() integrated into extraction pipeline, CoverageReport with per-concept resolution tracking, tag discovery CLI utility.
</objective>

### v3.1 XBRL-First Data Integrity (Phases 67-75)

**Milestone Goal:** Eliminate LLM hallucination from all quantitative data. Comprehensive XBRL extraction engine -- 120+ concepts, 8 quarters + 3 annual, forensic financial analysis, structured Form 4 extraction, SEC Frames API peer benchmarking -- with every metric tied to brain signals. ### Phase 67: XBRL Foundation -- Concept Expansion & Infrastructure
**Goal**: Expand XBRL concept coverage from 50 to 120+ concepts with sign normalization, derived computation, and coverage validation. The foundation everything else builds on.
**Depends on**: v3.0 (Phase 66)
**Requirements**: XBRL-01, XBRL-02, XBRL-03, XBRL-04, XBRL-05, XBRL-06
**Success Criteria** (what must be TRUE):
  1. `xbrl_concepts.json` has 120+ concepts with priority-ordered tag lists
  2. Every concept has `expected_sign` field; normalization layer applied
  3. Derived concepts (margins, ratios) computed from XBRL primitives
  4. Coverage validator logs resolution rates per concept per ticker
  5. All existing tests pass; existing tickers produce identical or better output
**Plans**: 3 plans
  - [ ] 67-01-PLAN.md -- Expand xbrl_concepts.json: 50 new income/balance/cash flow concepts with researched tag lists (XBRL-01, XBRL-06)
  - [ ] 67-02-PLAN.md -- Sign normalization layer + expected_sign config + coverage validator (XBRL-02, XBRL-04, XBRL-05)
  - [ ] 67-03-PLAN.md -- Derived concept computation module: margins, ratios, per-share metrics (XBRL-03)

**Depends on**: Phase 74 (pipeline must be fully wired first)

  2. All foundational signals cover 100% of actual Tier 1 acquisitions (8-K, Form 4, short interest, Frames included)

  <behavior>
    - test_signal_portability: Load all signal YAML files from brain/signals/**/*.yaml. For each signal with schema_version >= 2, assert acquisition is not None, evaluation is not None, presentation is not None. Collect failures as list and assert empty with descriptive message showing which signals fail.
    - test_manifest_coverage: Load manifest via load_manifest(). For each section, for each group, assert group.template file exists on disk under templates/html/. Collect missing templates and assert none missing.
    - test_template_purity: Scan all .html.j2 files under templates/html/sections/company/. For each file, read content and assert NO matches for forbidden patterns: numeric comparison operators with literal numbers in Jinja2 expressions (e.g., `{% if score > 3 %}`, `{% if value >= 15 %}`), threshold-like constants (e.g., `0.4`, `50%` used in conditionals), Python evaluation functions (e.g., `eval(`, `exec(`). Allow: `> 0` checks (presence/absence), loop counters, CSS classes with numbers, display formatting. The regex should target Jinja2 expression blocks `{% ... %}` and `{{ ... }}` containing numeric comparisons, not prose or comments.
  </behavior>
  <action>
1. Create test_signal_portability.py:
   - Use glob to find all brain/signals/**/*.yaml files
   - Parse each with yaml.safe_load
   - Handle multi-document YAML (some files have multiple signals)
   - For signals with schema_version >= 2: assert acquisition, evaluation, presentation are all present and non-None
   - Use parametrize or collect-and-assert pattern for clear failure messages

2. Create test_manifest_coverage.py:
   - Load manifest via load_manifest()
   - For each group in each section, resolve template path against templates/html/ directory
   - Assert file exists on disk
   - This extends/complements test_template_facet_audit.py's test_all_group_templates_resolve -- if that test already covers this exactly, add a comment and import/delegate rather than duplicating

3. Create test_template_purity.py:
   - Scan all .html.j2 files under templates/html/sections/company/
   - Also scan templates/html/sections/executive/complexity_dashboard.html.j2
   - Read each file's content
   - Search for forbidden patterns in Jinja2 blocks: numeric threshold comparisons like `> 3`, `>= 15`, `< 0.5`, `== 40` (but NOT `> 0` or `!= 0` which are presence checks, and NOT CSS numbers)
   - The regex should match: `{%.*(?:>|<|>=|<=|==)\s*(?:[1-9]\d*(?:\.\d+)?|\d+\.\d+).*%}` (numeric literals > 0 in conditional blocks)
   - Exclude known safe patterns: loop.index, forloop, CSS class names, display formatting
   - Assert zero matches with descriptive error showing file + line + match

4. Run all 3 tests and fix any failures in the test assertions (not in production code -- the tests should pass against the current codebase after Plan 01 and Plan 02 are complete).

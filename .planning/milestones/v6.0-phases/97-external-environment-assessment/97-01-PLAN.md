```python
def extract_environment_signals(state: AnalysisState) -> dict[str, Any]:
```

This function reads from `state.extracted.risk_factors`, `state.company.geographic_footprint`, `state.extracted.litigation`, and LLM extraction data to compute 5 numeric scores.

Implementation details per signal:

1. **regulatory_intensity_score**:
   - Count risk_factors where category == "REGULATORY"
   - Parse regulatory_environment text from LLM extraction for named regulators (use regex for SEC, FDA, EPA, DOJ, FTC, CFPB, FERC, NRC, OCC, FDIC, "state attorney")
   - Score = count of distinct regulators found (0-10+ scale)
   - Also store `regulatory_details` dict with regulator names and source passages

2. **geopolitical_risk_score**:
   - Define SANCTIONED_COUNTRIES = {"Cuba", "Iran", "North Korea", "Syria", "Russia", "Belarus", "Venezuela", "Myanmar"}
   - Define HIGH_RISK_COUNTRIES = {"China", "Saudi Arabia", "Turkey", "Brazil", "India", "South Africa", "Mexico", "Nigeria", "Pakistan", "Egypt"}
   - Scan geographic_footprint region/jurisdiction names for matches (case-insensitive substring)
   - Score: 3 if any sanctioned country, 2 if >3 high-risk, 1 if 1-3 high-risk, 0 otherwise
   - Store `geopolitical_details` with matched countries and risk level

3. **esg_gap_score**:
   - Count ESG risk factors (category == "ESG")
   - Check if ESG-related litigation exists (search litigation case names for "environmental", "climate", "ESG", "sustainability", "emissions", "discrimination", "diversity")
   - Score: 3 if ESG litigation + ESG risk factors (gap confirmed), 2 if ESG risk factors severity HIGH, 1 if ESG risk factors present, 0 if none
   - Store `esg_gap_details`

4. **cyber_risk_score**:
   - Count CYBER risk factors
   - Check severity of CYBER risk factors
   - Look for breach indicators in text_signals or web search results (keywords: "breach", "hack", "data leak", "unauthorized access", "cybersecurity incident")
   - Score: 3 if breach history, 2 if CYBER risk factors HIGH severity, 1 if CYBER factors present, 0 if none
   - Store `cyber_risk_details`

5. **macro_sensitivity_score**:
   - Check for interest_rate_risk from LLM extraction (non-null = 1 dimension)
   - Check for currency_risk from LLM extraction (non-null = 1 dimension)
   - Count FINANCIAL risk factors mentioning macro terms ("inflation", "recession", "commodity", "interest rate", "exchange rate", "tariff")
   - Score = count of distinct macro dimensions (0-5 scale)
   - Store `macro_sensitivity_details`

Helper: `_get_llm_ten_k_fields(state)` that extracts regulatory_environment, interest_rate_risk, currency_risk from the most recent LLM extraction (same pattern as `_enrich_from_llm` in company_profile.py).

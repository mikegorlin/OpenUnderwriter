Create `src/do_uw/templates/html/sections/company/environment_assessment.html.j2` that renders the 5 environment signals as a paired-column KV table with status indicators.

Follow the pattern of existing company section templates (e.g., risk_factors.html.j2). The template receives `company` context dict from extract_company().

Template structure:

```jinja2
{# External Environment Assessment - ENVR signals #}
{% set env = company.environment_assessment if company.environment_assessment else {} %}
{% if company.has_environment_data %}
<div class="subsection" id="external-environment">
  <h4>External Environment Assessment</h4>
  <table class="kv-table">
    <tbody>
      {# For each signal: row with name, level badge, and detail text #}
      {% for signal in [
        {"name": "Regulatory Intensity", "data": env.regulatory_intensity},
        {"name": "Geopolitical Exposure", "data": env.geopolitical_risk},
        {"name": "ESG Commitment Gap", "data": env.esg_gap},
        {"name": "Cyber Risk Profile", "data": env.cyber_risk},
        {"name": "Macro Sensitivity", "data": env.macro_sensitivity}
      ] %}
      {% if signal.data %}
      <tr>
        <td class="kv-label">{{ signal.name }}</td>
        <td class="kv-value">
          <span class="badge badge-{{ signal.data.level | lower }}">{{ signal.data.level }}</span>
          {% if signal.data.details %}
          <span class="detail-text">{{ signal.data.details }}</span>
          {% endif %}
        </td>
      </tr>
      {% endif %}
      {% endfor %}
    </tbody>
  </table>
</div>
{% endif %}
```

Use existing CSS classes: `kv-table`, `kv-label`, `kv-value`, `badge`, `badge-high` (red), `badge-moderate` (amber), `badge-low` (green).

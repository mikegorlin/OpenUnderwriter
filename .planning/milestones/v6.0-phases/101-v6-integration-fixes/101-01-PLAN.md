  Fix 4 bugs. Each fix is a single-line change with a known-good pattern to follow.

  **Fix 1a — corporate_events.html.j2 line 2:**
  CHANGE: `{% set events = corporate_events_signals if corporate_events_signals is defined else {} %}`
  TO: `{% set events = company.corporate_events_signals if company.corporate_events_signals else {} %}`
  WHY: `corporate_events_signals` is not a top-level Jinja2 variable — it's nested under the `company` dict passed to the template. The `is defined` guard silently returns `{}` making the section always empty. This matches the working pattern in environment_assessment.html.j2, sector_risk.html.j2, and operational_complexity.html.j2.

  **Fix 1b — structural_complexity.html.j2 line 2:**
  CHANGE: `{% set sc = structural_complexity_signals if structural_complexity_signals is defined else {} %}`
  TO: `{% set sc = company.structural_complexity_signals if company.structural_complexity_signals else {} %}`
  WHY: Same bug, same fix as Fix 1a.

  **Fix 2 — sector.yaml signal_class enum (2 locations):**
  Line 142: CHANGE `signal_class: reference` TO `signal_class: foundational`
  Line 223: CHANGE `signal_class: reference` TO `signal_class: foundational`
  WHY: `reference` is not a valid BrainSignalEntry enum value. These signals (SECT.claim_patterns and SECT.regulatory_overlay) contain static reference data without computation, making `foundational` the correct class. Currently these signals are silently dropped at YAML load, meaning they never evaluate or score.

  **Fix 3 — sector.yaml threshold_provenance.source:**
  Line 48: CHANGE `source: cornerstone_nera_scac` TO `source: sca_settlement_data`
  WHY: `cornerstone_nera_scac` is not in the valid source set defined in tests/brain/test_brain_contract.py. The data IS from Cornerstone/NERA SCA settlement studies, which maps to the `sca_settlement_data` category. This causes the brain contract CI test to fail.

  **Fix 4 — test_event_signals.py stale assertion:**
  Line 271: CHANGE `assert ce_group["render_as"] == "check_summary"` TO `assert ce_group["render_as"] == "kv_table"`
  WHY: Phase 100 changed corporate_events render_as from check_summary to kv_table to match the actual template pattern. The test was not updated.

### v8.0 Intelligence Dossier (Phases 115-120)

**Milestone Goal:** Transform every data point into D&O risk intelligence. Every data table gets a "so what for D&O?" column driven by brain YAML `do_context` blocks. Scoring factors get "What Was Found" + "Underwriting Commentary" sections. Forward-looking guidance claims map to SCA probability. Company intelligence dossier provides revenue model, competitive landscape, and concentration analysis. Stock drops get catalyst identification and D&O assessment. All commentary originates from brain YAML, not Python or templates.

**QUAL requirements (QUAL-01 through QUAL-07) are cross-cutting constraints that apply to ALL phases.** They are not assigned to a single phase -- every phase must satisfy them for its deliverables. QUAL compliance is verified during Phase 120 integration.

## Phase Details

### Phase 115: do_context Infrastructure
**Goal**: Every brain signal can carry template-driven D&O commentary that is evaluated once in ANALYZE and consumed as a pre-rendered string by dumb renderers
**Depends on**: Nothing (first phase of v8.0)
**Requirements**: INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05
**Success Criteria** (what must be TRUE):
  1. Brain signal YAML schema accepts `presentation.do_context` field with per-status commentary templates (TRIGGERED_RED, TRIGGERED_YELLOW, CLEAR) -- Pydantic-validated, backward compatible with existing 562 signals
  2. `do_context_engine.py` in ANALYZE evaluates do_context templates against signal results and state data, storing the rendered commentary string on `SignalResult.do_context`
  3. The 4 hardcoded D&O commentary functions in `_distress_do_context.py` (Altman, Beneish, Piotroski, Ohlson) are migrated to brain YAML `do_context` blocks and produce identical output
  4. Context builders consume `do_context` strings from signal results via a standard accessor -- zero evaluative logic in the builder itself
  5. CI gate fails when evaluative D&O language appears in template literals or context builder if/elif chains
**Plans:** 3/3 plans complete
Plans:
- [x] 115-01-PLAN.md -- Schema + engine: add do_context to PresentationSpec/SignalResult, build template engine, wire into signal pipeline
- [x] 115-02-PLAN.md -- Migration + CI: migrate 3 distress functions to YAML, wire consumer, golden parity tests, CI gate
- [x] 115-03-PLAN.md -- Gap closure: migrate Ohlson O-Score to brain YAML, delete Python fallback, remove CI gate exception

### Phase 117: Forward-Looking Risk Framework
**Goal**: Underwriters can see every forward-looking claim the company has made, its probability of being missed, and the SCA relevance if missed -- plus structured monitoring triggers and an underwriting recommendation
**Depends on**: Phase 116 (requires scoring commentary and do_context patterns established)
**Requirements**: FORWARD-01, FORWARD-02, FORWARD-03, FORWARD-04, FORWARD-05, FORWARD-06, SCORE-02, SCORE-03, SCORE-05, TRIGGER-01, TRIGGER-02, TRIGGER-03
**Success Criteria** (what must be TRUE):
  1. Forward-Looking Statement Risk Map renders: every publicly guided metric mapped to current value, guidance/claim, miss risk, and SCA relevance if missed
  2. Management Credibility table shows quarter-by-quarter guidance vs actual with beat/miss/magnitude, track record percentage, and credibility score
  3. Monitoring Triggers table renders: trigger/action/threshold for ongoing post-bind monitoring (SCA filing, stock below support, insider selling pace, CEO departure, earnings miss, peer SCA)

  5. Quick Screen / Trigger Matrix shows all RED/YELLOW flags with deep-dive section routing, prospective checks with traffic light status, and nuclear trigger verification

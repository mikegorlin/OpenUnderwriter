# Requirements: D&O Underwriting System v3.0

**Defined:** 2026-03-02
**Core Value:** Transform the D&O worksheet from a data report into an institutional-quality analytical briefing — CIQ-level density, professional narratives, interactive charts, and zero hidden data.

## v3.0 Requirements

56 requirements across 9 phases (39 MUST, 17 SHOULD). Three critical problems addressed:
1. **Massive duplication**: Word sections (10,628 lines, 28 files) reimplement ~6,000 lines of extraction logic already in md_renderer_helpers. Zero imports between them.
2. **Hidden data**: ~60% of extracted analytical detail (compensation, peer comparison, sentiment, hazard categories, executive shade factors, allegation detail) exists in state but never renders.
3. **Basic narratives**: Single-paragraph summaries where professional reports use 5-layer analytical structures with bull/bear framing and confidence-calibrated language.

### Shared Context Layer (CTX)

- [x] **CTX-01** [MUST]: Create `stages/render/context_builders/` package with one module per domain (company, financials, market, governance, litigation, scoring, analysis, calibration). Each module contains the `extract_*` functions currently in md_renderer_helpers.
- [x] **CTX-02** [MUST]: Each `extract_*` function moves with ZERO logic changes — pure file reorganization with re-exports from old locations.
- [x] **CTX-03** [MUST]: `md_renderer_helpers*.py` become thin re-export shims (<30 lines each). Grep confirms zero direct state access in shims.
- [x] **CTX-04** [MUST]: `build_template_context()` calls context_builders; `build_html_context()` calls `build_template_context()` unchanged.
- [x] **CTX-05** [MUST]: All 3,967+ tests pass; zero visual regression on SNA HTML output (diff).
- [x] **CTX-06** [MUST]: Every context_builder module under 500 lines.

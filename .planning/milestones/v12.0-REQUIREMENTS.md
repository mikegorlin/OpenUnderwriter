# v12.0 Requirements — Output Quality & Architectural Integrity

**Defined:** 2026-03-27
**Core Value:** The single source of truth for underwriters to make the most knowledgeable decisions on a risk.

## Canonical Metrics

- [ ] **METR-01**: CanonicalMetricsRegistry computes each metric exactly once with XBRL-first source priority (XBRL > yfinance > LLM extraction > web)
- [x] **METR-02**: Revenue, net income, market cap, stock price, employees, exchange name, CEO name, and growth rates are computed once and consumed by all sections
- [ ] **METR-03**: Every metric in the registry carries its source, as-of date, and confidence level
- [x] **METR-04**: Context builders import from the canonical registry instead of independently navigating state

## Typed Output Contracts

- [x] **TYPE-01**: Pydantic BaseModel defined for each section context (executive brief, financial, market, governance, litigation, company, scoring, meeting prep)
- [x] **TYPE-02**: All fields use `Optional[T] = None` defaults — no required fields for data that can be missing
- [x] **TYPE-03**: Validation wrapper pattern: try typed path, fall back to existing untyped dict on failure — zero risk of breaking existing render
- [x] **TYPE-04**: `model_dump(by_alias=True)` produces identical dict keys that templates already expect — zero template changes required
- [ ] **TYPE-05**: Priority migration: 5 highest-leakage builders first (exec summary, financial, market, governance, litigation)

## Contextual Signal Validation

- [ ] **SIG-01**: Post-ANALYZE validation pass cross-checks triggered signals against company state
- [ ] **SIG-02**: YAML-driven validation rules (not Python if/else) following brain portability principle
- [ ] **SIG-03**: IPO/offering signals suppressed for companies public > 5 years with no recent offerings
- [ ] **SIG-04**: Financial distress signals annotated when Z-Score and O-Score are both in Safe zone
- [ ] **SIG-05**: Negation detection — signals whose evidence says "do not have" / "no holdings" get annotated as negated
- [ ] **SIG-06**: Temporal validation — signals referencing departed executives annotated with departure date
- [ ] **SIG-07**: Validation annotates findings (adds context), never suppresses them — critical for D&O where hiding a finding is worse than a false positive

## Litigation Classification & Consolidation

- [x] **LIT-01**: Case type classified from legal theories (derivative vs SCA vs SEC enforcement vs regulatory) not source category
- [x] **LIT-02**: Substantially similar cases consolidated into single entry with list of related case names and filing dates
- [x] **LIT-03**: Same-name cases disambiguated by year (e.g., "In re Fastly (2020)" vs "In re Fastly (2024)")
- [x] **LIT-04**: Coverage side classification (A/B/C) derived from case type and named defendants
- [x] **LIT-05**: Missing critical fields (case number, court, class period, named defendants) flagged and acquisition attempted from available sources

## Output Sanitization

- [x] **SAN-01**: Single OutputSanitizer class consolidating existing scattered sanitization functions
- [x] **SAN-02**: Strip markdown artifacts (`**`, `---`, `###`) from LLM-generated text before render
- [x] **SAN-03**: Strip raw Python serialization (list `['...']`, dict `{'...'}`, debug evaluation strings) from final HTML
- [x] **SAN-04**: Strip system jargon that passes through existing filters (catch-all for any new leakage patterns)
- [x] **SAN-05**: Sanitizer logs what it strips — builds evidence for prevention rules upstream

## Quality Gates

- [ ] **GATE-01**: Cross-section consistency checker verifies same fact (revenue, CEO, exchange, case counts) is identical everywhere it appears
- [ ] **GATE-02**: Section completeness gate suppresses sections with >50% N/A values or shows "Insufficient Data" banner
- [x] **GATE-03**: Real-state integration tests for context builders using actual state.json files (not unspec'd MagicMocks)
- [x] **GATE-04**: Template variable type validation CI gate checking template variables against Pydantic output schema
- [ ] **GATE-05**: Consistency checker starts report-only, promoted to blocking after zero inconsistencies achieved

## Stock Charts

- [ ] **CHART-01**: 1-year stock chart displayed as first element in Stock & Market section
- [ ] **CHART-02**: 5-year stock chart displayed side by side with 1-year chart
- [ ] **CHART-03**: Both charts overlay sector ETF performance for relative comparison
- [ ] **CHART-04**: Charts include key event annotations (earnings, stock drops, litigation filings)

## Out of Scope

- LLM-based output sanitization — must be deterministic regex/BeautifulSoup (SAN-01)
- Blocking mode for consistency checker on day 1 — starts report-only (GATE-05)
- Template redesign — zero template changes in this milestone (TYPE-04)

**gap_searcher.py structure:**

```python
"""Phase E: Brain-driven gap search for SKIPPED checks.

After all structured ACQUIRE completes, checks with gap_bucket fields in brain
YAML that lack data_strategy routing receive targeted web searches.
Non-L1 checks only. Budget capped at min(budget_remaining, GAP_SEARCH_MAX).
"""
from __future__ import annotations
import logging
from pathlib import Path
from typing import Any
import yaml  # PyYAML — already in project

from do_uw.cache.sqlite_cache import AnalysisCache
from do_uw.models.state import AcquiredData, AnalysisState
from do_uw.stages.acquire.clients.web_search import WebSearchClient, WEB_SEARCH_TTL

logger = logging.getLogger(__name__)

GAP_SEARCH_MAX = 15  # Maximum gap searches per run (draws from shared 50-budget)

_BRAIN_CHECKS_DIR = Path(__file__).parent.parent.parent / "brain" / "checks"
```

**Core functions to implement:**

`_load_gap_eligible_checks() -> list[dict[str, Any]]`:
- Walk `brain/checks/**/*.yaml` with `Path(_BRAIN_CHECKS_DIR).glob("**/*.yaml")`
- For each check entry in each YAML file: include ONLY if both:
  1. `gap_bucket` field is present (was set in Plan 01)
  2. `acquisition_tier` is NOT "L1" (code-level L1 gate — independent of gap_bucket classification)
- Return list of check dicts with at minimum: id, name, acquisition_tier, gap_bucket, gap_keywords, tier (for priority)

`_rank_by_severity(checks: list[dict], max_count: int) -> list[dict]`:
- Sort by `tier` ascending (tier=1 = highest priority) then by `gap_bucket` order (routing-gap > aspirational > intentionally-unmapped)
- Return first `max_count` checks

`_evaluate_evidence(results: list[dict[str, str]], keywords: list[str]) -> tuple[bool, str]`:
- If keywords empty or results empty: return (False, "")
- For each result, check `snippet`/`description`/`title` (all lower-cased) for any keyword (lower-cased)
- First match: return (True, domain extracted from result's `url` via `urllib.parse.urlparse(url).netloc`)
- No match: return (False, "")

`_extract_domain(url: str) -> str`:
- `urllib.parse.urlparse(url).netloc` — strip "www." prefix
- Return empty string on parse failure

`run_gap_search(state, acquired, web_search, cache) -> None`:
- Public entry point called from orchestrator
- Compute `available = min(web_search.budget_remaining, GAP_SEARCH_MAX)`
- If available <= 0: log and return
- Load eligible checks, rank by severity
- For each check up to `available`:
  - Build cache key: `f"gap_search:{state.ticker}:{check['id']}"`
  - Check cache — on hit, load from cache into `acquired.brain_targeted_search` and SKIP the search (do not decrement `searched` counter for cache hits — only live searches count against budget)
  - Generate query via `generate_gap_query()` from gap_query_generator.py (with template fallback)
  - Execute `web_search.search(query, cache=cache)` — this increments `web_search._searches_used`
  - `keywords_matched, domain = _evaluate_evidence(results, check.get('gap_keywords', []))`
  - If `not keywords_matched and not results`: `suggested_status = None` — leave as SKIPPED, don't store
  - If results exist:
    - `suggested_status = "TRIGGERED" if keywords_matched else "CLEAR"`
    - Store gap_result dict in `acquired.brain_targeted_search[check_id]`
    - Cache: `cache.set(cache_key, gap_result, source="gap_search", ttl=WEB_SEARCH_TTL)` if cache
  - Increment `searched` counter only on live search
- Log: `"Gap search: %d/%d budget used, %d eligible checks unsearched"` with `searched, available, max(0, len(eligible)-searched_or_cached)`

Important pitfall to avoid: if keywords list is empty for a non-L1 check, log a warning and skip that check (don't mark as CLEAR — empty keywords means unevaluable). Per RESEARCH.md pitfall 3.

```python
"""LLM-based query generation for gap searches.

Two-step: LLM generates an optimized company-specific query string.
Falls back to template if LLM unavailable or fails.
"""
```

**Functions:**

`generate_gap_query(check: dict[str, Any], company_name: str, ticker: str) -> str`:
- First try `_generate_via_llm(check, company_name, ticker)`
- On any exception or empty result: fall back to `_generate_via_template(check, company_name, ticker)`

`_generate_via_template(check, company_name, ticker) -> str`:
- Template: `f'"{company_name}" ({ticker}) {" ".join(kw for kw in check.get("gap_keywords", [])[:3])}'`
- If no keywords: `f'"{company_name}" {ticker} {check.get("name", "")}'`

`_generate_via_llm(check, company_name, ticker) -> str`:
- Import `anthropic` lazily (don't fail at module load if not installed)
- Use `claude-haiku-3-5` model (cheap, fast, ~30 token output)
- Prompt (use Claude's judgment for exact wording):
  - Include company name, ticker
  - Include check name and required_data summary
  - Request ONLY the search query string, maximum 12 words
  - System prompt emphasizes recent news, regulatory actions, public disclosures
- max_tokens=50, no streaming
- Strip whitespace from response, return first line only
- If response is empty or too long (>100 chars): fall back to template
- Log cost at debug level: `"Gap query LLM call: model=%s"` (cost is negligible, ~$0.001 per call)

Note: Consider generating all queries in a single LLM call (one prompt listing all checks) to reduce API calls. Per RESEARCH.md pitfall 5, this is cheaper and faster. Use batch approach: if len(eligible) > 3, make one call with a numbered list of checks, parse numbered responses. Fall back to per-check calls if batch parsing fails.

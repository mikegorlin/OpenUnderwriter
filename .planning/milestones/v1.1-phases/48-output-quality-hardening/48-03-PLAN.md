    Read `src/do_uw/brain/red_flags.json` to understand the structure. The file has an `escalation_triggers` list where each entry has `"id"` and `"condition"` keys. Add this helper function near the top of the module (after imports):

    ```python
    def _load_crf_conditions() -> dict[str, str]:
        """Load CRF condition text from red_flags.json for threshold_context display.

        Returns {crf_id: condition_text} mapping. Empty dict on any failure — non-fatal.
        The condition text is the human-readable criterion for what triggers each CRF flag
        (e.g. "CRF-01" → "Company has pending securities class action lawsuit").
        """
        import json
        from pathlib import Path
        rf_path = Path(__file__).parent.parent.parent / "brain" / "red_flags.json"
        try:
            data = json.loads(rf_path.read_text())
            triggers = data.get("escalation_triggers", [])
            return {
                t["id"]: t.get("condition", "")
                for t in triggers
                if "id" in t
            }
        except Exception:
            return {}
    ```

    In `extract_scoring()`, find the section that builds the `red_flags` list from `sc.red_flags`. It currently produces dicts with keys: `id`, `name`, `description`, `ceiling`, `max_tier`.

    crf_conditions = _load_crf_conditions()

            "description": "; ".join(rf.evidence) if rf.evidence else "",
            "ceiling": str(rf.ceiling_applied) if rf.ceiling_applied else "N/A",
            "max_tier": rf.max_tier or "N/A",
            "threshold_context": crf_conditions.get(rf.flag_id, ""),  # QA-04

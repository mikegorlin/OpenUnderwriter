## CORE -- Core System

The system accepts a stock ticker and produces a comprehensive D&O underwriting worksheet.

- [ ] **CORE-01:** System accepts a stock ticker symbol as input and resolves it to a company identity (legal name, CIK, SIC, NAICS, exchange, sector, market cap)
- [ ] **CORE-02:** System produces a multi-section underwriting worksheet as primary output (7 analytical sections plus executive summary)
- [ ] **CORE-03:** System operates as CLI tool (`do-uw analyze <TICKER>`) with structured terminal output during execution
- [ ] **CORE-04:** System uses a 7-stage linear pipeline: RESOLVE -> ACQUIRE -> EXTRACT -> ANALYZE -> SCORE -> BENCHMARK -> RENDER
- [ ] **CORE-05:** System persists all intermediate state to a single `AnalysisState` Pydantic model serialized as JSON, enabling resume-from-failure
- [ ] **CORE-06:** System produces analysis for US-listed public companies (SEC filers), including ADRs
- [ ] **CORE-07:** System uses 100% publicly available data -- no proprietary feeds, no paid databases in v1

## DATA -- Data Acquisition and Sources

All external data acquisition, source definitions, fallback protocols, and data quality tracking.

- [ ] **DATA-01:** SEC EDGAR integration -- acquire 10-K, 10-Q, 8-K, DEF 14A, Form 4, S-1/S-3/424B, Exhibit 21, CORRESP, NT forms via EDGAR APIs (full-text search, XBRL Company Facts, filing history). Respect rate limit of 10 req/sec with proper User-Agent header. Use edgartools library as primary interface.

- [ ] **DATA-02:** Stock and market data -- Yahoo Finance (yfinance) for stock prices, historical returns, fundamental ratios, analyst consensus data, insider transaction summaries, short interest data. Provider abstraction layer to allow future swap.

- [ ] **DATA-03:** Litigation data with multi-tier fallback -- Stanford Securities Class Action Clearinghouse (SCAC) as primary source, with fallback chain: Stanford SCAC -> 10-K Item 3 -> CourtListener API -> PACER -> web search -> law firm press releases. Every finding cross-validated against 10-K disclosure. Never assume "no data found" = "no litigation exists."

- [ ] **DATA-04:** SEC enforcement data -- SEC Litigation Releases, Administrative and Cease-and-Desist Proceedings (AAER) database, EDGAR keyword search for Wells notices and subpoenas in filings.

- [ ] **DATA-05:** Regulatory databases -- FDA (openFDA API), EPA ECHO (REST API), OFAC SDN list, Transparency International Corruption Perceptions Index (CPI) scores by country, Stanford FCPA Clearinghouse, DOJ press releases for corporate enforcement.

- [ ] **DATA-06:** Earnings call transcripts -- Seeking Alpha, company investor relations pages, Financial Modeling Prep API. Required for sentiment analysis and management tone tracking.

- [ ] **DATA-07:** Sentiment and alternative data -- Glassdoor reviews and CEO approval ratings, LinkedIn headcount trends, Reddit/social media sentiment, news APIs, customer review aggregation. Acquired via web search with validation.

- [ ] **DATA-08:** Data freshness tracking -- every data point tagged with: (a) as-of date (when the data was current), (b) staleness flag: CURRENT (<90 days), AGING (90-180 days), STALE (>180 days), (c) confidence level: HIGH, MEDIUM, LOW, (d) source citation with filing type, date, and URL where applicable.

- [ ] **DATA-09:** Multi-tier fallback protocol -- for every data source, define primary -> secondary -> tertiary acquisition path. If primary source fails or returns no data, automatically fall through to next tier. Flag confidence level at each tier (HIGH for primary, MEDIUM for secondary, LOW for tertiary). Never present "no data" as "no issue" -- explicitly state what was not found and what sources were checked.

- [ ] **DATA-10:** Web search with validation -- web search is a legitimate data acquisition method for: sentiment signals, news coverage, short seller reports, alternative data, regulatory actions not in structured databases. Every web-sourced data point requires cross-validation with at least 2 independent sources before inclusion in the worksheet.

### DATA INTEGRITY ENFORCEMENT (applies to ALL data requirements above)

- [ ] **DATA-11:** Every data point in the Pydantic state model MUST have a `source` field (filing type + date + URL/CIK) and a `confidence` field (HIGH/MEDIUM/LOW). These fields are **not optional** — the model validation fails without them. No data point can be written to state without provenance.
- [ ] **DATA-12:** The system MUST NEVER generate, guess, or hallucinate financial data. If a data point cannot be sourced from a structured API response, filing, or cross-validated web search, the output displays "Not Available — [reason]" instead of a fabricated value. A gap in the worksheet is always better than a wrong number.
- [ ] **DATA-13:** Data acquisition gates — the pipeline MUST halt and flag if critical data is missing after all fallback tiers are exhausted. Six gates: (1) 10-K complete, (2) 10-Q series, (3) DEF 14A, (4) 8-K/Form 4 series, (5) litigation search, (6) market data. Gate failure produces a "DATA GAP" warning in the output, not silent omission.
- [ ] **DATA-14:** LLM-generated analysis (sentiment assessment, narrative coherence, earnings call tone) MUST be clearly labeled as "AI Assessment" in the output with methodology disclosed. It must never be presented as factual data.
- [ ] **DATA-15:** Source citation is a REQUIRED output element for every data point in the worksheet. Format: [Filing Type] [Date] [SEC URL or API]. No unsourced data points permitted in the final output. Auditable trail from every number to its origin.
- [ ] **DATA-16:** Multi-tier fallback chains — for every data source type, the system defines primary → secondary → tertiary acquisition paths. If primary fails, automatically fall through. Log which tier was used. Never assume "no data from primary" = "data doesn't exist."
- [ ] **DATA-17:** Blind Spot Detection via Broad Web Search -- after all structured data acquisition completes, the system MUST run broad exploratory web searches specifically designed to surface information that structured sources would miss. Includes: (a) company name + "lawsuit" / "investigation" / "scandal" / "fraud" / "SEC" / "subpoena" / "whistleblower" / "restatement" searches, (b) CEO/CFO name + "lawsuit" / "investigation" / "fired" / "terminated" searches, (c) company + "short seller report" / "Hindenburg" / "Muddy Waters" / "Citron" searches, (d) company + industry-specific risk terms (pharma: "FDA warning letter" / "clinical trial failure"; fintech: "CFPB action"; crypto: "enforcement"), (e) Reddit/social media sentiment sweeps. Results are cross-validated and logged as LOW confidence unless corroborated. The goal is to catch what structured APIs miss — news stories, blog posts, analyst notes, regulatory actions announced but not yet in databases, social media exposures. This is NOT optional: every analysis run MUST include a blind spot sweep.
- [ ] **DATA-18:** Web Search Breadth Policy -- broad web search is a FIRST-CLASS data acquisition method, not a fallback. The system runs proactive discovery searches at the START of the ACQUIRE phase (alongside structured source queries), not just when structured sources fail. Rationale: many D&O-relevant signals (short seller reports, employee lawsuits, state AG actions, FCPA investigations, consumer complaints, social media exposés) appear in news/web sources BEFORE they show up in SEC filings or structured databases. Early broad search ensures the system catches emerging risks.

## SECT1 -- Section 1: Executive Summary

The first page(s) of the worksheet. Provides at-a-glance risk assessment for rapid underwriter triage.

- [ ] **SECT1-01:** Company snapshot with key metrics pulled from Section 2 data -- market cap, revenue, employee count, industry classification, ticker symbol, exchange listing. Displayed as a structured header block.

- [ ] **SECT1-02:** Inherent Risk Baseline -- actuarial filing probability and severity range based on a market cap x industry matrix. This is the base rate BEFORE any company-specific analysis. Sources: Stanford SCAC filing rates by industry, settlement severity data by market cap tier.

- [ ] **SECT1-03:** Key Negatives (top 5) -- the five most material risk findings from the entire worksheet. Each negative includes: (a) evidence narrative (what was found and why it matters), (b) section origin (which worksheet section surfaced this finding), (c) scoring impact (how many points deducted and from which factor), (d) allegation theory mapping (which of the 5 complaint theories this finding supports: A=Disclosure, B=Guidance, C=Product/Ops, D=Governance, E=M&A).

- [ ] **SECT1-04:** Key Positives (top 5) -- the five most material positive findings. Same structure as negatives: (a) evidence narrative, (b) section origin, (c) scoring impact (points preserved or added), (d) defense theory mapping (which defense posture this finding supports: Truth on the Market, no scienter, PSLRA safe harbor, strong governance, etc.).

- [ ] **SECT1-05:** Subjective Risk Assessment -- claim probability band (Low <3%, Moderate 3-6%, Elevated 6-10%, High 10-15%, Very High >15%) plus loss severity scenarios at 25th/50th/75th/95th percentiles. Derived from SECT7 scoring output.

- [ ] **SECT1-06:** Tower Position Recommendation -- minimum recommended attachment point, position assessment by layer type (primary, low excess, mid excess, high excess), Side A separate assessment based on indemnification capacity. Derived from SECT7 tower analysis.

- [ ] **SECT1-07:** Deal context area -- structured fields for layer being quoted, premium, carrier lineup, tower structure, and other deal-specific information. These fields are populated when deal context is available (e.g., from broker submission) and left as placeholder fields when running ticker-only analysis.

## SECT2 -- Section 2: Company Profile & Industry Context

Comprehensive company identification, business operations analysis, and industry risk mapping.

- [ ] **SECT2-01:** Company identification fields -- legal company name, ticker symbol, CIK number, exchange listing, SIC code, NAICS code, state of incorporation, headquarters location, fiscal year end date, SEC filer category (large accelerated, accelerated, non-accelerated, smaller reporting, emerging growth), years since IPO/public listing.

- [ ] **SECT2-02:** Business Operations Profile -- plain-language business description extracted from 10-K Item 1; business model explanation (how the company makes money); revenue composition by reportable segment with growth/decline trends for each segment across available periods.

- [ ] **SECT2-03:** Geographic footprint -- where the company operates (domestic/international breakdown), revenue by geography (from segment disclosures), employee distribution by region, subsidiary footprint extracted from Exhibit 21 filings with country-level detail.

- [ ] **SECT2-04:** Customer and supplier concentration -- customers representing >10% of revenue (ASC 280 disclosure), single-source supplier dependencies, product line concentration (revenue dependence on single product or service line).

- [ ] **SECT2-05:** Operational complexity assessment -- total subsidiary count (from Exhibit 21), number of operating jurisdictions, Variable Interest Entity (VIE) structures, off-balance-sheet entities and arrangements, dual-class share structure identification, controlled company status determination.

- [ ] **SECT2-06:** Business model changes and pivots -- transformation activity (strategic pivots, digital transformation, new business lines), new market entry or geographic expansion, business exits or divestitures, segment additions or removals. Identified from 8-K filings, earnings call commentary, and year-over-year changes in segment reporting.

- [ ] **SECT2-07:** D&O Exposure Mapping from Operations -- systematic mapping of how the company's business operations create D&O claim vectors: (a) regulatory exposure by business line (FDA for pharma, CFPB for financial services, FCC for telecom, etc.), (b) product liability vectors (consumer products, medical devices, software/SaaS), (c) environmental and social exposure (manufacturing, energy, mining), (d) technology and IP litigation exposure, (e) government contracting risk (False Claims Act), (f) FCPA/anti-bribery exposure based on geographic footprint cross-referenced with Transparency International CPI scores for each operating country.

- [ ] **SECT2-08:** Industry Classification and D&O Risk Profile -- SCA filing rate for company's industry vs. all-company base rate, settlement severity distribution for industry, industry-specific claim theories (e.g., channel stuffing for tech, clinical trial disclosure for pharma), emerging industry-specific trends, regulatory pipeline affecting industry (pending legislation, new agency rules, enforcement priorities).

- [ ] **SECT2-09:** Peer Group Construction -- 3-tier peer group: (a) primary peers (same SIC code + similar market cap range, minimum 5 companies, named with ticker symbols), (b) sector ETF benchmark (identified by SIC/NAICS), (c) market cap cohort (companies in same market cap band regardless of industry). All peers used for benchmarking throughout the worksheet.

- [ ] **SECT2-10:** Company Event Timeline -- chronological timeline of material corporate events: IPO date, M&A activity (completed and pending deals), capital raises (equity and debt), restructurings, leadership changes (C-suite and board), regulatory milestones. Includes upcoming catalysts calendar (known future events: earnings dates, FDA decision dates, contract renewals, debt maturities, lock-up expirations).

- [ ] **SECT2-11:** Section Summary Paragraph -- narrative synthesis covering: operational complexity level, how dynamic/changing the business is (stable vs. transforming), positioning among peers (leader, middle of pack, laggard), key D&O exposure vectors from operations.

## SECT3 -- Section 3: Financial Health

Comprehensive financial analysis including statements, forensic quality analysis, distress indicators, and audit risk.

- [ ] **SECT3-01:** Financial Health Summary -- narrative paragraph at top of section synthesizing the financial picture: overall trajectory (improving/stable/deteriorating), key concerns, peer-relative positioning, and D&O relevance of financial findings.

- [ ] **SECT3-02:** Summary Income Statement table -- columns: FY Prior, FY Latest (YoY change), YTD from latest quarterly filings, Prior Year YTD (YoY change), Notes column flagging concerns. Metrics: revenue, gross profit, gross margin %, operating income, operating margin %, net income, net margin %, diluted EPS, non-GAAP EPS (if reported), stock-based compensation (SBC).

- [ ] **SECT3-03:** Summary Balance Sheet table -- columns: FY Prior, FY Latest (change), Latest Quarter, Change from FY, Notes column. Metrics: cash and short-term investments, accounts receivable, inventory, goodwill and intangibles, total assets, total debt (short-term + long-term), stockholders' equity.

- [ ] **SECT3-04:** Summary Cash Flow table -- same FY YoY + YTD structure as income statement. Metrics: operating cash flow (OCF), capital expenditures, free cash flow (FCF), OCF-to-net-income ratio, share buybacks, dividends paid.

- [ ] **SECT3-05:** Peer-Relative Benchmarking -- percentile rank for key financial ratios against primary peer group (from SECT2-09). Ratios: operating margin, net margin, debt-to-equity, net debt-to-EBITDA, free cash flow yield, return on equity (ROE), current ratio. Displayed as table with peer median and company percentile.

- [ ] **SECT3-06:** Earnings Quality and Forensic Analysis -- comprehensive earnings quality assessment including: (a) Sloan accrual ratio (high accruals = low earnings quality), (b) DSO/DIO/DPO trends and cash conversion cycle trajectory, (c) operating cash flow vs. net income divergence trend (growing gap is a red flag), (d) non-GAAP adjustment quality -- persistence, magnitude, and trend of non-GAAP add-backs, (e) SBC as percentage of revenue and as percentage of OCF, (f) revenue recognition complexity analysis (from 10-K Note 2 accounting policies), (g) revenue seasonality and quarter-end concentration patterns, (h) cookie jar reserve indicators (unusual reserve releases boosting earnings), (i) capex vs. depreciation ratio (underinvestment signal), (j) organic vs. acquired revenue growth decomposition, (k) receivables growth vs. revenue growth comparison, (l) allowance for doubtful accounts as percentage of AR and trend, (m) deferred revenue trend analysis.

- [ ] **SECT3-07:** Distress Indicators -- computed scores: Altman Z-Score (bankruptcy prediction), Beneish M-Score (earnings manipulation detection), Ohlson O-Score (bankruptcy probability), Piotroski F-Score (financial strength). All scores computed for the company AND for each primary peer (displayed as percentile rank against peers). 4-quarter trajectory shown for Z-Score and M-Score to identify deterioration trends.

- [ ] **SECT3-08:** Liquidity assessment -- cash and equivalents, current ratio, quick ratio, cash runway calculation (for cash-burning companies: months of cash at current burn rate), revolving credit facility details (total capacity, amount drawn, undrawn availability, maturity date, key covenants), working capital trend over 4 quarters.

- [ ] **SECT3-09:** Leverage assessment -- debt-to-equity ratio, net debt-to-EBITDA, total debt-to-total capitalization, interest coverage ratio, fixed charge coverage ratio, leverage trend over 4 quarters, leverage ratios vs. primary peers.

- [ ] **SECT3-10:** Debt structure analysis -- breakdown by instrument type (term loans, bonds/notes, convertible debt, revolver draws, other), secured vs. unsecured split, fixed-rate vs. floating-rate mix, maturity schedule for next 5 years, near-term maturities (amounts due within 12 months and 24 months), weighted average interest rate, weighted average maturity, covenant details and covenant headroom calculations, credit rating (S&P/Moody's/Fitch) with outlook, recent rating changes (upgrades/downgrades/watch placements).

- [ ] **SECT3-11:** Refinancing risk assessment -- upcoming maturities compared against available cash plus undrawn revolver capacity, current credit spreads for company's rating tier, existing coupon rates vs. current market rates (refinancing at higher/lower cost), wall of maturities visualization if significant near-term maturities exist.

- [ ] **SECT3-12:** Audit Risk -- (a) auditor identity, Big 4 status, and auditor tenure, (b) auditor changes (8-K Item 4.01 filings) with analysis of any disagreements or reportable events disclosed, (c) audit opinion type (unqualified, qualified, adverse, disclaimer), (d) going concern opinion (current or historical), (e) material weaknesses -- both remediated and unremediated, with detail on nature and remediation timeline, (f) restatements over last 5 years (from 10-K/A, 10-Q/A, 8-K Item 4.02) with classification (Big R restatement vs. Little r revision), (g) late filings (NT 10-K, NT 10-Q forms filed), (h) SEC comment letters (CORRESP filings on EDGAR) with topic categorization, (i) Critical Audit Matters (CAMs) from auditor's report, (j) PCAOB inspection results for the company's auditor.

- [ ] **SECT3-13:** Tax indicators -- effective tax rate vs. statutory rate gap analysis, cash effective tax rate vs. book effective tax rate divergence, unrecognized tax benefits balance and trend, tax haven subsidiary count (from Exhibit 21 cross-referenced with known low-tax jurisdictions).

## SECT4 -- Section 4: Market & Trading Signals

Stock performance, insider activity, short interest, earnings track record, and capital markets exposure.

- [ ] **SECT4-01:** Stock price charts -- 1-year and 5-year price charts showing company stock vs. sector ETF and primary peers. Charts include volume bars. Event markers on chart for: every single-day decline >5%, every multi-day decline >10%, earnings dates, and material 8-K events.

- [ ] **SECT4-02:** Stock performance data table -- current price, 52-week high, 52-week low, percentage decline from 52-week high, returns (1-year, 3-year, 5-year, YTD) compared against sector ETF returns and peer average returns for same periods, beta, annualized volatility, maximum drawdown over trailing 1 year.

- [ ] **SECT4-03:** Stock drop analysis (trailing 18 months) -- comprehensive catalog of every significant stock decline: (a) single-day drops >5%: date, magnitude, sector ETF return on same day (market vs. company-specific), company-specific attribution, triggering event identification (from 8-K, earnings release, or news), corrective disclosure identification if applicable, estimated Damages During Litigation (DDL) window, (b) multi-day declines: 2-day >10%, 5-day >15%, 20-day >25% with same detail as single-day analysis.

- [ ] **SECT4-04:** Insider trading analysis (trailing 18 months) -- all Form 4 filings cataloged: net buying/selling in dollars, 10b5-1 plan trades vs. discretionary (non-plan) trades, cluster selling identification (3 or more insiders selling within a 30-day window), timing of trades relative to material announcements (earnings, guidance, 8-K events), 10b5-1 plan adoptions/terminations/modifications, aggregate insider ownership level, pledged shares identified from proxy.

- [ ] **SECT4-05:** Short interest analysis -- current short interest as percentage of float, 6-month trend, days to cover ratio, short interest compared against EACH primary peer explicitly (not just peer average) with 6-month trend for each peer, activist short seller reports identified (Hindenburg Research, Muddy Waters, Citron Research, Spruce Point Capital, and others) with dates and key allegations.

- [ ] **SECT4-06:** Earnings guidance track record (trailing 16 quarters / 4 years) -- full table with one row per quarter: guidance range issued, mid-quarter revisions (raised/lowered/narrowed), actual result, beat/miss classification, miss magnitude as percentage of guided midpoint, stock price reaction to earnings. Summary statistics: overall beat rate, consecutive miss count (current streak), guidance withdrawals, guidance philosophy characterization (conservative/aggressive/no guidance).

- [ ] **SECT4-07:** Analyst sentiment -- analyst coverage count, consensus recommendation (buy/hold/sell distribution), mean and median price target, price target range (high/low), recent upgrades and downgrades (trailing 90 days), earnings estimate revisions (trailing 90 days -- up vs. down), estimate dispersion (standard deviation of analyst estimates as percentage of mean).

- [ ] **SECT4-08:** Capital markets activity and Section 11 exposure -- active shelf registration statements (S-3), recent equity and debt offerings (trailing 3 years) with dates and amounts, at-the-market (ATM) offering programs, Section 11 exposure window calculation (each offering date + 3 years = exposure window end), convertible securities outstanding (conversion price, maturity, dilution impact), PIPE transactions.

- [ ] **SECT4-09:** Adverse event scoring -- aggregated count and severity rating of all adverse corporate events over trailing 12 months. Events severity-weighted (e.g., earnings miss with 2% drop vs. restatement with 30% drop). Company score peer-ranked against primary peer group. This is a consolidated metric from the individual event analyses in SECT4-03 through SECT4-08.

## SECT5 -- Section 5: Governance & Leadership (Forensic)

Deep forensic analysis of leadership, board, and governance quality with emphasis on individual risk profiles.

- [ ] **SECT5-01:** Governance and Leadership Summary -- opening narrative assessment answering the key underwriting question: are these people a D&O asset or a D&O liability? Synthesizes findings from all subsections.

- [ ] **SECT5-02:** Leadership Forensic Profiles -- for each Named Executive Officer (CEO, CFO, CAO, CLO, COO at minimum, plus any others disclosed in DEF 14A): (a) biography and tenure from DEF 14A, (b) prior company litigation exposure -- search prior employers against Stanford SCAC database for securities class actions filed during or related to their tenure, (c) personal litigation and enforcement history -- search individual name against Stanford SCAC and SEC AAER database, (d) prior company associations -- restatements, SEC enforcement actions, or bankruptcies at prior companies during the individual's tenure, (e) shade factor assessment: credential verification (degrees, certifications), personal legal issues (via PACER search, news search), controversial public statements, financial pressure indicators (pledged shares, margin loans from proxy disclosure, personal bankruptcy filings).

- [ ] **SECT5-03:** Board Forensic Profiles -- same forensic research as SECT5-02 for each director, plus: (a) interlock analysis -- identify all other public company boards each director sits on and flag any currently in active litigation, (b) relationship mapping -- document interconnections between board members and between board and management (shared prior employers, co-investors, family relationships, social connections discoverable from public sources), (c) true independence assessment -- technically independent per exchange listing rules but functionally connected? Identify directors who are technically independent but have long tenure, social ties, or financial relationships with management.

- [ ] **SECT5-04:** Earnings Call Sentiment Analysis -- analysis of last 4-8 quarters of earnings call transcripts: (a) management tone trajectory (improving/stable/deteriorating), (b) hedging language trend using Loughran-McDonald financial uncertainty word dictionary, (c) CEO vs. CFO tone divergence (are they aligned or is one more cautious?), (d) Q&A evasion patterns (non-answers, redirections, "we'll get back to you"), (e) specificity trend (count of specific numbers/metrics per call -- declining specificity is a red flag), (f) analyst question aggression trend (are analysts becoming more challenging?), (g) forward-looking language certainty changes ("we will" vs. "we expect" vs. "we hope"), (h) emergence of legal or defensive language. Sources: Seeking Alpha transcripts, company IR pages, Financial Modeling Prep.

- [ ] **SECT5-05:** Executive Compensation and Incentive Risk -- (a) CEO total compensation (salary, bonus, equity awards, all other), (b) CEO pay ratio (vs. median employee), (c) CEO pay vs. peer group median, (d) compensation mix analysis (percentage salary/bonus/equity/other -- heavy short-term bonus incentivizes short-term behavior), (e) performance metrics used for incentive compensation (revenue targets, EPS targets, TSR -- do they incentivize earnings management?), (f) say-on-pay vote results (approval percentage, trend, any failed votes), (g) clawback policy assessment (mandatory Dodd-Frank minimum vs. broader voluntary clawback), (h) related party transactions (amounts, terms, board approval process), (i) notable perquisites.

- [ ] **SECT5-06:** Leadership Stability Assessment (trailing 18 months) -- (a) team stability snapshot table: each C-suite and senior role, current incumbent name, in-role-since date, whether departure/appointment was characterized as planned; (b) duration metrics: average executive tenure, longest-tenured, shortest-tenured; (c) 18-month change log: every departure and appointment with detail -- was it characterized as planned (retirement, personal reasons) or unplanned, 8-K language analysis (resignation vs. termination vs. mutual agreement), timing analysis (did departure coincide with earnings, filing deadlines, stock drops, or litigation?), pattern analysis across multiple changes; (d) red flag triggers: sudden CFO departure, CAO departure after filing season, CFO + another C-suite within 6 months, 3+ executive departures within 12 months, mid-term board resignation, General Counsel departure during active litigation; (e) leadership stability score peer-compared against primary peer group.

- [ ] **SECT5-07:** Board Structure and Governance Quality -- (a) composition: board size, independence ratio, diversity matrix (gender, ethnic, age, skills), average board tenure, board refreshment pattern (new directors added in last 3 years); (b) leadership: CEO/Chair duality status, lead independent director identified, (c) activity: meeting frequency (from proxy), (d) overboarding: directors serving on 4+ total public boards, (e) committee structure: audit, compensation, nominating/governance, risk, cybersecurity committees -- composition and independence of each, (f) derived governance quality score computed from public data (independence %, tenure mix, refreshment, committee structure, overboarding, meeting frequency) with peer-relative percentile rank.

- [ ] **SECT5-08:** Ownership Structure and Activist Risk -- (a) institutional ownership percentage, (b) top 10 institutional holders with identification of known activist investors, (c) insider ownership percentage (aggregate), (d) dual-class voting control analysis (if applicable: who controls what percentage of votes with what percentage of economic interest), (e) Schedule 13D filings (trailing 24 months) indicating activist or control intent, (f) 13G-to-13D conversions (passive to active intent -- a significant warning signal), (g) proxy contest history (trailing 3 years), (h) activist campaign risk assessment based on ownership structure, valuation, and governance gaps.

- [ ] **SECT5-09:** Broader Sentiment Signals -- (a) Glassdoor company rating and CEO approval trend (6-12 months), (b) LinkedIn headcount trend (growing/shrinking by department), (c) job posting patterns (is the company hiring in legal/compliance? -- potential signal of anticipated regulatory/litigation activity), (d) customer reviews (where applicable), (e) Reddit and social media sentiment summary, (f) news sentiment on the company and key named individuals, (g) mentions in congressional hearings or government reports.

- [ ] **SECT5-10:** Narrative Coherence Assessment -- does management's publicly stated strategy and messaging align with observable data? Specifically: (a) do financial results match the stated strategy? (b) does insider trading behavior match stated confidence? (c) does earnings call tone match financial trajectory? (d) does employee sentiment match management's public messaging? Assessment performed via rule-based checks (e.g., insider selling while CEO says "never been more bullish") supplemented by LLM analysis. Methodology fully disclosed -- no black-box assessment.

## SECT6 -- Section 6: Litigation & Regulatory Exposure

Comprehensive legal landscape analysis including active/historical litigation, regulatory proceedings, and defense posture.

- [ ] **SECT6-01:** Litigation and Regulatory Summary -- narrative paragraph synthesizing the overall legal exposure: active matters count and severity, historical pattern, regulatory pipeline, emerging exposure, and D&O relevance.

- [ ] **SECT6-02:** Litigation Timeline -- visual chronological display showing all legal/regulatory events, stock price drops, and executive changes on a single unified timeline. Enables pattern recognition (e.g., executive departure followed by litigation filing, or stock drop followed by regulatory inquiry).

- [ ] **SECT6-03:** Securities Class Actions -- active and historical cases (trailing 10 years) sourced from Stanford SCAC with full case details: (a) for each case: case name, case number, court and judge, filing date, class period (start/end), specific allegations, lead plaintiff and lead plaintiff type (institutional investor, public pension fund, individual), lead counsel and counsel tier assessment, current procedural status (filed, motion to dismiss pending, discovery, trial, settled, dismissed); (b) case strength assessment: claim type (10b-5, Section 11, Section 14(a)), scienter evidence strength, loss causation clarity, lead counsel quality (Robbins Geller, Bernstein Litowitz, etc.), judge/court track record on securities cases; (c) damages quantification: estimated DDL, settlement prediction at 25th/50th/75th/95th percentile based on empirical settlement data for similar cases, defense cost estimates. **Multi-tier data acquisition:** Stanford SCAC (primary) -> 10-K Item 3 litigation disclosure -> CourtListener API -> PACER -> web search -> law firm press releases. Every finding cross-validated against 10-K Item 3 and Item 8 contingency disclosures.

- [ ] **SECT6-04:** SEC Enforcement and Regulatory Pipeline -- (a) position in SEC enforcement pipeline: comment letter -> informal inquiry -> formal investigation (HO-number) -> Wells notice -> enforcement action -> private follow-on litigation; (b) enforcement actions: SEC complaints, administrative proceedings, penalties imposed, officer/director bars, consent decrees; (c) sources: SEC Litigation Releases, AAER database, 10-K/10-Q risk factor and legal proceedings disclosures, EDGAR full-text keyword search for "Wells notice," "subpoena," "investigation," and related terms, web search for SEC press releases.

- [ ] **SECT6-05:** Derivative Suits and Fiduciary Claims -- Delaware Court of Chancery filings, Section 220 books-and-records demands (often precursors to derivative suits), Caremark/oversight duty claims, demand refusal letters. Sources: Delaware court electronic filing system, PACER, 10-K Item 3 disclosure.

- [ ] **SECT6-06:** Regulatory Proceedings (Non-SEC) -- DOJ investigations and prosecutions, FCPA actions (Stanford FCPA Clearinghouse), FTC enforcement, industry-specific regulators: FDA (openFDA), EPA (ECHO database), CFPB, OCC, state banking regulators, EEOC, state attorneys general actions. Each with current status, exposure estimate, and D&O implications.

- [ ] **SECT6-07:** M&A and Deal Litigation -- merger objection lawsuits, appraisal actions (Delaware Section 262), disclosure-only settlements and their terms, Revlon duty claims, fiduciary duty challenges to completed or pending transactions.

- [ ] **SECT6-08:** Workforce, Product, and Environmental Litigation -- (a) employment litigation: EEOC charges, wage and hour class actions, workplace discrimination suits, (b) whistleblower indicators: SEC whistleblower awards referencing the company, qui tam / False Claims Act suits, (c) WARN Act notices (mass layoff filings), (d) product recalls (CPSC, FDA, NHTSA databases), (e) mass tort exposure, (f) EPA enforcement actions (ECHO database), (g) cybersecurity incidents (8-K Item 1.05 disclosures, HHS breach portal for healthcare), (h) climate-related litigation.

- [ ] **SECT6-09:** Defense Strength Assessment -- analysis of the company's defensive posture: (a) forum selection provisions in charter/bylaws (both Delaware exclusive forum and federal forum for Securities Act claims), (b) PSLRA safe harbor usage in forward-looking statements, (c) Truth on the Market defense viability (was corrective information publicly available?), (d) no-scienter posture (strength of argument that management did not know), (e) motion to dismiss judge track record (dismissal rates for assigned judge in securities cases), (f) prior dismissal success (has this company successfully dismissed securities claims before?).

- [ ] **SECT6-10:** Industry Claim Pattern Analysis -- research actual lawsuits filed against companies in the same industry (trailing 5-10 years): (a) categorize the specific claim theories used (e.g., channel stuffing for tech distributors, clinical trial data manipulation for pharma, loan loss reserve inadequacy for banks), (b) assess whether THIS company is exposed to each identified pattern based on its operations, (c) identify emerging or novel claim patterns not yet seen for this company but appearing in the industry. This is concrete case-based analysis, not statistical frequency data.

- [ ] **SECT6-11:** Statute of Limitations Map -- for each potential claim type, calculate the exposure window: (a) Section 10(b)/Rule 10b-5: 2-year discovery / 5-year repose, (b) Section 11: 1 year from discovery / 3 years from offering, (c) Section 14(a): varies by circuit, (d) derivative claims: demand futility analysis timeline, (e) FCPA: 5-year statute. Map when each window opens and closes based on known events and disclosures.

- [ ] **SECT6-12:** Known Matters and Contingent Liabilities -- (a) all pending litigation from 10-K Item 3 (Legal Proceedings), (b) contingent liability reserves from financial statement footnotes (ASC 450 disclosures -- both disclosed amounts and "reasonably possible" ranges), (c) existing D&O insurance claims (if disclosed in filings or public records), (d) indemnification obligations disclosed in filings.

## SECT7 -- Section 7: Risk Scoring & Synthesis

Quantitative scoring model, pattern detection, red flag gates, risk classification, and underwriting recommendations.

- [x] **SECT7-01:** 10-Factor Composite Score -- quality score computed as 100 minus total risk points. Factor weights (maximum point deductions): F.1 Prior Litigation History (max 20 points), F.2 Stock Decline Severity (max 15 points), F.3 Restatement/Audit Issues (max 12 points), F.4 Capital Markets Activity/M&A (max 10 points), F.5 Guidance/Earnings Track Record (max 10 points), F.6 Short Interest (max 8 points), F.7 Volatility/Market Signals (max 9 points), F.8 Financial Distress Indicators (max 8 points), F.9 Governance Quality (max 6 points), F.10 Officer/Leadership Stability (max 2 points). Each factor scored with transparent sub-component breakdowns showing exactly which findings contributed points.

- [x] **SECT7-02:** 17 Composite Pattern Detection -- detection of multi-factor risk patterns that are more significant than any individual factor alone. Patterns: EVENT_COLLAPSE (rapid multi-event deterioration), CASCADE (one problem triggering chain reaction), PEER_DIVERGENCE (company underperforming peers across multiple dimensions), DEATH_SPIRAL (reinforcing negative cycle of stock drops, litigation, executive departures), SHORT_ATTACK (coordinated short selling with public report), INFORMED_TRADING (insider selling before adverse events), SUSTAINABILITY_RISK (ESG-related risk patterns), CONCENTRATION_COMPOSITE (customer, supplier, and product concentration), AI_WASHING_RISK (AI claims vs. AI reality), DISCLOSURE_QUALITY_RISK (declining disclosure quality/specificity), NARRATIVE_COHERENCE_RISK (management story doesn't match data), CATALYST_RISK (upcoming events that could trigger claims), GUIDANCE_EROSION (systematic guidance deterioration), LIQUIDITY_STRESS (tightening liquidity across multiple indicators), EARNINGS_QUALITY_DETERIORATION (declining earnings quality metrics), TURNOVER_STRESS (leadership instability pattern), PROXY_ADVISOR_RISK (ISS/Glass Lewis negative recommendation risk).

- [x] **SECT7-03:** 11 Critical Red Flag gates -- hard score ceilings imposed when critical conditions are triggered, regardless of other scoring. Gates: (1) active securities class action -> ceiling, (2) Wells notice received -> ceiling, (3) DOJ criminal investigation -> ceiling, (4) going concern opinion -> ceiling, (5) financial restatement (Big R) -> ceiling, (6) unremediated material weakness -> ceiling, (7) auditor resignation (not non-renewal) -> ceiling, (8) activist short seller report published -> ceiling, (9) debt default or covenant violation -> ceiling, (10) stock price below IPO price -> ceiling, (11) disclosed subpoena from government agency -> ceiling. Specific ceiling values configurable via JSON config.

- [x] **SECT7-04:** D&O Risk Type Classification -- classify the company into one of 7 risk archetypes that drive downstream analysis weighting: BINARY_EVENT (outcome depends on single pending event, e.g., FDA approval), GROWTH_DARLING (high-growth, high-multiple, market expects perfection), GUIDANCE_DEPENDENT (market tracks company's own guidance closely), REGULATORY_SENSITIVE (heavily regulated industry, regulatory action is primary risk), TRANSFORMATION (company in major strategic pivot), STABLE_MATURE (low-growth, established, governance is primary risk vector), DISTRESSED (financial distress, bankruptcy risk, fiduciary duty is primary vector). Companies may have a secondary overlay (e.g., GROWTH_DARLING with REGULATORY_SENSITIVE overlay).

- [x] **SECT7-05:** Allegation Theory Mapping -- all findings from Sections 2-6 mapped to the 5 principal complaint theories: Theory A = Disclosure Fraud (material misrepresentation/omission), Theory B = Guidance Fraud (forward-looking statement liability), Theory C = Product/Operations (business operations causing harm leading to securities claims), Theory D = Governance/Fiduciary Breach (duty of care/loyalty violations), Theory E = M&A Related (deal litigation, appraisal, fiduciary duty in transactions). Distribution analysis showing the company's primary exposure type and concentration.

- [x] **SECT7-06:** Claims Correlation Scoring -- probability weights applied to individual findings based on their empirical correlation with D&O claim filing and severity. Methodology to be determined -- may integrate directly into factor scoring weights (SECT7-01) rather than as a separate scoring layer. Flagged for calibration.

- [x] **SECT7-07:** Claim Probability output -- probability band assignment derived from inherent industry/market-cap baseline (SECT1-02) adjusted by company-specific scoring. Bands: Low (<3% annual filing probability), Moderate (3-6%), Elevated (6-10%), High (10-15%), Very High (>15%). Expressed as a range, not a point estimate. Includes comparison to industry base rate.

- [x] **SECT7-08:** Loss Severity modeling -- (a) DDL scenario analysis: 10%, 20%, and 30% stock price decline applied to market cap to estimate investor losses, (b) settlement prediction: DDL multiplied by empirical settlement-to-DDL ratios adjusted for: lead counsel tier, lead plaintiff type (institutional vs. individual), case type (10b-5 vs. Section 11 vs. derivative), court/jurisdiction, presence of SEC co-enforcement, (c) defense cost estimates based on case complexity and jurisdiction, (d) four scenario outputs: 25th percentile (favorable), 50th percentile (median), 75th percentile (adverse), 95th percentile (catastrophic).

- [x] **SECT7-09:** Tower Position Recommendation -- (a) minimum recommended attachment point based on loss severity scenarios, (b) position assessment by layer type: primary layer (most exposed to frequency), low excess (frequency + moderate severity), mid excess (severity), high excess (catastrophic severity only), (c) Side A assessment: separate analysis of Side A exposure based on indemnification capacity (company's cash position, charter indemnification provisions, bankruptcy risk that would impair corporate indemnification). Side A is the coverage that protects individual directors/officers when the company cannot or will not indemnify.

- [x] **SECT7-10:** Red Flag Summary -- consolidated list of every flagged item from all sections, organized by severity tier: CRITICAL (immediate underwriting concern, likely deal-breaker or major pricing impact), HIGH (significant concern requiring underwriter attention), MODERATE (noteworthy but manageable), LOW (minor concern for monitoring). Each flagged item includes: finding description, source citation (filing type, date, URL), scoring impact (factor and points), allegation theory mapping (A/B/C/D/E), trajectory indicator (NEW, WORSENING, STABLE, IMPROVING).

- [x] **SECT7-11:** NEEDS CALIBRATION -- the following system parameters require calibration against real-world cases before production use: (a) scoring weights within each of the 10 factors, (b) factor maximum point allocations, (c) critical red flag ceiling values, (d) tier boundary thresholds (score ranges for WIN/WANT/WRITE/WATCH/WALK/NO TOUCH), (e) tower recommendation logic (attachment points by market cap tier and risk score), (f) composite pattern trigger conditions and score adjustments, (g) claims correlation weights. Calibration methodology: compare system output against historical underwriting decisions on known accounts. All calibration parameters stored in JSON config files, not hardcoded.

## OUT -- Output Generation

Document rendering across multiple formats from a single data source.

- [ ] **OUT-01:** Primary output format is Word document (.docx) generated via python-docx library. Professional formatting suitable for printing, annotation, email distribution, and filing. Includes all 7 worksheet sections with tables, charts, and narrative text.

- [ ] **OUT-02:** Secondary output formats: PDF generated via WeasyPrint, and Markdown generated via Jinja2 templates. All three formats (Word, PDF, Markdown) produced from the same underlying AnalysisState data -- no format-specific analysis logic.

- [ ] **OUT-03:** Every section begins with a summary paragraph that synthesizes the section's findings, key concerns, and D&O relevance. An underwriter reading only the summary paragraphs should get a complete (if abbreviated) picture of the risk.

- [ ] **OUT-04:** Every data point in the worksheet includes source citation: filing type (10-K, DEF 14A, Form 4, etc.), filing date, and URL or reference identifier. Underwriter can trace any number or claim back to its source.

- [ ] **OUT-05:** Every flagged item includes D&O underwriting context -- a brief explanation of why this particular finding matters specifically for D&O insurance (not just general corporate risk). Example: "CFO departure 3 weeks after 10-Q filing" should explain the D&O relevance (potential knowledge of undisclosed issues, scienter evidence in future litigation, historical pattern of CFO departures preceding restatements).

- [ ] **OUT-06:** Meeting prep companion document -- a separate structured document generated alongside the main worksheet. Contains questions for broker/client meetings categorized as: (a) CLARIFICATION -- questions about ambiguous or incomplete data, (b) FORWARD_INDICATOR -- questions about upcoming events or forward-looking information, (c) GAP_FILLER -- questions targeting data gaps the system could not fill from public sources, (d) CREDIBILITY_TEST -- questions designed to test management's narrative against the data (e.g., "Management has stated growth is organic, but our analysis shows 80% of revenue increase came from acquisitions -- can you reconcile?").

## ARCH -- Architecture and Anti-Context-Rot

Technical architecture decisions, development discipline, and predecessor failure prevention.

- [ ] **ARCH-01:** Python 3.12 with uv package manager. Locked dependencies via uv.lock. All dependencies version-pinned.

- [ ] **ARCH-02:** 7-stage linear pipeline architecture: RESOLVE (ticker -> company identity) -> ACQUIRE (all external data from all sources) -> EXTRACT (raw data -> structured facts) -> ANALYZE (structured facts -> check results across all sections) -> SCORE (check results -> 10-factor scores, tier classification) -> BENCHMARK (scores + peer data -> peer-relative positioning) -> RENDER (all results -> Word/PDF/Markdown output). Each stage has defined input/output contracts and validation gates.

- [ ] **ARCH-03:** Pydantic models for all state and data structures. Complete `AnalysisState` model defining the schema for all 7 pipeline stages. State persisted as JSON with atomic read/write and resume-from-failure support.

- [ ] **ARCH-04:** SQLite local cache for SEC filings, market data, and analysis results. Prevents redundant API calls. Cache invalidation based on data freshness rules (DATA-08).

- [ ] **ARCH-05:** No source file over 500 lines. Document generation uses one module per report section, each under 400 lines. Enforced via CI/linting check.

- [ ] **ARCH-06:** Anti-context-rot discipline -- mandatory practices to prevent the predecessor's 99-phase architectural decay: (a) explicit checklists for every multi-step operation, (b) data-before-analysis gates (no analysis stage runs until data acquisition is verified complete), (c) chunked processing for large operations (check execution in batches of ~20), (d) state files not memory (all intermediate results written to disk, not held in working memory across sessions), (e) verification checkpoints after each pipeline stage, (f) single source of truth for every concept (no concept defined in more than one place), (g) CLAUDE.md maintained with authoritative module map, (h) cleanup cadence -- every 5th development session focused on consolidation, dead code removal, and documentation refresh.

- [ ] **ARCH-07:** MCP server integration -- the following MCP servers are installed and available: brave-search (web search with 2,000 free/month via Brave API), playwright (headless browser automation for JavaScript-heavy sites like court databases and Stanford SCAC), fetch (HTTP content extraction), context7 (library documentation lookup), edgartools (SEC EDGAR filings and XBRL data), duckdb (analytical data cache at .cache/analysis.duckdb), github (dev workflow). All free-tier. Configuration in ~/.claude.json.

- [ ] **ARCH-08:** All data acquisition in main execution context -- MCP limitation: subagents/background agents cannot access MCP tools. Therefore, ALL external data acquisition (SEC EDGAR, court databases, market data, web searches) must complete in the main context before any subagent processing. This is a hard architectural boundary between ACQUIRE stage and all subsequent stages.

- [ ] **ARCH-09:** Scoring weights, thresholds, tier boundaries, pattern trigger conditions, and red flag ceiling values stored in JSON configuration files -- never hardcoded in Python source. Config files: `scoring.json` (10-factor weights and tier boundaries), `checks.json` (check registry -- THE single source defining all checks), `sector_baselines.json` (industry-specific base rates and adjustment factors), `critical_red_flags.json` (red flag gate definitions and ceiling values).

- [ ] **ARCH-10:** Predecessor BRAIN/ knowledge assets carried forward -- the following domain knowledge artifacts from Underwriting-2.0 are incorporated: `checks.json` (359 D&O-specific checks), `SCORING.md` (authoritative scoring model documentation), `PATTERNS.md` (17 composite risk patterns), `sector_baselines.json` (industry context), `critical_red_flags.json` (escalation triggers). These assets represent ~1.5MB of validated domain knowledge and are the intellectual foundation of the scoring and analysis system.

## Out of Scope

The following capabilities are explicitly excluded from this system -- not deferred, but architecturally excluded.

| Exclusion | Rationale |
|-----------|-----------|
| AI making accept/decline decisions | This is decision support, not decision making. The system surfaces red flags, scores risk, and recommends tower positioning -- the underwriter makes the final call. Regulatory frameworks require human-in-the-loop for insurance decisions. |
| Proprietary data feeds in v1 | System runs on 100% publicly available data. Architecture supports future integration of paid sources (ISS, Bloomberg, Advisen) but v1 has zero data licensing cost. |
| Real-time speed | Thoroughness over speed. Analysis may take minutes to hours depending on data acquisition volume. Sub-minute response time is not a design goal. |
| Pricing engine | System recommends tower positioning and estimates loss severity scenarios, but does NOT calculate premium. Premium pricing is the underwriter's professional judgment. |
| Claims management | Analysis and assessment only. No policy administration, claims tracking, or claims adjudication functionality. |
| Private company analysis | Public companies only (SEC filers). Private company D&O analysis requires fundamentally different data sources (no EDGAR, no stock price, no Form 4, no DEF 14A). |
| International-only companies | US-listed companies only (including foreign private issuers and ADRs that file with SEC). Non-US-listed companies without SEC filings are excluded. |
| Natural language chat interface | Structured, reproducible worksheet output. Not a conversational AI that produces different outputs based on different questions. |

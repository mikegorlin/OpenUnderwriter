# Requirements: D&O Underwriting System

**Defined:** 2026-03-09
**Core Value:** Surface every red flag and risk signal that exists in public data for any publicly traded company, benchmarked against industry peers, so an underwriter can make the most informed D&O decision possible.

## v6.0 Requirements

Requirements for Company Profile Completeness milestone. Each maps to roadmap phases.

### Architectural Constraint (applies to ALL requirements)

Every new data dimension MUST be expressed as brain signals in `brain/signals/*.yaml` with:
- `acquisition` block declaring data source and field mapping
- `evaluation` block with thresholds (red/amber/green) and scoring logic
- `presentation` block with facet assignment, display format, and callout templates
- Signal wired into output manifest (`brain/output_manifest.yaml`) for render dispatch
- Check engine evaluates the signal (not just display — triggers red flags when thresholds breached)

No data enters the worksheet outside the brain signal framework. Extract → Signal → Evaluate → Render.

- [x] **EVENT-05**: Business changes rendered (pivots, divestitures, restructurings — already extracted as `business_changes`) — wired to brain signal with presentation block

### External Environment (ENVR)

- [x] **ENVR-01**: Regulatory environment intensity classified per jurisdiction (high/medium/low) with sector-specific regulators identified — brain signal with multi-regulator thresholds
- [x] **ENVR-02**: Geopolitical exposure mapped (operations in sanctioned/high-risk countries, trade war sensitivity, tariff exposure) — brain signal with sanctions/high-risk country evaluation
- [x] **ENVR-03**: ESG commitment-vs-performance gap assessed (stated commitments from proxy vs actual litigation/regulatory actions) — brain signal with gap severity evaluation
- [x] **ENVR-04**: Cyber risk profile extracted (breach history, data sensitivity classification, cyber risk factor language from 10-K) — brain signal with breach-history evaluation
- [x] **ENVR-05**: Macroeconomic sensitivity scored (interest rate sensitivity, inflation exposure, recession vulnerability from 10-K risk factors) — brain signal with composite sensitivity score

### Sector Risk Classification (SECT)

- [x] **SECT-01**: Sector hazard tier classification (Highest/High/Moderate/Lower) mapped from SIC with D&O filing rate data — brain signal with tier-based evaluation
- [x] **SECT-02**: Sector-specific claim patterns identified (top 3 claim theories per sector from SCAC/settlement data) — brain signal with presentation block (reference data)
- [x] **SECT-03**: Sector regulatory intensity overlay (number + severity of active sector regulators) — brain signal with high-regulation thresholds
- [x] **SECT-04**: Sector peer risk comparison (company's risk profile vs sector median across key D&O dimensions) — brain signal with outlier evaluation

### Structural Complexity (STRUC)

- [x] **STRUC-01**: Disclosure complexity scored (risk factor count + length, critical accounting estimate count, forward-looking statement density) — brain signal with complexity thresholds
- [x] **STRUC-02**: Non-GAAP metric usage assessed (number of non-GAAP measures, reconciliation quality, SEC comment history) — brain signal with non-GAAP count thresholds
- [x] **STRUC-03**: Related party transaction density extracted (count, materiality, recurring vs one-time) — brain signal with materiality evaluation
- [x] **STRUC-04**: Off-balance-sheet exposure identified (operating leases pre-ASC 842, guarantees, commitments, VIE consolidation) — brain signal with exposure evaluation
- [x] **STRUC-05**: Holding company structure analyzed (layers between parent and operating entities, intercompany complexity) — brain signal with layer-depth thresholds

## Out of Scope

| Feature | Reason |
|---------|--------|
| Real-time data feeds | Thoroughness over speed — batch analysis sufficient |
| Proprietary data sources | System runs on 100% public data |
| Sector-conditional signal activation | Deferred to future milestone (ISRA-02) — v6.0 adds sector classification, not sector-conditional logic |
| Automated underwriting decisions | System surfaces red flags, human decides |
| Custom sector peer groups | SECT-04 uses SIC-based peers; custom peer selection deferred |
